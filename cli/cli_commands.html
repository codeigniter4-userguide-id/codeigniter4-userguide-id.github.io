Membuat Perintah (Command) Spark

Meskipun kemampuan untuk menggunakan Controller melalui CLI seperti route lainnya nyaman, Anda mungkin menemukan saat-saat di mana Anda membutuhkan sesuatu yang sedikit berbeda. Di situlah perintah Spark berperan. Mereka adalah class sederhana yang tidak perlu memiliki route yang ditentukan, menjadikannya sempurna untuk membangun alat yang dapat digunakan pengembang untuk membuat pekerjaan mereka lebih sederhana, baik dengan menangani migrasi atau seeding database, memeriksa status cronjob, atau bahkan membangun generator kode kustom untuk perusahaan Anda.

Membuat Perintah Baru

Anda dapat dengan sangat mudah membuat perintah baru untuk digunakan dalam pengembangan Anda sendiri. Setiap class harus berada dalam berkasnya sendiri, dan harus meng-extend CodeIgniter\CLI\BaseCommand, dan mengimplementasikan metode run().

Properti berikut harus digunakan agar terdaftar dalam perintah CLI dan untuk menambahkan fungsionalitas bantuan ke perintah Anda:

  • $group: string untuk mendeskripsikan grup tempat perintah dikelompokkan saat mencantumkan perintah. Contoh: Database

  • $name: string untuk mendeskripsikan nama perintah. Contoh: make:controller

  • $description: string untuk mendeskripsikan perintah. Contoh: Menghasilkan berkas controller baru.

  • $usage: string untuk mendeskripsikan penggunaan perintah. Contoh: make:controller <nama> [opsi]

  • $arguments: array string untuk mendeskripsikan setiap argumen perintah. Contoh: 'name' => 'Nama class controller.'

  • $options: array string untuk mendeskripsikan setiap opsi perintah. Contoh: '--force' => 'Paksa timpa berkas yang sudah ada.'

Deskripsi bantuan akan secara otomatis dihasilkan sesuai dengan parameter di atas.

Lokasi Berkas

Perintah harus disimpan dalam direktori bernama Commands. Namun, direktori tersebut harus berada di namespace PSR-4 agar Autoloader dapat menemukannya. Ini bisa di app/Commands, atau direktori tempat Anda menyimpan perintah untuk digunakan dalam semua pengembangan proyek Anda, seperti Acme/Commands.

Catatan

Ketika perintah dieksekusi, seluruh environment CLI CodeIgniter telah dimuat, sehingga memungkinkan untuk mendapatkan informasi environment, informasi path, dan menggunakan alat apa pun yang akan Anda gunakan saat membuat Controller.

Contoh Perintah

Mari kita telusuri contoh perintah yang fungsi satu-satunya adalah melaporkan informasi dasar tentang aplikasi itu sendiri, untuk tujuan demonstrasi. Mulailah dengan membuat berkas baru di app/Commands/AppInfo.php. Ini harus berisi kode berikut:

<?php

namespace App\Commands;

use CodeIgniter\CLI\BaseCommand;
use CodeIgniter\CLI\CLI;

class AppInfo extends BaseCommand
{
    protected $group       = 'Demo';
    protected $name        = 'app:info';
    protected $description = 'Displays basic application information.';

    public function run(array $params)
    {
        // ...
    }
}

Jika Anda menjalankan perintah list, Anda akan melihat perintah baru terdaftar di bawah grup Demo-nya sendiri. Jika Anda melihat lebih dekat, Anda akan melihat bagaimana ini bekerja dengan cukup mudah. Properti $group hanya memberi tahu bagaimana mengorganisir perintah ini dengan semua perintah lain yang ada, memberi tahu di bawah judul apa untuk mencantumkannya.

Properti $name adalah nama perintah ini dapat dipanggil. Satu-satunya persyaratan adalah tidak boleh mengandung spasi, dan semua karakter harus valid di baris perintah itu sendiri. Namun, berdasarkan konvensi, perintah adalah huruf kecil, dengan pengelompokan perintah lebih lanjut dilakukan dengan menggunakan titik dua dengan nama perintah itu sendiri. Ini membantu menjaga beberapa perintah agar tidak mengalami tabrakan nama.

Properti terakhir, $description adalah string singkat yang ditampilkan dalam perintah list dan harus mendeskripsikan apa yang dilakukan perintah.

run()

Metode run() adalah metode yang dipanggil saat perintah sedang dijalankan. Array $params adalah daftar argumen CLI apa pun setelah nama perintah untuk penggunaan Anda. Jika string CLI adalah:

php spark foo bar baz

Maka foo adalah nama perintah, dan array $params akan menjadi:

<?php

$params = ['bar', 'baz'];

Ini juga dapat diakses melalui library CLI, tetapi ini sudah menghapus perintah Anda dari string. Parameter ini dapat digunakan untuk menyesuaikan bagaimana skrip Anda berperilaku.

Perintah demo kami mungkin memiliki metode run() seperti:

<?php

namespace App\Commands;

use CodeIgniter\CLI\BaseCommand;
use CodeIgniter\CLI\CLI;

class AppInfo extends BaseCommand
{
    // ...

    public function run(array $params)
    {
        CLI::write('PHP Version: ' . CLI::color(PHP_VERSION, 'yellow'));
        CLI::write('CI Version: ' . CLI::color(\CodeIgniter\CodeIgniter::CI_VERSION, 'yellow'));
        CLI::write('APPPATH: ' . CLI::color(APPPATH, 'yellow'));
        CLI::write('SYSTEMPATH: ' . CLI::color(SYSTEMPATH, 'yellow'));
        CLI::write('ROOTPATH: ' . CLI::color(ROOTPATH, 'yellow'));
        CLI::write('Included files: ' . CLI::color(count(get_included_files()), 'yellow'));
    }
}

Lihat halaman CLI Library untuk informasi rinci.

Penghentian Perintah

Secara default, perintah keluar dengan kode sukses 0. Jika error ditemukan saat menjalankan perintah, Anda dapat menghentikan perintah dengan menggunakan konstruksi bahasa return dengan kode keluar di metode run().

Misalnya, return EXIT_ERROR;

Pendekatan ini dapat membantu dalam debugging di tingkat sistem, jika perintah, misalnya, dijalankan melalui crontab.

Anda dapat menggunakan konstanta kode keluar EXIT_* yang didefinisikan dalam berkas app/Config/Constants.php.

BaseCommand

Class BaseCommand yang harus di-extend oleh semua perintah memiliki beberapa metode utilitas yang berguna yang harus Anda biasakan saat membuat perintah Anda sendiri. Ia juga memiliki Logger yang tersedia di $this->logger.

class CodeIgniter\CLI\BaseCommand
call(string $command[, array $params = []])
Parameter:

  • $command (string) -- Nama perintah lain yang akan dipanggil.

  • $params (array) -- Argumen CLI tambahan untuk tersedia bagi perintah tersebut.

Metode ini memungkinkan Anda untuk menjalankan perintah lain selama eksekusi perintah Anda saat ini:

<?php

$this->call('command_one');
$this->call('command_two', $params);
showError(Throwable $e)
Parameter:

  • $e (Throwable) -- Exception yang akan digunakan untuk pelaporan error.

Metode kenyamanan untuk menjaga output error yang konsisten dan jelas ke CLI:

<?php

try {
    // ...
} catch (\Exception $e) {
    $this->showError($e);
}
showHelp()

Metode untuk menampilkan bantuan perintah: (penggunaan, argumen, deskripsi, opsi)

setPad(string $item, int $max, int $extra = 2, int $indent = 0) → string
Parameter:

  • $item (string) -- Item string.

  • $max (integer) -- Ukuran maksimum.

  • $extra (integer) -- Berapa banyak spasi tambahan yang akan ditambahkan di akhir.

  • $indent (integer) -- Spasi indentasi.

Mengisi string kita sehingga semua judul memiliki panjang yang sama untuk berbaris dengan rapi deskripsi:

use CodeIgniter\CLI\CLI;

$length = max(array_map('strlen', array_keys($this->options)));

foreach ($this->options as $option => $description) {
    CLI::write(CLI::color($this->setPad($option, $length, 2, 2), 'green') . $description);
}
/*
 * Output will be:
 *  -n     Set migration namespace
 *  -g     Set database group
 *  --all  Set for all namespaces, will ignore (-n) option
 */
getPad($array, $pad)

Ditinggalkan sejak versi 4.0.5: Gunakan CodeIgniter\CLI\BaseCommand::setPad() sebagai gantinya.

Parameter:

  • $array (array) -- Array $key => $value.

  • $pad (integer) -- Spasi padding.

Metode untuk menghitung padding untuk output array $key => $value. Padding dapat digunakan untuk menghasilkan tabel yang diformat dengan baik di CLI.
