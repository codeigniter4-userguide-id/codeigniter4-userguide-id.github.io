Menjalankan Controller via CLI

Selain memanggil Controller aplikasi melalui URL di browser, mereka juga dapat dimuat melalui antarmuka baris perintah (CLI).

Catatan

Disarankan untuk menggunakan Perintah Spark untuk skrip CLI daripada memanggil controller melalui CLI. Lihat halaman Command Spark dan Creating Spark Commands untuk informasi rinci.

    • Definisikan Route

Mari coba: Hello World!

Buat Controller

Mari kita buat controller sederhana agar Anda bisa melihatnya beraksi. Menggunakan editor teks Anda, buat berkas bernama Tools.php, dan masukkan kode berikut di dalamnya:

<?php

namespace App\Controllers;

use CodeIgniter\Controller;

class Tools extends Controller
{
    public function message($to = 'World')
    {
        return "Hello {$to}!" . PHP_EOL;
    }
}

Catatan

Jika Anda menggunakan Auto Routing (Improved), ubah nama metode menjadi cliMessage().

Kemudian simpan berkas tersebut ke direktori app/Controllers/ Anda.

Definisikan Route

Jika Anda menggunakan Auto Routing, lewati bagian ini.

Dalam berkas app/Config/Routes.php Anda dapat membuat route yang hanya dapat diakses dari CLI semudah Anda membuat route lainnya. Alih-alih menggunakan metode get(), post(), atau metode serupa, Anda akan menggunakan metode cli(). Segala sesuatu yang lain berfungsi persis seperti definisi route normal:

<?php

$routes->cli('tools/message/(:segment)', 'Tools::message/$1');

Untuk informasi lebih lanjut, lihat halaman Routes.

Peringatan

Jika Anda mengaktifkan Auto Routing (Legacy) dan menempatkan berkas perintah di app/Controllers, siapa pun dapat mengakses perintah dengan bantuan Auto Routing (Legacy) melalui HTTP.

Jalankan via CLI

Sekarang biasanya Anda akan mengunjungi situs Anda menggunakan URL yang mirip dengan ini:

example.com/index.php/tools/message/to

Sebagai gantinya, kita akan membuka Terminal di Mac/Linux atau pergi ke Run > "cmd" di Windows dan navigasikan ke root web proyek CodeIgniter kita.

$ cd /path/to/project/public
$ php index.php tools message

Jika Anda melakukannya dengan benar, Anda akan melihat "Hello World!" tercetak.

$ php index.php tools message "John Smith"

Di sini kita meneruskan argumen dengan cara yang sama seperti parameter URL bekerja. "John Smith" diteruskan sebagai argumen dan outputnya adalah:

Hello John Smith!

Itu Dasar-dasarnya!

Itu, secara singkat, adalah semua yang perlu diketahui tentang controller di baris perintah. Ingatlah bahwa ini hanyalah controller normal, jadi routing dan _remap() berfungsi dengan baik.

Catatan

_remap() tidak berfungsi dengan Auto Routing (Improved).

Jika Anda ingin memastikan berjalan via CLI, periksa nilai pengembalian dari is_cli().

Namun, CodeIgniter menyediakan alat tambahan untuk membuat skrip yang dapat diakses CLI bahkan lebih menyenangkan, termasuk routing khusus CLI, dan library yang membantu Anda dengan alat khusus CLI.
