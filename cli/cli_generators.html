Generator CLI

CodeIgniter4 kini dilengkapi dengan generator untuk memudahkan pembuatan controller, model, entity, dll. Anda juga dapat membuat perancah (scaffold) set lengkap berkas hanya dengan satu perintah.

  • Pendahuluan

  • Generator Bawaan

    • make:cell

    • make:command

    • make:config

    • make:controller

    • make:entity

    • make:filter

    • make:model

    • make:seeder

    • make:test

    • make:migration

    • make:validation

  • Membuat Perancah (Scaffolding) Set Lengkap Kode Stok

  • GeneratorTrait

  • Mendeklarasikan Lokasi Template Perintah Generator Kustom

Pendahuluan

Semua generator bawaan berada di bawah grup Generators saat dicantumkan menggunakan php spark list. Untuk melihat deskripsi lengkap dan informasi penggunaan pada generator tertentu, gunakan perintah:

php spark help <generator_command>

di mana <generator_command> akan diganti dengan perintah yang akan diperiksa.

Catatan

Apakah Anda perlu memiliki kode yang dihasilkan dalam subfolder? Katakanlah jika Anda ingin membuat class controller untuk berada di subfolder Admin dari folder Controllers utama, Anda hanya perlu menambahkan subfolder ke nama class, seperti ini: php spark make:controller admin/login. Perintah ini akan membuat controller Login di subfolder Controllers/Admin dengan namespace App\Controllers\Admin.

Catatan

Bekerja dengan modul? Pembuatan kode akan mengatur namespace root ke default APP_NAMESPACE. Jika Anda perlu memiliki kode yang dihasilkan di tempat lain dalam namespace modul Anda, pastikan untuk mengatur opsi --namespace dalam perintah Anda, mis., php spark make:model blog --namespace Acme\\Blog.

Peringatan

Pastikan saat mengatur opsi --namespace bahwa namespace yang diberikan adalah namespace yang valid yang didefinisikan dalam array $psr4 Anda di Config\Autoload atau didefinisikan dalam berkas autoload composer Anda. Jika tidak, pembuatan kode akan terganggu.

Generator Bawaan

CodeIgniter4 dilengkapi dengan generator berikut secara default.

make:cell

Baru pada versi 4.3.0.

Membuat berkas Cell baru dan view-nya.

Penggunaan:

make:cell <nama> [opsi]

Argumen:

  • name: Nama class cell. Seharusnya dalam PascalCase. [WAJIB]

Opsi:

  • --namespace: Mengatur namespace root. Default ke nilai APP_NAMESPACE.

  • --force: Mengatur flag ini untuk menimpa berkas yang ada di tujuan.

make:command

Membuat perintah spark baru.

Penggunaan:

make:command <nama> [opsi]

Argumen:

  • name: Nama class perintah. [WAJIB]

Opsi:

  • --command: Nama perintah untuk dijalankan di spark. Default ke command:name.

  • --group: Grup/namespace dari perintah. Default ke App untuk perintah dasar, dan Generators untuk perintah generator.

  • --type: Jenis perintah, apakah perintah basic atau perintah generator. Default ke basic.

  • --namespace: Mengatur namespace root. Default ke nilai APP_NAMESPACE.

  • --suffix: Menambahkan suffix komponen ke nama class yang dihasilkan.

  • --force: Mengatur flag ini untuk menimpa berkas yang ada di tujuan.

make:config

Membuat berkas config baru.

Penggunaan:

make:config <nama> [opsi]

Argumen:

  • name: Nama class config. [WAJIB]

Opsi:

  • --namespace: Mengatur namespace root. Default ke nilai APP_NAMESPACE.

  • --suffix: Menambahkan suffix komponen ke nama class yang dihasilkan.

  • --force: Mengatur flag ini untuk menimpa berkas yang ada di tujuan.

make:controller

Membuat berkas controller baru.

Penggunaan:

make:controller <nama> [opsi]

Argumen:

  • name: Nama class controller. [WAJIB]

Opsi:

  • --bare: Mengextends dari CodeIgniter\Controller alih-alih BaseController.

  • --restful: Mengextends dari sumber daya RESTful. Pilihan adalah controller dan presenter. Default ke controller.

  • --namespace: Mengatur namespace root. Default ke nilai APP_NAMESPACE.

  • --suffix: Menambahkan suffix komponen ke nama class yang dihasilkan.

  • --force: Mengatur flag ini untuk menimpa berkas yang ada di tujuan.

Catatan

Jika Anda menggunakan --suffix, nama controller yang dihasilkan akan seperti ProductController. Itu melanggar konvensi penamaan Controller saat menggunakan Auto Routing (nama class Controller HARUS dimulai dengan huruf kapital dan HANYA karakter pertama yang bisa huruf kapital). Jadi --suffix dapat digunakan ketika Anda menggunakan Defined Routes.

Membuat Perancah (Scaffolding) Set Lengkap Kode Stok

Terkadang dalam fase pengembangan kita membuat fungsionalitas berdasarkan grup, seperti membuat grup Admin. Grup ini akan berisi berkas controller, model, migrasi, atau bahkan entity-nya sendiri. Anda mungkin tergoda untuk mengetik setiap perintah generator satu per satu di terminal dan berharap akan sangat bagus untuk memiliki satu perintah generator untuk menguasai semuanya.

Jangan khawatir lagi! CodeIgniter4 juga dilengkapi dengan perintah make:scaffold khusus yang pada dasarnya adalah pembungkus untuk perintah generator controller, model, entity, migrasi, dan seeder. Yang Anda butuhkan hanyalah nama class yang akan digunakan untuk menamai semua class yang dihasilkan. Juga, opsi individual yang didukung oleh setiap perintah generator dikenali oleh perintah scaffold.

Menjalankan ini di terminal Anda:

php spark make:scaffold user

akan membuat berkas-berkas berikut:

  1. app/Controllers/User.php

  2. app/Models/User.php

  3. app/Database/Migrations/<tanggal di sini>_User.php dan

  4. app/Database/Seeds/User.php

Untuk menyertakan class Entity dalam berkas yang di-scaffold, cukup sertakan --return entity ke perintah dan itu akan diteruskan ke generator model.

GeneratorTrait

Semua perintah generator harus menggunakan GeneratorTrait untuk sepenuhnya memanfaatkan metodenya yang digunakan dalam pembuatan kode.

Mendeklarasikan Lokasi Template Perintah Generator Kustom

Urutan pencarian default untuk template generator adalah (1) template yang didefinisikan dalam berkas app/Config/Generators.php, dan (2) jika tidak ditemukan, template yang ditemukan di namespace CodeIgniter\Commands\Generators\Views.

Untuk mendeklarasikan lokasi template untuk perintah generator kustom Anda, Anda perlu menambahkannya ke berkas app/Config/Generators.php. Misalnya, jika Anda memiliki perintah make:awesome-command dan template generator Anda terletak di dalam direktori app Anda app/Commands/Generators/Views/awesomecommand.tpl.php, Anda akan memperbarui berkas config seperti ini:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class Generators extends BaseConfig
{
    public array $views = [
        // ..
        'make:awesome-command' => 'App\Commands\Generators\Views\awesomecommand.tpl.php',
    ];
}
