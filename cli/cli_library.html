Library CLI

Library CLI CodeIgniter membuat pembuatan skrip baris perintah interaktif menjadi sederhana, termasuk:

  • Meminta informasi lebih lanjut dari pengguna

  • Menulis teks multi-warna ke terminal

  • Membunyikan suara (beep)

  • Menampilkan bilah kemajuan selama tugas yang panjang

  • Membungkus baris teks panjang agar sesuai dengan jendela.

Menginisialisasi class

Anda tidak perlu membuat instance dari library CLI, karena semua metodenya bersifat statis. Sebaliknya, Anda hanya perlu memastikan controller Anda dapat menemukannya melalui pernyataan use di atas class Anda:

<?php

namespace App\Commands;

use CodeIgniter\CLI\BaseCommand;
use CodeIgniter\CLI\CLI;

class MyCommand extends BaseCommand
{
    // ...

    public function run(array $params)
    {
        // ...
    }
}

class secara otomatis diinisialisasi saat berkas dimuat pertama kali.

Mendapatkan Input dari Pengguna

Terkadang Anda perlu meminta informasi lebih lanjut dari pengguna. Mereka mungkin tidak memberikan argumen baris perintah opsional, atau skrip mungkin menemukan berkas yang sudah ada dan memerlukan konfirmasi sebelum menimpa. Ini ditangani dengan metode prompt() atau promptByKey().

Catatan

Sejak v4.3.0, Anda dapat menulis tes untuk metode ini dengan PhpStreamWrapper. Lihat Testing CLI Input.

prompt()

Anda dapat memberikan pertanyaan dengan meneruskannya sebagai parameter pertama:

<?php

use CodeIgniter\CLI\CLI;

$color = CLI::prompt('What is your favorite color?');

Anda dapat memberikan jawaban default yang akan digunakan jika pengguna hanya menekan enter dengan meneruskan default di parameter kedua:

<?php

use CodeIgniter\CLI\CLI;

$color = CLI::prompt('What is your favorite color?', 'blue');

Anda dapat membatasi jawaban yang dapat diterima dengan meneruskan array jawaban yang diizinkan sebagai parameter kedua:

<?php

use CodeIgniter\CLI\CLI;

$overwrite = CLI::prompt('File exists. Overwrite?', ['y', 'n']);

Terakhir, Anda dapat meneruskan aturan validation ke input jawaban sebagai parameter ketiga:

<?php

use CodeIgniter\CLI\CLI;

$email = CLI::prompt('What is your email?', null, 'required|valid_email');

Aturan validation juga dapat ditulis dalam sintaks array:

<?php

use CodeIgniter\CLI\CLI;

$email = CLI::prompt('What is your email?', null, ['required', 'valid_email']);

promptByKey()

Jawaban yang telah ditentukan (opsi) untuk prompt terkadang perlu dijelaskan atau terlalu kompleks untuk dipilih melalui nilainya. promptByKey() memungkinkan pengguna untuk memilih opsi berdasarkan kuncinya alih-alih nilainya:

<?php

use CodeIgniter\CLI\CLI;

$fruit = CLI::promptByKey('These are your choices:', ['The red apple', 'The plump orange', 'The ripe banana']);
/*
 * These are your choices:
 *   [0]  The red apple
 *   [1]  The plump orange
 *   [2]  The ripe banana
 *
 * [0, 1, 2]:
 */

Kunci bernama juga dimungkinkan:

<?php

use CodeIgniter\CLI\CLI;

$fruit = CLI::promptByKey(['These are your choices:', 'Which would you like?'], [
    'apple'  => 'The red apple',
    'orange' => 'The plump orange',
    'banana' => 'The ripe banana',
]);
/*
 * These are your choices:
 *   [apple]   The red apple
 *   [orange]  The plump orange
 *   [banana]  The ripe banana
 *
 * Which would you like? [apple, orange, banana]:
 */

Terakhir, Anda dapat meneruskan aturan validation ke input jawaban sebagai parameter ketiga, jawaban yang dapat diterima secara otomatis dibatasi pada opsi yang diteruskan.

promptByMultipleKeys()

Baru pada versi 4.3.0.

Metode ini sama dengan promptByKey(), tetapi mendukung banyak nilai.

<?php

use CodeIgniter\CLI\CLI;

$hobbies = CLI::promptByMultipleKeys('Select your hobbies:', ['Playing game', 'Sleep', 'Badminton']);
/*
 * Select your hobbies:
 *   [0]  Playing game
 *   [1]  Sleep
 *   [2]  Badminton
 *
 * You can specify multiple values separated by commas.
 * [0, 1, 2]:
 *
 * if your answer is '0,2', the return is the key and the value of the options :
 * [
 *   [0] => "Playing game",
 *   [2] => "Badminton"
 * ]
 */

Penting

Metode promptByMultipleKeys(), tidak seperti promptByKey(), tidak mendukung kunci bernama atau validation.

Memberikan Umpan Balik

write()

Beberapa metode disediakan agar Anda dapat memberikan umpan balik kepada pengguna Anda. Ini bisa sesederhana pembaruan status tunggal atau tabel informasi kompleks yang membungkus ke jendela terminal pengguna. Inti dari ini adalah metode write() yang mengambil string untuk output sebagai parameter pertama:

<?php

use CodeIgniter\CLI\CLI;

CLI::write('The rain in Spain falls mainly on the plains.');

Anda dapat mengubah warna teks dengan meneruskan nama warna sebagai parameter kedua:

<?php

use CodeIgniter\CLI\CLI;

CLI::write('File created.', 'green');

Ini dapat digunakan untuk membedakan pesan berdasarkan status, atau membuat 'header' dengan menggunakan warna yang berbeda. Anda bisa bahkan mengatur warna latar belakang dengan meneruskan nama warna sebagai parameter ketiga:

<?php

use CodeIgniter\CLI\CLI;

CLI::write('File overwritten.', 'light_red', 'dark_gray');

Warna foreground berikut tersedia:

  • black

  • dark_gray

  • blue

  • dark_blue

  • light_blue

  • green

  • light_green

  • cyan

  • light_cyan

  • red

  • light_red

  • purple

  • light_purple

  • light_yellow

  • yellow

  • light_gray

  • white

Dan sejumlah kecil tersedia sebagai warna latar belakang:

  • black

  • blue

  • green

  • cyan

  • red

  • yellow

  • light_gray

  • magenta

print()

print berfungsi identik dengan metode write(), kecuali bahwa itu tidak memaksa baris baru baik sebelum atau sesudah. Sebaliknya, ia mencetaknya ke layar di mana pun kursor berada saat ini. Ini memungkinkan Anda untuk mencetak beberapa item semua di baris yang sama, dari panggilan yang berbeda. Ini sangat membantu ketika Anda ingin menampilkan status, melakukan sesuatu, lalu mencetak "Selesai" pada baris yang sama:

<?php

use CodeIgniter\CLI\CLI;

for ($i = 0; $i <= 10; $i++) {
    CLI::print($i);
}

color()

Sementara perintah write() akan menulis satu baris ke terminal, mengakhirinya dengan karakter EOL, Anda dapat menggunakan metode color() untuk membuat fragmen string yang dapat digunakan dengan cara yang sama, kecuali bahwa itu tidak akan memaksa EOL setelah mencetak. Ini memungkinkan Anda untuk membuat beberapa output pada baris yang sama. Atau, lebih umum, Anda dapat menggunakan itu di dalam metode write() untuk membuat string dengan warna berbeda di dalamnya:

<?php

use CodeIgniter\CLI\CLI;

CLI::write("fileA \t" . CLI::color('/path/to/file', 'white'), 'yellow');

Contoh ini akan menulis satu baris ke jendela, dengan fileA dalam warna kuning, diikuti oleh tab, dan kemudian /path/to/file dalam teks putih.

error()

Jika Anda perlu menampilkan error, Anda harus menggunakan metode error() yang diberi nama dengan tepat. Ini menulis teks merah muda ke STDERR, bukan STDOUT, seperti yang dilakukan write() dan color(). Ini dapat berguna jika Anda memiliki skrip yang memantau error sehingga mereka tidak perlu menyaring semua informasi, hanya pesan error yang sebenarnya. Anda menggunakannya persis seperti Anda akan menggunakan metode write():

<?php

use CodeIgniter\CLI\CLI;

CLI::error('Cannot write to file: ' . $file);

wrap()

Perintah ini akan mengambil string, mulai mencetaknya pada baris saat ini, dan membungkusnya ke panjang yang ditentukan pada baris baru. Ini mungkin berguna saat menampilkan daftar opsi dengan deskripsi yang ingin Anda bungkus di jendela saat ini dan tidak keluar dari layar:

<?php

use CodeIgniter\CLI\CLI;

CLI::color("task1\t", 'yellow');
CLI::wrap('Some long description goes here that might be longer than the current window.');

Secara default, string akan membungkus pada lebar terminal. Windows saat ini tidak menyediakan cara untuk menentukan ukuran jendela, jadi kami default ke 80 karakter. Jika Anda ingin membatasi lebar ke sesuatu yang lebih pendek yang Anda cukup yakin sesuai dengan jendela, teruskan panjang baris maksimum sebagai parameter kedua. Ini akan memecah string pada batas kata terdekat sehingga kata-kata tidak terputus.

<?php

use CodeIgniter\CLI\CLI;

// Wrap the text at max 20 characters wide
CLI::wrap($description, 20);

Anda mungkin menemukan bahwa Anda menginginkan kolom di sebelah kiri judul, berkas, atau tugas, sementara Anda menginginkan kolom teks di sebelah kanan dengan deskripsinya. Secara default, ini akan membungkus kembali ke tepi kiri jendela, yang tidak memungkinkan hal-hal untuk berbaris dalam kolom. Dalam kasus seperti ini, Anda dapat meneruskan sejumlah spasi untuk mengisi setiap baris setelah baris pertama, sehingga Anda akan memiliki tepi kolom yang rapi di sebelah kiri:

<?php

use CodeIgniter\CLI\CLI;

$titles = [
    'task1a',
    'task1abc',
];
$descriptions = [
    'Lorem Ipsum is simply dummy text of the printing and typesetting industry.',
    "Lorem Ipsum has been the industry's standard dummy text ever since the",
];

// Determine the maximum length of all titles
// to determine the width of the left column
$maxlen = max(array_map('strlen', $titles));

for ($i = 0; $i < count($titles); $i++) {
    CLI::write(
        // Display the title on the left of the row
        substr(
            $titles[$i] . str_repeat(' ', $maxlen + 3),
            0,
            $maxlen + 3,
        ) .
        // Wrap the descriptions in a right-hand column
        // with its left side 3 characters wider than
        // the longest item on the left.
        CLI::wrap($descriptions[$i], 40, $maxlen + 3),
    );
}

Akan membuat sesuatu seperti ini:

task1a     Lorem Ipsum is simply dummy
           text of the printing and
           typesetting industry.
task1abc   Lorem Ipsum has been the
           industry's standard dummy
           text ever since the

newLine()

Metode newLine() menampilkan baris kosong kepada pengguna. Ini tidak mengambil parameter apa pun:

<?php

use CodeIgniter\CLI\CLI;

CLI::newLine();

clearScreen()

Anda dapat membersihkan jendela terminal saat ini dengan metode clearScreen(). Di sebagian besar versi Windows, ini akan cukup menyisipkan 40 baris kosong karena Windows tidak mendukung fitur ini. Integrasi bash Windows 10 seharusnya mengubah ini:

<?php

use CodeIgniter\CLI\CLI;

CLI::clearScreen();

showProgress()

Jika Anda memiliki tugas yang berjalan lama yang ingin Anda terus perbarui kemajuannya kepada pengguna, Anda dapat menggunakan metode showProgress() yang menampilkan sesuatu seperti berikut:

[####......] 40% Selesai

Blok ini dianimasikan di tempat untuk efek yang sangat bagus.

Untuk menggunakannya, teruskan langkah saat ini sebagai parameter pertama, dan jumlah total langkah sebagai parameter kedua. Persentase selesai dan panjang tampilan akan ditentukan berdasarkan angka tersebut. Ketika Anda selesai, teruskan false sebagai parameter pertama dan bilah kemajuan akan dihapus.

<?php

use CodeIgniter\CLI\CLI;

$totalSteps = count($tasks);
$currStep   = 1;

foreach ($tasks as $task) {
    CLI::showProgress($currStep++, $totalSteps);
    $task->run();
}

// Done, so erase it...
CLI::showProgress(false);

table()

<?php

use CodeIgniter\CLI\CLI;

$thead = ['ID', 'Title', 'Updated At', 'Active'];
$tbody = [
    [7, 'A great item title', '2017-11-15 10:35:02', 1],
    [8, 'Another great item title', '2017-11-16 13:46:54', 0],
];

CLI::table($tbody, $thead);

+----+--------------------------+---------------------+--------+
| ID | Judul                    | Diperbarui Pada     | Aktif  |
+----+--------------------------+---------------------+--------+
| 7  | Judul item yang bagus    | 2017-11-16 10:35:02 | 1      |
| 8  | Judul item bagus lainnya | 2017-11-16 13:46:54 | 0      |
+----+--------------------------+---------------------+--------+

wait()

Menunggu sejumlah detik, secara opsional menampilkan pesan tunggu dan menunggu penekanan tombol.

<?php

use CodeIgniter\CLI\CLI;

// wait for specified interval, with countdown displayed
CLI::wait($seconds, true);

// show continuation message and wait for input
CLI::wait(0, false);

// wait for specified interval
CLI::wait($seconds, false);
