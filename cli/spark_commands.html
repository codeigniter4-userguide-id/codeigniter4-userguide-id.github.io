Menjalankan Command (Perintah)

Menjalankan via CLI

Perintah dijalankan dari baris perintah, di direktori root proyek. Berkas perintah spark telah disediakan yang digunakan untuk menjalankan perintah CLI apa pun.

Menampilkan Daftar Perintah

Ketika memanggil spark tanpa menentukan perintah, halaman bantuan sederhana akan ditampilkan yang juga menyediakan daftar perintah yang tersedia dan deskripsinya, diurutkan berdasarkan kategori:

php spark
spark list

php spark sama persis dengan perintah list:

php spark list

Anda juga dapat menggunakan opsi --simple untuk mendapatkan daftar mentah semua perintah yang tersedia, diurutkan secara alfabetis:

php spark list --simple

Menampilkan Bantuan

Anda bisa mendapatkan bantuan tentang perintah CLI apa pun menggunakan perintah help sebagai berikut:

php spark help db:seed

Sejak v4.3.0, Anda juga dapat menggunakan opsi --help alih-alih perintah help:

php spark db:seed --help

Menjalankan Perintah

Anda harus meneruskan nama perintah sebagai argumen pertama untuk menjalankan perintah tersebut:

php spark migrate

Beberapa perintah memerlukan argumen tambahan, yang harus disediakan langsung setelah perintah, dipisahkan oleh spasi:

php spark db:seed DevUserSeeder

Untuk semua perintah yang disediakan CodeIgniter, jika Anda tidak memberikan argumen yang diperlukan, Anda akan diminta untuk informasi yang dibutuhkan agar berjalan dengan benar:

php spark make:controller

Controller class name :

Menekan Output Header

Ketika Anda menjalankan perintah, header dengan versi CodeIgniter dan waktu saat ini akan ditampilkan:

php spark env

CodeIgniter v4.3.5 Command Line Tool - Server Time: 2023-06-16 12:45:31 UTC+00:00

Lingkungan Anda saat ini diatur sebagai development.

Anda selalu dapat menambahkan --no-header untuk menghilangkan output header:

php spark env --no-header

Lingkungan Anda saat ini diatur sebagai development.

Memanggil Perintah

Perintah juga dapat dijalankan dari dalam kode Anda sendiri. Ini paling sering dilakukan dalam sebuah controller untuk tugas cronjob, tetapi dapat digunakan kapan saja. Anda melakukannya dengan menggunakan fungsi command(). Fungsi ini selalu tersedia.

<?php

echo command('make:migration TestMigration');

Satu-satunya argumen adalah string yaitu perintah yang dipanggil dan parameter apa pun. Ini muncul persis seperti yang akan Anda panggil dari baris perintah.

Semua output dari perintah yang dijalankan ditangkap ketika tidak dijalankan dari baris perintah. Ini dikembalikan dari perintah sehingga Anda dapat memilih untuk menampilkannya atau tidak.
