Autoloading File

Setiap aplikasi terdiri dari sejumlah besar class di banyak lokasi yang berbeda. Framework menyediakan class untuk fungsionalitas inti. Aplikasi Anda akan memiliki sejumlah library, model, dan entitas lain untuk membuatnya berfungsi. Anda mungkin memiliki class pihak ketiga yang digunakan proyek Anda. Melacak di mana setiap file berada, dan mengkodekan lokasi tersebut ke dalam file Anda dalam serangkaian requires() adalah sakit kepala yang besar dan sangat rawan kesalahan. Di situlah autoloader berperan.

Autoloader CodeIgniter4

CodeIgniter menyediakan autoloader yang sangat fleksibel yang dapat digunakan dengan sedikit konfigurasi. Ini dapat menemukan class namespaced individual yang mematuhi struktur direktori autoloading PSR-4.

Autoloader bekerja dengan baik dengan sendirinya, tetapi juga dapat bekerja dengan autoloader lain, seperti Composer, atau bahkan autoloader kustom Anda sendiri, jika diperlukan. Karena semuanya terdaftar melalui spl_autoload_register, mereka bekerja secara berurutan dan tidak saling mengganggu.

Autoloader selalu aktif, didaftarkan dengan spl_autoload_register() di awal eksekusi framework.

Penting

Anda harus selalu berhati-hati tentang huruf besar/kecil nama file. Banyak pengembang mengembangkan di sistem file yang tidak peka huruf besar/kecil di Windows atau macOS. Namun, sebagian besar lingkungan server menggunakan sistem file yang peka huruf besar/kecil. Jika huruf besar/kecil nama file salah, autoloader tidak dapat menemukan file di server.

Konfigurasi

Konfigurasi awal dilakukan di app/Config/Autoload.php. File ini berisi dua array utama: satu untuk classmap, dan satu untuk namespace yang kompatibel dengan PSR-4.

Namespaces

Metode yang direkomendasikan untuk mengatur class Anda adalah dengan membuat satu atau lebih namespace untuk file aplikasi Anda.

Array $psr4 dalam file konfigurasi memungkinkan Anda untuk memetakan namespace ke direktori tempat class-class tersebut dapat ditemukan:

<?php

namespace Config;

use CodeIgniter\Config\AutoloadConfig;

class Autoload extends AutoloadConfig
{
    // ...
    public $psr4 = [
        APP_NAMESPACE => APPPATH,
    ];

    // ...
}

Kunci setiap baris adalah namespace itu sendiri. Ini tidak memerlukan garis miring terbalik di akhir. Nilainya adalah lokasi direktori tempat class-class tersebut dapat ditemukan.

Secara default, namespace App terletak di direktori app, dan namespace Config terletak di direktori app/Config.

Jika Anda membuat file class di lokasi dan sesuai dengan PSR-4, autoloader akan meng-autoload-nya.

Mengkonfirmasi Namespaces

Anda dapat memeriksa konfigurasi namespace dengan perintah spark namespaces:

php spark namespaces

Namespace Aplikasi

Secara default, direktori aplikasi di-namespace ke namespace App. Anda harus men-namespace controller, library, atau model di direktori aplikasi, dan mereka akan ditemukan di bawah namespace App.

Namespace Config

File Config di-namespace di namespace Config, bukan di App\Config seperti yang Anda harapkan. Ini memungkinkan file sistem inti untuk selalu dapat menemukannya, bahkan ketika namespace aplikasi telah berubah.

Catatan

Sejak v4.5.3 appstarter, namespace Config\\ telah ditambahkan ke autoload.psr-4 composer.json.

Mengubah Namespace Aplikasi

Anda dapat mengubah namespace ini dengan mengedit file app/Config/Constants.php dan mengatur nilai namespace baru di bawah pengaturan APP_NAMESPACE:

defined('APP_NAMESPACE') || define('APP_NAMESPACE', 'App');

Dan jika Anda menggunakan autoloader Composer, Anda juga perlu mengubah namespace App di composer.json Anda, dan jalankan composer dump-autoload.

{
    ...
    "autoload": {
        "psr-4": {
            "App\\": "app/"    <-- Ubah
        },
        ...
    },
    ...
}

Catatan

Sejak v4.5.0 appstarter, namespace App\\ telah ditambahkan ke autoload.psr-4 composer.json. Jika composer.json Anda tidak memilikinya, menambahkannya dapat meningkatkan kinerja autoloading aplikasi Anda.

Anda perlu memodifikasi file yang ada yang mereferensikan namespace saat ini.

Classmap

Jika Anda menggunakan library pihak ketiga yang bukan paket Composer dan tidak di-namespace, Anda dapat memuat class-class tersebut menggunakan classmap:

<?php

namespace Config;

use CodeIgniter\Config\AutoloadConfig;

class Autoload extends AutoloadConfig
{
    // ...
    public $classmap = [
        'Markdown' => APPPATH . 'ThirdParty/markdown.php',
    ];

    // ...
}

Kunci setiap baris adalah nama class yang ingin Anda temukan. Nilainya adalah jalur untuk menemukannya.

Dukungan Composer

Dukungan Composer secara otomatis diinisialisasi secara default.

Secara default, ia mencari file autoload Composer di ROOTPATH . 'vendor/autoload.php'. Jika Anda perlu mengubah lokasi file tersebut karena alasan apa pun, Anda dapat memodifikasi nilai yang didefinisikan di app/Config/Constants.php.

Prioritas Autoloader

Jika namespace yang sama didefinisikan di CodeIgniter dan Composer, autoloader Composer akan menjadi yang pertama mendapatkan kesempatan untuk menemukan file.

Catatan

Sebelum v4.5.0, jika namespace yang sama didefinisikan di CodeIgniter dan Composer, autoloader CodeIgniter adalah yang pertama mendapatkan kesempatan untuk menemukan file.

Caching FileLocator

Baru pada versi 4.5.0.

FileLocator bertanggung jawab untuk menemukan file atau mendapatkan nama class dari file, yang tidak dapat dicapai dengan autoloading PHP.

Untuk meningkatkan kinerjanya, Caching FileLocator telah diimplementasikan.

Bagaimana Cara Kerjanya

  • Simpan semua data yang ditemukan oleh FileLocator ke dalam file cache saat dihancurkan, jika data cache diperbarui.

  • Pulihkan data yang di-cache saat menginstansiasi jika data cache tersedia.

Data yang di-cache digunakan secara permanen.

Bagaimana Cara Menghapus Data yang Di-cache

Setelah disimpan, data yang di-cache tidak pernah kedaluwarsa.

Jadi jika Anda menambah atau menghapus file atau mengubah jalur file yang ada, atau namespace, data cache lama akan dikembalikan dan aplikasi Anda mungkin tidak berfungsi dengan benar.

Dalam kasus tersebut, Anda harus menghapus file cache secara manual. Jika Anda menambahkan paket CodeIgniter melalui Composer, Anda juga perlu menghapus file cache.

Anda dapat menggunakan perintah spark cache:clear:

php spark cache:clear

Atau cukup hapus file writable/cache/FileLocatorCache.

Catatan

Perintah spark optimize membersihkan cache.

Bagaimana Cara Mengaktifkan Caching FileLocator

Atur properti berikut ke true di app/Config/Optimize.php:

public bool $locatorCacheEnabled = true;

Atau Anda dapat mengaktifkannya dengan perintah spark optimize.

Catatan

Properti ini tidak dapat ditimpa oleh variabel environment.
