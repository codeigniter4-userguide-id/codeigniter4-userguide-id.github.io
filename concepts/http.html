Bekerja dengan Permintaan HTTP

Untuk mendapatkan hasil maksimal dari CodeIgniter, Anda perlu memiliki pemahaman dasar tentang bagaimana HTTP request dan HTTP response bekerja. Karena inilah yang Anda kerjakan saat mengembangkan aplikasi web, memahami konsep di balik HTTP adalah keharusan bagi semua pengembang yang ingin berhasil.

Bagian pertama dari bab ini memberikan gambaran umum. Setelah konsep-konsepnya jelas, kita akan membahas bagaimana bekerja dengan HTTP request dan HTTP response di dalam CodeIgniter.

    • Request (Permintaan)

    • Response

  • Bekerja dengan Request dan Response

Apa itu HTTP?

HTTP hanyalah konvensi berbasis teks yang memungkinkan dua mesin untuk saling berbicara. Ketika browser membuat request halaman, ia bertanya kepada server apakah ia bisa mendapatkan halaman tersebut. Server kemudian menyiapkan halaman dan mengirimkan response kembali ke browser yang memintanya. Itu saja. Jelas, ada beberapa kompleksitas yang bisa Anda gunakan, tetapi dasarnya sangat sederhana.

HTTP adalah istilah yang digunakan untuk menggambarkan konvensi pertukaran tersebut. Ini adalah singkatan dari HyperText Transfer Protocol. Tujuan Anda saat mengembangkan aplikasi web adalah untuk selalu memahami apa yang di-request browser, dan dapat merespons dengan tepat.

Request (Permintaan)

Setiap kali klien (browser web, aplikasi smartphone, dll.) membuat request, ia mengirimkan pesan teks kecil ke server dan menunggu response.

Request akan terlihat seperti ini:

GET / HTTP/1.1
Host codeigniter.com
Accept: text/html
User-Agent: Chrome/46.0.2490.80

Pesan ini menampilkan semua informasi yang diperlukan untuk mengetahui apa yang di-request klien. Ini memberitahu metode untuk request (GET, POST, DELETE, dll.), dan versi HTTP yang didukungnya.

Request juga menyertakan sejumlah header request opsional yang dapat berisi berbagai macam informasi seperti bahasa apa yang diinginkan klien untuk ditampilkan kontennya, jenis format yang diterima klien, dan banyak lagi. Wikipedia memiliki artikel yang mencantumkan semua bidang header jika Anda ingin melihatnya.

Response

Setelah server menerima request, aplikasi Anda akan mengambil informasi itu dan menghasilkan beberapa output. Server akan menggabungkan output Anda sebagai bagian dari response-nya kepada klien. Ini juga direpresentasikan sebagai pesan teks sederhana yang terlihat seperti ini:

HTTP/1.1 200 OK
Server: nginx/1.8.0
Date: Thu, 05 Nov 2015 05:33:22 GMT
Content-Type: text/html; charset=UTF-8

<html>
    . . .
</html>

Response memberitahu klien versi spesifikasi HTTP yang digunakannya dan, mungkin yang paling penting, kode status (200). Kode status adalah salah satu dari sejumlah kode yang telah distandarisasi untuk memiliki arti yang sangat spesifik bagi klien. Ini dapat memberitahu mereka bahwa itu berhasil (200), atau bahwa halaman tidak ditemukan (404). Kunjungi IANA untuk daftar lengkap kode status HTTP.

Bekerja dengan Request dan Response

Meskipun PHP menyediakan cara untuk berinteraksi dengan header request dan response, CodeIgniter, seperti kebanyakan framework, mengabstraksinya sehingga Anda memiliki antarmuka yang konsisten dan sederhana untuk mereka. Class IncomingRequest adalah representasi berorientasi objek dari HTTP request. Ini menyediakan semua yang Anda butuhkan:

<?php

use CodeIgniter\HTTP\IncomingRequest;

$request = request();

// the URI path being requested (i.e., /about)
$request->getUri()->getPath();

// Retrieve $_GET and $_POST variables
$request->getGet('foo');
$request->getPost('foo');

// Retrieve from $_REQUEST which should include
// both $_GET and $_POST contents
$request->getVar('foo');

// Retrieve JSON from AJAX calls
$request->getJSON();

// Retrieve server variables
$request->getServer('Host');

// Retrieve an HTTP Request header, with case-insensitive names
$request->header('host');
$request->header('Content-Type');

// Checks the HTTP method
$request->is('get');
$request->is('post');

Class request melakukan banyak pekerjaan di latar belakang untuk Anda, yang tidak perlu Anda khawatirkan. Metode isAJAX() dan isSecure() memeriksa beberapa metode berbeda untuk menentukan jawaban yang benar.

Catatan

Metode isAJAX() bergantung pada header X-Requested-With, yang dalam beberapa kasus tidak dikirim secara default dalam XHR request melalui JavaScript (yaitu, fetch). Lihat bagian AJAX Request tentang cara menghindari masalah ini.

CodeIgniter juga menyediakan class Response yang merupakan representasi berorientasi objek dari HTTP response. Ini memberi Anda cara yang mudah dan kuat untuk membangun response Anda kepada klien:

<?php

use CodeIgniter\HTTP\Response;

$response = response();

$response->setStatusCode(Response::HTTP_OK);
$response->setBody($output);
$response->setHeader('Content-Type', 'text/html');
$response->noCache();

// Sends the output to the browser
// This is typically handled by the framework
$response->send();

Selain itu, class Response memungkinkan Anda untuk bekerja dengan lapisan cache HTTP untuk kinerja terbaik.
