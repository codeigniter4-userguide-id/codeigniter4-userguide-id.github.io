Apa itu MVC?

Setiap kali Anda membuat aplikasi, Anda harus menemukan cara untuk mengatur kode agar mudah menemukan file yang tepat dan mudah untuk dipelihara. Seperti kebanyakan framework web, CodeIgniter menggunakan pola Model, View, Controller (MVC) untuk mengatur file. Ini menjaga data, presentasi, dan alur melalui aplikasi sebagai bagian yang terpisah.

Perlu dicatat bahwa ada banyak pandangan tentang peran pasti setiap elemen, tetapi dokumen ini menjelaskan pandangan kami tentang hal itu. Jika Anda memikirkannya secara berbeda, Anda bebas untuk memodifikasi cara Anda menggunakan setiap bagian sesuai kebutuhan Anda.

Model mengelola data aplikasi dan membantu menegakkan aturan bisnis khusus apa pun yang mungkin dibutuhkan aplikasi.

View adalah file sederhana, dengan sedikit atau tanpa logika, yang menampilkan informasi kepada pengguna.

Controller bertindak sebagai kode penghubung, mengatur data bolak-balik antara view (atau pengguna yang melihatnya) dan penyimpanan data.

Pada dasarnya, controller dan model hanyalah class yang memiliki pekerjaan spesifik. Mereka bukan satu-satunya jenis class yang dapat Anda gunakan, tentu saja, tetapi mereka membentuk inti dari bagaimana framework ini dirancang untuk digunakan. Mereka bahkan memiliki direktori yang ditunjuk di direktori app untuk penyimpanannya, meskipun Anda bebas untuk menyimpannya di mana pun Anda inginkan, selama mereka di-namespace dengan benar. Kita akan membahasnya lebih detail di bawah.

Mari kita lihat lebih dekat masing-masing dari tiga komponen utama ini.

Komponen-komponen

View

View adalah file paling sederhana dan biasanya HTML dengan sedikit PHP. PHP harus sangat sederhana, biasanya hanya menampilkan isi variabel, atau mengulang beberapa item dan menampilkan informasinya dalam tabel.

View mendapatkan data untuk ditampilkan dari controller, yang meneruskannya ke view sebagai variabel yang dapat ditampilkan dengan panggilan echo sederhana. Anda juga dapat menampilkan view lain di dalam view, membuatnya cukup sederhana untuk menampilkan header atau footer umum di setiap halaman.

View umumnya disimpan di app/Views, tetapi dapat dengan cepat menjadi tidak terkendali jika tidak diatur dalam beberapa cara. CodeIgniter tidak memberlakukan jenis organisasi apa pun, tetapi aturan praktis yang baik adalah membuat direktori baru di direktori Views untuk setiap controller. Kemudian, beri nama view berdasarkan nama metode. Ini membuatnya sangat mudah ditemukan nanti. Misalnya, profil pengguna mungkin ditampilkan di controller bernama User, dan metode bernama profile. Anda mungkin menyimpan file view untuk metode ini di app/Views/user/profile.php.

Jenis organisasi itu berfungsi dengan baik sebagai kebiasaan dasar. Terkadang Anda mungkin perlu mengaturnya secara berbeda. Itu tidak masalah. Selama CodeIgniter dapat menemukan file, itu dapat menampilkannya.

Cari tahu lebih lanjut tentang view

Model

Tugas model adalah memelihara satu jenis data untuk aplikasi. Ini mungkin pengguna, posting blog, transaksi, dll. Dalam kasus ini, tugas model memiliki dua bagian: menegakkan aturan bisnis pada data saat ditarik dari, atau dimasukkan ke dalam, database; dan menangani penyimpanan dan pengambilan data yang sebenarnya dari database.

Bagi banyak pengembang, kebingungan muncul ketika menentukan aturan bisnis apa yang diberlakukan. Ini berarti bahwa setiap pembatasan atau persyaratan pada data ditangani oleh model. Ini mungkin termasuk menormalkan data mentah sebelum disimpan untuk memenuhi standar perusahaan, atau memformat kolom dengan cara tertentu sebelum menyerahkannya ke controller. Dengan menjaga persyaratan bisnis ini di model, Anda tidak akan mengulang kode di beberapa controller dan secara tidak sengaja melewatkan pembaruan area.

Model biasanya disimpan di app/Models, meskipun mereka dapat menggunakan namespace untuk dikelompokkan sesuai kebutuhan Anda.

Cari tahu lebih lanjut tentang model

Controller

Controller memiliki beberapa peran yang berbeda. Yang paling jelas adalah bahwa mereka menerima input dari pengguna dan kemudian menentukan apa yang harus dilakukan dengannya. Ini sering melibatkan meneruskan data ke model untuk menyimpannya, atau meminta data dari model yang kemudian diteruskan ke view untuk ditampilkan. Ini juga termasuk memuat class utilitas lain, jika diperlukan, untuk menangani tugas-tugas khusus yang berada di luar lingkup model.

Tanggung jawab lain dari controller adalah menangani segala sesuatu yang berkaitan dengan permintaan HTTP - pengalihan, otentikasi, keamanan web, pengkodean, dll. Singkatnya, controller adalah tempat Anda memastikan bahwa orang diizinkan untuk berada di sana, dan mereka mendapatkan data yang mereka butuhkan dalam format yang dapat mereka gunakan.

Controller biasanya disimpan di app/Controllers, meskipun mereka dapat menggunakan namespace untuk dikelompokkan sesuai kebutuhan Anda.
