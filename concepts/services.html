Pendahuluan

Apa itu Service?

Service di CodeIgniter 4 menyediakan fungsionalitas untuk membuat dan berbagi instance class baru. Ini diimplementasikan sebagai class Config\Services.

Semua class inti dalam CodeIgniter disediakan sebagai "service". Ini berarti bahwa, alih-alih mengkodekan nama class untuk dimuat, class yang akan dipanggil didefinisikan dalam file konfigurasi yang sangat sederhana. File ini bertindak sebagai jenis pabrik untuk membuat instance baru dari class yang diperlukan.

Mengapa Menggunakan Service?

Contoh singkat mungkin akan membuat segalanya lebih jelas, jadi bayangkan Anda perlu menarik instance dari class Timer. Metode paling sederhana adalah dengan membuat instance baru dari class tersebut:

<?php

$timer = new \CodeIgniter\Debug\Timer();

Dan ini berfungsi dengan baik. Sampai Anda memutuskan bahwa Anda ingin menggunakan class timer yang berbeda sebagai gantinya. Mungkin yang ini memiliki pelaporan canggih yang tidak disediakan oleh timer default. Untuk melakukan ini, Anda sekarang harus menemukan semua lokasi di aplikasi Anda tempat Anda menggunakan class timer. Karena Anda mungkin meninggalkannya di tempat untuk menyimpan log kinerja aplikasi Anda yang terus-menerus berjalan, ini mungkin cara yang memakan waktu dan rawan kesalahan untuk menanganinya. Di situlah service berguna.

Alih-alih membuat instance sendiri, kita membiarkan class pusat membuat instance class untuk kita. Class ini dijaga sangat sederhana. Ini hanya berisi metode untuk setiap class yang ingin kita gunakan sebagai service. Metode ini biasanya mengembalikan instance bersama dari class tersebut, meneruskan dependensi apa pun yang mungkin dimilikinya. Kemudian, kita akan mengganti kode pembuatan timer kita dengan kode yang memanggil fungsi global ini atau class Services:

<?php

$timer = service('timer');

// The code above is the same as the code below.
$timer = \Config\Services::timer();

Ketika Anda perlu mengubah implementasi yang digunakan, Anda dapat memodifikasi file konfigurasi service, dan perubahan terjadi secara otomatis di seluruh aplikasi Anda tanpa Anda harus melakukan apa pun. Sekarang Anda hanya perlu memanfaatkan fungsionalitas baru apa pun dan Anda siap. Sangat sederhana dan tahan kesalahan.

Catatan

Disarankan untuk hanya membuat service di dalam controller. File lain, seperti model dan library harus memiliki dependensi yang diteruskan ke konstruktor atau melalui metode setter.

Bagaimana Cara Mendapatkan Service

Karena banyak class CodeIgniter disediakan sebagai service, Anda bisa mendapatkannya seperti berikut:

<?php

$timer = service('timer');

$timer adalah instance dari class Timer, dan jika Anda memanggil service('timer') lagi, Anda akan mendapatkan instance yang sama persis.

Service biasanya mengembalikan instance bersama dari class. Kode berikut membuat instance CURLRequest pada panggilan pertama. Dan panggilan kedua mengembalikan instance yang sama persis.

<?php

$options1 = [
    'baseURI' => 'http://example.com/api/v1/',
    'timeout' => 3,
];
$client1 = service('curlrequest', $options1);

$options2 = [
    'baseURI' => 'http://another.example.com/api/v2/',
    'timeout' => 10,
];
$client2 = service('curlrequest', $options2);
// $options2 does not work.
// $client2 is the exactly same instance as $client1.

Oleh karena itu, parameter $options2 untuk $client2 tidak berfungsi. Itu hanya diabaikan.

Mendapatkan Instance Baru

Jika Anda ingin mendapatkan instance baru dari class Timer, Anda perlu meneruskan false ke argumen $getShared:

<?php

$timer = \Config\Services::timer(false);

Fungsi Kemudahan

Dua fungsi telah disediakan untuk mendapatkan service. Fungsi-fungsi ini selalu tersedia.

service()

Yang pertama adalah service() yang mengembalikan instance baru dari service yang diminta. Satu-satunya parameter yang diperlukan adalah nama service. Ini sama dengan nama metode dalam file Services selalu mengembalikan instance SHARED dari class, jadi memanggil fungsi beberapa kali seharusnya selalu mengembalikan instance yang sama:

<?php

$logger = service('logger');

// The code above is the same as the code below.
$logger = \Config\Services::logger();

Catatan

Sejak v4.5.0, ketika Anda tidak meneruskan parameter ke service, fungsi global service() direkomendasikan karena peningkatan kinerja.

Jika metode pembuatan memerlukan parameter tambahan, mereka dapat diteruskan setelah nama service:

<?php

$renderer = service('renderer', APPPATH . 'views/');

// The code above is the same as the code below.
$renderer = \Config\Services::renderer(APPPATH . 'views/');

single_service()

Fungsi kedua, single_service() berfungsi persis seperti service() tetapi mengembalikan instance baru dari class:

<?php

$logger = single_service('logger');

// The code above is the same as the code below.
$logger = \Config\Services::logger(false);

Mendefinisikan Service

Agar service berfungsi dengan baik, Anda harus dapat mengandalkan setiap class yang memiliki API konstan, atau interface, untuk digunakan. Hampir semua class CodeIgniter menyediakan interface yang mereka patuhi. Ketika Anda ingin memperluas atau mengganti class inti, Anda hanya perlu memastikan Anda memenuhi persyaratan interface dan Anda tahu bahwa class tersebut kompatibel.

Misalnya, class RouteCollection mengimplementasikan RouteCollectionInterface. Ketika Anda ingin membuat pengganti yang menyediakan cara berbeda untuk membuat rute, Anda hanya perlu membuat class baru yang mengimplementasikan RouteCollectionInterface:

<?php

namespace App\Router;

use CodeIgniter\Router\RouteCollectionInterface;

class MyRouteCollection implements RouteCollectionInterface
{
    // Implement required methods here.
}

Terakhir, tambahkan metode routes() ke app/Config/Services.php untuk membuat instance baru dari MyRouteCollection alih-alih CodeIgniter\Router\RouteCollection:

<?php

namespace Config;

use CodeIgniter\Config\BaseService;

class Services extends BaseService
{
    // ...

    public static function routes()
    {
        return new \App\Router\MyRouteCollection(static::locator(), config('Modules'));
    }
}

Mengizinkan Parameter

Dalam beberapa kasus, Anda ingin memiliki opsi untuk meneruskan pengaturan ke class selama instansiasi. Karena file service adalah class yang sangat sederhana, mudah untuk membuat ini berfungsi.

Contoh yang baik adalah service renderer. Secara default, kita ingin class ini dapat menemukan view di APPPATH . 'views/'. Kita ingin pengembang memiliki opsi untuk mengubah jalur itu, jika kebutuhan mereka mengharuskannya. Jadi class menerima $viewPath sebagai parameter konstruktor. Metode service terlihat seperti ini:

<?php

namespace Config;

use CodeIgniter\Config\BaseService;

class Services extends BaseService
{
    // ...

    public static function renderer($viewPath = APPPATH . 'views/')
    {
        return new \CodeIgniter\View\View($viewPath);
    }
}

Ini menetapkan jalur default dalam metode konstruktor, tetapi memungkinkan untuk dengan mudah mengubah jalur yang digunakannya:

<?php

$renderer = \Config\Services::renderer('/shared/views/');

Class Bersama

Ada kalanya Anda perlu mensyaratkan bahwa hanya satu instance service yang dibuat. Ini mudah ditangani dengan metode getSharedInstance() yang dipanggil dari dalam metode pabrik. Ini menangani pemeriksaan apakah instance telah dibuat dan disimpan di dalam class, dan, jika tidak, membuat yang baru. Semua metode pabrik menyediakan nilai $getShared = true sebagai parameter terakhir. Anda harus tetap pada metode juga:

<?php

namespace Config;

use CodeIgniter\Config\BaseService;

class Services extends BaseService
{
    // ...

    public static function routes($getShared = true)
    {
        if ($getShared) {
            return static::getSharedInstance('routes');
        }

        return new \App\Router\MyRouteCollection(static::locator(), config('Modules'));
    }
}

Diskoveri Service

CodeIgniter dapat secara otomatis menemukan file Config/Services.php apa pun yang mungkin telah Anda buat dalam namespace yang ditentukan. Ini memungkinkan penggunaan sederhana dari file Service modul apa pun. Agar file Service kustom ditemukan, mereka harus memenuhi persyaratan ini:

  • Namespace-nya harus didefinisikan di app/Config/Autoload.php

  • Di dalam namespace, file harus ditemukan di Config/Services.php

  • Itu harus memperluas CodeIgniter\Config\BaseService

Contoh kecil akan memperjelas ini.

Bayangkan Anda telah membuat direktori baru, Blog di direktori root proyek Anda. Ini akan berisi modul Blog dengan controller, model, dll, dan Anda ingin membuat beberapa class tersedia sebagai service. Langkah pertama adalah membuat file baru: Blog/Config/Services.php. Kerangka file seharusnya:

<?php

namespace Blog\Config;

use CodeIgniter\Config\BaseService;

class Services extends BaseService
{
    public static function postManager()
    {
        // ...
    }
}

Sekarang Anda dapat menggunakan file ini seperti yang dijelaskan di atas. Ketika Anda ingin mengambil service posting dari controller mana pun, Anda cukup menggunakan class Config\Services dari framework untuk mengambil service Anda:

<?php

$postManager = service('postManager');

Catatan

Jika beberapa file Service memiliki nama metode yang sama, yang pertama ditemukan akan menjadi instance yang dikembalikan.

Mengatur Ulang Cache Service

Baru pada versi 4.6.0.

Ketika class Services pertama kali dipanggil cukup awal dalam proses inisialisasi framework, class Services yang ditemukan oleh penemuan otomatis di-cache dalam properti, dan itu tidak akan diperbarui.

Jika modul dimuat secara dinamis nanti, dan ada Service di modul, cache harus diperbarui.

Ini dapat dilakukan dengan menjalankan Config\Services::resetServicesCache(). Ini akan membersihkan cache, dan memaksa penemuan service lagi saat dibutuhkan.
