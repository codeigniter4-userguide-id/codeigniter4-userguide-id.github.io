Struktur Aplikasi

Untuk mendapatkan hasil maksimal dari CodeIgniter, Anda perlu memahami bagaimana aplikasi distrukturkan, secara default, dan apa yang dapat Anda ubah untuk memenuhi kebutuhan aplikasi Anda.

  • Direktori Default

    • app

    • system

    • public

    • writable

    • tests

  • Memodifikasi Lokasi Direktori

Direktori Default

Instalasi baru memiliki lima direktori:

  • app

  • public

  • writable

  • tests

  • vendor atau system

Setiap direktori ini memiliki peran yang sangat spesifik.

app

Direktori app adalah tempat semua kode aplikasi Anda berada. Ini dilengkapi dengan struktur direktori default yang berfungsi dengan baik untuk banyak aplikasi.

Folder-folder berikut membentuk konten dasar:

app/
    Config/         Menyimpan file konfigurasi
    Controllers/    Controller menentukan alur program
    Database/       Menyimpan file migrasi dan seed database
    Filters/        Menyimpan class filter yang dapat berjalan sebelum dan sesudah controller
    Helpers/        Helper menyimpan koleksi fungsi mandiri
    Language/       Dukungan multi-bahasa membaca string bahasa dari sini
    Libraries/      Class berguna yang tidak cocok di kategori lain
    Models/         Model bekerja dengan database untuk merepresentasikan entitas bisnis
    ThirdParty/     Library Pihak Ketiga yang dapat digunakan dalam aplikasi
    Views/          View membentuk HTML yang ditampilkan kepada klien

Karena direktori app sudah di-namespace, Anda bebas untuk memodifikasi struktur direktori ini agar sesuai dengan kebutuhan aplikasi Anda.

Misalnya, Anda mungkin memutuskan untuk mulai menggunakan pola Repository dan Entitas untuk bekerja dengan data Anda. Dalam kasus ini, Anda dapat mengganti nama direktori Models menjadi Repositories, dan menambahkan direktori Entities baru.

Semua file di direktori ini berada di bawah namespace App, meskipun Anda bebas untuk mengubahnya di app/Config/Constants.php.

system

Catatan

Jika Anda menginstal CodeIgniter dengan Composer, system terletak di vendor/codeigniter4/framework/system.

Direktori ini menyimpan file-file yang membentuk framework itu sendiri. Meskipun Anda memiliki banyak fleksibilitas dalam cara Anda menggunakan direktori aplikasi, file-file di direktori sistem tidak boleh dimodifikasi. Sebaliknya, Anda harus memperluas class, atau membuat class baru, untuk menyediakan fungsionalitas yang diinginkan.

Semua file di direktori ini berada di bawah namespace CodeIgniter.

public

Folder public berisi bagian aplikasi web Anda yang dapat diakses browser, mencegah akses langsung ke kode sumber Anda.

Ini berisi file .htaccess utama, index.php, dan aset aplikasi apa pun yang Anda tambahkan, seperti CSS, javascript, atau gambar.

Folder ini dimaksudkan sebagai "web root" situs Anda, dan server web Anda akan dikonfigurasi untuk menunjuk ke sana.

writable

Direktori ini menyimpan direktori apa pun yang mungkin perlu ditulis selama masa pakai aplikasi. Ini termasuk direktori untuk menyimpan file cache, log, dan unggahan apa pun yang mungkin dikirim pengguna.

Anda harus menambahkan direktori lain yang perlu ditulis oleh aplikasi Anda di sini. Ini memungkinkan Anda untuk menjaga direktori utama lainnya tidak dapat ditulis sebagai tindakan keamanan tambahan.

tests

Direktori ini diatur untuk menyimpan file pengujian Anda. Direktori _support menyimpan berbagai class mock dan utilitas lain yang dapat Anda gunakan saat menulis pengujian Anda.

Direktori ini tidak perlu ditransfer ke server produksi Anda.

Memodifikasi Lokasi Direktori

Jika Anda telah memindahkan salah satu direktori utama, Anda dapat mengubah pengaturan konfigurasi di dalam app/Config/Paths.php.

Silakan baca Mengelola Aplikasi Anda.
