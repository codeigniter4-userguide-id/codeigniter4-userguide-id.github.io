File Konfigurasi

CodeIgniter memiliki file konfigurasi yang memungkinkan Anda menyimpan nilai koneksi database Anda (username, password, nama database, dll.). File konfigurasi terletak di app/Config/Database.php. Anda juga dapat mengatur nilai koneksi database di file .env. Lihat di bawah untuk detail lebih lanjut.

Mengatur Database Default

Pengaturan konfigurasi disimpan dalam properti class yang merupakan array dengan prototipe ini:

<?php

namespace Config;

use CodeIgniter\Database\Config;

class Database extends Config
{
    // ...

    public array $default = [
        'DSN'      => '',
        'hostname' => 'localhost',
        'username' => 'root',
        'password' => '',
        'database' => 'database_name',
        'DBDriver' => 'MySQLi',
        'DBPrefix' => '',
        'pConnect' => false,
        'DBDebug'  => true,
        'charset'  => 'utf8mb4',
        'DBCollat' => 'utf8mb4_general_ci',
        'swapPre'  => '',
        'encrypt'  => false,
        'compress' => false,
        'strictOn' => false,
        'failover' => [],
        'port'     => 3306,
    ];

    // ...
}

Nama properti class adalah nama koneksi, dan dapat digunakan saat menghubungkan untuk menentukan nama grup.

Catatan

Lokasi default database SQLite3 adalah folder writable. Jika Anda ingin mengubah lokasi, Anda harus mengatur jalur lengkap ke folder baru (misalnya, 'database' => WRITEPATH . 'db/database_name.db').

DSN

Beberapa driver database (seperti Postgre, OCI8) memerlukan string DSN (Data Source Name) lengkap untuk terhubung. Tetapi jika Anda tidak menentukan string DSN untuk driver yang memerlukannya, CodeIgniter akan mencoba membangunnya dengan pengaturan lain yang disediakan.

Jika Anda menentukan DSN, Anda harus menggunakan pengaturan konfigurasi 'DSN', seolah-olah Anda menggunakan ekstensi PHP asli driver, seperti ini:

    // OCI8
    public array $default = [
        'DSN' => '//localhost/XE',
        // ...
    ];
DSN dalam Cara Universal

Anda juga dapat mengatur DSN secara universal (seperti URL). Dalam kasus tersebut, DSN harus memiliki prototipe ini:

    public array $default = [
        'DSN' => 'DBDriver://username:password@hostname:port/database',
        // ...
    ];

Untuk menimpa nilai konfigurasi default saat terhubung dengan versi universal dari string DSN, tambahkan variabel konfigurasi sebagai query string:

    // MySQLi
    public array $default = [
        'DSN' => 'MySQLi://username:password@hostname:3306/database?charset=utf8mb4&DBCollat=utf8mb4_general_ci',
        // ...
    ];

    // Postgre
    public array $default = [
        'DSN' => 'Postgre://username:password@hostname:5432/database?charset=utf8&connect_timeout=5&sslmode=require',
        // ...
    ];

Catatan

Jika Anda memberikan string DSN dan ada beberapa pengaturan yang valid yang hilang (misalnya, set karakter database), yang ada di field konfigurasi lainnya, CodeIgniter akan menambahkannya.

Failover

Anda juga dapat menentukan failover untuk situasi ketika koneksi utama tidak dapat terhubung karena alasan tertentu. Failover ini dapat ditentukan dengan mengatur failover untuk koneksi seperti ini:

<?php

namespace Config;

use CodeIgniter\Database\Config;

class Database extends Config
{
    // ...

    public array $default = [
        // ...
        'failover' => [
            [
                'hostname' => 'localhost1',
                'username' => '',
                'password' => '',
                'database' => '',
                'DBDriver' => 'MySQLi',
                'DBPrefix' => '',
                'pConnect' => true,
                'DBDebug'  => true,
                'charset'  => 'utf8mb4',
                'DBCollat' => 'utf8mb4_general_ci',
                'swapPre'  => '',
                'encrypt'  => false,
                'compress' => false,
                'strictOn' => false,
            ],
            [
                'hostname' => 'localhost2',
                'username' => '',
                'password' => '',
                'database' => '',
                'DBDriver' => 'MySQLi',
                'DBPrefix' => '',
                'pConnect' => true,
                'DBDebug'  => true,
                'charset'  => 'utf8mb4',
                'DBCollat' => 'utf8mb4_general_ci',
                'swapPre'  => '',
                'encrypt'  => false,
                'compress' => false,
                'strictOn' => false,
            ],
        ],
        // ...
    ];

    // ...
}

Anda dapat menentukan failover sebanyak yang Anda inginkan.

Mengatur Beberapa Database

Anda dapat secara opsional menyimpan beberapa set nilai koneksi. Jika, misalnya, Anda menjalankan beberapa environment (development, production, test, dll.) di bawah satu instalasi, Anda dapat menyiapkan grup koneksi untuk masing-masing, lalu beralih antar grup sesuai kebutuhan. Misalnya, untuk menyiapkan environment "test", Anda akan melakukan ini:

<?php

namespace Config;

use CodeIgniter\Database\Config;

class Database extends Config
{
    // ...

    public array $test = [
        'DSN'      => '',
        'hostname' => 'localhost',
        'username' => 'root',
        'password' => '',
        'database' => 'database_name',
        'DBDriver' => 'MySQLi',
        'DBPrefix' => '',
        'pConnect' => true,
        'DBDebug'  => true,
        'charset'  => 'utf8mb4',
        'DBCollat' => 'utf8mb4_general_ci',
        'swapPre'  => '',
        'compress' => false,
        'encrypt'  => false,
        'strictOn' => false,
        'failover' => [],
    ];

    // ...
}

Kemudian, untuk secara global memberi tahu sistem untuk menggunakan grup tersebut, Anda akan mengatur variabel ini yang terletak di file konfigurasi:

<?php

namespace Config;

use CodeIgniter\Database\Config;

class Database extends Config
{
    // ...

    public string $defaultGroup = 'test';

    // ...
}

Catatan

Nama test adalah arbitrer. Bisa apa saja yang Anda inginkan. Secara default kami telah menggunakan kata default untuk koneksi utama, tetapi itu juga dapat diganti namanya menjadi sesuatu yang lebih relevan dengan proyek Anda.

Mengubah Database Secara Otomatis

Anda dapat memodifikasi file konfigurasi untuk mendeteksi environment dan secara otomatis memperbarui nilai defaultGroup ke yang benar dengan menambahkan logika yang diperlukan dalam konstruktor class:

<?php

namespace Config;

use CodeIgniter\Database\Config;

/**
 * Database Configuration
 */
class Database extends Config
{
    // ...
    public $development = [/* ... */];
    public $test        = [/* ... */];
    public $production  = [/* ... */];

    public function __construct()
    {
        // ...

        $this->defaultGroup = ENVIRONMENT;
    }
}

Mengkonfigurasi dengan File .env

Anda juga dapat menyimpan nilai konfigurasi Anda dalam file .env dengan pengaturan database server saat ini. Anda hanya perlu memasukkan nilai yang berubah dari apa yang ada di pengaturan konfigurasi grup default. Nilai-nilai harus mengikuti format ini, di mana default adalah nama grup:

database.default.username = 'root';
database.default.password = '';
database.default.database = 'ci4';

Tetapi Anda tidak dapat menambahkan properti baru dengan mengatur variabel environment, maupun mengubah nilai skalar menjadi array. Lihat Environment Variables as Replacements for Data untuk detailnya.

Jadi jika Anda ingin menggunakan SSL dengan MySQL, Anda memerlukan hack. Misalnya, atur nilai array sebagai string JSON di file .env Anda:

database.default.encrypt = {"ssl_verify":true,"ssl_ca":"/var/www/html/BaltimoreCyberTrustRoot.crt.pem"}

dan dekode di konstruktor di Config class:

<?php

namespace Config;

use CodeIgniter\Database\Config;

/**
 * Database Configuration
 */
class Database extends Config
{
    // ...

    public function __construct()
    {
        // ...

        $array = json_decode($this->default['encrypt'], true);
        if (is_array($array)) {
            $this->default['encrypt'] = $array;
        }
    }
}

Deskripsi Nilai

Nama Konfigurasi

Deskripsi

DSN

String koneksi DSN (urutan konfigurasi all-in-one).

hostname

Hostname server database Anda. Seringkali ini adalah 'localhost'.

username

Username yang digunakan untuk terhubung ke database. (SQLite3 tidak menggunakannya.)

password

Password yang digunakan untuk terhubung ke database. (SQLite3 tidak menggunakannya.)

database

Nama database yang ingin Anda sambungkan.

Catatan

CodeIgniter tidak mendukung titik (.) dalam nama tabel dan kolom. Sejak v4.5.0, nama database dengan titik didukung.

DBDriver

Nama driver database. Huruf besar/kecil harus sesuai dengan nama driver. Anda dapat mengatur fully qualified classname untuk menggunakan driver kustom Anda. Driver yang didukung: MySQLi, Postgre, SQLite3, SQLSRV, dan OCI8.

DBPrefix

Prefiks tabel opsional yang akan ditambahkan ke nama tabel saat menjalankan kueri Query Builder. Ini memungkinkan beberapa instalasi CodeIgniter untuk berbagi satu database.

pConnect

true/false (boolean) - Apakah akan menggunakan koneksi persisten.

DBDebug

true/false (boolean) - Apakah akan memunculkan exception saat terjadi error database.

charset

Set karakter yang digunakan dalam berkomunikasi dengan database.

DBCollat

(hanya MySQLi) Kolasi karakter yang digunakan dalam berkomunikasi dengan database.

swapPre

Prefiks tabel default yang harus ditukar dengan DBPrefix. Ini berguna untuk aplikasi terdistribusi di mana Anda mungkin menjalankan kueri yang ditulis secara manual, dan membutuhkan prefiks agar tetap dapat disesuaikan oleh pengguna akhir.

schema

(hanya Postgre dan SQLSRV) Skema database, nilai default bervariasi berdasarkan driver.

encrypt

(hanya MySQLi dan SQLSRV) Apakah akan menggunakan koneksi terenkripsi. Lihat MySQLi encrypt untuk pengaturan MySQLi. Driver SQLSRV menerima true/false.

compress

(hanya MySQLi) Apakah akan menggunakan kompresi klien.

strictOn

(hanya MySQLi) true/false (boolean) - Apakah akan memaksa koneksi "Strict Mode", bagus untuk memastikan SQL yang ketat saat mengembangkan aplikasi.

port

Nomor port database - String kosong '' untuk port default (atau port dinamis dengan SQLSRV).

foreignKeys

(hanya SQLite3) true/false (boolean) - Apakah akan mengaktifkan batasan Foreign Key.

Penting

Batasan Foreign Key SQLite3 dinonaktifkan secara default. Lihat dokumentasi SQLite. Untuk memberlakukan batasan Foreign Key, atur item konfigurasi ini ke true.

busyTimeout

(hanya SQLite3) milidetik (int) - Tidur selama jumlah waktu yang ditentukan ketika tabel terkunci.

synchronous

(hanya SQLite3) flag (int) - Seberapa ketat SQLite akan melakukan flushing ke disk selama transaksi. Gunakan null untuk tetap dengan pengaturan default. Ini dapat digunakan sejak v4.6.0.

numberNative

(hanya MySQLi) true/false (boolean) - Apakah akan mengaktifkan MYSQLI_OPT_INT_AND_FLOAT_NATIVE.

foundRows

(hanya MySQLi) true/false (boolean) - Apakah akan mengaktifkan MYSQLI_CLIENT_FOUND_ROWS.

dateFormat

Format tanggal/waktu default sebagai format DateTime PHP. * date - format tanggal * datetime - format tanggal dan waktu * datetime-ms - format tanggal dan waktu dengan milidetik * datetime-us - format tanggal dan waktu dengan mikrodetik * time - format waktu Ini dapat digunakan sejak v4.5.0, dan Anda bisa mendapatkan nilainya, misalnya, $db->dateFormat['datetime']. Saat ini, driver database tidak menggunakan nilai-nilai ini secara langsung, tetapi Model menggunakannya.

Catatan

Tergantung pada driver database yang Anda gunakan (MySQLi, Postgre, dll.) tidak semua nilai akan dibutuhkan. Misalnya, saat menggunakan SQLite3, Anda tidak perlu menyediakan username atau password, dan nama database akan menjadi jalur ke file database Anda.

MySQLi

hostname

Mengkonfigurasi Koneksi Soket

Untuk terhubung ke server MySQL melalui soket sistem file, jalur ke soket harus ditentukan dalam pengaturan 'hostname'. Driver MySQLi CodeIgniter akan menyadari hal ini dan mengkonfigurasi koneksi dengan benar.

    // MySQLi over a socket
    public array $default = [
        // ...
        'hostname' => '/cloudsql/toolbox-tests:europe-north1:toolbox-db',
        // ...
        'DBDriver' => 'MySQLi',
        // ...
    ];

encrypt

Driver MySQLi menerima array dengan opsi-opsi berikut:

  • ssl_key - Jalur ke file kunci privat

  • ssl_cert - Jalur ke file sertifikat kunci publik

  • ssl_ca - Jalur ke file otoritas sertifikat

  • ssl_capath - Jalur ke direktori yang berisi sertifikat CA terpercaya dalam format PEM

  • ssl_cipher - Daftar cipher yang diizinkan untuk digunakan dalam enkripsi, dipisahkan oleh titik dua (:)

  • ssl_verify - true/false (boolean) - Apakah akan memverifikasi sertifikat server atau tidak
