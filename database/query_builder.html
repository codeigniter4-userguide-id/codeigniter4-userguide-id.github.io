Class Query Builder

CodeIgniter memberi Anda akses ke kelas Query Builder. Pola ini memungkinkan informasi diambil, dimasukkan, dan diperbarui di database Anda dengan skrip minimal. Dalam beberapa kasus, hanya satu atau dua baris kode yang diperlukan untuk melakukan tindakan database. CodeIgniter tidak mengharuskan setiap tabel database menjadi file kelasnya sendiri. Sebaliknya, ia menyediakan antarmuka yang lebih sederhana.

Selain kesederhanaan, manfaat utama menggunakan fitur Query Builder adalah memungkinkan Anda membuat aplikasi independen database, karena sintaks kueri dihasilkan oleh setiap adapter database. Ini juga memungkinkan kueri yang lebih aman, karena nilai-nilai di-escape secara otomatis oleh sistem.

Catatan

CodeIgniter tidak mendukung titik (.) dalam nama tabel dan kolom. Sejak v4.5.0, nama database dengan titik didukung.

Proteksi SQL Injection

Anda dapat membuat pernyataan SQL dengan cukup aman menggunakan Query Builder. Namun, itu tidak dirancang untuk mencegah SQL injection tidak peduli data apa yang Anda berikan.

Argumen yang diteruskan ke Query Builder dapat berupa:

  1. identifier seperti nama field (atau tabel)

  2. nilai mereka

  3. bagian dari string SQL

Query Builder akan meng-escape semua nilai secara default.

Ini juga akan mencoba melindungi identifier dan identifier dalam string SQL dengan benar secara default. Namun, ini diimplementasikan untuk bekerja dengan baik dalam banyak kasus penggunaan dan tidak dirancang untuk mencegah semua serangan. Oleh karena itu, Anda tidak boleh memasukkan input pengguna ke dalamnya tanpa validasi yang tepat.

Juga, banyak metode memiliki parameter $escape yang dapat diatur untuk menonaktifkan escaping. Jika $escape diatur ke false, tidak ada perlindungan yang diberikan oleh Query Builder, jadi Anda harus memastikan sendiri bahwa mereka di-escape atau dilindungi dengan benar sebelum meneruskannya ke Query Builder. Hal yang sama berlaku saat menggunakan RawSql, yang menentukan pernyataan SQL mentah.

Memuat Query Builder

Query Builder dimuat melalui metode table() pada koneksi database. Ini mengatur bagian FROM dari kueri untuk Anda dan mengembalikan instance baru dari kelas Query Builder:

<?php

$db      = \Config\Database::connect();
$builder = $db->table('users');

Query Builder hanya dimuat ke memori saat Anda secara khusus meminta kelas, jadi tidak ada sumber daya yang digunakan secara default.

Memilih Data

Metode berikut memungkinkan Anda membangun pernyataan SQL SELECT.

Get

$builder->get()

Menjalankan kueri pemilihan dan mengembalikan hasilnya. Dapat digunakan sendiri untuk mengambil semua record dari sebuah tabel:

<?php

$builder = $db->table('mytable');
$query   = $builder->get();  // Produces: SELECT * FROM mytable

Parameter pertama dan kedua memungkinkan Anda untuk mengatur klausa limit dan offset:

<?php

$query = $builder->get(10, 20);
/*
 * Executes: SELECT * FROM mytable LIMIT 20, 10
 * (in MySQL. Other databases have slightly different syntax)
 */

Anda akan melihat bahwa metode di atas ditetapkan ke variabel bernama $query, yang dapat digunakan untuk menampilkan hasilnya:

<?php

$query = $builder->get();

foreach ($query->getResult() as $row) {
    echo $row->title;
}

Silakan kunjungi metode getResult() untuk diskusi lengkap mengenai pembuatan hasil.

$builder->getCompiledSelect()

Mengkompilasi kueri pemilihan seperti $builder->get() tetapi tidak menjalankan kueri. Metode ini hanya mengembalikan kueri SQL sebagai string.

Contoh:

<?php

$sql = $builder->getCompiledSelect();
echo $sql;
// Prints string: SELECT * FROM mytable

Parameter (false) dalam kueri pertama di bawah ini memungkinkan Anda untuk mengatur apakah query builder akan direset atau tidak (karena nilai default parameter adalah true, getCompiledSelect(bool $reset = true), secara default akan direset seperti saat menggunakan $builder->get()):

<?php

echo $builder->limit(10, 20)->getCompiledSelect(false);
/*
 * Prints string: SELECT * FROM mytable LIMIT 20, 10
 * (in MySQL. Other databases have slightly different syntax)
 */

echo $builder->select('title, content, date')->getCompiledSelect();
// Prints string: SELECT title, content, date FROM mytable LIMIT 20, 10

Hal utama yang perlu diperhatikan dalam contoh di atas adalah bahwa kueri kedua tidak menggunakan limit(10, 20) tetapi kueri SQL yang dihasilkan memiliki LIMIT 20, 10. Alasan untuk hasil ini adalah karena parameter dalam kueri pertama diatur ke false, limit(10, 20) tetap ada di kueri kedua.

$builder->getWhere()

Identik dengan metode get() kecuali bahwa ia memungkinkan Anda untuk menambahkan klausa "where" di parameter pertama, alih-alih menggunakan metode $builder->where():

<?php

$query = $builder->getWhere(['id' => $id], $limit, $offset);

Silakan baca tentang metode where() di bawah ini untuk informasi lebih lanjut.

Select

$builder->select()

Memungkinkan Anda untuk menulis bagian SELECT dari kueri Anda:

<?php

$builder->select('title, content, date');
$query = $builder->get();
// Executes: SELECT title, content, date FROM mytable

Catatan

Jika Anda memilih semua (*) dari sebuah tabel, Anda tidak perlu menggunakan metode ini. Ketika dihilangkan, CodeIgniter mengasumsikan bahwa Anda ingin memilih semua field dan secara otomatis menambahkan SELECT *.

$builder->select() menerima parameter kedua opsional. Jika Anda mengaturnya ke false, CodeIgniter tidak akan mencoba melindungi nama field atau tabel Anda. Ini berguna jika Anda memerlukan pernyataan select gabungan di mana escaping otomatis field dapat merusaknya.

<?php

$builder->select('(SELECT SUM(payments.amount) FROM payments WHERE payments.invoice_id=4) AS amount_paid', false);
$query = $builder->get();
RawSql

Baru pada versi 4.2.0.

Sejak v4.2.0, $builder->select() menerima instance CodeIgniter\Database\RawSql, yang menyatakan string SQL mentah.

<?php

use CodeIgniter\Database\RawSql;

$sql = 'REGEXP_SUBSTR(ral_anno,"[0-9]{1,2}([,.][0-9]{1,3})([,.][0-9]{1,3})") AS ral';
$builder->select(new RawSql($sql));
$query = $builder->get();

Peringatan

Saat Anda menggunakan RawSql, Anda HARUS meng-escape nilai-nilai dan melindungi identifier secara manual. Kegagalan untuk melakukannya dapat mengakibatkan SQL injection.

$builder->selectMax()

Menulis bagian SELECT MAX(field) untuk kueri Anda. Anda dapat secara opsional menyertakan parameter kedua untuk mengganti nama field yang dihasilkan.

<?php

$builder->selectMax('age');
$query = $builder->get();
// Produces: SELECT MAX(age) as age FROM mytable

$builder->selectMax('age', 'member_age');
$query = $builder->get();
// Produces: SELECT MAX(age) as member_age FROM mytable

$builder->selectMin()

Menulis bagian SELECT MIN(field) untuk kueri Anda. Seperti selectMax(), Anda dapat secara opsional menyertakan parameter kedua untuk mengganti nama field yang dihasilkan.

<?php

$builder->selectMin('age');
$query = $builder->get();
// Produces: SELECT MIN(age) as age FROM mytable

$builder->selectAvg()

Menulis bagian SELECT AVG(field) untuk kueri Anda. Seperti selectMax(), Anda dapat secara opsional menyertakan parameter kedua untuk mengganti nama field yang dihasilkan.

<?php

$builder->selectAvg('age');
$query = $builder->get();
// Produces: SELECT AVG(age) as age FROM mytable

$builder->selectSum()

Menulis bagian SELECT SUM(field) untuk kueri Anda. Seperti selectMax(), Anda dapat secara opsional menyertakan parameter kedua untuk mengganti nama field yang dihasilkan.

<?php

$builder->selectSum('age');
$query = $builder->get();
// Produces: SELECT SUM(age) as age FROM mytable

$builder->selectCount()

Menulis bagian SELECT COUNT(field) untuk kueri Anda. Seperti selectMax(), Anda dapat secara opsional menyertakan parameter kedua untuk mengganti nama field yang dihasilkan.

Catatan

Metode ini sangat membantu ketika digunakan dengan groupBy(). Untuk menghitung hasil secara umum, lihat countAll() atau countAllResults().

<?php

$builder->selectCount('age');
$query = $builder->get();
// Produces: SELECT COUNT(age) as age FROM mytable

$builder->selectSubquery()

Menambahkan subkueri ke bagian SELECT.

$subquery = $db->table('countries')->select('name')->where('id', 1);
$builder  = $db->table('users')->select('name')->selectSubquery($subquery, 'country');
$query    = $builder->get();
// Produces: SELECT `name`, (SELECT `name` FROM `countries` WHERE `id` = 1) `country` FROM `users`

From

$builder->from()

Memungkinkan Anda untuk menulis bagian FROM dari kueri Anda:

<?php

$builder = $db->table('users');
$builder->select('title, content, date');
$builder->from('mytable');
$query = $builder->get();
// Produces: SELECT title, content, date FROM users, mytable

Catatan

Seperti yang ditunjukkan sebelumnya, bagian FROM dari kueri Anda dapat ditentukan dalam metode $db->table(). Panggilan tambahan ke from() akan menambahkan lebih banyak tabel ke bagian FROM dari kueri Anda.

Subqueries

$builder->fromSubquery()

Memungkinkan Anda untuk menulis bagian dari kueri FROM sebagai subkueri.

Di sinilah kita menambahkan subkueri ke tabel yang sudah ada:

<?php

$subquery = $db->table('users');
$builder  = $db->table('jobs')->fromSubquery($subquery, 'alias');
$query    = $builder->get();
// Produces: SELECT * FROM `jobs`, (SELECT * FROM `users`) `alias`

Gunakan metode $db->newQuery() untuk menjadikan subkueri sebagai tabel utama:

<?php

$subquery = $db->table('users')->select('id, name');
$builder  = $db->newQuery()->fromSubquery($subquery, 't');
$query    = $builder->get();
// Produces: SELECT * FROM (SELECT `id`, `name` FROM users) `t`

Join

$builder->join()

Memungkinkan Anda untuk menulis bagian JOIN dari kueri Anda:

<?php

$builder = $db->table('blogs');
$builder->select('*');
$builder->join('comments', 'comments.id = blogs.id');
$query = $builder->get();
/*
 * Produces:
 * SELECT * FROM blogs JOIN comments ON comments.id = blogs.id
 */

Beberapa panggilan metode dapat dilakukan jika Anda memerlukan beberapa join dalam satu kueri.

Jika Anda memerlukan jenis JOIN tertentu, Anda dapat menentukannya melalui parameter ketiga dari metode. Pilihannya adalah: left, right, outer, inner, left outer, dan right outer.

<?php

$builder->join('comments', 'comments.id = blogs.id', 'left');
// Produces: LEFT JOIN comments ON comments.id = blogs.id
RawSql

Baru pada versi 4.2.0.

Sejak v4.2.0, $builder->join() menerima instance CodeIgniter\Database\RawSql sebagai kondisi JOIN ON, yang menyatakan string SQL mentah.

<?php

use CodeIgniter\Database\RawSql;

$sql = 'user.id = device.user_id AND ((1=1 OR 1=1) OR (1=1 OR 1=1))';
$builder->join('user', new RawSql($sql), 'LEFT');
// Produces: LEFT JOIN "user" ON user.id = device.user_id AND ((1=1 OR 1=1) OR (1=1 OR 1=1))

Peringatan

Saat Anda menggunakan RawSql, Anda HARUS meng-escape nilai-nilai dan melindungi identifier secara manual. Kegagalan untuk melakukannya dapat mengakibatkan SQL injection.

Mencari Data Spesifik

Where

$builder->where()

Metode ini memungkinkan Anda untuk mengatur klausa WHERE menggunakan salah satu dari lima metode:

Catatan

Semua nilai yang diteruskan ke metode ini di-escape secara otomatis, menghasilkan kueri yang lebih aman, kecuali saat menggunakan string kustom.

Catatan

$builder->where() menerima parameter ketiga opsional. Jika Anda mengaturnya ke false, CodeIgniter tidak akan mencoba melindungi nama field atau tabel Anda.

1. Metode kunci/nilai sederhana
<?php

$builder->where('name', $name);
// Produces: WHERE name = 'Joe'

Perhatikan bahwa tanda sama dengan ditambahkan untuk Anda.

Jika Anda menggunakan beberapa panggilan metode, mereka akan dirangkai bersama dengan AND di antara mereka:

<?php

$builder->where('name', $name);
$builder->where('title', $title);
$builder->where('status', $status);
// WHERE name = 'Joe' AND title = 'boss' AND status = 'active'
2. Metode kunci/nilai kustom

Anda dapat menyertakan operator di parameter pertama untuk mengontrol perbandingan:

<?php

$builder->where('name !=', $name);
$builder->where('id <', $id);
// Produces: WHERE name != 'Joe' AND id < 45
3. Metode array asosiatif
<?php

$array = ['name' => $name, 'title' => $title, 'status' => $status];
$builder->where($array);
// Produces: WHERE name = 'Joe' AND title = 'boss' AND status = 'active'

Anda juga dapat menyertakan operator Anda sendiri menggunakan metode ini:

<?php

$array = ['name !=' => $name, 'id <' => $id, 'date >' => $date];
$builder->where($array);
4. String kustom

Anda dapat menulis klausa Anda sendiri secara manual:

<?php

$where = "name='Joe' AND status='boss' OR status='active'";
$builder->where($where);

Peringatan

Jika Anda menggunakan data yang disediakan pengguna dalam string, Anda HARUS meng-escape nilai-nilai dan melindungi identifier secara manual. Kegagalan untuk melakukannya dapat mengakibatkan SQL injection.

<?php

$name  = $builder->db->escape('Joe');
$where = "name={$name} AND status='boss' OR status='active'";
$builder->where($where);
5. RawSql

Baru pada versi 4.2.0.

Sejak v4.2.0, $builder->where() menerima instance CodeIgniter\Database\RawSql, yang menyatakan string SQL mentah.

<?php

use CodeIgniter\Database\RawSql;

$sql = "id > 2 AND name != 'Accountant'";
$builder->where(new RawSql($sql));

Peringatan

Saat Anda menggunakan RawSql, Anda HARUS meng-escape nilai-nilai dan melindungi identifier secara manual. Kegagalan untuk melakukannya dapat mengakibatkan SQL injection.

6. Subqueries
<?php

// With closure
use CodeIgniter\Database\BaseBuilder;

$builder->where('advance_amount <', static function (BaseBuilder $builder) {
    $builder->select('MAX(advance_amount)', false)->from('orders')->where('id >', 2);
});
// Produces: WHERE "advance_amount" < (SELECT MAX(advance_amount) FROM "orders" WHERE "id" > 2)

// With builder directly
$subQuery = $db->table('orders')->select('MAX(advance_amount)', false)->where('id >', 2);
$builder->where('advance_amount <', $subQuery);

$builder->orWhere()

Metode ini identik dengan yang di atas, kecuali bahwa beberapa instance digabungkan dengan OR:

<?php

$builder->where('name !=', $name);
$builder->orWhere('id >', $id);
// Produces: WHERE name != 'Joe' OR id > 50

$builder->whereIn()

Menghasilkan kueri SQL WHERE field IN ('item', 'item') yang digabungkan dengan AND jika sesuai:

<?php

$names = ['Frank', 'Todd', 'James'];
$builder->whereIn('username', $names);
// Produces: WHERE username IN ('Frank', 'Todd', 'James')

Anda dapat menggunakan subkueri alih-alih array nilai:

<?php

// With closure
use CodeIgniter\Database\BaseBuilder;

$builder->whereIn('id', static function (BaseBuilder $builder) {
    $builder->select('job_id')->from('users_jobs')->where('user_id', 3);
});
// Produces: WHERE "id" IN (SELECT "job_id" FROM "users_jobs" WHERE "user_id" = 3)

// With builder directly
$subQuery = $db->table('users_jobs')->select('job_id')->where('user_id', 3);
$builder->whereIn('id', $subQuery);

$builder->orWhereIn()

Menghasilkan kueri SQL WHERE field IN ('item', 'item') yang digabungkan dengan OR jika sesuai:

<?php

$names = ['Frank', 'Todd', 'James'];
$builder->orWhereIn('username', $names);
// Produces: OR username IN ('Frank', 'Todd', 'James')

Anda dapat menggunakan subkueri alih-alih array nilai:

<?php

// With closure
use CodeIgniter\Database\BaseBuilder;

$builder->orWhereIn('id', static function (BaseBuilder $builder) {
    $builder->select('job_id')->from('users_jobs')->where('user_id', 3);

// Produces: OR "id" IN (SELECT "job_id" FROM "users_jobs" WHERE "user_id" = 3)

// With builder directly
$subQuery = $db->table('users_jobs')->select('job_id')->where('user_id', 3);
$builder->orWhereIn('id', $subQuery);

$builder->whereNotIn()

Menghasilkan kueri SQL WHERE field NOT IN ('item', 'item') yang digabungkan dengan AND jika sesuai:

<?php

$names = ['Frank', 'Todd', 'James'];

$builder->orWhereNotIn()

Menghasilkan kueri SQL WHERE field NOT IN ('item', 'item') yang digabungkan dengan OR jika sesuai:

<?php

Mencari Data Serupa

Like

$builder->like()

Metode ini memungkinkan Anda untuk menghasilkan klausa LIKE, berguna untuk melakukan pencarian.

Catatan

Semua nilai yang diteruskan ke metode ini di-escape secara otomatis.

Catatan

Semua variasi metode like* dapat dipaksa untuk melakukan pencarian yang tidak peka huruf besar/kecil dengan meneruskan parameter kelima true ke metode. Ini akan menggunakan fitur khusus platform jika tersedia jika tidak, akan memaksa nilai menjadi huruf kecil, yaitu, WHERE LOWER(column) LIKE '%search%'. Ini mungkin memerlukan indeks untuk dibuat untuk LOWER(column) alih-alih column agar efektif.

1. Metode kunci/nilai sederhana
<?php

$builder->like('title', 'match');
// Produces: WHERE `title` LIKE '%match%' ESCAPE '!'

Jika Anda menggunakan beberapa panggilan metode, mereka akan dirangkai bersama dengan AND di antara mereka:

<?php

$builder->like('title', 'match');
$builder->like('body', 'match');
// WHERE `title` LIKE '%match%' ESCAPE '!' AND  `body` LIKE '%match%' ESCAPE '!'

Jika Anda ingin mengontrol di mana wildcard (%) ditempatkan, Anda dapat menggunakan argumen ketiga opsional. Pilihan Anda adalah before, after dan both (yang merupakan default).

<?php

$builder->like('title', 'match', 'before'); // Produces: WHERE `title` LIKE '%match' ESCAPE '!'
$builder->like('title', 'match', 'after');  // Produces: WHERE `title` LIKE 'match%' ESCAPE '!'
$builder->like('title', 'match', 'both');   // Produces: WHERE `title` LIKE '%match%' ESCAPE '!'
2. Metode array asosiatif
<?php

$array = ['title' => $match, 'page1' => $match, 'page2' => $match];

/*
 * WHERE `title` LIKE '%match%' ESCAPE '!'
 *     AND  `page1` LIKE '%match%' ESCAPE '!'
 *     AND  `page2` LIKE '%match%' ESCAPE '!'
 */
3. RawSql

Baru pada versi 4.2.0.

Sejak v4.2.0, $builder->like() menerima instance CodeIgniter\Database\RawSql, yang menyatakan string SQL mentah.

<?php

use CodeIgniter\Database\RawSql;

$sql    = "CONCAT(users.name, ' ', IF(users.surname IS NULL OR users.surname = '', '', users.surname))";
$rawSql = new RawSql($sql);
$builder->like($rawSql, 'value', 'both');

Peringatan

Saat Anda menggunakan RawSql, Anda HARUS meng-escape nilai-nilai dan melindungi identifier secara manual. Kegagalan untuk melakukannya dapat mengakibatkan SQL injection.

$builder->havingLike()

Metode ini memungkinkan Anda untuk menghasilkan klausa LIKE untuk bagian HAVING dari kueri, berguna untuk melakukan pencarian.

Catatan

Semua nilai yang diteruskan ke metode ini di-escape secara otomatis.

Catatan

Semua variasi metode havingLike*() dapat dipaksa untuk melakukan pencarian yang tidak peka huruf besar/kecil dengan meneruskan parameter kelima true ke metode. Ini akan menggunakan fitur khusus platform jika tersedia jika tidak, akan memaksa nilai menjadi huruf kecil, yaitu, HAVING LOWER(column) LIKE '%search%'. Ini mungkin memerlukan indeks untuk dibuat untuk LOWER(column) alih-alih column agar efektif.

1. Metode kunci/nilai sederhana

$builder->havingLike('title', 'match');

$builder->havingLike('title', 'match');

// HAVING `title` LIKE '%match%' ESCAPE '!' AND  `body` LIKE '%match% ESCAPE '!'

$builder->havingLike('title', 'match', 'before'); // Produces: HAVING `title` LIKE '%match' ESCAPE '!'

$builder->havingLike('title', 'match', 'both');   // Produces: HAVING `title` LIKE '%match%' ESCAPE '!'

OrderBy

$builder->orderBy()

Memungkinkan Anda mengatur klausa ORDER BY.

Parameter pertama berisi nama kolom yang ingin Anda urutkan.

Parameter kedua memungkinkan Anda mengatur arah hasil. Pilihannya adalah ASC, DESC DAN RANDOM.

<?php

$builder->orderBy('title', 'DESC');

$builder->orderBy('title DESC, name ASC');

$builder->orderBy('title', 'DESC');

Membatasi atau Menghitung Hasil

Limit

$builder->limit()

Memungkinkan Anda membatasi jumlah baris yang ingin Anda kembalikan oleh kueri:

<?php

$builder->limit(10);
// Produces: LIMIT 10

Catatan

Jika LIMIT 0 ditentukan dalam pernyataan SQL, 0 record dikembalikan. Namun, ada bug di Query Builder, dan jika limit(0) ditentukan, pernyataan SQL yang dihasilkan tidak akan memiliki klausa LIMIT dan semua record akan dikembalikan. Untuk memperbaiki perilaku yang salah, pengaturan ditambahkan di v4.5.0. Lihat limit(0) Behavior untuk detailnya. Perilaku yang salah akan diperbaiki di versi mendatang, jadi disarankan Anda mengubah pengaturan default.

Parameter kedua memungkinkan Anda mengatur offset hasil.

<?php

$builder->limit(10, 20);
// Produces: LIMIT 20, 10 (in MySQL. Other databases have slightly different syntax)

$builder->countAllResults()

<?php

echo $builder->countAllResults(); // Produces an integer, like 25

$builder->from('my_table');
echo $builder->countAllResults(); // Produces an integer, like 17

Namun, metode ini juga mereset nilai field apa pun yang mungkin telah Anda berikan ke select(). Jika Anda perlu menyimpannya, Anda dapat meneruskan false sebagai parameter pertama.

<?php

echo $builder->countAllResults(false); // Produces an integer, like 17

<?php

Seperti dalam metode countAllResult(), metode ini juga mereset nilai field apa pun yang mungkin telah Anda berikan ke select(). Jika Anda perlu menyimpannya, Anda dapat meneruskan false sebagai parameter pertama.

Kueri Union

Union

$builder->union()

Digunakan untuk menggabungkan kumpulan hasil dari dua atau lebih pernyataan SELECT. Ini hanya akan mengembalikan hasil yang unik.

<?php

$builder = $db->table('users')->select('id, name')->limit(10);
$union   = $db->table('groups')->select('id, name');
$builder->union($union)->get();
/*
 * Produces:
 * SELECT * FROM (SELECT `id`, `name` FROM `users` LIMIT 10) uwrp0
 * UNION SELECT * FROM (SELECT `id`, `name` FROM `groups`) uwrp1
 */

Catatan

Untuk bekerja dengan benar dengan DBMS (seperti MSSQL dan Oracle) kueri dibungkus dalam SELECT * FROM ( ... ) alias Kueri utama akan selalu memiliki alias uwrp0. Setiap kueri berikutnya yang ditambahkan melalui union() akan memiliki alias uwrpN+1.

Semua kueri union akan ditambahkan setelah kueri utama, terlepas dari urutan metode union() dipanggil. Artinya, metode limit() atau orderBy() akan relatif terhadap kueri utama, meskipun dipanggil setelah union().

Dalam beberapa kasus, mungkin diperlukan, misalnya, untuk mengurutkan atau membatasi jumlah record dari hasil kueri. Solusinya adalah menggunakan wrapper yang dibuat melalui $db->newQuery(). Dalam contoh di bawah ini, kita mendapatkan 5 pengguna pertama + 5 pengguna terakhir dan mengurutkan hasilnya berdasarkan id:

<?php

$union   = $db->table('users')->select('id, name')->orderBy('id', 'DESC')->limit(5);
$builder = $db->table('users')->select('id, name')->orderBy('id', 'ASC')->limit(5)->union($union);

$db->newQuery()->fromSubquery($builder, 'q')->orderBy('id', 'DESC')->get();
/*
 * Produces:
 * SELECT * FROM (
 *      SELECT * FROM (SELECT `id`, `name` FROM `users` ORDER BY `id` ASC LIMIT 5) uwrp0
 *      UNION
 *      SELECT * FROM (SELECT `id`, `name` FROM `users` ORDER BY `id` DESC LIMIT 5) uwrp1
 * ) q ORDER BY `id` DESC
 */

$builder->unionAll()

Perilaku ini sama dengan metode union(). Namun, semua hasil akan dikembalikan, tidak hanya yang unik.

Pengelompokan Kueri

Group

Pengelompokan kueri memungkinkan Anda membuat kelompok klausa WHERE dengan mengelompokkannya dalam tanda kurung. Ini akan memungkinkan Anda membuat kueri dengan klausa WHERE yang kompleks. Kelompok bersarang didukung. Contoh:

<?php

$builder->select('*')->from('my_table')
    ->groupStart()
        ->where('a', 'a')
        ->orGroupStart()
            ->where('b', 'b')
            ->where('c', 'c')
        ->groupEnd()
    ->groupEnd()
    ->where('d', 'd')
    ->get();
/*
 * Generates:
 * SELECT * FROM (`my_table`) WHERE ( `a` = 'a' OR ( `b` = 'b' AND `c` = 'c' ) ) AND `d` = 'd'
 */

Catatan

Kelompok perlu diseimbangkan, pastikan setiap groupStart() cocok dengan groupEnd().

$builder->groupStart()

Memulai grup baru dengan menambahkan tanda kurung pembuka ke klausa WHERE dari kueri.

$builder->orGroupStart()

Memulai grup baru dengan menambahkan tanda kurung pembuka ke klausa WHERE dari kueri, memberinya awalan OR.

$builder->notGroupStart()

Memulai grup baru dengan menambahkan tanda kurung pembuka ke klausa WHERE dari kueri, memberinya awalan NOT.

$builder->orNotGroupStart()

Memulai grup baru dengan menambahkan tanda kurung pembuka ke klausa WHERE dari kueri, memberinya awalan OR NOT.

$builder->groupEnd()

Mengakhiri grup saat ini dengan menambahkan tanda kurung penutup ke klausa WHERE dari kueri.

$builder->havingGroupStart()

Memulai grup baru dengan menambahkan tanda kurung pembuka ke klausa HAVING dari kueri.

$builder->orHavingGroupStart()

Memulai grup baru dengan menambahkan tanda kurung pembuka ke klausa HAVING dari kueri, memberinya awalan OR.

$builder->notHavingGroupStart()

Memulai grup baru dengan menambahkan tanda kurung pembuka ke klausa HAVING dari kueri, memberinya awalan NOT.

$builder->orNotHavingGroupStart()

Memulai grup baru dengan menambahkan tanda kurung pembuka ke klausa HAVING dari kueri, memberinya awalan OR NOT.

$builder->havingGroupEnd()

Mengakhiri grup saat ini dengan menambahkan tanda kurung penutup ke klausa HAVING dari kueri.

Memasukkan Data

Insert

$builder->insert()

Menghasilkan string insert berdasarkan data yang Anda berikan, dan menjalankan kueri. Anda dapat meneruskan array atau object ke metode. Berikut adalah contoh menggunakan array:

<?php

use CodeIgniter\Database\RawSql;

$data = [
    'id'          => new RawSql('DEFAULT'),
    'title'       => 'My title',
    'name'        => 'My Name',
    'date'        => '2022-01-01',
    'last_update' => new RawSql('CURRENT_TIMESTAMP()'),
];

$builder->insert($data);
/* Produces:
    INSERT INTO mytable (id, title, name, date, last_update)
    VALUES (DEFAULT, 'My title', 'My name', '2022-01-01', CURRENT_TIMESTAMP())
*/

Parameter pertama adalah array asosiatif dari nilai-nilai.

Catatan

Semua nilai kecuali RawSql di-escape secara otomatis menghasilkan kueri yang lebih aman.

Peringatan

Saat Anda menggunakan RawSql, Anda HARUS meng-escape data secara manual. Kegagalan untuk melakukannya dapat mengakibatkan SQL injection.

Berikut adalah contoh menggunakan object:

<?php

namespace App\Libraries;

class MyClass
{
    public $title   = 'My Title';
    public $content = 'My Content';
    public $date    = 'My Date';
}

<?php

use App\Libraries\MyClass;

$object = new MyClass();
$builder->insert($object);
// Produces: INSERT INTO mytable (title, content, date) VALUES ('My Title', 'My Content', 'My Date')

Parameter pertama adalah object.

$builder->ignore()

Menghasilkan string insert ignore berdasarkan data yang Anda berikan, dan menjalankan kueri. Jadi jika entri dengan primary key yang sama sudah ada, kueri tidak akan dimasukkan. Anda dapat secara opsional meneruskan boolean ke metode. Juga dapat digunakan pada insertBatch, update dan delete (bila didukung). Berikut adalah contoh menggunakan array dari contoh di atas:

<?php

$data = [
    'title' => 'My title',
    'name'  => 'My Name',
    'date'  => 'My date',
];

$builder->ignore(true)->insert($data);
// Produces: INSERT OR IGNORE INTO mytable (title, name, date) VALUES ('My title', 'My name', 'My date')

$builder->getCompiledInsert()

Mengkompilasi kueri penyisipan seperti $builder->insert() tetapi tidak menjalankan kueri. Metode ini hanya mengembalikan kueri SQL sebagai string.

Contoh:

<?php

$data = [
    'title' => 'My title',
    'name'  => 'My Name',
    'date'  => 'My date',
];

$sql = $builder->set($data)->getCompiledInsert();
echo $sql;
// Produces string: INSERT INTO mytable (`title`, `name`, `date`) VALUES ('My title', 'My name', 'My date')

Parameter pertama memungkinkan Anda untuk mengatur apakah kueri query builder akan direset atau tidak (secara default akan direset--sama seperti $builder->insert()):

<?php

echo $builder->set('title', 'My Title')->getCompiledInsert(false);
// Produces string: INSERT INTO mytable (`title`) VALUES ('My Title')

echo $builder->set('content', 'My Content')->getCompiledInsert();
// Produces string: INSERT INTO mytable (`title`, `content`) VALUES ('My Title', 'My Content')

Alasan kueri kedua berfungsi adalah karena parameter pertama diatur ke false.

Catatan

Metode ini tidak berfungsi untuk insert batch.

insertBatch

$builder->insertBatch()

Insert dari Data

Menghasilkan string insert berdasarkan data yang Anda berikan, dan menjalankan kueri. Anda dapat meneruskan array atau object ke metode. Berikut adalah contoh menggunakan array:

<?php

$data = [
    [
        'title' => 'My title',
        'name'  => 'My Name',
        'date'  => 'My date',
    ],
    [
        'title' => 'Another title',
        'name'  => 'Another Name',
        'date'  => 'Another date',
    ],
];

$builder->insertBatch($data);
/*
 * Produces:
 * INSERT INTO mytable (title, name, date)
 *      VALUES ('My title', 'My name', 'My date'),
 *      ('Another title', 'Another name', 'Another date')
 */

Parameter pertama adalah array asosiatif dari nilai-nilai.

Catatan

Semua nilai kecuali RawSql di-escape secara otomatis menghasilkan kueri yang lebih aman.

Peringatan

Saat Anda menggunakan RawSql, Anda HARUS meng-escape data secara manual. Kegagalan untuk melakukannya dapat mengakibatkan SQL injection.

Insert dari Kueri

Anda juga dapat memasukkan dari kueri:

<?php

use CodeIgniter\Database\RawSql;

$query = 'SELECT user2.name, user2.email, user2.country
          FROM user2
          LEFT JOIN user ON user.email = user2.email
          WHERE user.email IS NULL';

$sql = $builder
    ->ignore(true)
    ->setQueryAsData(new RawSql($query), null, 'name, country, email')
    ->insertBatch();
/* MySQLi produces:
    INSERT IGNORE INTO `db_user` (`name`, `country`, `email`)
    SELECT user2.name, user2.email, user2.country
    FROM user2
    LEFT JOIN user ON user.email = user2.email
    WHERE user.email IS NULL
*/

Catatan

setQueryAsData() dapat digunakan sejak v4.3.0.

Catatan

Diperlukan untuk memberi alias pada kolom kueri select agar sesuai dengan kolom tabel target.

Upserting Data

Upsert

$builder->upsert()

Baru pada versi 4.3.0.

Menghasilkan string upsert berdasarkan data yang Anda berikan, dan menjalankan kueri. Anda dapat meneruskan array atau object ke metode. Secara default, batasan akan ditentukan secara berurutan. Primary key akan dipilih terlebih dahulu dan kemudian unique key. MySQL akan menggunakan batasan apa pun secara default. Berikut adalah contoh menggunakan array:

<?php

$data = [
    'email'   => 'ahmadinejad@example.com',
    'name'    => 'Ahmadinejad',
    'country' => 'Iran',
];

$builder->upsert($data);
// MySQLi  produces: INSERT INTO.. ON DUPLICATE KEY UPDATE..
// Postgre produces: INSERT INTO.. ON CONFLICT.. DO UPDATE..
// SQLite3 produces: INSERT INTO.. ON CONFLICT.. DO UPDATE..
// SQLSRV  produces: MERGE INTO.. WHEN MATCHED THEN UPDATE.. WHEN NOT MATCHED THEN INSERT..
// OCI8    produces: MERGE INTO.. WHEN MATCHED THEN UPDATE.. WHEN NOT MATCHED THEN INSERT..

Catatan

Untuk database selain MySQL, jika sebuah tabel memiliki beberapa key (primary atau unique), primary key akan diprioritaskan secara default saat menangani batasan. Jika Anda lebih suka menggunakan unique key yang berbeda alih-alih primary key, gunakan metode onConstraint().

Parameter pertama adalah array asosiatif dari nilai-nilai.

Berikut adalah contoh menggunakan object:

<?php

namespace App\Libraries;

class MyClass
{
    public $email   = 'ahmadinejad@example.com';
    public $name    = 'Ahmadinejad';
    public $country = 'Iran';
}

<?php

use App\Libraries\MyClass;

$object = new MyClass();
$builder->upsert($object);

Parameter pertama adalah object.

Catatan

Semua nilai di-escape secara otomatis menghasilkan kueri yang lebih aman.

$builder->getCompiledUpsert()

Baru pada versi 4.3.0.

Mengkompilasi kueri upsert seperti $builder->upsert() tetapi tidak menjalankan kueri. Metode ini hanya mengembalikan kueri SQL sebagai string.

Contoh:

<?php

$data = [
    'email'   => 'ahmadinejad@example.com',
    'name'    => 'Ahmadinejad',
    'country' => 'Iran',
];

$sql = $builder->setData($data)->getCompiledUpsert();
echo $sql;
/* MySQLi produces:
    INSERT INTO `db_user` (`country`, `email`, `name`)
    VALUES ('Iran','ahmadinejad@example.com','Ahmadinejad')
    ON DUPLICATE KEY UPDATE
    `country` = VALUES(`country`),
    `email` = VALUES(`email`),
    `name` = VALUES(`name`)
*/

Catatan

Metode ini tidak berfungsi untuk upsert batch.

upsertBatch

$builder->upsertBatch()

Baru pada versi 4.3.0.

Upsert dari Data

Menghasilkan string upsert berdasarkan data yang Anda berikan, dan menjalankan kueri. Anda dapat meneruskan array atau object ke metode. Secara default, batasan akan ditentukan secara berurutan. Primary key akan dipilih terlebih dahulu dan kemudian unique key. MySQL akan menggunakan batasan apa pun secara default.

Berikut adalah contoh menggunakan array:

<?php

$data = [
    [
        'id'      => 2,
        'email'   => 'ahmadinejad@example.com',
        'name'    => 'Ahmadinejad',
        'country' => 'Iran',
    ],
    [
        'id'      => null,
        'email'   => 'pedro@example.com',
        'name'    => 'Pedro',
        'country' => 'El Salvador',
    ],
];

$builder->upsertBatch($data);
// MySQLi  produces: INSERT INTO.. ON DUPLICATE KEY UPDATE..
// Postgre produces: INSERT INTO.. ON CONFLICT.. DO UPDATE..
// SQLite3 produces: INSERT INTO.. ON CONFLICT.. DO UPDATE..
// SQLSRV  produces: MERGE INTO.. WHEN MATCHED THEN UPDATE.. WHEN NOT MATCHED THEN INSERT..
// OCI8    produces: MERGE INTO.. WHEN MATCHED THEN UPDATE.. WHEN NOT MATCHED THEN INSERT..

Parameter pertama adalah array asosiatif dari nilai-nilai.

Catatan

Semua nilai di-escape secara otomatis menghasilkan kueri yang lebih aman.

Upsert dari Kueri

Anda juga dapat melakukan upsert dari kueri:

<?php

use CodeIgniter\Database\RawSql;

$query = $this->db->table('user2')
    ->select('user2.name, user2.email, user2.country')
    ->join('user', 'user.email = user2.email', 'left')
    ->where('user.email IS NULL');

$additionalUpdateField = ['updated_at' => new RawSql('CURRENT_TIMESTAMP')];

$sql = $builder->setQueryAsData($query)
    ->onConstraint('email')
    ->updateFields($additionalUpdateField, true)
    ->upsertBatch();
/* MySQLi produces:
    INSERT INTO `db_user` (`country`, `email`, `name`)
    SELECT user2.name, user2.email, user2.country
    FROM user2
    LEFT JOIN user ON user.email = user2.email
    WHERE user.email IS NULL
    ON DUPLICATE KEY UPDATE
    `country` = VALUES(`country`),
    `email` = VALUES(`email`),
    `name` = VALUES(`name`),
    `updated_at` = CURRENT_TIMESTAMP
*/

Catatan

Metode setQueryAsData(), onConstraint(), dan updateFields() dapat digunakan sejak v4.3.0.

Catatan

Diperlukan untuk memberi alias pada kolom kueri select agar sesuai dengan kolom tabel target.

$builder->onConstraint()

Baru pada versi 4.3.0.

Memungkinkan pengaturan batasan secara manual untuk digunakan dalam upsert. Ini tidak berfungsi dengan MySQL karena MySQL memeriksa semua batasan secara default.

<?php

$data = [
    'id'      => 2,
    'email'   => 'ahmadinejad@example.com',
    'name'    => 'Ahmadinejad',
    'country' => 'Iran',
];

$builder->onConstraint('email')->upsert($data);
/* Postgre produces:
    INSERT INTO "db_user"("country", "email", "id", "name")
    VALUES ('Iran','ahmadinejad@example.com',2,'Ahmadinejad')
    ON CONFLICT("email")
    DO UPDATE SET
    "country" = "excluded"."country",
    "id" = "excluded"."id",
    "name" = "excluded"."name"
*/

Metode ini menerima string atau array kolom.

$builder->updateFields()

Baru pada versi 4.3.0.

Memungkinkan pengaturan field yang akan diperbarui secara manual saat melakukan upsert.

<?php

$data = [
    'id'      => 2,
    'email'   => 'ahmadinejad@example.com',
    'name'    => 'Ahmadinejad Zaghari',
    'country' => 'Afghanistan',
];

$builder->updateFields('name, country')->setData($data, null, '_upsert')->upsert();
/* SQLSRV produces:
    MERGE INTO "test"."dbo"."db_user"
    USING (
     VALUES ('Iran','ahmadinejad@example.com',2,'Ahmadinejad')
    ) "_upsert" ("country", "email", "id", "name")
    ON ("test"."dbo"."db_user"."id" = "_upsert"."id")
    WHEN MATCHED THEN UPDATE SET
    "country" = "_upsert"."country",
    "name" = "_upsert"."name"
    WHEN NOT MATCHED THEN INSERT ("country", "email", "id", "name")
    VALUES ("_upsert"."country", "_upsert"."email", "_upsert"."id", "_upsert"."name");
*/

Metode ini menerima string, array kolom, atau RawSql. Anda juga dapat menentukan kolom tambahan yang akan diperbarui yang tidak termasuk dalam dataset. Ini dapat dilakukan dengan mengatur parameter kedua ke true.

<?php

use CodeIgniter\Database\RawSql;

$data = [
    [
        'id'      => 2,
        'email'   => 'ahmadinejad@example.com',
        'name'    => 'Ahmadinejad',
        'country' => 'Iran',
    ],
    [
        'id'      => null,
        'email'   => 'pedro@example.com',
        'name'    => 'Pedro',
        'country' => 'El Salvador',
    ],
];

$additionalUpdateField = ['updated_at' => new RawSql('CURRENT_TIMESTAMP')];

$sql = $builder->setData($data)->updateFields($additionalUpdateField, true)->upsertBatch();
/* MySQLi produces:
    INSERT INTO `db_user` (`country`, `email`, `name`)
    VALUES ('Iran','ahmadinejad@example.com','Ahmadinejad'),('El Salvador','pedro@example.com','Pedro')
    ON DUPLICATE KEY UPDATE
    `country` = VALUES(`country`),
    `email` = VALUES(`email`),
    `name` = VALUES(`name`),
    `updated_at` = CURRENT_TIMESTAMP
*/

Perhatikan bahwa field updated_at tidak dimasukkan tetapi digunakan saat update.

Memperbarui Data

Update

$builder->replace()

Metode ini menjalankan pernyataan REPLACE, yang pada dasarnya adalah standar SQL untuk (opsional) DELETE + INSERT, menggunakan kunci PRIMARY dan UNIQUE sebagai faktor penentu. Dalam kasus kami, ini akan menyelamatkan Anda dari kebutuhan untuk mengimplementasikan logika kompleks dengan kombinasi berbeda dari panggilan select(), update(), delete(), dan insert().

Contoh:

<?php

$data = [
    'title' => 'My title',
    'name'  => 'My Name',
    'date'  => 'My date',
];

$builder->replace($data);
// Executes: REPLACE INTO mytable (title, name, date) VALUES ('My title', 'My name', 'My date')

Dalam contoh di atas, jika kita mengasumsikan bahwa field title adalah primary key kita, maka jika ada baris yang berisi My title sebagai nilai title, baris tersebut akan dihapus dan diganti dengan data baris baru kita.

Penggunaan metode set() juga diizinkan dan semua nilai di-escape secara otomatis, sama seperti dengan insert().

$builder->set()

Metode ini memungkinkan Anda untuk mengatur nilai untuk insert atau update.

Ini dapat digunakan sebagai pengganti meneruskan array data langsung ke metode insert() atau update():

<?php

$builder->set('name', $name);
$builder->insert();
// Produces: INSERT INTO mytable (`name`) VALUES ('{$name}')

Jika Anda menggunakan beberapa panggilan metode, mereka akan dirangkai dengan benar berdasarkan apakah Anda melakukan insert atau update:

<?php

$builder->set('name', $name);
$builder->set('title', $title);
$builder->set('status', $status);
$builder->insert();

set() juga akan menerima parameter ketiga opsional ($escape), yang akan mencegah nilai-nilai di-escape jika diatur ke false. Untuk mengilustrasikan perbedaannya, berikut adalah set() yang digunakan baik dengan maupun tanpa parameter escape.

<?php

$builder->set('field', 'field+1', false);
$builder->where('id', 2);
$builder->update();
// gives UPDATE mytable SET field = field+1 WHERE `id` = 2

$builder->set('field', 'field+1');
$builder->where('id', 2);
$builder->update();
// gives UPDATE `mytable` SET `field` = 'field+1' WHERE `id` = 2

Anda juga dapat meneruskan array asosiatif ke metode ini:

<?php

$array = [
    'name'   => $name,
    'title'  => $title,
    'status' => $status,
];

$builder->set($array);
$builder->insert();

Atau sebuah object:

<?php

namespace App\Libraries;

class MyClass
{
    public $title   = 'My Title';
    public $content = 'My Content';
    public $date    = 'My Date';
}

<?php

use App\Libraries\MyClass;

$object = new MyClass();
$builder->set($object);
$builder->insert();

$builder->update()

Menghasilkan string update dan menjalankan kueri berdasarkan data yang Anda berikan. Anda dapat meneruskan array atau object ke metode. Berikut adalah contoh menggunakan array:

<?php

$data = [
    'title' => $title,
    'name'  => $name,
    'date'  => $date,
];

$builder->where('id', $id);
$builder->update($data);
/*
 * Produces:
 * UPDATE mytable
 * SET title = '{$title}', name = '{$name}', date = '{$date}'
 * WHERE id = $id
 */

Atau Anda dapat memberikan object:

<?php

namespace App\Libraries;

class MyClass
{
    public $title   = 'My Title';
    public $content = 'My Content';
    public $date    = 'My Date';
}

<?php

use App\Libraries\MyClass;

$object = new MyClass();
$builder->where('id', $id);
$builder->update($object);
/*
 * Produces:
 * UPDATE `mytable`
 * SET `title` = '{$title}', `content` = '{$content}', `date` = '{$date}'
 * WHERE id = `$id`
 */

Catatan

Semua nilai kecuali RawSql di-escape secara otomatis menghasilkan kueri yang lebih aman.

Peringatan

Saat Anda menggunakan RawSql, Anda HARUS meng-escape data secara manual. Kegagalan untuk melakukannya dapat mengakibatkan SQL injection.

Anda akan melihat penggunaan metode $builder->where(), yang memungkinkan Anda mengatur klausa WHERE. Anda dapat secara opsional meneruskan informasi ini langsung ke metode update() sebagai string:

<?php

$builder->update($data, 'id = 4');

Atau sebagai array:

<?php

$builder->update($data, ['id' => $id]);

Anda juga dapat menggunakan metode $builder->set() yang dijelaskan di atas saat melakukan update.

$builder->getCompiledUpdate()

Ini berfungsi persis sama dengan $builder->getCompiledInsert() kecuali bahwa ia menghasilkan string SQL UPDATE alih-alih string SQL INSERT.

Untuk informasi lebih lanjut, lihat dokumentasi untuk $builder->getCompiledInsert().

Catatan

Metode ini tidak berfungsi untuk update batch.

UpdateBatch

$builder->updateBatch()

Catatan

Sejak v4.3.0, parameter kedua $index dari updateBatch() telah berubah menjadi $constraints. Sekarang menerima tipe array, string, atau RawSql.

Update dari Data

Menghasilkan string update berdasarkan data yang Anda berikan, dan menjalankan kueri. Anda dapat meneruskan array atau object ke metode. Berikut adalah contoh menggunakan array:

<?php

$data = [
    [
        'title'  => 'Title 1',
        'author' => 'Author 1',
        'name'   => 'Name 1',
        'date'   => 'Date 1',
    ],
    [
        'title'  => 'Title 2',
        'author' => 'Author 2',
        'name'   => 'Name 2',
        'date'   => 'Date 2',
    ],
];
$builder->updateBatch($data, ['title', 'author']);
/*
 * Produces:
 * UPDATE `mytable`
 * INNER JOIN (
 * SELECT 'Title 1' `title`, 'Author 1' `author`, 'Name 1' `name`, 'Date 1' `date` UNION ALL
 * SELECT 'Title 2' `title`, 'Author 2' `author`, 'Name 2' `name`, 'Date 2' `date`
 * ) `u`
 * ON `mytable`.`title` = `u`.`title` AND `mytable`.`author` = `u`.`author`
 * SET
 * `mytable`.`title` = `u`.`title`,
 * `mytable`.`name` = `u`.`name`,
 * `mytable`.`date` = `u`.`date`
 */

Parameter pertama adalah array asosiatif dari nilai-nilai, parameter kedua adalah kunci where.

Catatan

Sejak v4.3.0, struktur SQL yang dihasilkan telah ditingkatkan.

Sejak v4.3.0, Anda juga dapat menggunakan metode onConstraint() dan updateFields():

<?php

use CodeIgniter\Database\RawSql;

$builder->setData($data)->onConstraint('title, author')->updateBatch();

// OR
$builder->setData($data, null, 'u')
    ->onConstraint(['`mytable`.`title`' => '`u`.`title`', 'author' => new RawSql('`u`.`author`')])
    ->updateBatch();

// OR
foreach ($data as $row) {
    $builder->setData($row);
}
$builder->onConstraint('title, author')->updateBatch();

// OR
$builder->setData($data, true, 'u')
    ->onConstraint(new RawSql('`mytable`.`title` = `u`.`title` AND `mytable`.`author` = `u`.`author`'))
    ->updateFields(['last_update' => new RawSql('CURRENT_TIMESTAMP()')], true)
    ->updateBatch();
/*
 * Produces:
 * UPDATE `mytable`
 * INNER JOIN (
 * SELECT 'Title 1' `title`, 'Author 1' `author`, 'Name 1' `name`, 'Date 1' `date` UNION ALL
 * SELECT 'Title 2' `title`, 'Author 2' `author`, 'Name 2' `name`, 'Date 2' `date`
 * ) `u`
 * ON `mytable`.`title` = `u`.`title` AND `mytable`.`author` = `u`.`author`
 * SET
 * `mytable`.`title` = `u`.`title`,
 * `mytable`.`name` = `u`.`name`,
 * `mytable`.`date` = `u`.`date`,
 * `mytable`.`last_update` = CURRENT_TIMESTAMP() // this only applies to the last scenario
 */

Catatan

Semua nilai kecuali RawSql di-escape secara otomatis menghasilkan kueri yang lebih aman.

Peringatan

Saat Anda menggunakan RawSql, Anda HARUS meng-escape data secara manual. Kegagalan untuk melakukannya dapat mengakibatkan SQL injection.

Catatan

affectedRows() tidak akan memberi Anda hasil yang tepat dengan metode ini, karena sifat kerjanya. Sebaliknya, updateBatch() mengembalikan jumlah baris yang terpengaruh.

Update dari Kueri

Sejak v4.3.0, Anda juga dapat melakukan update dari kueri dengan metode setQueryAsData():

<?php

use CodeIgniter\Database\RawSql;

$query = $this->db->table('user2')
    ->select('user2.name, user2.email, user2.country')
    ->join('user', 'user.email = user2.email', 'inner')

 * Produces:
 * UPDATE `user`
 * INNER JOIN (
 * SELECT user2.name, user2.email, user2.country
 * FROM user2
 * INNER JOIN user ON user.email = user2.email

 * ) `u`
 * ON `user`.`email` = `u`.`email`
 * SET
 * `mytable`.`name` = `u`.`name`,
 * `mytable`.`email` = `u`.`email`,
 * `mytable`.`country` = `u`.`country`,
 * `mytable`.`updated_at` = CURRENT_TIMESTAMP()

Diperlukan untuk memberi alias pada kolom kueri select agar sesuai dengan kolom tabel target.

Delete dari Data

Menghasilkan pernyataan DELETE batch berdasarkan sekumpulan data.

<?php

$data = [
    [
        'order'   => 48372,
        'line'    => 3,
        'product' => 'Keyboard',
        'qty'     => 1,
    ],
    [
        'order'   => 48372,

Pernyataan Kondisional

When

$builder->when()

Baru pada versi 4.3.0.

Ini memungkinkan modifikasi kueri berdasarkan kondisi tanpa keluar dari rantai query builder. Parameter pertama adalah kondisi, dan dievaluasi menggunakan logika boolean asli PHP - yang berarti bahwa nilai-nilai seperti false, null, 0, '0', 0.0, string kosong '' dan array kosong [] akan dianggap false. Parameter kedua adalah callable yang akan dijalankan ketika kondisi bernilai true.

Misalnya, Anda mungkin hanya ingin menerapkan pernyataan WHERE tertentu berdasarkan nilai yang dikirim dalam permintaan HTTP:

<?php

$status = service('request')->getPost('status');

$users = $this->db->table('users')
    ->when($status, static function ($query, $status) {
        $query->where('status', $status);
    })
    ->get();

Karena kondisi dievaluasi sebagai true, callable akan dipanggil. Nilai yang diatur dalam kondisi akan diteruskan sebagai parameter kedua ke callable sehingga dapat digunakan dalam kueri.

Terkadang Anda mungkin ingin menerapkan pernyataan yang berbeda jika kondisi dievaluasi sebagai false. Ini dapat dicapai dengan menyediakan closure kedua:

<?php

$onlyInactive = service('request')->getPost('return_inactive');

$users = $this->db->table('users')
    ->when($onlyInactive, static function ($query, $onlyInactive) {
        $query->where('status', 'inactive');
    }, static function ($query) {
        $query->where('status', 'active');
    })
    ->get();

WhenNot

$builder->whenNot()

Baru pada versi 4.3.0.

Ini berfungsi persis sama dengan $builder->when() kecuali bahwa ia akan hanya menjalankan callable ketika kondisi dievaluasi sebagai false, alih-alih true seperti when().

<?php

$status = service('request')->getPost('status');

$users = $this->db->table('users')
    ->whenNot($status, static function ($query, $status) {
        $query->where('active', 0);
    })
    ->get();

Ketika lebih dari $batch_size baris disediakan, beberapa kueri UPSERT akan dieksekusi, masing-masing mencoba melakukan upsert hingga $batch_size baris.

Mengkompilasi dan menjalankan pernyataan UPDATE batch. Parameter $constraints menerima string kolom yang dipisahkan koma, array, array asosiatif, atau RawSql.

Catatan

Ketika lebih dari $batchSize pasangan field/nilai disediakan, beberapa kueri akan dieksekusi, masing-masing menangani hingga $batchSize pasangan field/nilai. Jika kita mengatur $batchSize ke 0, maka semua pasangan field/nilai akan dieksekusi dalam satu kueri.
