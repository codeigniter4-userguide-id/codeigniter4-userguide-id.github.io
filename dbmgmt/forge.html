Menginisialisasi Class Forge

Penting

Untuk menginisialisasi class Forge, driver database Anda harus sudah berjalan, karena class Forge bergantung padanya.

Muat Class Forge sebagai berikut:

<?php

$forge = \Config\Database::forge();

Anda juga dapat meneruskan nama grup database lain ke loader DB Forge, jika database yang ingin Anda kelola bukan yang default:

<?php

$this->myforge = \Config\Database::forge('other_db');

Dalam contoh di atas, kami meneruskan nama grup database yang berbeda untuk disambungkan sebagai parameter pertama.

Membuat dan Menghapus Database

$forge->createDatabase('db_name')

Memungkinkan Anda untuk membuat database yang ditentukan dalam parameter pertama. Mengembalikan true/false berdasarkan keberhasilan atau kegagalan:

<?php

if ($forge->createDatabase('my_db')) {
    echo 'Database created!';
}

Parameter kedua opsional yang diatur ke true akan menambahkan pernyataan IF EXISTS atau akan memeriksa apakah database ada sebelum membuatnya (tergantung pada DBMS).

<?php

$forge->createDatabase('my_db', true);
/*
 * gives CREATE DATABASE IF NOT EXISTS `my_db`
 * or will check if a database exists
 */

$forge->dropDatabase('db_name')

Memungkinkan Anda untuk menghapus database yang ditentukan dalam parameter pertama. Mengembalikan true/false berdasarkan keberhasilan atau kegagalan:

<?php

if ($forge->dropDatabase('my_db')) {
    echo 'Database deleted!';
}

Membuat Database di Baris Perintah

CodeIgniter mendukung pembuatan database langsung dari terminal favorit Anda menggunakan perintah db:create yang khusus. Dengan menggunakan perintah ini diasumsikan bahwa database belum ada. Jika tidak, CodeIgniter akan mengeluh bahwa pembuatan database gagal.

Untuk memulai, cukup ketik perintah dan nama database (misalnya, foo):

php spark db:create foo

Jika semuanya berjalan lancar, Anda akan melihat pesan Database "foo" successfully created. ditampilkan.

Jika Anda berada di lingkungan pengujian atau Anda menggunakan driver SQLite3, Anda dapat meneruskan ekstensi berkas untuk berkas tempat database akan dibuat menggunakan opsi --ext. Nilai yang valid adalah db dan sqlite dan defaultnya adalah db. Ingat bahwa ini tidak boleh didahului oleh titik. :

php spark db:create foo --ext sqlite

Perintah di atas akan membuat berkas db di WRITEPATH/foo.sqlite.

Catatan

Saat menggunakan nama database SQLite3 khusus :memory:, harapkan perintah masih menghasilkan pesan sukses tetapi tidak ada berkas database yang dibuat. Ini karena SQLite3 hanya akan menggunakan database di memori.

Membuat Tabel

Ada beberapa hal yang mungkin ingin Anda lakukan saat membuat tabel. Menambahkan field, menambahkan kunci ke tabel, mengubah kolom. CodeIgniter menyediakan mekanisme untuk ini.

Menambahkan Field

$forge->addField()

Field biasanya dibuat melalui array asosiatif. Dalam array, Anda harus menyertakan kunci type yang berkaitan dengan tipe data field.

Contoh, INT, VARCHAR, TEXT, dll. Banyak tipe data (misalnya VARCHAR) juga memerlukan kunci constraint.

<?php

$fields = [
    'users' => [
        'type'       => 'VARCHAR',
        'constraint' => 100,
    ],
];
// will translate to "users VARCHAR(100)" when the field is added.

Selain itu, kunci/nilai berikut dapat digunakan:

  • unsigned/true : untuk menghasilkan UNSIGNED dalam definisi field.

  • default/value : untuk menghasilkan batasan DEFAULT dalam definisi field.

  • null/true : untuk menghasilkan NULL dalam definisi field. Tanpa ini, field akan default ke NOT NULL.

  • auto_increment/true : menghasilkan flag auto_increment pada field. Perhatikan bahwa tipe field harus merupakan tipe yang mendukung ini, seperti INTEGER.

  • unique/true : untuk menghasilkan kunci unik untuk definisi field.

<?php

$fields = [
    'id' => [
        'type'           => 'INT',
        'constraint'     => 5,
        'unsigned'       => true,
        'auto_increment' => true,
    ],
    'title' => [
        'type'       => 'VARCHAR',
        'constraint' => '100',
        'unique'     => true,
    ],
    'author' => [
        'type'       => 'VARCHAR',
        'constraint' => 100,
        'default'    => 'King of Town',
    ],
    'description' => [
        'type' => 'TEXT',
        'null' => true,
    ],
    'status' => [
        'type'       => 'ENUM',
        'constraint' => ['publish', 'pending', 'draft'],
        'default'    => 'pending',
    ],
];
$forge->addField($fields);

Setelah field didefinisikan, mereka dapat ditambahkan menggunakan $forge->addField($fields) diikuti dengan panggilan ke method createTable().

Catatan tentang Tipe Data

Tipe Floating-Point

Tipe Floating-Point seperti FLOAT dan DOUBLE mewakili nilai perkiraan. Oleh karena itu, mereka tidak boleh digunakan ketika nilai yang tepat diperlukan.

mysql> CREATE TABLE t (f FLOAT, d DOUBLE);
mysql> INSERT INTO t VALUES(99.9, 99.9);

mysql> SELECT * FROM t WHERE f=99.9;
Empty set (0.00 sec)

mysql> SELECT * FROM t WHERE f > 99.89 AND f < 99.91;
+------+------+
| f    | d    |
+------+------+
| 99.9 | 99.9 |
+------+------+
1 row in set (0.01 sec)

Ketika penting untuk mempertahankan presisi yang tepat, misalnya dengan data moneter, DECIMAL atau NUMERIC harus digunakan.

TEXT

TEXT tidak boleh digunakan pada SQLSRV. Ini sudah tidak digunakan lagi. Lihat ntext, text, and image (Transact-SQL) - SQL Server | Microsoft Learn.

ENUM

Tidak semua database mendukung ENUM.

Mulai v4.5.0, SQLSRV Forge mengonversi tipe data ENUM ke VARCHAR(n). Versi sebelumnya dikonversi ke TEXT.

String SQL Mentah sebagai Nilai Default

Baru pada versi 4.2.0.

Sejak v4.2.0, $forge->addField() menerima instance CodeIgniter\Database\RawSql, yang mengekspresikan string SQL mentah.

<?php

use CodeIgniter\Database\RawSql;

$fields = [
    'id' => [
        'type'           => 'INT',
        'constraint'     => 5,
        'unsigned'       => true,
        'auto_increment' => true,
    ],
    'created_at' => [
        'type'    => 'TIMESTAMP',
        'default' => new RawSql('CURRENT_TIMESTAMP'),
    ],
];
$forge->addField($fields);
/*
gives:
    "id" INT(5) UNSIGNED NOT NULL AUTO_INCREMENT,
    "created_at" TIMESTAMP DEFAULT CURRENT_TIMESTAMP NOT NULL
*/

Peringatan

Saat Anda menggunakan RawSql, Anda HARUS meng-escape data secara manual. Kegagalan untuk melakukannya dapat mengakibatkan injeksi SQL.

Meneruskan String sebagai Field

Jika Anda tahu persis bagaimana Anda ingin sebuah field dibuat, Anda dapat meneruskan string ke dalam definisi field dengan addField():

<?php

$forge->addField("label varchar(100) NOT NULL DEFAULT 'default label'");

Catatan

Meneruskan string mentah sebagai field tidak dapat diikuti oleh panggilan addKey() pada field tersebut.

Catatan

Beberapa panggilan ke addField() bersifat kumulatif.

Membuat Field id

Ada pengecualian khusus untuk membuat field id. Sebuah field dengan tipe id akan secara otomatis ditetapkan sebagai Primary Key INT(9) auto_incrementing.

<?php

$forge->addField('id');
// gives `id` INT(9) NOT NULL AUTO_INCREMENT

Menambahkan Kunci

$forge->addKey()

Secara umum, Anda ingin tabel Anda memiliki Kunci. Ini dicapai dengan $forge->addKey('field'). Parameter kedua opsional yang diatur ke true akan menjadikannya primary key dan parameter ketiga yang diatur ke true akan menjadikannya unique key. Anda dapat menentukan nama dengan parameter keempat. Perhatikan bahwa addKey() harus diikuti oleh panggilan ke createTable() atau processIndexes() ketika tabel sudah ada.

Kunci non-primary dengan beberapa kolom harus dikirim sebagai array. Contoh output di bawah ini adalah untuk MySQL.

<?php

$forge->addKey('blog_id', true);
// gives PRIMARY KEY `blog_id` (`blog_id`)

$forge->addKey('blog_id', true);
$forge->addKey('site_id', true);
// gives PRIMARY KEY `blog_id_site_id` (`blog_id`, `site_id`)

$forge->addKey('blog_name');
// gives KEY `blog_name` (`blog_name`)

$forge->addKey(['blog_name', 'blog_label'], false, false, 'my_key_name');
// gives KEY `my_key_name` (`blog_name`, `blog_label`)

$forge->addKey(['blog_id', 'uri'], false, true, 'my_key_name');
// gives UNIQUE KEY `my_key_name` (`blog_id`, `uri`)

$forge->addPrimaryKey()

$forge->addUniqueKey()

Untuk membuat pembacaan kode lebih objektif, dimungkinkan juga untuk menambahkan primary dan unique key dengan method khusus:

<?php

$forge->addPrimaryKey('blog_id', 'pd_name');
// gives PRIMARY KEY `pd_name` (`blog_id`)

$forge->addUniqueKey(['blog_id', 'uri'], 'key_name');
// gives UNIQUE KEY `key_name` (`blog_id`, `uri`)

Catatan

Saat Anda menambahkan primary key, MySQL dan SQLite akan menganggap nama PRIMARY meskipun nama disediakan.

Menambahkan Foreign Key

Foreign Key membantu menegakkan hubungan dan tindakan di seluruh tabel Anda. Untuk tabel yang mendukung Foreign Key, Anda dapat menambahkannya langsung di forge:

<?php

$forge->addForeignKey('users_id', 'users', 'id');
// gives CONSTRAINT `TABLENAME_users_id_foreign` FOREIGN KEY(`users_id`) REFERENCES `users`(`id`)

$forge->addForeignKey(['users_id', 'users_name'], 'users', ['id', 'name']);
// gives CONSTRAINT `TABLENAME_users_id_foreign` FOREIGN KEY(`users_id`, `users_name`) REFERENCES `users`(`id`, `name`)

Anda dapat menentukan tindakan yang diinginkan untuk properti "on update" dan "on delete" dari batasan serta namanya:

<?php

$forge->addForeignKey('users_id', 'users', 'id', 'CASCADE', 'CASCADE', 'my_fk_name');
// gives CONSTRAINT `my_fk_name` FOREIGN KEY(`users_id`) REFERENCES `users`(`id`) ON DELETE CASCADE ON UPDATE CASCADE

$forge->addForeignKey('users_id', 'users', 'id', '', 'CASCADE');
// gives CONSTRAINT `TABLENAME_users_foreign` FOREIGN KEY(`users_id`) REFERENCES `users`(`id`) ON DELETE CASCADE

$forge->addForeignKey(['users_id', 'users_name'], 'users', ['id', 'name'], 'CASCADE', 'CASCADE', 'my_fk_name');
// gives CONSTRAINT `my_fk_name` FOREIGN KEY(`users_id`, `users_name`) REFERENCES `users`(`id`, `name`) ON DELETE CASCADE ON UPDATE CASCADE

Catatan

SQLite3 tidak mendukung penamaan foreign key. CodeIgniter akan merujuknya dengan prefix_table_column_foreign.

Membuat Sebuah Tabel

Setelah field dan kunci dideklarasikan, Anda dapat membuat tabel baru dengan

<?php

$forge->createTable('table_name');
// gives CREATE TABLE table_name

Parameter kedua opsional yang diatur ke true akan membuat tabel hanya jika belum ada.

<?php

$forge->createTable('table_name', true);
// creates table only if table does not exist

Anda juga dapat meneruskan atribut tabel opsional, seperti ENGINE MySQL:

<?php

$attributes = ['ENGINE' => 'InnoDB'];
$forge->createTable('table_name', false, $attributes);
// produces: CREATE TABLE `table_name` (...) ENGINE = InnoDB DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci

Catatan

Kecuali Anda menentukan atribut CHARACTER SET dan/atau COLLATE, createTable() akan selalu menambahkannya dengan nilai charset dan DBCollat yang Anda konfigurasi, selama tidak kosong (hanya MySQL).

Menghapus Tabel

Menghapus Sebuah Tabel

Jalankan pernyataan DROP TABLE dan secara opsional tambahkan klausa IF EXISTS.

<?php

// Produces: DROP TABLE `table_name`
$forge->dropTable('table_name');

// Produces: DROP TABLE IF EXISTS `table_name`
$forge->dropTable('table_name', true);

Parameter ketiga dapat diteruskan untuk menambahkan opsi CASCADE, yang mungkin diperlukan untuk beberapa driver untuk menangani penghapusan tabel dengan foreign key.

<?php

// Produces: DROP TABLE `table_name` CASCADE
$forge->dropTable('table_name', false, true);

Mengubah Tabel

Menambahkan Field ke Tabel

$forge->addColumn()

Method addColumn() digunakan untuk mengubah tabel yang sudah ada. Ini menerima array field yang sama dengan Creating Tables, dan dapat digunakan untuk menambahkan field tambahan.

Catatan

Tidak seperti saat membuat tabel, jika null tidak ditentukan, kolom akan menjadi NULL, bukan NOT NULL.

<?php

$fields = [
    'preferences' => ['type' => 'TEXT'],
];
$forge->addColumn('table_name', $fields);
// Executes: ALTER TABLE `table_name` ADD `preferences` TEXT

Jika Anda menggunakan MySQL atau CUBIRD, maka Anda dapat memanfaatkan klausa AFTER dan FIRST untuk memposisikan kolom baru.

Contoh:

<?php

// Will place the new column after the `another_field` column:
$fields = [
    'preferences' => ['type' => 'TEXT', 'after' => 'another_field'],
];

// Will place the new column at the start of the table definition:
$fields = [
    'preferences' => ['type' => 'TEXT', 'first' => true],
];

Menghapus Field dari Tabel

$forge->dropColumn()

Digunakan untuk menghapus kolom dari tabel.

<?php

$forge->dropColumn('table_name', 'column_to_drop'); // to drop one single column

Digunakan untuk menghapus beberapa kolom dari tabel.

<?php

$forge->dropColumn('table_name', 'column_1,column_2');      // by proving comma separated column names
$forge->dropColumn('table_name', ['column_1', 'column_2']); // by proving array of column names

Mengubah Field dalam Tabel

$forge->modifyColumn()

Penggunaan method ini identik dengan addColumn(), kecuali itu mengubah kolom yang sudah ada daripada menambahkan yang baru. Untuk mengubah nama, Anda dapat menambahkan kunci "name" ke dalam array definisi field.

<?php

$fields = [
    'old_name' => [
        'name' => 'new_name',
        'type' => 'TEXT',
        'null' => false,
    ],
];
$forge->modifyColumn('table_name', $fields);
// gives ALTER TABLE `table_name` CHANGE `old_name` `new_name` TEXT NOT NULL

Catatan

modifyColumn() mungkin secara tak terduga mengubah NULL/NOT NULL. Jadi disarankan untuk selalu menentukan nilai untuk kunci null. Tidak seperti saat membuat tabel, jika null tidak ditentukan, kolom akan menjadi NULL, bukan NOT NULL.

Catatan

Karena bug, sebelum v4.3.4, SQLite3 mungkin tidak mengatur NOT NULL meskipun Anda menentukan 'null' => false.

Catatan

Karena bug, sebelum v4.3.4, Postgres dan SQLSRV mengatur NOT NULL meskipun Anda menentukan 'null' => true.

Menambahkan Kunci ke Tabel

Baru pada versi 4.3.0.

Anda dapat menambahkan kunci ke tabel yang sudah ada dengan menggunakan addKey(), addPrimaryKey(), addUniqueKey() atau addForeignKey() dan processIndexes():

<?php

$this->forge->addKey(['category', 'name'], false, false, 'category_name');
$this->forge->addPrimaryKey('id', 'pk_actions');
$this->forge->addForeignKey('userid', 'user', 'id', '', '', 'userid_fk');
$this->forge->processIndexes('actions');
/* gives:
ALTER TABLE `actions` ADD KEY `category_name` (`category`, `name`);
ALTER TABLE `actions` ADD CONSTRAINT `pk_actions` PRIMARY KEY(`id`);
ALTER TABLE `actions` ADD CONSTRAINT `userid_fk` FOREIGN KEY (`userid`) REFERENCES `user`(`id`);
*/

Menghapus Primary Key

Baru pada versi 4.3.0.

Jalankan DROP PRIMARY KEY.

<?php

// MySQLi Produces: ALTER TABLE `tablename` DROP PRIMARY KEY
// Others Produces: ALTER TABLE `tablename` DROP CONSTRAINT `pk_tablename`
$forge->dropPrimaryKey('tablename');

Menghapus Kunci

Jalankan DROP KEY.

<?php

// For Indexes Produces:        DROP INDEX `users_index` ON `tablename`
// For Unique Indexes Produces: ALTER TABLE `tablename` DROP CONSTRAINT `users_index`
$forge->dropKey('tablename', 'users_index', false);

Menghapus Foreign Key

Jalankan DROP FOREIGN KEY.

<?php

// Produces: ALTER TABLE `tablename` DROP FOREIGN KEY `users_foreign`
$forge->dropForeignKey('tablename', 'users_foreign');

Mengganti Nama Tabel

Menjalankan penggantian nama TABLE

<?php

$forge->renameTable('old_table_name', 'new_table_name');
// gives ALTER TABLE `old_table_name` RENAME TO `new_table_name`
