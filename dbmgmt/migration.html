Database Migrations

Migrasi adalah cara yang nyaman bagi Anda untuk mengubah database Anda secara terstruktur dan terorganisir. Anda bisa mengedit fragmen SQL secara manual tetapi Anda kemudian akan bertanggung jawab untuk memberitahu pengembang lain bahwa mereka perlu menjalankan perubahan tersebut. Anda juga harus melacak perubahan mana yang perlu dijalankan terhadap mesin produksi saat Anda melakukan deploy berikutnya.

Tabel database migrations melacak migrasi mana yang telah dijalankan, jadi yang perlu Anda lakukan hanyalah memastikan migrasi Anda sudah ada dan menjalankan perintah spark migrate untuk membawa database ke kondisi terbaru. Anda juga dapat menggunakan spark migrate --all untuk menyertakan migrasi dari semua namespace.

Nama Berkas Migrasi

Nama berkas migrasi terdiri dari prefiks timestamp, garis bawah (_), dan nama deskriptif (nama class).

  • 2024-09-08-013653_AddBlogTable.php

Setiap migrasi diberi nomor menggunakan timestamp (2024-09-08-013653) saat migrasi dibuat, dalam format YYYY-MM-DD-HHIISS.

Nama deskriptif (AddBlogTable) untuk migrasi adalah nama class dalam PHP. Jadi Anda harus menamai nama class yang valid.

Tahun, bulan, hari, dan waktu dalam prefiks dapat dipisahkan satu sama lain oleh tanda hubung (-), garis bawah (_), atau tidak sama sekali. Contoh:

  • 2012-10-31-100538_AlterBlogTrackViews.php

  • 2012_10_31_100539_AlterBlogAddTranslations.php

  • 20121031100537_AddBlog.php

Setiap Migrasi dijalankan dalam urutan numerik maju atau mundur tergantung pada method yang diambil. Ini membantu mencegah konflik penomoran saat bekerja dalam lingkungan tim.

Membuat Migrasi

Ini akan menjadi migrasi pertama untuk situs baru yang memiliki blog. Semua migrasi masuk ke direktori app/Database/Migrations/ dan memiliki nama seperti 2022-01-31-013057_AddBlog.php.

<?php

namespace App\Database\Migrations;

use CodeIgniter\Database\Migration;

class AddBlog extends Migration
{
    public function up()
    {
        $this->forge->addField([
            'blog_id' => [
                'type'           => 'INT',
                'constraint'     => 5,
                'unsigned'       => true,
                'auto_increment' => true,
            ],
            'blog_title' => [
                'type'       => 'VARCHAR',
                'constraint' => '100',
            ],
            'blog_description' => [
                'type' => 'TEXT',
                'null' => true,
            ],
        ]);
        $this->forge->addKey('blog_id', true);
        $this->forge->createTable('blog');
    }

    public function down()
    {
        $this->forge->dropTable('blog');
    }
}

Koneksi database dan class database Forge keduanya tersedia untuk Anda melalui $this->db dan $this->forge, secara berurutan.

Sebagai alternatif, Anda dapat menggunakan panggilan baris perintah untuk menghasilkan berkas migrasi kerangka. Lihat make:migration di Alat Baris Perintah untuk detail lebih lanjut.

Catatan

Karena class migrasi adalah class PHP, nama class harus unik di setiap berkas migrasi.

Foreign Keys

Ketika tabel Anda menyertakan Foreign Key, migrasi seringkali dapat menyebabkan masalah saat Anda mencoba menghapus tabel dan kolom. Untuk sementara melewati pemeriksaan foreign key saat menjalankan migrasi, gunakan method disableForeignKeyChecks() dan enableForeignKeyChecks() pada koneksi database.

<?php

namespace App\Database\Migrations;

use CodeIgniter\Database\Migration;

class AddBlog extends Migration
{
    public function up()
    {
        $this->db->disableForeignKeyChecks();

        // Migration rules would go here..

        $this->db->enableForeignKeyChecks();
    }
}

Grup Database

Sebuah migrasi hanya akan dijalankan terhadap satu grup database. Jika Anda memiliki beberapa grup yang didefinisikan dalam app/Config/Database.php, maka secara default akan dijalankan terhadap $defaultGroup seperti yang ditentukan dalam berkas konfigurasi yang sama.

Mungkin ada saat-saat ketika Anda membutuhkan skema yang berbeda untuk grup database yang berbeda. Mungkin Anda memiliki satu database yang digunakan untuk semua informasi situs umum, sementara database lain digunakan untuk data penting misi.

Anda dapat memastikan bahwa migrasi hanya dijalankan terhadap grup yang tepat dengan mengatur properti $DBGroup pada migrasi Anda. Nama ini harus sesuai dengan nama grup database secara tepat:

<?php

namespace App\Database\Migrations;

use CodeIgniter\Database\Migration;

class AddBlog extends Migration
{
    protected $DBGroup = 'alternate_db_group';

    public function up()
    {
        // ...
    }

    public function down()
    {
        // ...
    }
}

Catatan

Tabel migrations yang melacak migrasi mana yang telah dijalankan akan selalu dibuat dalam grup database default.

Namespaces

Library migrasi dapat secara otomatis memindai semua namespace yang telah Anda definisikan dalam app/Config/Autoload.php atau dimuat dari sumber eksternal seperti Composer, menggunakan properti $psr4 untuk mencocokkan nama direktori. Ini akan menyertakan semua migrasi yang ditemukannya di Database/Migrations.

Setiap namespace memiliki urutan versinya sendiri, ini akan membantu Anda mengupgrade dan mendowngrade setiap modul (namespace) tanpa memengaruhi namespace lain.

Contoh, asumsikan bahwa kita memiliki namespace berikut yang didefinisikan dalam berkas konfigurasi Autoload kita:

$psr4 = [
    APP_NAMESPACE => APPPATH,
    'MyCompany'   => ROOTPATH . 'MyCompany',
];

Ini akan mencari migrasi apa pun yang terletak di APPPATH/Database/Migrations dan ROOTPATH/MyCompany/Database/Migrations. Ini membuat mudah untuk menyertakan migrasi dalam suite kode modular yang dapat digunakan kembali.

Alat Baris Perintah

CodeIgniter dilengkapi dengan beberapa perintah yang tersedia dari baris perintah untuk membantu Anda bekerja dengan migrasi. Alat-alat ini membuat segalanya lebih mudah bagi Anda yang ingin menggunakannya. Alat-alat ini terutama menyediakan akses ke method yang sama yang tersedia dalam class MigrationRunner.

migrate

Memigrasi grup database dengan semua migrasi yang tersedia:

php spark migrate

Anda dapat menggunakan migrate dengan opsi berikut:

  • -g - untuk menentukan grup database. Jika ditentukan, hanya migrasi untuk grup database yang ditentukan yang akan dijalankan. Jika tidak ditentukan, semua migrasi akan dijalankan.

  • -n - untuk memilih namespace, jika tidak namespace App akan digunakan.

  • --all - untuk memigrasi semua namespace ke migrasi terbaru.

Contoh ini akan memigrasi namespace Acme\Blog dengan migrasi baru apa pun pada grup database test:

Untuk Unix:

php spark migrate -g test -n Acme\\Blog

Untuk Windows:

php spark migrate -g test -n Acme\Blog

Saat menggunakan opsi --all, ini akan memindai semua namespace untuk menemukan migrasi yang belum dijalankan. Ini semua akan dikumpulkan dan kemudian diurutkan sebagai grup berdasarkan tanggal pembuatan. Ini akan membantu meminimalkan potensi konflik antara aplikasi utama dan modul apa pun.

migrate:rollback

Mengembalikan semua migrasi ke keadaan kosong, secara efektif migrasi 0:

php spark migrate:rollback

Anda dapat menggunakan migrate:rollback dengan opsi berikut:

  • -b - untuk memilih batch: angka alami menentukan batch.

  • -f - untuk memaksa melewati pertanyaan konfirmasi, ini hanya ditanyakan di lingkungan produksi.

migrate:refresh

Memperbarui status database dengan terlebih dahulu mengembalikan semua migrasi, dan kemudian memigrasi semua:

php spark migrate:refresh

Anda dapat menggunakan migrate:refresh dengan opsi berikut:

  • -g - untuk menentukan grup database. Jika ditentukan, hanya migrasi untuk grup database yang ditentukan yang akan dijalankan. Jika tidak ditentukan, semua migrasi akan dijalankan.

  • -n - untuk memilih namespace, jika tidak namespace App akan digunakan.

  • --all - untuk memperbarui semua namespace.

  • -f - untuk memaksa melewati pertanyaan konfirmasi, ini hanya ditanyakan di lingkungan produksi.

migrate:status

Menampilkan daftar semua migrasi dan tanggal serta waktu mereka dijalankan, atau '--' jika belum dijalankan:

php spark migrate:status

...

+----------------------+-------------------+-----------------------+---------+---------------------+-------+
| Namespace            | Version           | Filename              | Group   | Migrated On         | Batch |
+----------------------+-------------------+-----------------------+---------+---------------------+-------+
| App                  | 2022-04-06-234508 | CreateCiSessionsTable | default | 2022-04-06 18:45:14 | 2     |
| CodeIgniter\Settings | 2021-07-04-041948 | CreateSettingsTable   | default | 2022-04-06 01:23:08 | 1     |
| CodeIgniter\Settings | 2021-11-14-143905 | AddContextColumn      | default | 2022-04-06 01:23:08 | 1     |
+----------------------+-------------------+-----------------------+---------+---------------------+-------+

Anda dapat menggunakan migrate:status dengan opsi berikut:

  • -g - untuk menentukan grup database. Jika ditentukan, hanya migrasi untuk grup database yang ditentukan yang akan diperiksa. Jika tidak ditentukan, semua migrasi akan diperiksa.

make:migration

Membuat berkas migrasi kerangka di app/Database/Migrations. Secara otomatis menambahkan timestamp saat ini. Nama class yang dibuat adalah versi Pascal case dari nama berkas.

php spark make:migration <class> [options]

Anda dapat menggunakan make:migration dengan opsi berikut:

  • --namespace - Mengatur namespace root. Default: APP_NAMESPACE.

  • --suffix - Menambahkan judul komponen ke nama class.

Opsi berikut juga tersedia untuk menghasilkan berkas migrasi untuk sesi database:

  • --session - Menghasilkan berkas migrasi untuk sesi database.

  • --table - Nama tabel yang akan digunakan untuk sesi database. Default: ci_sessions.

  • --dbgroup - Grup database yang akan digunakan untuk sesi database. Default: default.

Preferensi Migrasi

Berikut adalah tabel semua opsi konfigurasi untuk migrasi, yang tersedia di app/Config/Migrations.php.

Referensi Class

class CodeIgniter\Database\MigrationRunner
findMigrations()
Kembali:

Array berkas migrasi

Return type:

array

Array nama berkas migrasi yang ditemukan di properti path dikembalikan.

latest($group)
Parameter:

  • $group (mixed) -- nama grup database, jika null grup database default akan digunakan.

Kembali:

true jika berhasil, false jika gagal

Return type:

bool

Ini menemukan migrasi untuk sebuah namespace (atau semua namespace), menentukan migrasi mana yang belum dijalankan, dan menjalankannya sesuai urutan versinya (namespace bercampur).

regress($targetBatch, $group)
Parameter:

  • $targetBatch (int) -- batch sebelumnya untuk migrasi mundur; 1+ menentukan batch, 0 mengembalikan semua, negatif mengacu pada batch relatif (misalnya, -3 berarti "tiga batch ke belakang")

  • $group (?string) -- nama grup database, jika null grup database default akan digunakan.

Kembali:

true jika berhasil, false jika gagal atau tidak ada migrasi yang ditemukan

Return type:

bool

Regress dapat digunakan untuk mengembalikan perubahan ke keadaan sebelumnya, batch demi batch.

<?php

$migration->regress(5);
$migration->regress(-1);
force($path, $namespace, $group)
Parameter:

  • $path (mixed) -- path ke berkas migrasi yang valid.

  • $namespace (mixed) -- namespace dari migrasi yang disediakan.

  • $group (mixed) -- nama grup database, jika null grup database default akan digunakan.

Kembali:

true jika berhasil, false jika gagal

Return type:

bool

Ini memaksa satu berkas untuk dimigrasi terlepas dari urutan atau batch. Method up() atau down() dideteksi berdasarkan apakah sudah dimigrasi.

Catatan

Method ini hanya direkomendasikan untuk pengujian dan dapat menyebabkan masalah konsistensi data.

setNamespace($namespace)
Parameter:

  • $namespace (string|null) -- namespace aplikasi. null adalah semua namespace.

Kembali:

Instance MigrationRunner saat ini

Return type:

CodeIgniter\Database\MigrationRunner

Mengatur namespace yang harus dicari oleh library untuk berkas migrasi:

<?php

$migration->setNamespace($namespace)->latest();

Catatan

Jika Anda mengatur null, itu mencari berkas migrasi di semua namespace.

setGroup($group)
Parameter:

  • $group (string) -- nama grup database.

Kembali:

Instance MigrationRunner saat ini

Return type:

CodeIgniter\Database\MigrationRunner

Mengatur grup yang harus dicari oleh library untuk berkas migrasi:

<?php

$migration->setGroup($group)->latest();
