Apa itu Perutean Otomatis?

Secara default, semua router harus didefinisikan di berkas konfigurasi.

Namun, dengan Perutean Otomatis, Anda dapat mendefinisikan nama controller dan nama methodnya sesuai dengan konvensi dan itu akan dirutekan secara otomatis. Dengan kata lain, tidak perlu mendefinisikan router secara manual.

Jika Anda mengaktifkan Perutean Otomatis, ketika tidak ada router yang didefinisikan yang cocok dengan URI, sistem akan mencoba mencocokkan URI tersebut dengan controller dan method.

Penting

Untuk alasan keamanan, jika sebuah controller digunakan dalam router yang didefinisikan, Perutean Otomatis tidak merutekan ke controller tersebut.

Catatan

Perutean Otomatis dinonaktifkan secara default. Untuk menggunakannya, lihat Mengaktifkan Perutean Otomatis.

Perbedaan dari Perutean Otomatis (Versi Lawas)

Perutean Otomatis (Legacy) adalah sistem perutean dari CodeIgniter 3. Jika Anda tidak terbiasa dengannya, lanjutkan ke bagian berikutnya.

Jika Anda sudah mengetahuinya dengan baik, berikut adalah beberapa perubahan dalam Perutean Otomatis:

  • Sebuah method controller membutuhkan prefiks HTTP verb seperti getIndex(), postCreate().

    • Karena pengembang selalu mengetahui method HTTP, permintaan dengan method HTTP yang tidak terduga tidak akan pernah mengeksekusi controller.

  • Controller Default (Home secara default) dan Method Default (index secara default) harus dihilangkan dalam URI.

    • Ini membatasi korespondensi satu-ke-satu antara method controller dan URI.

    • Misal, secara default, Anda dapat mengakses /, tetapi /home dan /home/index akan menjadi 404 Not Found.

  • Ini memeriksa jumlah parameter method.

    • Jika ada lebih banyak parameter di URI daripada parameter method, itu menghasilkan 404 Not Found.

  • Ini tidak mendukung method _remap().

    • Ini membatasi korespondensi satu-ke-satu antara method controller dan URI.

    • Tetapi ia memiliki fitur Fallback Method Default sebagai gantinya.

  • Tidak dapat mengakses controller di Router yang Didefinisikan.

    • Ini sepenuhnya memisahkan controller yang dapat diakses melalui Perutean Otomatis dari yang dapat diakses melalui Router yang Didefinisikan.

Mengaktifkan Perutean Otomatis

Untuk menggunakannya, Anda perlu mengubah pengaturan opsi $autoRoute menjadi true di app/Config/Routing.php:

public bool $autoRoute = true;

Dan Anda perlu mengubah properti $autoRoutesImproved menjadi true di app/Config/Feature.php:

public bool $autoRoutesImproved = true;

Penting

Saat Anda menggunakan Perutean Otomatis, Anda harus menghapus baris $routes->get('/', 'Home::index'); di app/Config/Routes.php. Karena router yang didefinisikan lebih diutamakan daripada Perutean Otomatis, dan controller yang didefinisikan di router yang didefinisikan ditolak aksesnya oleh Perutean Otomatis untuk alasan keamanan.

Segmen URI

Segmen dalam URL, mengikuti pendekatan Model-View-Controller, biasanya mewakili:

http://example.com/{class}/{method}/{param1}

  1. Segmen pertama mewakili class controller yang harus dipanggil.

  2. Segmen kedua mewakili method class yang harus dipanggil.

  3. Segmen ketiga, dan segmen tambahan apa pun, mewakili parameter apa pun yang akan diteruskan ke method controller.

Pertimbangkan URI ini:

http://example.com/hello-world/hello/1

Dalam contoh di atas, ketika Anda mengirim permintaan HTTP dengan method GET, Perutean Otomatis akan mencoba menemukan controller bernama App\Controllers\HelloWorld dan mengeksekusi method getHello() dengan meneruskan '1' sebagai parameter pertama.

Catatan

Sebuah method controller yang akan dieksekusi oleh Perutean Otomatis (Ditingkatkan) membutuhkan prefiks HTTP verb (get, post, put, dll.) seperti getIndex(), postCreate().

Catatan

Ketika nama pendek controller cocok dengan segmen pertama URI, itu akan dimuat.

Mari kita coba: Hello World!

Mari kita buat controller sederhana agar Anda dapat melihatnya beraksi.

Buat Controller

Menggunakan editor teks Anda, buat berkas bernama HelloWorld.php di direktori app/Controllers Anda, dan masukkan kode berikut di dalamnya.

<?php

namespace App\Controllers;

class HelloWorld extends BaseController
{
    public function getIndex()
    {
        return 'Hello World!';
    }
}

Penting

Berkas harus diberi nama HelloWorld.php. Saat Anda menggunakan Perutean Otomatis (Ditingkatkan), nama class controller HARUS CamelCase.

Anda akan melihat bahwa Controller HelloWorld memperluas BaseController. Anda juga dapat memperluas CodeIgniter\Controller jika Anda tidak membutuhkan fungsionalitas BaseController.

BaseController menyediakan tempat yang nyaman untuk memuat komponen dan melakukan fungsi yang dibutuhkan oleh semua controller Anda. Anda dapat memperluas class ini di controller baru mana pun.

Penting

Sebuah method controller yang akan dieksekusi oleh Perutean Otomatis (Ditingkatkan) membutuhkan prefiks HTTP verb (get, post, put, dll.) seperti getIndex(), postCreate().

Periksa Router

Anda dapat memeriksa router Anda dengan perintah spark routes.

php spark routes

Jika Anda melakukannya dengan benar, Anda akan melihat:

+-----------+-------------+------+---------------------------------------+----------------+---------------+
| Method    | Router      | Nama | Penangan                              | Filter Sebelum | Filter Sesudah|
+-----------+-------------+------+---------------------------------------+----------------+---------------+
| GET(auto) | hello-world |      | \App\Controllers\HelloWorld::getIndex |                |               |
+-----------+-------------+------+---------------------------------------+----------------+---------------+

Lihat spark routes untuk detail output.

Kunjungi Situs Anda

Sekarang kunjungi situs Anda menggunakan URL yang mirip dengan ini:

http://example.com/hello-world

Sistem secara otomatis menerjemahkan URI dengan tanda hubung (-) ke CamelCase di segmen URI controller dan method.

Misalnya, URI sub-dir/hello-controller/some-method akan mengeksekusi method SubDir\HelloController::getSomeMethod().

Jika Anda melakukannya dengan benar, Anda akan melihat:

Hello World!

Contoh Nama Controller

Berikut ini adalah nama controller yang valid. Karena App\Controllers\HelloWorld adalah CamelCase.

<?php

namespace App\Controllers;

class HelloWorld extends BaseController
{
    // ...
}

Berikut ini tidak valid. Karena huruf pertama (h) bukan huruf kapital.

<?php

namespace App\Controllers;

class helloworld extends BaseController
{
    // ...
}

Berikut ini juga tidak valid. Karena huruf pertama (h) bukan huruf kapital.

<?php

namespace App\Controllers;

class helloWorld extends BaseController
{
    // ...
}

Method Controller

Visibilitas Method

Ketika Anda mendefinisikan method yang dapat dieksekusi melalui permintaan HTTP, method tersebut harus dideklarasikan sebagai public.

Peringatan

Untuk alasan keamanan, pastikan untuk mendeklarasikan method utilitas baru apa pun sebagai protected atau private.

Method Default

Dalam contoh di atas, nama method adalah getIndex(). Method (HTTP verb + Index()) disebut Method Default, dan dimuat jika segmen kedua dari URI kosong.

Method Normal

Segmen kedua dari URI menentukan method mana di controller yang dipanggil.

Mari kita coba. Tambahkan method baru ke controller Anda:

<?php

namespace App\Controllers;

class HelloWorld extends BaseController
{
    public function getIndex()
    {
        return 'Hello World!';
    }

    public function getComment()
    {
        return 'I am not flat!';
    }
}

Sekarang muat URL berikut untuk melihat method getComment():

http://example.com/hello-world/comment/

Anda akan melihat pesan baru Anda.

Meneruskan Segmen URI ke Method Anda

Jika URI Anda berisi lebih dari dua segmen, mereka akan diteruskan ke method Anda sebagai parameter.

Misalnya, katakanlah Anda memiliki URI seperti ini:

http://example.com/products/shoes/sandals/123

Method Anda akan diteruskan segmen URI 3 dan 4 ('sandals' dan '123'):

<?php

namespace App\Controllers;

class Products extends BaseController
{
    public function getShoes($type, $id)
    {
        return $type . $id;
    }
}

Catatan

Jika ada lebih banyak parameter di URI daripada parameter method, Perutean Otomatis tidak mengeksekusi method, dan itu menghasilkan 404 Not Found.

Controller Default

Controller Default adalah controller khusus yang digunakan ketika URI berakhir dengan nama direktori atau ketika URI tidak ada, seperti yang akan terjadi ketika hanya URL root situs Anda diminta.

Secara default, Controller Default adalah Home.

Catatan

Definisikan hanya method default (getIndex() untuk permintaan GET) di controller default. Jika Anda mendefinisikan method publik lainnya, method tersebut tidak akan dieksekusi.

Untuk informasi lebih lanjut, silakan merujuk ke Opsi Konfigurasi.

Fallback Method Default

Baru pada versi 4.4.0.

Jika method controller yang sesuai dengan segmen URI dari nama method tidak ada, dan jika method default didefinisikan, segmen URI yang tersisa diteruskan ke method default untuk dieksekusi.

<?php

namespace App\Controllers;

class Product extends BaseController
{
    public function getIndex($id = null, $action = '')
    {
        // ...
    }
}

Muat URL berikut:

http://example.com/product/15/edit

Method akan diteruskan segmen URI 2 dan 3 ('15' dan 'edit'):

Penting

Jika ada lebih banyak parameter di URI daripada parameter method, Perutean Otomatis (Ditingkatkan) tidak mengeksekusi method, dan itu menghasilkan 404 Not Found.

Fallback ke Controller Default

Jika controller yang sesuai dengan segmen URI dari nama controller tidak ada, dan jika controller default (Home secara default) ada di direktori, segmen URI yang tersisa diteruskan ke method default controller default.

Misalnya, ketika Anda memiliki controller default Home berikut di direktori app/Controllers/News:

<?php

namespace App\Controllers\News;

use App\Controllers\BaseController;

class Home extends BaseController
{
    public function getIndex($id = null)
    {
        // ...
    }
}

Muat URL berikut:

http://example.com/news/101

Controller News\Home dan method default getIndex() akan ditemukan. Jadi method default akan mendapatkan segmen URI kedua ('101'):

Catatan

Jika ada controller App\Controllers\News, itu lebih diutamakan. Segmen URI dicari secara berurutan dan controller pertama yang ditemukan digunakan.

Catatan

Jika ada lebih banyak parameter di URI daripada parameter method, Perutean Otomatis tidak mengeksekusi method, dan itu menghasilkan 404 Not Found.

Mengorganisir Controller Anda ke dalam Sub-direktori

Jika Anda membangun aplikasi besar, Anda mungkin ingin secara hierarkis mengatur atau menyusun controller Anda ke dalam sub-direktori. CodeIgniter mengizinkan Anda untuk melakukan ini.

Cukup buat sub-direktori di bawah app/Controllers utama, dan tempatkan class controller Anda di dalamnya.

Penting

Nama direktori HARUS dimulai dengan huruf kapital dan CamelCase.

Saat menggunakan fitur ini, segmen pertama URI Anda harus menentukan direktori. Misalnya, katakanlah Anda memiliki controller yang terletak di sini:

app/Controllers/Products/Shoes.php

Untuk memanggil controller di atas, URI Anda akan terlihat seperti ini:

http://example.com/products/shoes/show/123

Catatan

Anda tidak dapat memiliki direktori dengan nama yang sama di app/Controllers dan public. Ini karena jika ada direktori, web server akan mencarinya dan itu tidak akan dirutekan ke CodeIgniter.

Setiap sub-direktori Anda dapat berisi controller default yang akan dipanggil jika URL hanya berisi sub-direktori. Cukup tempatkan controller di sana yang cocok dengan nama controller default Anda seperti yang ditentukan di berkas app/Config/Routing.php Anda.

Contoh Controller/Method dan URI

Dalam kasus permintaan GET dengan konfigurasi default, pemetaan antara controller/method dan URI adalah sebagai berikut:

Controller/Method

URI

Deskripsi

Home::getIndex()

/

Controller default dan method default.

Blog::getIndex()

/blog

Method default.

UserProfile::getIndex()

/user-profile

Method default.

Blog::getTags()

/blog/tags

Blog::getNews($id)

/blog/news/123

Blog\Home::getIndex()

/blog

Sub-direktori Blog dan controller default serta method default. Jika ada controller Blog, itu lebih diutamakan.

Blog\Tags::getIndex()

/blog/tags

Sub-direktori Blog dan method default. Jika ada controller Blog, itu lebih diutamakan.

Blog\News::getIndex($id)

/blog/news/123

Sub-direktori Blog dan method default fallback. Jika ada controller Blog, itu lebih diutamakan.

Menerapkan Filter

Menerapkan filter controller memungkinkan Anda untuk menambahkan pemrosesan sebelum dan sesudah eksekusi method controller. Ini sangat berguna selama otentikasi atau pencatatan API.

Jika Anda menggunakan Perutean Otomatis, atur filter yang akan diterapkan di app/Config/Filters.php. Lihat Filter Controller untuk informasi lebih lanjut tentang pengaturan filter.

Opsi Konfigurasi

Opsi-opsi ini tersedia di berkas app/Config/Routing.php.

Controller Default

Untuk URI Root Situs

Ketika pengguna mengunjungi root situs Anda (yaitu, http://example.com) controller yang akan digunakan ditentukan oleh nilai yang diatur ke properti $defaultController, kecuali jika ada router yang secara eksplisit ada untuk itu.

Nilai default untuk ini adalah Home yang cocok dengan controller di app/Controllers/Home.php:

public string $defaultController = 'Home';

Untuk URI Direktori

Controller default juga digunakan ketika tidak ada router yang cocok ditemukan, dan URI akan menunjuk ke direktori di direktori controller. Misalnya, jika pengguna mengunjungi http://example.com/admin, jika controller ditemukan di app/Controllers/Admin/Home.php, itu akan digunakan.

Penting

Anda tidak dapat mengakses controller default dengan URI dari nama controller. Ketika controller default adalah Home, Anda dapat mengakses http://example.com/, tetapi jika Anda mengakses http://example.com/home, itu tidak akan ditemukan.

Method Default

Ini berfungsi mirip dengan pengaturan controller default, tetapi digunakan untuk menentukan method default yang digunakan ketika controller ditemukan yang cocok dengan URI, tetapi tidak ada segmen yang ada untuk method. Nilai default adalah index.

Dalam contoh ini, jika pengguna mengunjungi example.com/products, dan controller Products ada, method Products::getListAll() akan dieksekusi:

public string $defaultMethod = 'listAll';

Penting

Anda tidak dapat mengakses controller dengan URI dari nama method default. Dalam contoh di atas, Anda dapat mengakses example.com/products, tetapi jika Anda mengakses example.com/products/listall, itu tidak akan ditemukan.

Terjemahkan URI ke CamelCase

Baru pada versi 4.5.0.

Catatan

Sejak v4.6.0, opsi $translateUriToCamelCase diaktifkan secara default.

Sejak v4.5.0, opsi $translateUriToCamelCase telah diimplementasikan, yang berfungsi baik dengan standar pengkodean CodeIgniter saat ini.

Opsi ini memungkinkan Anda untuk secara otomatis menerjemahkan URI dengan tanda hubung (-) ke CamelCase di segmen URI controller dan method.

Misalnya, URI sub-dir/hello-controller/some-method akan mengeksekusi method SubDir\HelloController::getSomeMethod().

Catatan

Ketika opsi ini diaktifkan, opsi $translateURIDashes diabaikan.

Nonaktifkan Terjemahan URI ke CamelCase

Catatan

Opsi untuk menonaktifkan "Terjemahan URI ke CamelCase" hanya ada untuk kompatibilitas mundur. Kami tidak merekomendasikan untuk menonaktifkannya.

Untuk menonaktifkannya, Anda perlu mengubah pengaturan opsi $translateUriToCamelCase menjadi false di app/Config/Routing.php:

public bool $translateUriToCamelCase = false;

Perutean Modul

Baru pada versi 4.4.0.

Anda dapat menggunakan perutean otomatis bahkan jika Anda menggunakan Code Modules dan menempatkan controller di namespace yang berbeda.

Untuk merutekan ke modul, properti $moduleRoutes di app/Config/Routing.php harus diatur:

public array $moduleRoutes = [
    'blog' => 'Acme\Blog\Controllers',
];

Kunci adalah segmen URI pertama untuk modul, dan nilainya adalah namespace controller. Dalam konfigurasi di atas, http://localhost:8080/blog/foo/bar akan dirutekan ke Acme\Blog\Controllers\Foo::getBar().

Catatan

Jika Anda mendefinisikan $moduleRoutes, perutean untuk modul akan lebih diutamakan. Dalam contoh di atas, meskipun Anda memiliki controller App\Controllers\Blog, http://localhost:8080/blog akan dirutekan ke controller default Acme\Blog\Controllers\Home.
