Apa itu Controller?

Controller hanyalah sebuah berkas class yang menangani permintaan HTTP. Perutean URI mengaitkan sebuah URI dengan sebuah controller. Ini mengembalikan string tampilan atau objek Response.

Setiap controller yang Anda buat harus memperluas class BaseController. Class ini menyediakan beberapa fitur yang tersedia untuk semua controller Anda.

Konstruktor

Controller CodeIgniter memiliki konstruktor khusus initController(). Ini akan dipanggil oleh framework setelah eksekusi konstruktor PHP __construct().

Jika Anda ingin menimpa initController(), jangan lupa untuk menambahkan parent::initController($request, $response, $logger); di method:

<?php

namespace App\Controllers;

use CodeIgniter\HTTP\RequestInterface;
use CodeIgniter\HTTP\ResponseInterface;
use Psr\Log\LoggerInterface;

class Product extends BaseController
{
    public function initController(
        RequestInterface $request,
        ResponseInterface $response,
        LoggerInterface $logger,
    ) {
        parent::initController($request, $response, $logger);

        // Add your code here.
    }

    // ...
}

Penting

Anda tidak dapat menggunakan return di konstruktor. Jadi return redirect()->to('route'); tidak berfungsi.

Method initController() mengatur tiga properti berikut.

Properti yang Disertakan

Controller CodeIgniter menyediakan properti-properti ini.

Objek Request

Instance Request utama aplikasi selalu tersedia sebagai properti class, $this->request.

Objek Response

Instance Response utama aplikasi selalu tersedia sebagai properti class, $this->response.

Objek Logger

Sebuah instance dari class Logger tersedia sebagai properti class, $this->logger.

Pembantu

Anda dapat mendefinisikan array berkas pembantu sebagai properti class. Setiap kali controller dimuat berkas pembantu ini akan secara otomatis dimuat ke memori sehingga Anda dapat menggunakan method mereka di mana saja di dalam controller:

<?php

namespace App\Controllers;

class MyController extends BaseController
{
    protected $helpers = ['url', 'form'];
}

forceHTTPS

Method kenyamanan untuk memaksa method diakses melalui HTTPS tersedia di semua controller:

<?php

if (! $this->request->isSecure()) {
    $this->forceHTTPS();
}

Secara default, dan di browser modern yang mendukung header HTTP Strict Transport Security, panggilan ini akan memaksa browser untuk mengubah panggilan non-HTTPS menjadi panggilan HTTPS selama satu tahun. Anda dapat memodifikasi ini dengan meneruskan durasi (dalam detik) sebagai parameter pertama:

<?php

if (! $this->request->isSecure()) {
    $this->forceHTTPS(31536000); // one year
}

Catatan

Sejumlah konstanta berbasis waktu selalu tersedia untuk Anda gunakan, termasuk YEAR, MONTH, dan banyak lagi.

Memvalidasi Data

$this->validateData()

Baru pada versi 4.2.0.

Untuk menyederhanakan pemeriksaan data, controller juga menyediakan method kenyamanan validateData().

Method ini menerima (1) array data yang akan divalidasi, (2) array aturan, (3) array opsional pesan error kustom untuk ditampilkan jika item tidak valid, (4) grup database opsional untuk digunakan.

Dokumen Pustaka Validasi memiliki detail tentang format array aturan dan pesan, serta aturan yang tersedia:

<?php

namespace App\Controllers;

class StoreController extends BaseController
{
    public function product(int $id)
    {
        $data = [
            'id'   => $id,
            'name' => $this->request->getPost('name'),
        ];

        $rule = [
            'id'   => 'integer',
            'name' => 'required|max_length[255]',
        ];

        if (! $this->validateData($data, $rule)) {
            return view('store/product', [
                'errors' => $this->validator->getErrors(),
            ]);
        }

        // ...
    }
}

$this->validate()

Penting

Method ini hanya ada untuk kompatibilitas mundur. Jangan menggunakannya dalam proyek baru. Bahkan jika Anda sudah menggunakannya, kami merekomendasikan Anda untuk menggunakan method validateData() sebagai gantinya.

Controller juga menyediakan method kenyamanan validate().

Peringatan

Daripada validate(), gunakan validateData() untuk memvalidasi data POST saja. validate() menggunakan $request->getVar() yang mengembalikan data $_GET, $_POST atau $_COOKIE dalam urutan tersebut (tergantung pada request-order php.ini). Nilai yang lebih baru menimpa nilai yang lebih lama. Nilai POST dapat ditimpa oleh cookie jika memiliki nama yang sama.

Method ini menerima array aturan pada parameter pertama, dan pada parameter opsional kedua, array pesan error kustom untuk ditampilkan jika item tidak valid.

Secara internal, ini menggunakan instance $this->request controller untuk mendapatkan data yang akan divalidasi.

Dokumen Pustaka Validasi memiliki detail tentang format array aturan dan pesan, serta aturan yang tersedia:

<?php

namespace App\Controllers;

class UserController extends BaseController
{
    public function updateUser(int $userID)
    {
        if (! $this->validate([
            'email' => "required|is_unique[users.email,id,{$userID}]",
            'name'  => 'required|alpha_numeric_spaces',
        ])) {
            // The validation failed.
            return view('users/update', [
                'errors' => $this->validator->getErrors(),
            ]);
        }

        // The validation was successful.

        // Get the validated data.
        $validData = $this->validator->getValidated();

        // ...
    }
}

Peringatan

Saat Anda menggunakan method validate(), Anda harus menggunakan method getValidated() untuk mendapatkan data yang divalidasi. Karena method validate() menggunakan method Validation::withRequest() secara internal, dan itu memvalidasi data dari $request->getJSON() atau $request->getRawInput() atau $request->getVar(), dan penyerang dapat mengubah data apa yang divalidasi.

Catatan

Method $this->validator->getValidated() dapat digunakan sejak v4.4.0.

Jika Anda merasa lebih sederhana untuk menyimpan aturan dalam berkas konfigurasi, Anda dapat mengganti array $rules dengan nama grup seperti yang didefinisikan di app/Config/Validation.php:

<?php

namespace App\Controllers;

class UserController extends BaseController
{
    public function updateUser(int $userID)
    {
        if (! $this->validate('userRules')) {
            // The validation failed.
            return view('users/update', [
                'errors' => $this->validator->getErrors(),
            ]);
        }

        // The validation was successful.

        // Get the validated data.
        $validData = $this->validator->getValidated();

        // ...
    }
}

Catatan

Validasi juga dapat ditangani secara otomatis di model, tetapi terkadang lebih mudah melakukannya di controller. Di mana pun itu terserah Anda.

Melindungi Method

Dalam beberapa kasus, Anda mungkin ingin method tertentu disembunyikan dari akses publik. Untuk mencapai ini, cukup deklarasikan method sebagai private atau protected. Itu akan mencegahnya dilayani oleh permintaan URL.

Misalnya, jika Anda mendefinisikan method seperti ini untuk controller Helloworld:

<?php

namespace App\Controllers;

class Helloworld extends BaseController
{
    protected function utility()
    {
        // some code
    }
}

dan untuk mendefinisikan router (helloworld/utitilty) untuk method. Maka mencoba mengaksesnya menggunakan URL berikut tidak akan berfungsi:

example.com/index.php/helloworld/utility

Perutean otomatis juga tidak akan berfungsi.

Perutean Otomatis (Ditingkatkan)

Baru pada versi 4.2.0.

Perutean Otomatis (Ditingkatkan) adalah sistem perutean otomatis baru yang lebih aman.

Lihat Auto Routing (Improved) untuk detailnya.

Perutean Otomatis (Legacy)

Penting

Fitur ini hanya ada untuk kompatibilitas mundur. Jangan menggunakannya dalam proyek baru. Bahkan jika Anda sudah menggunakannya, kami merekomendasikan Anda untuk menggunakan Auto Routing (Improved) sebagai gantinya.

Bagian ini menjelaskan fungsionalitas Perutean Otomatis (Legacy) yang merupakan sistem perutean dari CodeIgniter 3. Ini secara otomatis merutekan permintaan HTTP, dan mengeksekusi method controller yang sesuai tanpa definisi router. Perutean otomatis dinonaktifkan secara default.

Peringatan

Untuk mencegah kesalahan konfigurasi dan kesalahan pengkodean, kami merekomendasikan agar Anda tidak menggunakan Perutean Otomatis (Legacy). Mudah untuk membuat aplikasi yang rentan di mana filter controller atau perlindungan CSRF dilewati.

Penting

Perutean Otomatis (Legacy) merutekan permintaan HTTP dengan method HTTP apa pun ke method controller.

Penting

Sejak v4.5.0, jika Perutean Otomatis (Legacy) tidak menemukan controller, itu akan memunculkan pengecualian PageNotFoundException sebelum Filter Controller dieksekusi.

Pertimbangkan URI ini:

example.com/index.php/helloworld/

Dalam contoh di atas, CodeIgniter akan mencoba menemukan controller bernama Helloworld.php dan memuatnya.

Catatan

Ketika nama pendek controller cocok dengan segmen pertama URI, itu akan dimuat.

Mari kita coba: Hello World! (Legacy)

Mari kita buat controller sederhana agar Anda dapat melihatnya beraksi. Menggunakan editor teks Anda, buat berkas bernama Helloworld.php, dan masukkan kode berikut di dalamnya. Anda akan melihat bahwa Controller Helloworld memperluas BaseController. Anda juga dapat memperluas CodeIgniter\Controller jika Anda tidak membutuhkan fungsionalitas BaseController.

BaseController menyediakan tempat yang nyaman untuk memuat komponen dan melakukan fungsi yang dibutuhkan oleh semua controller Anda. Anda dapat memperluas class ini di controller baru mana pun.

Untuk alasan keamanan, pastikan untuk mendeklarasikan method utilitas baru apa pun sebagai protected atau private:

<?php

namespace App\Controllers;

class Helloworld extends BaseController
{
    public function index()
    {
        return 'Hello World!';
    }
}

Kemudian simpan berkas ke direktori app/Controllers Anda.

Penting

Berkas harus diberi nama Helloworld.php, dengan huruf kapital H. Saat Anda menggunakan Perutean Otomatis, nama class Controller HARUS dimulai dengan huruf kapital dan HANYA karakter pertama yang boleh huruf kapital.

Sekarang kunjungi situs Anda menggunakan URL yang mirip dengan ini:

example.com/index.php/helloworld

Jika Anda melakukannya dengan benar, Anda akan melihat:

Hello World!

Ini valid:

<?php

namespace App\Controllers;

class Helloworld extends BaseController
{
    // ...
}

Ini tidak valid:

<?php

namespace App\Controllers;

class helloworld extends BaseController
{
    // ...
}

Ini tidak valid:

<?php

namespace App\Controllers;

class HelloWorld extends BaseController
{
    // ...
}

Juga, selalu pastikan controller Anda memperluas class controller induk sehingga dapat mewarisi semua methodnya.

Catatan

Sistem akan mencoba mencocokkan URI dengan Controller dengan mencocokkan setiap segmen dengan direktori/berkas di app/Controllers, ketika kecocokan tidak ditemukan dengan router yang didefinisikan. Itulah mengapa nama direktori/berkas Anda HARUS dimulai dengan huruf kapital dan sisanya HARUS huruf kecil.

Jika Anda ingin konvensi penamaan lain, Anda perlu mendefinisikannya secara manual menggunakan Perutean Router yang Didefinisikan. Berikut adalah contoh berdasarkan PSR-4 Autoloader:

<?php

/*
 * Folder and file structure:
 * \<NamespaceName>(\<SubNamespaceNames>)*\<ClassName>
 */

$routes->get('helloworld', '\App\Controllers\HelloWorld::index');

Method (Legacy)

Dalam contoh di atas, nama method adalah index(). Method index() selalu dimuat secara default jika segmen kedua dari URI kosong. Cara lain untuk menampilkan pesan "Hello World" Anda adalah ini:

example.com/index.php/helloworld/index/

Segmen kedua dari URI menentukan method mana di controller yang dipanggil.

Mari kita coba. Tambahkan method baru ke controller Anda:

<?php

namespace App\Controllers;

class Helloworld extends BaseController
{
    public function index()
    {
        return 'Hello World!';
    }

    public function comment()
    {
        return 'I am not flat!';
    }
}

Sekarang muat URL berikut untuk melihat method komentar:

example.com/index.php/helloworld/comment/

Anda akan melihat pesan baru Anda.

Meneruskan Segmen URI ke Method Anda (Legacy)

Jika URI Anda berisi lebih dari dua segmen, mereka akan diteruskan ke method Anda sebagai parameter.

Misalnya, katakanlah Anda memiliki URI seperti ini:

example.com/index.php/products/shoes/sandals/123

Method Anda akan diteruskan segmen URI 3 dan 4 ('sandals' dan '123'):

<?php

namespace App\Controllers;

class Products extends BaseController
{
    public function shoes($sandals, $id)
    {
        return $sandals . $id;
    }
}

Controller Default (Legacy)

Controller Default adalah controller khusus yang digunakan ketika URI berakhir dengan nama direktori atau ketika URI tidak ada, seperti yang akan terjadi ketika hanya URL root situs Anda diminta.

Mendefinisikan Controller Default (Legacy)

Mari kita coba dengan controller Helloworld.

Untuk menentukan controller default, buka berkas app/Config/Routing.php Anda dan atur properti ini:

public string $defaultController = 'Helloworld';

Di mana Helloworld adalah nama class controller yang ingin Anda gunakan.

Dan komentari baris di app/Config/Routes.php:

$routes->get('/', 'Home::index');

Jika Anda sekarang menjelajahi situs Anda tanpa menentukan segmen URI apa pun, Anda akan melihat pesan "Hello World".

Catatan

Baris $routes->get('/', 'Home::index'); adalah optimasi yang ingin Anda gunakan dalam aplikasi "dunia nyata". Tetapi untuk tujuan demonstrasi kami tidak ingin menggunakan fitur itu. $routes->get() dijelaskan dalam Perutean URI

Untuk informasi lebih lanjut, silakan merujuk ke dokumentasi Opsi Konfigurasi (Legacy).

Mengorganisir Controller Anda ke dalam Sub-direktori (Legacy)

Jika Anda membangun aplikasi besar, Anda mungkin ingin secara hierarkis mengatur atau menyusun controller Anda ke dalam sub-direktori. CodeIgniter mengizinkan Anda untuk melakukan ini.

Cukup buat sub-direktori di bawah app/Controllers utama, dan tempatkan class controller Anda di dalamnya.

Penting

Nama direktori HARUS dimulai dengan huruf kapital dan HANYA karakter pertama yang boleh huruf kapital.

Saat menggunakan fitur ini, segmen pertama URI Anda harus menentukan direktori. Misalnya, katakanlah Anda memiliki controller yang terletak di sini:

app/Controllers/Products/Shoes.php

Untuk memanggil controller di atas, URI Anda akan terlihat seperti ini:

example.com/index.php/products/shoes/show/123

Catatan

Anda tidak dapat memiliki direktori dengan nama yang sama di app/Controllers dan public/. Ini karena jika ada direktori, web server akan mencarinya dan itu tidak akan dirutekan ke CodeIgniter.

Setiap sub-direktori Anda dapat berisi controller default yang akan dipanggil jika URL hanya berisi sub-direktori. Cukup tempatkan controller di sana yang cocok dengan nama controller default Anda seperti yang ditentukan di berkas app/Config/Routing.php Anda.

CodeIgniter juga mengizinkan Anda untuk memetakan URI Anda menggunakan Perutean Router yang Didefinisikan..

Memetakan Ulang Panggilan Method

Catatan

Perutean Otomatis (Ditingkatkan) tidak mendukung fitur ini secara sengaja.

Seperti yang disebutkan di atas, segmen kedua dari URI biasanya menentukan method mana di controller yang dipanggil. CodeIgniter mengizinkan Anda untuk menimpa perilaku ini melalui penggunaan method _remap():

<?php

namespace App\Controllers;

class Products extends BaseController
{
    public function _remap()
    {
        // Some code here...
    }
}

Penting

Jika controller Anda berisi method bernama _remap(), itu akan selalu dipanggil terlepas dari apa yang terkandung dalam URI Anda. Ini menimpa perilaku normal di mana URI menentukan method mana yang dipanggil, memungkinkan Anda untuk mendefinisikan aturan perutean method Anda sendiri.

Panggilan method yang ditimpa (biasanya segmen kedua dari URI) akan diteruskan sebagai parameter ke method _remap():

<?php

namespace App\Controllers;

class Products extends BaseController
{
    public function _remap($method)
    {
        if ($method === 'some_method') {
            return $this->{$method}();
        }

        return $this->default_method();
    }
}

Segmen tambahan apa pun setelah nama method diteruskan ke _remap(). Parameter ini dapat diteruskan ke method untuk meniru perilaku default CodeIgniter.

Contoh:

<?php

namespace App\Controllers;

class Products extends BaseController
{
    public function _remap($method, ...$params)
    {
        $method = 'process_' . $method;

        if (method_exists($this, $method)) {
            return $this->{$method}(...$params);
        }

        throw \CodeIgniter\Exceptions\PageNotFoundException::forPageNotFound();
    }
}
