Filter Controller memungkinkan Anda untuk melakukan tindakan baik sebelum atau sesudah eksekusi controller. Tidak seperti event, Anda dapat memilih URI atau router spesifik di mana filter akan diterapkan. Filter sebelum dapat memodifikasi Request sementara filter sesudah dapat bertindak dan bahkan memodifikasi Response, memungkinkan banyak fleksibilitas dan kekuatan.

Beberapa contoh umum tugas yang mungkin dilakukan dengan filter adalah:

  • Melakukan perlindungan CSRF pada permintaan masuk

  • Membatasi area situs Anda berdasarkan Role mereka

  • Melakukan pembatasan laju pada endpoint tertentu

  • Menampilkan halaman "Sedang dalam Perbaikan"

  • Melakukan negosiasi konten otomatis

  • dan banyak lagi...

Membuat Filter

Filter adalah class sederhana yang mengimplementasikan CodeIgniter\Filters\FilterInterface. Mereka berisi dua method: before() dan after() yang menampung kode yang akan berjalan sebelum dan sesudah controller secara berurutan. Class Anda harus berisi kedua method tetapi dapat membiarkan method kosong jika tidak diperlukan. Kerangka class filter terlihat seperti:

<?php

namespace App\Filters;

use CodeIgniter\Filters\FilterInterface;
use CodeIgniter\HTTP\RequestInterface;
use CodeIgniter\HTTP\ResponseInterface;

class MyFilter implements FilterInterface
{
    public function before(RequestInterface $request, $arguments = null)
    {
        // Do something here
    }

    public function after(RequestInterface $request, ResponseInterface $response, $arguments = null)
    {
        // Do something here
    }
}

Filter Sebelum

Mengganti Request

Dari filter mana pun, Anda dapat mengembalikan objek $request dan itu akan mengganti Request saat ini, memungkinkan Anda untuk membuat perubahan yang masih akan ada saat controller dieksekusi.

Menghentikan Filter Selanjutnya

Juga, ketika Anda memiliki serangkaian filter, Anda mungkin juga ingin menghentikan eksekusi filter selanjutnya setelah filter tertentu. Anda dapat dengan mudah melakukannya dengan mengembalikan hasil non-kosong apa pun. Jika filter sebelum mengembalikan hasil kosong, tindakan controller atau filter selanjutnya masih akan dieksekusi.

Pengecualian untuk aturan hasil non-kosong adalah instance Request. Mengembalikannya di filter sebelum tidak akan menghentikan eksekusi tetapi hanya mengganti objek $request saat ini.

Mengembalikan Response

Karena filter sebelum dieksekusi sebelum controller Anda dieksekusi, Anda mungkin kadang-kadang ingin menghentikan tindakan di controller agar tidak terjadi.

Ini biasanya digunakan untuk melakukan pengalihan, seperti dalam contoh ini:

<?php

namespace App\Filters;

use CodeIgniter\Filters\FilterInterface;
use CodeIgniter\HTTP\RequestInterface;
use CodeIgniter\HTTP\ResponseInterface;

class MyFilter implements FilterInterface
{
    public function before(RequestInterface $request, $arguments = null)
    {
        $auth = service('auth');

        if (! $auth->isLoggedIn()) {
            return redirect()->to(site_url('login'));
        }
    }
}

Jika instance Response dikembalikan, Response akan dikirim kembali ke klien dan eksekusi skrip akan berhenti. Ini dapat berguna untuk mengimplementasikan pembatasan laju untuk API. Lihat Throttler untuk sebuah contoh.

Filter Sesudah

Filter sesudah hampir identik dengan filter sebelum, kecuali bahwa Anda hanya dapat mengembalikan objek $response, dan Anda tidak dapat menghentikan eksekusi skrip. Ini memungkinkan Anda untuk memodifikasi output akhir, atau hanya melakukan sesuatu dengan output akhir. Ini dapat digunakan untuk memastikan header keamanan tertentu diatur dengan cara yang benar, atau untuk menyimpan output akhir, atau bahkan untuk memfilter output akhir dengan filter kata-kata buruk.

Mengkonfigurasi Filter

Ada dua cara untuk mengkonfigurasi filter saat mereka dijalankan. Salah satunya dilakukan di app/Config/Filters.php, yang lainnya dilakukan di app/Config/Routes.php.

Jika Anda ingin menentukan filter ke router yang didefinisikan, gunakan app/Config/Routes.php dan lihat Perutean URI.

Catatan

Cara teraman untuk menerapkan filter adalah dengan menonaktifkan perutean otomatis, dan mengatur filter ke router.

app/Config/Filters.php

Berkas app/Config/Filters.php berisi empat properti yang memungkinkan Anda untuk mengkonfigurasi kapan tepatnya filter berjalan.

Peringatan

Disarankan agar Anda selalu menambahkan * di akhir URI dalam pengaturan filter. Karena method controller mungkin dapat diakses oleh URL yang berbeda dari yang Anda kira. Misalnya, ketika Perutean Otomatis (Legacy) diaktifkan, jika Anda memiliki Blog::index(), itu dapat diakses dengan blog, blog/index, dan blog/index/1, dll.

$aliases

Array $aliases digunakan untuk mengaitkan nama sederhana dengan satu atau lebih nama class yang sepenuhnya memenuhi syarat yang merupakan filter yang akan dijalankan:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class Filters extends BaseConfig
{
    public array $aliases = [
        'csrf' => \CodeIgniter\Filters\CSRF::class,
    ];

    // ...
}

Alias adalah wajib dan jika Anda mencoba menggunakan nama class lengkap nanti, sistem akan memunculkan error.

Mendefinisikannya dengan cara ini membuatnya sederhana untuk mengganti class yang digunakan. Sangat bagus untuk ketika Anda memutuskan Anda perlu beralih ke sistem otentikasi yang berbeda karena Anda hanya mengubah class filter dan Anda selesai.

Anda dapat menggabungkan beberapa filter menjadi satu alias, membuat set filter yang kompleks mudah diterapkan:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class Filters extends BaseConfig
{
    public array $aliases = [
        'api-prep' => [
            \App\Filters\Negotiate::class,
            \App\Filters\ApiAuth::class,
        ],
    ];

    // ...
}

Anda harus mendefinisikan alias sebanyak yang Anda butuhkan.

$required

Baru pada versi 4.5.0.

Bagian kedua memungkinkan Anda untuk mendefinisikan Filter yang Diperlukan. Mereka adalah filter khusus yang diterapkan pada setiap permintaan yang dibuat oleh framework. Mereka diterapkan sebelum dan sesudah jenis filter lain yang dijelaskan di bawah ini.

Catatan

Filter yang Diperlukan selalu dieksekusi. Namun, jika router tidak ada, hanya Filter Sebelum yang dieksekusi.

Anda harus berhati-hati dengan berapa banyak yang Anda gunakan di sini, karena itu bisa memiliki implikasi kinerja untuk terlalu banyak berjalan pada setiap permintaan. Tetapi filter yang diatur secara default menyediakan fungsionalitas framework. Jika dihapus, fungsi-fungsi tersebut tidak akan berfungsi lagi. Lihat Filter yang Disediakan untuk detailnya.

Filter dapat ditentukan dengan menambahkan alias mereka ke array before atau after:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class Filters extends BaseConfig
{
    // ...
    public array $required = [
        'before' => [
            'forcehttps', // Force Global Secure Requests
            'pagecache',  // Web Page Caching
        ],
        'after' => [
            'pagecache',   // Web Page Caching
            'performance', // Performance Metrics
            'toolbar',     // Debug Toolbar
        ],
    ];

    // ...
}

$globals

Bagian ketiga memungkinkan Anda untuk mendefinisikan filter apa pun yang harus diterapkan pada setiap permintaan valid yang dibuat oleh framework.

Anda harus berhati-hati dengan berapa banyak yang Anda gunakan di sini, karena itu bisa memiliki implikasi kinerja untuk terlalu banyak berjalan pada setiap permintaan.

Filter dapat ditentukan dengan menambahkan alias mereka ke array before atau after:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class Filters extends BaseConfig
{
    // ...

    public array $globals = [
        'before' => [
            'csrf',
        ],
        'after' => [],
    ];

    // ...
}
Kecuali untuk Beberapa URI

Ada kalanya Anda ingin menerapkan filter ke hampir setiap permintaan, tetapi ada beberapa yang harus dibiarkan. Salah satu contoh umum adalah jika Anda perlu mengecualikan beberapa URI dari filter perlindungan CSRF untuk memungkinkan permintaan dari situs web pihak ketiga untuk mencapai satu atau dua URI spesifik, sambil menjaga sisanya tetap terlindungi.

Untuk melakukan ini, tambahkan array dengan kunci except dan jalur URI (relatif terhadap BaseURL) untuk dicocokkan sebagai nilai di samping alias:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class Filters extends BaseConfig
{
    // ...

    public array $globals = [
        'before' => [
            'csrf' => ['except' => 'api/*'],
        ],
        'after' => [],
    ];

    // ...
}

Peringatan

Sebelum v4.4.7, karena bug, jalur URI yang diproses oleh filter tidak di-URL-decode. Dengan kata lain, jalur URI yang ditentukan dalam perutean dan jalur URI yang ditentukan dalam filter bisa berbeda. Lihat Paths in Controller Filters untuk detailnya.

Di mana pun Anda dapat menggunakan jalur URI (relatif terhadap BaseURL) dalam pengaturan filter, Anda dapat menggunakan ekspresi reguler atau, seperti dalam contoh di atas, menggunakan tanda bintang (*) untuk wildcard yang akan mencocokkan semua karakter setelah itu. Dalam contoh ini, jalur URI apa pun yang dimulai dengan api/ akan dikecualikan dari perlindungan CSRF, tetapi form situs semuanya akan dilindungi.

Jika Anda perlu menentukan beberapa jalur URI, Anda dapat menggunakan array pola jalur URI:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class Filters extends BaseConfig
{
    // ...

    public array $globals = [
        'before' => [
            'csrf' => ['except' => ['foo/*', 'bar/*']],
        ],
        'after' => [],
    ];

    // ...
}

$methods

Peringatan

Jika Anda menggunakan filter $methods, Anda harus menonaktifkan Perutean Otomatis (Legacy) karena Perutean Otomatis (Legacy) mengizinkan method HTTP apa pun untuk mengakses controller. Mengakses controller dengan method yang tidak Anda harapkan dapat melewati filter.

Anda dapat menerapkan filter ke semua permintaan dari method HTTP tertentu, seperti POST, GET, PUT, dll. Nilainya akan menjadi array filter yang akan dijalankan:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class Filters extends BaseConfig
{
    // ...

    public array $methods = [
        'POST' => ['invalidchars', 'csrf'],
        'GET'  => ['csrf'],
    ];

    // ...
}

Catatan

Tidak seperti properti $globals atau $filters, ini hanya akan berjalan sebagai filter sebelum.

Selain method HTTP standar, ini juga mendukung satu kasus khusus: CLI. Method CLI akan berlaku untuk semua permintaan yang dijalankan dari baris perintah.

Catatan

Sebelum v4.5.0, karena bug, Anda perlu menentukan nama method HTTP dalam huruf kecil.

$filters

Properti ini adalah array alias filter. Untuk setiap alias, Anda dapat menentukan array before dan after yang berisi daftar pola jalur URI (relatif terhadap BaseURL) yang harus diterapkan oleh filter:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class Filters extends BaseConfig
{
    // ...

    public array $filters = [
        'foo' => ['before' => ['admin/*'], 'after' => ['users/*']],
        'bar' => ['before' => ['api/*', 'admin/*']],
    ];

    // ...
}

Peringatan

Sebelum v4.4.7, karena bug, jalur URI yang diproses oleh filter tidak di-URL-decode. Dengan kata lain, jalur URI yang ditentukan dalam perutean dan jalur URI yang ditentukan dalam filter bisa berbeda. Lihat Paths in Controller Filters untuk detailnya.

Argumen Filter

Baru pada versi 4.4.0.

Saat mengkonfigurasi $filters, argumen tambahan dapat diteruskan ke filter:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class Filters extends BaseConfig
{
    // ...

    public $filters = [
        'group:admin,superadmin'  => ['before' => ['admin/*']],
        'permission:users.manage' => ['before' => ['admin/users/*']],
    ];

    // ...
}

Dalam contoh ini, ketika URI cocok dengan admin/*', array ['admin', 'superadmin'] akan diteruskan dalam $arguments ke method before() filter group. Ketika URI cocok dengan admin/users/*', array ['users.manage'] akan diteruskan dalam $arguments ke method before() filter permission.

Catatan

Sebelum v4.6.0, filter yang sama tidak dapat dijalankan beberapa kali dengan argumen yang berbeda.

Urutan Eksekusi Filter

Penting

Mulai v4.5.0, urutan eksekusi filter telah berubah. Jika Anda ingin mempertahankan urutan eksekusi yang sama seperti pada versi sebelumnya, Anda harus mengatur true ke Config\Feature::$oldFilterOrder.

Filter dieksekusi dalam urutan berikut:

  • Filter Sebelum: required → globals → methods → filters → route

  • Filter Sesudah: route → filters → globals → required

Catatan

Filter required dapat digunakan sejak v4.5.0.

Catatan

Sebelum v4.5.0, filter yang ditentukan ke router (di app/Config/Routes.php) dieksekusi sebelum filter yang ditentukan di app/Config/Filters.php. Dan urutan eksekusi Filter Sesudah dalam filter Router dan filter Filters tidak dibalik. Lihat Panduan Peningkatan untuk detailnya.

Mengkonfirmasi Filter

CodeIgniter memiliki perintah berikut untuk memeriksa filter untuk sebuah router.

filter:check

Baru pada versi 4.3.0.

Misalnya, periksa filter untuk router / dengan method GET:

php spark filter:check get /

Outputnya seperti berikut:

+--------+-------+----------------------+-------------------------------+
| Method | Router| Filter Sebelum       | Filter Sesudah                |
+--------+-------+----------------------+-------------------------------+
| GET    | /     | forcehttps pagecache | pagecache performance toolbar |
+--------+-------+----------------------+-------------------------------+

Class Filter Sebelum:
CodeIgniter\Filters\ForceHTTPS → CodeIgniter\Filters\PageCache
Class Filter Sesudah:
CodeIgniter\Filters\PageCache → CodeIgniter\Filters\PerformanceMetrics → CodeIgniter\Filters\DebugToolbar

Catatan

Sejak v4.6.0, argumen filter telah ditampilkan dalam tabel output. Juga, nama class filter yang sebenarnya telah ditampilkan di akhir.

Anda juga dapat melihat router dan filter dengan perintah spark routes, tetapi mungkin tidak menampilkan filter yang akurat ketika Anda menggunakan ekspresi reguler untuk router. Lihat Perutean URI untuk detailnya.

Filter yang Disediakan

Filter yang dibundel dengan CodeIgniter4 adalah:

  • cors => Cross-Origin Resource Sharing (CORS)

  • csrf => CSRF

  • toolbar => DebugToolbar

  • honeypot => Honeypot

  • invalidchars => InvalidChars

  • secureheaders => SecureHeaders

  • forcehttps => ForceHTTPS

  • pagecache => PageCache

  • performance => PerformanceMetrics

Catatan

Filter dieksekusi dalam urutan yang ditentukan dalam berkas konfigurasi. Namun, jika diaktifkan, DebugToolbar selalu dieksekusi terakhir karena harus dapat menangkap semua yang terjadi di filter lain.

ForceHTTPS

Baru pada versi 4.5.0.

Filter ini menyediakan fitur "Paksa Permintaan Aman Global".

Jika Anda mengatur Config\App:$forceGlobalSecureRequests ke true, ini akan memaksa setiap permintaan yang dibuat ke aplikasi ini untuk dilakukan melalui koneksi aman (HTTPS). Jika permintaan masuk tidak aman, pengguna akan diarahkan ke versi aman halaman dan header HTTP Strict Transport Security (HSTS) akan diatur.

PerformanceMetrics

Baru pada versi 4.5.0.

Filter ini menyediakan pseudo-variabel untuk metrik kinerja.

Jika Anda ingin menampilkan total waktu yang berlalu dari saat CodeIgniter dimulai hingga saat tepat sebelum output akhir dikirim ke browser, cukup tempatkan pseudo-variabel ini di salah satu tampilan Anda:

{elapsed_time}

Jika Anda ingin menampilkan penggunaan memori Anda di berkas tampilan Anda, gunakan pseudo-variabel ini:

{memory_usage}

Jika Anda tidak membutuhkan fitur ini, hapus 'performance' dari $required['after'].

InvalidChars

Filter ini melarang data input pengguna ($_GET, $_POST, $_COOKIE, php://input) mengandung karakter berikut:

  • karakter UTF-8 tidak valid

  • karakter kontrol kecuali baris baru dan kode tab

SecureHeaders

Filter ini menambahkan header respons HTTP yang dapat digunakan aplikasi Anda untuk meningkatkan keamanan aplikasi Anda.

Jika Anda ingin menyesuaikan header, perluas CodeIgniter\Filters\SecureHeaders dan timpa properti $headers. Dan ubah properti $aliases di app/Config/Filters.php:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class Filters extends BaseConfig
{
    public array $aliases = [
        // ...
        'secureheaders' => \App\Filters\SecureHeaders::class,
    ];

    // ...
}

Jika Anda ingin tahu tentang header aman, lihat OWASP Secure Headers Project.
