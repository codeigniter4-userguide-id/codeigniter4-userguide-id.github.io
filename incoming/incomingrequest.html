Class IncomingRequest

Class IncomingRequest menyediakan representasi berorientasi objek dari permintaan HTTP dari klien, seperti browser. Ini memperluas dari, dan memiliki akses ke semua method dari Class Request dan Class Message , selain method yang tercantum di bawah ini.

Mengakses Permintaan

Sebuah instance dari class request sudah diisi untuk Anda jika class saat ini adalah turunan dari CodeIgniter\Controller dan dapat diakses sebagai properti class:

<?php

namespace App\Controllers;

use CodeIgniter\Controller;

class UserController extends Controller
{
    public function index()
    {
        if ($this->request->isAJAX()) {
            // ...
        }
    }
}

Jika Anda tidak berada dalam controller, tetapi masih membutuhkan akses ke objek Request aplikasi, Anda dapat mendapatkan salinannya melalui class Services:

<?php

$request = service('request');

Namun, lebih baik untuk meneruskan permintaan sebagai dependensi jika class tersebut bukan controller, di mana Anda dapat menyimpannya sebagai properti class:

<?php

namespace App\Libraries;

use CodeIgniter\HTTP\RequestInterface;

class SomeClass
{
    protected $request;

    public function __construct(RequestInterface $request)
    {
        $this->request = $request;
    }
}

$someClass = new SomeClass(service('request'));

Menentukan Jenis Permintaan

Sebuah permintaan bisa dari beberapa jenis, termasuk permintaan AJAX atau permintaan dari baris perintah. Ini dapat diperiksa dengan method isAJAX() dan isCLI():

<?php

// Check for AJAX request.
if ($request->isAJAX()) {
    // ...
}

// Check for CLI Request
if ($request->isCLI()) {
    // ...
}

Catatan

Method isAJAX() bergantung pada header X-Requested-With, yang dalam beberapa kasus tidak dikirim secara default dalam permintaan XHR melalui JavaScript (yaitu, fetch). Lihat bagian Permintaan AJAX tentang cara menghindari masalah ini.

is()

Baru pada versi 4.3.0.

Sejak v4.3.0, Anda dapat menggunakan method is(). Ini menerima method HTTP, 'ajax', atau 'json', dan mengembalikan boolean.

Catatan

Method HTTP harus peka huruf besar/kecil, tetapi parameternya tidak peka huruf besar/kecil.

<?php

// Checks HTTP methods. Returns boolean.
$request->is('get');
$request->is('post');
$request->is('put');
$request->is('delete');
$request->is('head');
$request->is('patch');
$request->is('options');

// Checks if it is an AJAX request. The same as `$request->isAJAX()`.
$request->is('ajax');

// Checks if it is a JSON request.
$request->is('json');

getMethod()

Anda dapat memeriksa method HTTP yang diwakili oleh permintaan ini dengan method getMethod():

<?php

// Returns 'POST'
$method = $request->getMethod();

Method HTTP peka huruf besar/kecil, dan menurut konvensi, method standar didefinisikan dalam huruf besar US-ASCII.

Catatan

Sebelum v4.5.0, secara default, method dikembalikan sebagai string huruf kecil (yaitu, 'get', 'post', dll). Tapi itu adalah bug.

Anda bisa mendapatkan versi huruf kecil dengan membungkus panggilan di strtolower():

// Mengembalikan 'get'
$method = strtolower($request->getMethod());

Anda juga dapat memeriksa apakah permintaan dibuat melalui koneksi HTTPS dengan method isSecure():

<?php

if (! $request->isSecure()) {
    force_https();
}

Mengambil Input

Anda dapat mengambil input dari $_GET, $_POST, $_COOKIE, $_SERVER dan $_ENV melalui objek Request. Data tidak secara otomatis difilter dan mengembalikan data input mentah seperti yang diteruskan dalam permintaan.

Catatan

Merupakan praktik yang buruk untuk menggunakan variabel global. Pada dasarnya, itu harus dihindari dan disarankan untuk menggunakan method objek Request.

Keuntungan utama menggunakan method ini daripada mengaksesnya secara langsung ($_POST['something']), adalah bahwa method ini akan mengembalikan null jika item tidak ada, dan Anda dapat memfilter data. Ini memungkinkan Anda dengan mudah menggunakan data tanpa harus menguji apakah item ada terlebih dahulu. Dengan kata lain, biasanya Anda mungkin melakukan sesuatu seperti ini:

<?php

$something = $_POST['foo'] ?? null;

Dengan method bawaan CodeIgniter Anda cukup melakukan ini:

<?php

$something = $request->getPost('foo');

Mendapatkan Data

getGet()

Method getGet() akan menarik dari $_GET.

  • $request->getGet()

getPost()

Method getPost() akan menarik dari $_POST.

  • $request->getPost()

getCookie()

Method getCookie() akan menarik dari $_COOKIE.

  • $request->getCookie()

getServer()

Method getServer() akan menarik dari $_SERVER.

  • $request->getServer()

getEnv()

Ditinggalkan sejak versi 4.4.4: Method ini tidak berfungsi sejak awal. Gunakan env() sebagai gantinya.

Method getEnv() akan menarik dari $_ENV.

  • $request->getEnv()

getPostGet()

Selain itu, ada beberapa method utilitas untuk mengambil informasi dari $_GET atau $_POST, sambil mempertahankan kemampuan untuk mengontrol urutan Anda mencarinya:

  • $request->getPostGet() - memeriksa $_POST terlebih dahulu, lalu $_GET

getGetPost()

  • $request->getGetPost() - memeriksa $_GET terlebih dahulu, lalu $_POST

getVar()

Penting

Method ini hanya ada untuk kompatibilitas mundur. Jangan menggunakannya dalam proyek baru. Bahkan jika Anda sudah menggunakannya, kami merekomendasikan Anda untuk menggunakan method lain yang lebih sesuai.

Method getVar() akan menarik dari $_REQUEST, jadi akan mengembalikan data apa pun dari $_GET, $POST, atau $_COOKIE (tergantung pada request-order php.ini).

Peringatan

Jika Anda ingin memvalidasi data POST saja, jangan gunakan getVar(). Nilai yang lebih baru menimpa nilai yang lebih lama. Nilai POST dapat ditimpa oleh cookie jika memiliki nama yang sama, dan Anda mengatur "C" setelah "P" di request-order.

Catatan

Jika permintaan masuk memiliki header Content-Type yang diatur ke application/json, method getVar() mengembalikan data JSON daripada data $_REQUEST.

Mendapatkan Data JSON

Anda dapat mengambil konten php://input sebagai aliran JSON dengan getJSON().

Catatan

Ini tidak memiliki cara untuk memeriksa apakah data masuk adalah JSON yang valid atau tidak, Anda hanya boleh menggunakan method ini jika Anda tahu bahwa Anda mengharapkan JSON.

<?php

$json = $request->getJSON();

Secara default, ini akan mengembalikan objek apa pun dalam data JSON sebagai objek. Jika Anda ingin itu dikonversi ke array asosiatif, teruskan true sebagai parameter pertama.

Parameter kedua dan ketiga cocok dengan argumen $depth dan $flags dari fungsi PHP json_decode().

Mendapatkan Data Spesifik dari JSON

Anda bisa mendapatkan bagian data tertentu dari aliran JSON dengan meneruskan nama variabel ke getJsonVar() untuk data yang Anda inginkan atau Anda dapat menggunakan notasi "titik" untuk menggali JSON untuk mendapatkan data yang tidak berada di tingkat root.

<?php

/*
 * With a request body of:
 * {
 *     "foo": "bar",
 *     "fizz": {
 *         "buzz": "baz"
 *     }
 * }
 */

$data = $request->getJsonVar('foo');
// $data = "bar"

$data = $request->getJsonVar('fizz.buzz');
// $data = "baz"

Jika Anda ingin hasilnya menjadi array asosiatif daripada objek, Anda dapat meneruskan true di parameter kedua:

<?php

// With the same request as above
$data = $request->getJsonVar('fizz');
// $data->buzz = "baz"

$data = $request->getJsonVar('fizz', true);
// $data = ["buzz" => "baz"]

Catatan

Lihat dokumentasi untuk dot_array_search() di pembantu Array untuk informasi lebih lanjut tentang notasi "titik".

Mengambil Data Mentah (PUT, PATCH, DELETE)

Akhirnya, Anda dapat mengambil konten php://input sebagai aliran mentah dengan getRawInput():

<?php

$data = $request->getRawInput();

Ini akan mengambil data dan mengubahnya menjadi array. Seperti ini:

<?php

var_dump($request->getRawInput());

/*
 * Outputs:
 * [
 *     'Param1' => 'Value1',
 *     'Param2' => 'Value2',
 * ]
 */

Anda juga dapat menggunakan getRawInputVar(), untuk mendapatkan variabel yang ditentukan dari aliran mentah dan memfilternya.

<?php

// When the request body is 'foo=one&bar=two&baz[]=10&baz[]=20'
var_dump($request->getRawInputVar('bar'));
// Outputs: two

// foo=one&bar=two&baz[]=10&baz[]=20
var_dump($request->getRawInputVar(['foo', 'bar']));
/*
 * Outputs:
 * [
 *      'foo' => 'one',
 *      'bar' => 'two'
 * ]
 */

// foo=one&bar=two&baz[]=10&baz[]=20
var_dump($request->getRawInputVar('baz'));
/*
 * Outputs:
 * [
 *      '10',
 *      '20'
 * ]
 */

// foo=one&bar=two&baz[]=10&baz[]=20
var_dump($request->getRawInputVar('baz.0'));
// Outputs: 10

Memfilter Data Input

Untuk menjaga keamanan aplikasi Anda, Anda akan ingin memfilter semua input saat Anda mengaksesnya. Anda dapat meneruskan jenis filter yang akan digunakan sebagai parameter kedua dari method ini. Fungsi asli filter_var() digunakan untuk pemfilteran. Kunjungi manual PHP untuk daftar jenis filter yang valid.

Memfilter variabel POST akan terlihat seperti ini:

<?php

$email = $request->getPost('email', FILTER_SANITIZE_EMAIL);

Semua method yang disebutkan di atas mendukung jenis filter yang diteruskan sebagai parameter kedua, dengan pengecualian getJSON() dan getRawInput().

Mengambil Header

Anda dapat mengakses header apa pun yang dikirim dengan permintaan dengan method headers(), yang mengembalikan array semua header, dengan kunci sebagai nama header, dan nilainya adalah instance dari CodeIgniter\HTTP\Header:

<?php

var_dump($request->headers());

/*
 * Outputs:
 * [
 *     'Host'          => CodeIgniter\HTTP\Header,
 *     'Cache-Control' => CodeIgniter\HTTP\Header,
 *     'Accept'        => CodeIgniter\HTTP\Header,
 * ]
 */

Jika Anda hanya membutuhkan satu header, Anda dapat meneruskan nama ke method header(). Ini akan mengambil objek header yang ditentukan secara tidak peka huruf besar/kecil jika ada. Jika tidak, maka akan mengembalikan null:

<?php

// these are all equivalent
$host = $request->header('host');
$host = $request->header('Host');
$host = $request->header('HOST');

Anda selalu dapat menggunakan hasHeader() untuk melihat apakah header ada dalam permintaan ini:

<?php

if ($request->hasHeader('DNT')) {
    // Don't track something...
}

Jika Anda membutuhkan nilai header sebagai string dengan semua nilai dalam satu baris, Anda dapat menggunakan method getHeaderLine():

<?php

// Accept-Encoding: gzip, deflate, sdch
echo 'Accept-Encoding: ' . $request->getHeaderLine('accept-encoding');

Jika Anda membutuhkan seluruh header, dengan nama dan nilai dalam satu string, cukup ubah header menjadi string:

<?php

echo (string) $header;

URL Permintaan

Anda dapat mengambil objek URI yang mewakili URI saat ini untuk permintaan ini melalui method $request->getUri(). Anda dapat mengubah objek ini menjadi string untuk mendapatkan URL lengkap untuk permintaan saat ini:

<?php

$uri = (string) $request->getUri();

Objek memberi Anda kemampuan penuh untuk mengambil bagian mana pun dari permintaan secara mandiri:

<?php

$uri = $request->getUri();

echo $uri->getScheme();         // http
echo $uri->getAuthority();      // snoopy:password@example.com:88
echo $uri->getUserInfo();       // snoopy:password
echo $uri->getHost();           // example.com
echo $uri->getPort();           // 88
echo $uri->getPath();           // /path/to/page
echo $uri->getRoutePath();      // path/to/page
echo $uri->getQuery();          // foo=bar&bar=baz
print_r($uri->getSegments());   // Array ( [0] => path [1] => to [2] => page )
echo $uri->getSegment(1);       // path
echo $uri->getTotalSegments();  // 3

Anda dapat bekerja dengan string URI saat ini (jalur relatif terhadap baseURL Anda) menggunakan getRoutePath().

Catatan

Method getRoutePath() dapat digunakan sejak v4.4.0. Sebelum v4.4.0, method getPath() mengembalikan jalur relatif terhadap baseURL Anda.

Berkas yang Diunggah

Informasi tentang semua berkas yang diunggah dapat diambil melalui $request->getFiles(), yang mengembalikan array instance CodeIgniter\HTTP\Files\UploadedFile. Ini membantu meringankan kesulitan bekerja dengan berkas yang diunggah, dan menggunakan praktik terbaik untuk meminimalkan risiko keamanan.

<?php

$files = $request->getFiles();

Lihat Bekerja dengan Berkas yang Diunggah untuk detailnya.

Anda dapat mengambil satu berkas yang diunggah secara terpisah, berdasarkan nama berkas yang diberikan dalam input berkas HTML:

<?php

$file = $request->getFile('userfile');

Anda dapat mengambil array berkas dengan nama yang sama yang diunggah sebagai bagian dari pengunggahan multi-berkas, berdasarkan nama berkas yang diberikan dalam input berkas HTML:

<?php

$files = $request->getFileMultiple('userfile');

Catatan

Berkas-berkas di sini sesuai dengan $_FILES. Bahkan jika pengguna hanya mengklik tombol submit form dan tidak mengunggah berkas apa pun, berkas tersebut akan tetap ada. Anda dapat memeriksa bahwa berkas tersebut benar-benar diunggah dengan method isValid() di UploadedFile. Lihat Verify a File untuk detail lebih lanjut.

Negosiasi Konten

Anda dapat dengan mudah menegosiasikan jenis konten dengan permintaan melalui method negotiate():

<?php

$language    = $request->negotiate('language', ['en-US', 'en-GB', 'fr', 'es-mx']);
$imageType   = $request->negotiate('media', ['image/png', 'image/jpg']);
$charset     = $request->negotiate('charset', ['UTF-8', 'UTF-16']);
$contentType = $request->negotiate('media', ['text/html', 'text/xml']);
$encoding    = $request->negotiate('encoding', ['gzip', 'compress']);

Lihat halaman Negosiasi Konten untuk detail lebih lanjut.

class CodeIgniter\HTTP\IncomingRequest
isCLI()
Kembali:

True jika permintaan dimulai dari baris perintah, jika tidak false.

Return type:

bool

isAJAX()
Kembali:

True jika permintaan adalah permintaan AJAX, jika tidak false.

Return type:

bool

isSecure()
Kembali:

True jika permintaan adalah permintaan HTTPS, jika tidak false.

Return type:

bool

getVar([$index = null[, $filter = null[, $flags = null]]])
Parameter:

  • $index (string) -- Nama variabel/kunci yang akan dicari.

  • $filter (int) -- Jenis filter yang akan diterapkan. Daftar filter dapat ditemukan di Jenis filter.

  • $flags (int) -- Flag yang akan diterapkan. Daftar flag dapat ditemukan di Flag filter.

Kembali:

$_REQUEST jika tidak ada parameter yang diberikan, jika tidak nilai REQUEST jika ditemukan, atau null jika tidak

Return type:

array|bool|float|int|object|string|null

Penting

Method ini hanya ada untuk kompatibilitas mundur. Jangan menggunakannya dalam proyek baru. Bahkan jika Anda sudah menggunakannya, kami merekomendasikan Anda untuk menggunakan method lain yang lebih sesuai.

Method ini identik dengan getGet(), hanya saja ini mengambil data REQUEST.

getGet([$index = null[, $filter = null[, $flags = null]]])
Parameter:

  • $index (string) -- Nama variabel/kunci yang akan dicari.

  • $filter (int) -- Jenis filter yang akan diterapkan. Daftar filter dapat ditemukan di Jenis filter.

  • $flags (int) -- Flag yang akan diterapkan. Daftar flag dapat ditemukan di Flag filter.

Kembali:

$_GET jika tidak ada parameter yang diberikan, jika tidak nilai GET jika ditemukan, atau null jika tidak

Return type:

array|bool|float|int|object|string|null

Parameter pertama akan berisi nama item GET yang Anda cari:

<?php

$request->getGet('some_data');

Method mengembalikan null jika item yang Anda coba ambil tidak ada.

Parameter opsional kedua memungkinkan Anda menjalankan data melalui filter PHP. Berikan jenis filter yang diinginkan sebagai parameter kedua:

<?php

$request->getGet('some_data', FILTER_SANITIZE_FULL_SPECIAL_CHARS);

Untuk mengembalikan array dari semua item GET panggil tanpa parameter apa pun.

Untuk mengembalikan semua item GET dan meneruskannya melalui filter, atur parameter pertama ke null sambil mengatur parameter kedua ke filter yang ingin Anda gunakan:

<?php

$request->getGet(null, FILTER_SANITIZE_FULL_SPECIAL_CHARS);
// returns all GET items with string sanitation

Untuk mengembalikan array dari beberapa parameter GET, berikan semua kunci yang diperlukan sebagai array:

<?php

$request->getGet(['field1', 'field2']);

Aturan yang sama berlaku di sini, untuk mengambil parameter dengan pemfilteran, atur parameter kedua ke jenis filter yang akan diterapkan:

<?php

$request->getGet(['field1', 'field2'], FILTER_SANITIZE_FULL_SPECIAL_CHARS);
getPost([$index = null[, $filter = null[, $flags = null]]])
Parameter:

  • $index (string) -- Nama variabel/kunci yang akan dicari.

  • $filter (int) -- Jenis filter yang akan diterapkan. Daftar filter dapat ditemukan di sini.

  • $flags (int) -- Flag yang akan diterapkan. Daftar flag dapat ditemukan di sini.

Kembali:

$_POST jika tidak ada parameter yang diberikan, jika tidak nilai POST jika ditemukan, atau null jika tidak

Return type:

array|bool|float|int|object|string|null

Method ini identik dengan getGet(), hanya saja ini mengambil data POST.

getPostGet([$index = null[, $filter = null[, $flags = null]]])
Parameter:

  • $index (string) -- Nama variabel/kunci yang akan dicari.

  • $filter (int) -- Jenis filter yang akan diterapkan. Daftar filter dapat ditemukan di Jenis filter.

  • $flags (int) -- Flag yang akan diterapkan. Daftar flag dapat ditemukan di Flag filter.

Kembali:

$_POST dan $_GET digabungkan jika tidak ada parameter yang ditentukan (lebih suka nilai POST jika terjadi konflik), jika tidak mencari nilai POST, jika tidak ada yang ditemukan mencari nilai GET, jika tidak ada nilai yang ditemukan mengembalikan null
