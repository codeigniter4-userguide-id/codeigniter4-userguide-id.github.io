Penanganan Resource RESTful

Representational State Transfer (REST) adalah gaya arsitektur untuk aplikasi terdistribusi, pertama kali dijelaskan oleh Roy Fielding dalam disertasi PhD-nya tahun 2000, Architectural Styles and the Design of Network-based Software Architectures. Itu mungkin bacaan yang agak kering, dan Anda mungkin menemukan Richardson Maturity Model dari Martin Fowler sebagai pengantar yang lebih lembut.

REST telah diinterpretasikan, dan disalahartikan, dengan lebih banyak cara daripada kebanyakan arsitektur perangkat lunak, dan mungkin lebih mudah untuk mengatakan bahwa semakin banyak prinsip Roy Fielding yang Anda terapkan dalam arsitektur, semakin "RESTful" aplikasi Anda akan dianggap.

CodeIgniter memudahkan pembuatan API RESTful untuk resource Anda, dengan router resource dan ResourceController-nya.

Router Resource

Anda dapat dengan cepat membuat beberapa router RESTful untuk satu resource dengan method resource(). Ini membuat lima router paling umum yang dibutuhkan untuk CRUD lengkap sebuah resource: membuat resource baru, memperbarui yang sudah ada, mencantumkan semua resource tersebut, menampilkan satu resource, dan menghapus satu resource. Parameter pertama adalah nama resource:

<?php

$routes->resource('photos');

// Equivalent to the following:
$routes->get('photos/new', 'Photos::new');
$routes->post('photos', 'Photos::create');
$routes->get('photos', 'Photos::index');
$routes->get('photos/(:segment)', 'Photos::show/$1');
$routes->get('photos/(:segment)/edit', 'Photos::edit/$1');
$routes->put('photos/(:segment)', 'Photos::update/$1');
$routes->patch('photos/(:segment)', 'Photos::update/$1');
$routes->delete('photos/(:segment)', 'Photos::delete/$1');

Catatan

Urutan di atas adalah untuk kejelasan, sedangkan urutan sebenarnya router dibuat di RouteCollection, memastikan resolusi router yang tepat

Penting

Router dicocokkan dalam urutan yang ditentukan, jadi jika Anda memiliki resource foto di atas get 'photos/poll', router tindakan show untuk baris resource akan dicocokkan sebelum baris get. Untuk memperbaikinya, pindahkan baris get di atas baris resource agar dicocokkan terlebih dahulu.

Parameter kedua menerima array opsi yang dapat digunakan untuk memodifikasi router yang dihasilkan. Meskipun router ini ditujukan untuk penggunaan API, di mana lebih banyak method diizinkan, Anda dapat meneruskan opsi websafe agar menghasilkan method update dan delete yang berfungsi dengan form HTML:

<?php

$routes->resource('photos', ['websafe' => 1]);

// The following equivalent routes are created:
$routes->post('photos/(:segment)/delete', 'Photos::delete/$1');
$routes->post('photos/(:segment)', 'Photos::update/$1');

Mengubah Controller yang Digunakan

Anda dapat menentukan controller yang harus digunakan dengan meneruskan opsi controller dengan nama controller yang harus digunakan:

<?php

$routes->resource('photos', ['controller' => 'Gallery']);
// Would create routes like:
$routes->get('photos', '\App\Controllers\Gallery::index');

<?php

$routes->resource('photos', ['controller' => '\App\Gallery']);
// Would create routes like:
$routes->get('photos', '\App\Gallery::index');

<?php

use App\Controllers\Gallery;

$routes->resource('photos', ['namespace' => '', 'controller' => Gallery::class]);
// Would create routes like:
$routes->get('photos', '\App\Controllers\Gallery::index');

Lihat juga Namespace Controller.

Mengubah Placeholder yang Digunakan

Secara default, placeholder (:segment) digunakan ketika ID resource dibutuhkan. Anda dapat mengubah ini dengan meneruskan opsi placeholder dengan string baru yang akan digunakan:

<?php

$routes->resource('photos', ['placeholder' => '(:num)']);

// Generates routes like:
$routes->get('photos/(:num)', 'Photos::show/$1');

Membatasi Router yang Dibuat

Anda dapat membatasi router yang dihasilkan dengan opsi only. Ini harus berupa array atau daftar yang dipisahkan koma dari nama method yang harus dibuat. Hanya router yang cocok dengan salah satu method ini yang akan dibuat. Sisanya akan diabaikan:

<?php

$routes->resource('photos', ['only' => ['index', 'show']]);

Jika tidak, Anda dapat menghapus router yang tidak digunakan dengan opsi except. Ini juga harus berupa array atau daftar yang dipisahkan koma dari nama method. Opsi ini berjalan setelah only:

<?php

$routes->resource('photos', ['except' => 'new,edit']);

Method yang valid adalah: index, show, create, update, new, edit dan delete.

ResourceController

ResourceController menyediakan titik awal yang nyaman untuk API RESTful Anda, dengan method yang sesuai dengan router resource di atas.

Perluaslah, dengan menimpa properti modelName dan format, lalu implementasikan method-method yang ingin Anda tangani:

<?php

namespace App\Controllers;

use CodeIgniter\RESTful\ResourceController;

class Photos extends ResourceController
{
    protected $modelName = 'App\Models\Photos';
    protected $format    = 'json';

    public function index()
    {
        return $this->respond($this->model->findAll());
    }

    // ...
}

Perutean untuk ini adalah:

<?php

$routes->resource('photos');

Router Presenter

Anda dapat dengan cepat membuat controller presentasi yang sejajar dengan controller resource, menggunakan method presenter(). Ini membuat router untuk method controller yang akan mengembalikan tampilan untuk resource Anda, atau memproses form yang dikirim dari tampilan tersebut.

Ini tidak diperlukan, karena presentasi dapat ditangani dengan controller konvensional - ini adalah kenyamanan. Penggunaannya mirip dengan perutean resource:

<?php

$routes->presenter('photos');

// Equivalent to the following:
$routes->get('photos/new', 'Photos::new');
$routes->post('photos/create', 'Photos::create');
$routes->post('photos', 'Photos::create');   // alias
$routes->get('photos', 'Photos::index');
$routes->get('photos/show/(:segment)', 'Photos::show/$1');
$routes->get('photos/(:segment)', 'Photos::show/$1');  // alias
$routes->get('photos/edit/(:segment)', 'Photos::edit/$1');
$routes->post('photos/update/(:segment)', 'Photos::update/$1');
$routes->get('photos/remove/(:segment)', 'Photos::remove/$1');
$routes->post('photos/delete/(:segment)', 'Photos::delete/$1');

Catatan

Urutan di atas adalah untuk kejelasan, sedangkan urutan sebenarnya router dibuat di RouteCollection, memastikan resolusi router yang tepat

Anda tidak akan memiliki router untuk photos untuk controller resource dan presenter. Anda perlu membedakannya, misalnya:

<?php

$routes->resource('api/photo');
$routes->presenter('admin/photos');

Parameter kedua menerima array opsi yang dapat digunakan untuk memodifikasi router yang dihasilkan.

Mengubah Controller yang Digunakan

Anda dapat menentukan controller yang harus digunakan dengan meneruskan opsi controller dengan nama controller yang harus digunakan:

$routes->presenter('photos', ['controller' => 'Gallery']);

<?php

ResourcePresenter

ResourcePresenter menyediakan titik awal yang nyaman untuk menyajikan tampilan resource Anda, dan memproses data dari form di tampilan tersebut, dengan method yang sejajar dengan router resource di atas.

Perluaslah, dengan menimpa properti modelName, lalu implementasikan method-method yang ingin Anda tangani:

<?php

namespace App\Controllers;

use CodeIgniter\RESTful\ResourcePresenter;

class Photos extends ResourcePresenter
{
    protected $modelName = 'App\Models\Photos';

    public function index()
    {
        return view('templates/list', $this->model->findAll());
    }

    // ...
}

Perutean untuk ini adalah:

<?php

$routes->presenter('photos');
