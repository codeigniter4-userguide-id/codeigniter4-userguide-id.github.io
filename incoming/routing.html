Apa itu Perutean URI?

Perutean URI mengaitkan sebuah URI dengan sebuah method controller.

CodeIgniter memiliki dua jenis perutean. Salah satunya adalah Perutean Router yang Didefinisikan, dan yang lainnya adalah Perutean Otomatis. Dengan Perutean Router yang Didefinisikan, Anda dapat mendefinisikan router secara manual. Ini memungkinkan URL yang fleksibel. Perutean Otomatis secara otomatis merutekan permintaan HTTP berdasarkan konvensi dan mengeksekusi method controller yang sesuai. Tidak perlu mendefinisikan router secara manual.

Pertama, mari kita lihat Perutean Router yang Didefinisikan. Jika Anda ingin menggunakan Perutean Otomatis, lihat Auto Routing (Improved).

Mengatur Aturan Perutean

Aturan perutean didefinisikan dalam berkas app/Config/Routes.php. Di dalamnya Anda akan melihat bahwa itu membuat sebuah instance dari class RouteCollection ($routes) yang memungkinkan Anda untuk menentukan kriteria perutean Anda sendiri. Router dapat ditentukan menggunakan placeholder atau Regular Expressions.

Ketika Anda menentukan sebuah router, Anda memilih sebuah method yang sesuai dengan HTTP verbs (request method). Jika Anda mengharapkan permintaan GET, Anda menggunakan method get():

<?php

$routes->get('/', 'Home::index');

Sebuah router mengambil Jalur Router (jalur URI relatif terhadap BaseURL. /) di sebelah kiri, dan memetakannya ke Penangan Router (controller dan method Home::index) di sebelah kanan, bersama dengan parameter apa pun yang harus diteruskan ke controller.

Controller dan method harus dicantumkan dengan cara yang sama seperti Anda akan menggunakan method statis, dengan memisahkan class dan method-nya dengan titik dua ganda, seperti Users::list.

Jika method tersebut memerlukan parameter untuk diteruskan kepadanya, maka parameter tersebut akan dicantumkan setelah nama method, dipisahkan oleh garis miring:

<?php

// Calls $Users->list()
$routes->get('users', 'Users::list');

// Calls $Users->list(1, 23)
$routes->get('users/1/23', 'Users::list/1/23');

Contoh

Berikut adalah beberapa contoh perutean dasar.

Sebuah URL yang berisi kata journals di segmen pertama akan dipetakan ke class \App\Controllers\Blogs, dan method default, yang biasanya index():

<?php

$routes->get('journals', 'Blogs');

Sebuah URL yang berisi segmen blog/joe akan dipetakan ke class \App\Controllers\Blogs dan method users(). ID akan diatur ke 34:

<?php

$routes->get('blog/joe', 'Blogs::users/34');

Sebuah URL dengan product sebagai segmen pertama, dan apa pun di segmen kedua akan dipetakan ke class \App\Controllers\Catalog dan method productLookup():

<?php

$routes->get('product/(:segment)', 'Catalog::productLookup');

Sebuah URL dengan product sebagai segmen pertama, dan sebuah angka di segmen kedua akan dipetakan ke class \App\Controllers\Catalog dan method productLookupByID() yang meneruskan kecocokan sebagai variabel ke method:

<?php

$routes->get('product/(:num)', 'Catalog::productLookupByID/$1');

Router HTTP verb

Anda dapat menggunakan HTTP verb standar apa pun (GET, POST, PUT, DELETE, OPTIONS, dll):

<?php

$routes->post('products', 'Product::feature');
$routes->put('products/1', 'Product::feature');
$routes->delete('products/1', 'Product::feature');

Anda dapat menyediakan beberapa verb yang harus dicocokkan oleh sebuah router dengan meneruskannya sebagai array ke method match():

<?php

$routes->match(['GET', 'PUT'], 'products', 'Product::feature');

Menentukan Penangan Router

Namespace Controller

Ketika Anda menentukan nama controller dan method sebagai string, jika sebuah controller ditulis tanpa \ di depannya, Namespace Default akan ditambahkan di depan:

<?php

// Routes to \App\Controllers\Api\Users::update()
$routes->post('api/users', 'Api\Users::update');

Jika Anda meletakkan \ di awal, itu diperlakukan sebagai nama class yang sepenuhnya memenuhi syarat:

<?php

// Routes to \Acme\Blog\Controllers\Home::list()
$routes->get('blog', '\Acme\Blog\Controllers\Home::list');

Anda juga dapat menentukan namespace dengan opsi namespace:

<?php

// Routes to \Admin\Users::index()
$routes->get('admin/users', 'Users::index', ['namespace' => 'Admin']);

Lihat Menetapkan Namespace untuk detailnya.

Sintaks Panggilan Array

Baru pada versi 4.2.0.

Sejak v4.2.0, Anda dapat menggunakan sintaks panggilan array untuk menentukan controller:

$routes->get('/', [\App\Controllers\Home::class, 'index']);

Atau menggunakan kata kunci use:

use App\Controllers\Home;

$routes->get('/', [Home::class, 'index']);

Jika Anda lupa menambahkan use App\Controllers\Home;, nama class controller diinterpretasikan sebagai \Home, bukan App\Controllers\Home.

Catatan

Ketika Anda menggunakan Sintaks Panggilan Array, nama class selalu diinterpretasikan sebagai nama class yang sepenuhnya memenuhi syarat. Jadi Namespace Default dan namespace option tidak berpengaruh.

Sintaks Panggilan Array dan Placeholder

Jika ada placeholder, itu akan secara otomatis mengatur parameter dalam urutan yang ditentukan:

use App\Controllers\Product;

$routes->get('product/(:num)/(:num)', [Product::class, 'index']);

// The above code is the same as the following:
$routes->get('product/(:num)/(:num)', 'Product::index/$1/$2');

Tetapi parameter yang dikonfigurasi secara otomatis mungkin tidak benar jika Anda menggunakan ekspresi reguler dalam router. Dalam kasus seperti itu, Anda dapat menentukan parameter secara manual:

use App\Controllers\Product;

$routes->get('product/(:num)/(:num)', [[Product::class, 'index'], '$2/$1']);

// The above code is the same as the following:
$routes->get('product/(:num)/(:num)', 'Product::index/$2/$1');

Menggunakan Closure

Anda dapat menggunakan fungsi anonim, atau Closure, sebagai tujuan yang dipetakan oleh sebuah router. Fungsi ini akan dieksekusi ketika pengguna mengunjungi URI tersebut. Ini berguna untuk dengan cepat mengeksekusi tugas-tugas kecil, atau bahkan hanya menampilkan tampilan sederhana:

<?php

use App\Libraries\RSSFeeder;

$routes->get('feed', static function () {
    $rss = new RSSFeeder();

    return $rss->feed('general');
});

Menentukan Jalur Router

Placeholder

Sebuah router tipikal mungkin terlihat seperti ini:

<?php

$routes->get('product/(:num)', 'Catalog::productLookup');

Dalam sebuah router, parameter pertama berisi URI yang akan dicocokkan, sedangkan parameter kedua berisi tujuan yang harus dirutekan. Dalam contoh di atas, jika kata literal "product" ditemukan di segmen pertama jalur URL, dan sebuah angka ditemukan di segmen kedua, class Catalog dan method productLookup digunakan sebagai gantinya.

Placeholder hanyalah string yang mewakili pola Regular Expression. Selama proses perutean, placeholder ini diganti dengan nilai Regular Expression. Placeholder ini terutama digunakan untuk keterbacaan.

Placeholder berikut tersedia untuk Anda gunakan dalam router Anda:

Placeholder

Deskripsi

(:any)

akan mencocokkan semua karakter dari titik tersebut hingga akhir URI. Ini mungkin termasuk beberapa segmen URI.

(:segment)

akan mencocokkan karakter apa pun kecuali garis miring (/) membatasi hasilnya ke satu segmen.

(:num)

akan mencocokkan bilangan bulat positif apa pun.

(:alpha)

akan mencocokkan string karakter alfabet apa pun

(:alphanum)

akan mencocokkan string karakter alfabet atau bilangan bulat apa pun, atau kombinasi keduanya.

(:hash)

sama dengan (:segment), tetapi dapat digunakan untuk dengan mudah melihat router mana yang menggunakan ID hash.

Catatan

{locale} tidak dapat digunakan sebagai placeholder atau bagian lain dari router, karena itu dicadangkan untuk digunakan dalam lokalisasi.

Perilaku (:any)

Perhatikan bahwa satu (:any) akan mencocokkan beberapa segmen di URL jika ada.

Misalnya router:

<?php

$routes->get('product/(:any)', 'Catalog::productLookup/$1');

akan cocok dengan product/123, product/123/456, product/123/456/789 dan seterusnya.

Secara default, dalam contoh di atas, jika placeholder $1 berisi garis miring (/), itu masih akan dibagi menjadi beberapa parameter ketika diteruskan ke Catalog::productLookup().

Catatan

Sejak v4.5.0, Anda dapat mengubah perilaku dengan opsi konfigurasi. Lihat Beberapa Segmen URI sebagai Satu Parameter untuk detailnya.

Implementasi di Controller harus mempertimbangkan parameter maksimum:

<?php

namespace App\Controllers;

class Catalog extends BaseController
{
    public function productLookup($seg1 = false, $seg2 = false, $seg3 = false)
    {
        echo $seg1; // Will be 123 in all examples
        echo $seg2; // false in first, 456 in second and third example
        echo $seg3; // false in first and second, 789 in third
    }
}

Atau Anda dapat menggunakan daftar argumen dengan panjang variabel:

<?php

namespace App\Controllers;

class Catalog extends BaseController
{
    public function productLookup(...$params)
    {
        echo $params[0] ?? null; // Will be 123 in all examples
        echo $params[1] ?? null; // null in first, 456 in second and third example
        echo $params[2] ?? null; // null in first and second, 789 in third
    }
}

Penting

Jangan meletakkan placeholder setelah (:any). Karena jumlah parameter yang diteruskan ke method controller dapat berubah.

Jika pencocokan beberapa segmen bukan perilaku yang dimaksudkan, (:segment) harus digunakan saat mendefinisikan router. Dengan contoh URL dari atas:

<?php

$routes->get('product/(:segment)', 'Catalog::productLookup/$1');

hanya akan cocok dengan product/123 dan menghasilkan error 404 untuk contoh lainnya.

Placeholder Kustom

Anda dapat membuat placeholder Anda sendiri yang dapat digunakan dalam berkas router Anda untuk sepenuhnya menyesuaikan pengalaman dan keterbacaan.

Anda menambahkan placeholder baru dengan method addPlaceholder(). Parameter pertama adalah string yang akan digunakan sebagai placeholder. Parameter kedua adalah pola Regular Expression yang harus diganti. Ini harus dipanggil sebelum Anda menambahkan router:

<?php

$routes->addPlaceholder('uuid', '[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}');
$routes->get('users/(:uuid)', 'Users::show/$1');

Regular Expressions

Jika Anda mau, Anda dapat menggunakan ekspresi reguler untuk mendefinisikan aturan perutean Anda. Setiap ekspresi reguler yang valid diizinkan, begitu juga back-references.

Penting

Catatan: Jika Anda menggunakan back-references, Anda harus menggunakan sintaks dolar daripada sintaks garis miring ganda. Sebuah router RegEx tipikal mungkin terlihat seperti ini:

<?php

$routes->get('products/([a-z]+)/(\d+)', 'Products::show/$1/id_$2');

Dalam contoh di atas, URI yang mirip dengan products/shirts/123 akan memanggil method show() dari class controller Products, dengan segmen pertama dan kedua asli diteruskan sebagai argumen kepadanya.

Dengan ekspresi reguler, Anda juga dapat menangkap segmen yang berisi garis miring (/), yang biasanya akan mewakili pembatas antara beberapa segmen.

Misalnya, jika pengguna mengakses area aplikasi web Anda yang dilindungi kata sandi dan Anda ingin dapat mengarahkan mereka kembali ke halaman yang sama setelah mereka masuk, Anda mungkin menemukan contoh ini berguna:

<?php

$routes->get('login/(.+)', 'Auth::login/$1');

Secara default, dalam contoh di atas, jika placeholder $1 berisi garis miring (/), itu masih akan dibagi menjadi beberapa parameter ketika diteruskan ke Auth::login().

Catatan

Sejak v4.5.0, Anda dapat mengubah perilaku dengan opsi konfigurasi. Lihat Beberapa Segmen URI sebagai Satu Parameter untuk detailnya.

Bagi Anda yang tidak tahu ekspresi reguler dan ingin belajar lebih banyak tentangnya, regular-expressions.info mungkin merupakan titik awal yang baik.

Catatan

Anda juga dapat mencampur dan mencocokkan placeholder dengan ekspresi reguler.

Router Tampilan

Baru pada versi 4.3.0.

Jika Anda hanya ingin merender tampilan yang tidak memiliki logika terkait dengannya, Anda dapat menggunakan method view(). Ini selalu diperlakukan sebagai permintaan GET. Method ini menerima nama tampilan yang akan dimuat sebagai parameter kedua.

<?php

// Displays the view in /app/Views/pages/about.php
$routes->view('about', 'pages/about');

Jika Anda menggunakan placeholder dalam router Anda, Anda dapat mengaksesnya dalam tampilan dalam variabel khusus, $segments. Variabel tersebut tersedia sebagai array, diindeks dalam urutan kemunculannya di router.

<?php

// Displays the view in /app/Views/map.php
$routes->view('map/(:segment)/(:segment)', 'map');

// Within the view, you can access the segments with
// $segments[0] and $segments[1] respectively.

Mengarahkan Router

Situs mana pun yang hidup cukup lama pasti akan memiliki halaman yang berpindah. Anda dapat menentukan router yang harus mengarahkan ke router lain dengan method addRedirect(). Parameter pertama adalah pola URI untuk router lama. Parameter kedua adalah URI baru untuk mengarahkan, atau nama dari router bernama. Parameter ketiga adalah kode status HTTP yang harus dikirim bersama dengan pengalihan. Nilai default adalah 302 yang merupakan pengalihan sementara dan direkomendasikan dalam banyak kasus:

<?php

$routes->get('users/profile', 'Users::profile', ['as' => 'profile']);

// Redirect to a named route
$routes->addRedirect('users/about', 'profile');
// Redirect to a URI
$routes->addRedirect('users/about', 'users/profile');

// Redirect with placeholder
$routes->get('post/(:num)/comment/(:num)', 'PostsComments::index', ['as' => 'post.comment']);

// Redirect to a URI
$routes->addRedirect('article/(:num)/(:num)', 'post/$1/comment/$2');
// Redirect to a named route
$routes->addRedirect('article/(:num)/(:num)', 'post.comment');

Catatan

Sejak v4.2.0, addRedirect() dapat menggunakan placeholder.

Jika router pengalihan dicocokkan selama pemuatan halaman, pengguna akan segera diarahkan ke halaman baru sebelum controller dapat dimuat.

Pembatasan Lingkungan

Anda dapat membuat sekumpulan router yang hanya akan dapat dilihat di lingkungan tertentu. Ini memungkinkan Anda untuk membuat tools yang hanya dapat digunakan oleh pengembang di mesin lokal mereka yang tidak dapat dijangkau di server pengujian atau produksi. Ini dapat dilakukan dengan method environment(). Parameter pertama adalah nama lingkungan. Setiap router yang didefinisikan dalam closure ini hanya dapat diakses dari lingkungan yang diberikan:

<?php

$routes->environment('development', static function ($routes) {
    $routes->get('builder', 'Tools\Builder::index');
});

Router dengan HTTP verb apa pun

Penting

Method ini hanya ada untuk kompatibilitas mundur. Jangan menggunakannya dalam proyek baru. Bahkan jika Anda sudah menggunakannya, kami merekomendasikan Anda untuk menggunakan method lain yang lebih sesuai.

Peringatan

Meskipun method add() tampaknya nyaman, disarankan untuk selalu menggunakan router berbasis HTTP-verb, yang dijelaskan di atas, karena lebih aman. Jika Anda menggunakan perlindungan CSRF, itu tidak melindungi permintaan GET. Jika URI yang ditentukan dalam method add() dapat diakses oleh method GET, perlindungan CSRF tidak akan berfungsi.

Dimungkinkan untuk mendefinisikan router dengan HTTP verb apa pun. Anda dapat menggunakan method add():

<?php

$routes->add('products', 'Product::feature');

Catatan

Menggunakan router berbasis HTTP-verb juga akan memberikan sedikit peningkatan kinerja, karena hanya router yang cocok dengan method permintaan saat ini yang disimpan, menghasilkan lebih sedikit router untuk dipindai saat mencoba menemukan kecocokan.

Memetakan Beberapa Router

Penting

Method ini hanya ada untuk kompatibilitas mundur. Jangan menggunakannya dalam proyek baru. Bahkan jika Anda sudah menggunakannya, kami merekomendasikan Anda untuk menggunakan method lain yang lebih sesuai.

Peringatan

Method map() juga tidak direkomendasikan seperti halnya add() karena secara internal memanggil add().

Meskipun method add() mudah digunakan, seringkali lebih praktis untuk bekerja dengan beberapa router sekaligus, menggunakan method map(). Daripada memanggil method add() untuk setiap router yang perlu Anda tambahkan, Anda dapat mendefinisikan array router dan kemudian meneruskannya sebagai parameter pertama ke method map():

<?php

$multipleRoutes = [
    'product/(:num)'      => 'Catalog::productLookupById',
    'product/(:alphanum)' => 'Catalog::productLookupByName',
];

$routes->map($multipleRoutes);

Router Khusus Baris Perintah

Catatan

Disarankan untuk menggunakan Spark Command untuk skrip CLI daripada memanggil controller melalui CLI. Lihat halaman Membuat Perintah (Command) Spark untuk informasi lebih rinci.

Setiap router yang dibuat oleh method router berbasis HTTP-verb juga tidak akan dapat diakses dari CLI, tetapi router yang dibuat oleh method add() masih akan tersedia dari baris perintah.

Anda dapat membuat router yang hanya berfungsi dari baris perintah, dan tidak dapat diakses dari browser web, dengan method cli():

<?php

$routes->cli('migrate', 'App\Database::migrate');

Peringatan

Jika Anda mengaktifkan Perutean Otomatis (Legacy) dan menempatkan berkas perintah di app/Controllers, siapa pun dapat mengakses perintah dengan bantuan Perutean Otomatis (Legacy) melalui HTTP.

Opsi Global

Semua method untuk membuat router (get(), post(), resource() dll) dapat mengambil array opsi yang dapat memodifikasi router yang dihasilkan, atau lebih membatasinya. Array $options selalu menjadi parameter terakhir:

<?php

$routes->add('from', 'to', $options);
$routes->get('from', 'to', $options);
$routes->post('from', 'to', $options);
$routes->put('from', 'to', $options);
$routes->head('from', 'to', $options);
$routes->options('from', 'to', $options);
$routes->delete('from', 'to', $options);
$routes->patch('from', 'to', $options);
$routes->match(['GET', 'PUT'], 'from', 'to', $options);
$routes->resource('photos', $options);
$routes->map($array, $options);
$routes->group('name', $options, static function () {});

Menerapkan Filter

Anda dapat mengubah perilaku router tertentu dengan menyediakan filter untuk dijalankan sebelum atau sesudah controller. Ini sangat berguna selama otentikasi atau pencatatan API.

Nilai untuk filter dapat berupa string atau array string:

  • yang cocok dengan alias yang didefinisikan dalam app/Config/Filters.php.

  • nama class filter

Lihat Filter Controller untuk informasi lebih lanjut tentang mendefinisikan alias.

Peringatan

Jika Anda mengatur filter ke router di app/Config/Routes.php (bukan di app/Config/Filters.php), disarankan untuk menonaktifkan Perutean Otomatis (Legacy). Ketika Perutean Otomatis (Legacy) diaktifkan, mungkin saja sebuah controller dapat diakses melalui URL yang berbeda dari router yang dikonfigurasi, dalam hal ini filter yang Anda tentukan ke router tidak akan diterapkan. Lihat Gunakan Hanya Router yang Didefinisikan untuk menonaktifkan perutean otomatis.

Filter Alias

Anda menentukan alias yang didefinisikan dalam app/Config/Filters.php untuk nilai filter:

<?php

$routes->get('admin', ' AdminController::index', ['filter' => 'admin-auth']);

Anda juga dapat menyediakan argumen untuk diteruskan ke method before() dan after() filter alias:

<?php

$routes->post('users/delete/(:segment)', 'AdminController::index', ['filter' => 'admin-auth:dual,noreturn']);

Filter Nama Class

Baru pada versi 4.1.5.

Anda dapat menentukan nama class filter untuk nilai filter:

<?php

$routes->get('admin', ' AdminController::index', ['filter' => \App\Filters\SomeFilter::class]);

Beberapa Filter

Baru pada versi 4.1.5.

Penting

Sejak v4.5.0, Beberapa Filter selalu diaktifkan. Sebelum v4.5.0, Beberapa Filter dinonaktifkan secara default. Jika Anda ingin menggunakannya dengan sebelum v4.5.0, Lihat Meningkatkan dari 4.1.4 ke 4.1.5 untuk detailnya.

Anda dapat menentukan array untuk nilai filter:

<?php

$routes->get('admin', ' AdminController::index', ['filter' => ['admin-auth', \App\Filters\SomeFilter::class]]);
Argumen Filter

Argumen tambahan dapat diteruskan ke filter:

<?php

$routes->add('users/delete/(:segment)', 'AdminController::index', ['filter' => 'admin-auth:dual,noreturn']);

Dalam contoh ini, array ['dual', 'noreturn'] akan diteruskan dalam $arguments ke method implementasi before() dan after() filter.

Menetapkan Namespace

Meskipun Namespace Default akan ditambahkan di depan controller yang dihasilkan, Anda juga dapat menentukan namespace yang berbeda untuk digunakan dalam array opsi apa pun, dengan opsi namespace. Nilainya harus berupa namespace yang ingin Anda modifikasi:

<?php

// Routes to \Admin\Users::index()
$routes->get('admin/users', 'Users::index', ['namespace' => 'Admin']);

Namespace baru hanya diterapkan selama panggilan tersebut untuk method apa pun yang membuat satu router, seperti get, post, dll. Untuk method apa pun yang membuat beberapa router, namespace baru dilampirkan ke semua router yang dihasilkan oleh fungsi tersebut atau, dalam kasus group(), semua router yang dihasilkan saat dalam closure.

Batasi ke Hostname

Anda dapat membatasi grup router untuk berfungsi hanya di domain atau subdomain tertentu dari aplikasi Anda dengan meneruskan opsi "hostname" bersama dengan domain yang diinginkan untuk mengizinkannya sebagai bagian dari array opsi:

<?php

$routes->get('from', 'to', ['hostname' => 'accounts.example.com']);

Contoh ini hanya akan mengizinkan host yang ditentukan untuk berfungsi jika domainnya sama persis dengan accounts.example.com. Itu tidak akan berfungsi di bawah situs utama di example.com.

Batasi berdasarkan Beberapa Hostname

Baru pada versi 4.6.0.

Anda juga dapat membatasi berdasarkan beberapa hostname, contohnya:

<?php

$routes->get('from', 'to', ['hostname' => ['s1.example.com', 's2.example.com']]);

Batasi ke Subdomain

Ketika opsi subdomain ada, sistem akan membatasi router agar hanya tersedia di subdomain tersebut. Router hanya akan dicocokkan jika subdomain adalah tempat aplikasi sedang dilihat:

<?php

// Limit to media.example.com
$routes->get('from', 'to', ['subdomain' => 'media']);

Anda dapat membatasinya ke subdomain apa pun dengan mengatur nilai ke tanda bintang, (*). Jika Anda melihat dari URL yang tidak memiliki subdomain apa pun, ini tidak akan dicocokkan:

<?php

// Limit to any sub-domain
$routes->get('from', 'to', ['subdomain' => '*']);

Penting

Sistem ini tidak sempurna dan harus diuji untuk domain spesifik Anda sebelum digunakan dalam produksi. Sebagian besar domain harus berfungsi dengan baik tetapi beberapa kasus tepi, terutama dengan titik di domain itu sendiri (tidak digunakan untuk memisahkan akhiran atau www) berpotensi menyebabkan positif palsu.

Menggeser Parameter yang Cocok

Anda dapat menggeser parameter yang cocok dalam router Anda dengan nilai numerik apa pun dengan opsi offset, dengan nilai menjadi jumlah segmen yang akan digeser.

Ini dapat bermanfaat saat mengembangkan API dengan segmen URI pertama menjadi nomor versi. Ini juga dapat digunakan ketika parameter pertama adalah string bahasa:

<?php

$routes->get('users/(:num)', 'users/show/$1', ['offset' => 1]);

// Creates:
$routes['users/(:num)'] = 'users/show/$2';

Perutean Balik

Perutean balik memungkinkan Anda untuk mendefinisikan controller dan method, serta parameter apa pun, yang harus dituju oleh sebuah tautan, dan membiarkan router mencari router saat ini. Ini memungkinkan definisi router berubah tanpa Anda harus memperbarui kode aplikasi Anda. Ini biasanya digunakan dalam tampilan untuk membuat tautan.

Misalnya, jika Anda memiliki router ke galeri foto yang ingin Anda tautkan, Anda dapat menggunakan fungsi pembantu url_to() untuk mendapatkan router yang harus digunakan. Parameter pertama adalah Controller dan method yang sepenuhnya memenuhi syarat, dipisahkan oleh titik dua ganda (::), mirip dengan cara Anda akan menggunakannya saat menulis router awal itu sendiri. Parameter apa pun yang harus diteruskan ke router diteruskan selanjutnya:

<?php

// The route is defined as:
$routes->get('users/(:num)/gallery/(:num)', 'Galleries::showUserGallery/$1/$2');

?>

<!-- Generate the URI to link to user ID 15, gallery 12: -->
<a href="<?= url_to('Galleries::showUserGallery', 15, 12) ?>">View Gallery</a>
<!-- Result: 'http://example.com/users/15/gallery/12' -->

Router Bernama

Anda dapat menamai router untuk membuat aplikasi Anda tidak terlalu rapuh. Ini menerapkan nama ke router yang dapat dipanggil nanti, dan bahkan jika definisi router berubah, semua tautan di aplikasi Anda yang dibangun dengan url_to() akan tetap berfungsi tanpa Anda harus melakukan perubahan apa pun. Sebuah router dinamai dengan meneruskan opsi as dengan nama router:

<?php

// The route is defined as:
$routes->get('users/(:num)/gallery/(:num)', 'Galleries::showUserGallery/$1/$2', ['as' => 'user_gallery']);

?>

<!-- Generate the URI to link to user ID 15, gallery 12: -->
<a href="<?= url_to('user_gallery', 15, 12) ?>">View Gallery</a>
<!-- Result: 'http://example.com/users/15/gallery/12' -->

Ini memiliki manfaat tambahan membuat tampilan lebih mudah dibaca juga.

Catatan

Secara default, semua router yang didefinisikan memiliki nama yang cocok dengan jalurnya, dengan placeholder diganti dengan ekspresi reguler yang sesuai. Misalnya, jika Anda mendefinisikan router seperti $routes->get('edit/(:num)', 'PostController::edit/$1');, Anda dapat menghasilkan URL yang sesuai menggunakan route_to('edit/([0-9]+)', 12).

Peringatan

Menurut Prioritas Router, jika router yang tidak bernama didefinisikan terlebih dahulu (misalnya, $routes->get('edit', 'PostController::edit');) dan router bernama lain didefinisikan kemudian dengan nama yang sama dengan jalur router pertama (misalnya, $routes->get('edit/(:num)', 'PostController::edit/$1', ['as' => 'edit']);), router kedua tidak akan didaftarkan karena namanya akan bertentangan dengan nama yang secara otomatis ditetapkan dari router pertama.

Mengelompokkan Router

Anda dapat mengelompokkan router Anda di bawah nama umum dengan method group(). Nama grup menjadi segmen yang muncul sebelum router yang didefinisikan di dalam grup. Ini memungkinkan Anda untuk mengurangi pengetikan yang diperlukan untuk membangun sekumpulan router yang luas yang semuanya berbagi string pembuka, seperti saat membangun area admin:

<?php

$routes->group('admin', static function ($routes) {
    $routes->get('users', 'Admin\Users::index');
    $routes->get('blog', 'Admin\Blog::index');
});

Ini akan menambahkan awalan admin ke URI users dan blog, menangani URL seperti admin/users dan admin/blog.

Mengatur Namespace

Jika Anda perlu menetapkan opsi ke grup, seperti Menetapkan Namespace, lakukan sebelum callback:

<?php

$routes->group('api', ['namespace' => 'App\API\v1'], static function ($routes) {
    $routes->resource('users');
});

Ini akan menangani router resource ke controller App\API\v1\Users dengan URI api/users.

Mengatur Filter

Anda juga dapat menggunakan filter tertentu untuk sekelompok router. Ini akan selalu menjalankan filter sebelum atau sesudah controller. Ini sangat berguna selama otentikasi atau pencatatan API:

<?php

$routes->group('api', ['filter' => 'api-auth'], static function ($routes) {
    $routes->resource('users');
});

Nilai untuk filter harus cocok dengan salah satu alias yang didefinisikan dalam app/Config/Filters.php.

Catatan

Sebelum v4.5.4, karena bug, filter yang diteruskan ke group() tidak digabungkan ke dalam filter yang diteruskan ke router internal.

Mengatur Opsi Lain

Pada titik tertentu, Anda mungkin ingin mengelompokkan router untuk tujuan menerapkan filter atau opsi konfigurasi router lainnya seperti namespace, subdomain, dll. Tanpa perlu menambahkan awalan ke grup, Anda dapat meneruskan string kosong sebagai ganti awalan dan router dalam grup akan dirutekan seolah-olah grup tidak pernah ada tetapi dengan opsi konfigurasi router yang diberikan:

<?php

$routes->group('', ['namespace' => 'Myth\Auth\Controllers'], static function ($routes) {
    $routes->get('login', 'AuthController::login', ['as' => 'login']);
    $routes->post('login', 'AuthController::attemptLogin');
    $routes->get('logout', 'AuthController::logout');
});

Mengelompokkan Bersarang

Dimungkinkan untuk mengelompokkan bersarang di dalam grup untuk organisasi yang lebih baik jika Anda membutuhkannya:

<?php

$routes->group('admin', ['filter' => 'myfilter1:config'], static function ($routes) {
    $routes->get('/', 'Admin\Admin::index');

    $routes->group('users', ['filter' => 'myfilter2:region'], static function ($routes) {
        $routes->get('list', 'Admin\Users::list');
    });
});

Ini akan menangani URL di admin/users/list.

Opsi filter yang diteruskan ke group() luar digabungkan dengan opsi filter group() dalam. Kode di atas menjalankan myfilter1:config untuk router admin, dan myfilter1:config serta myfilter2:region untuk router admin/users/list.

Catatan

Sebelum v4.6.0, filter yang sama tidak dapat dijalankan beberapa kali dengan argumen yang berbeda.

Setiap opsi tumpang tindih lainnya yang diteruskan ke group() dalam akan menimpa nilainya.

Catatan

Sebelum v4.5.0, karena bug, opsi yang diteruskan ke group() luar tidak digabungkan dengan opsi group() dalam.

Prioritas Router

Router didaftarkan dalam tabel perutean dalam urutan di mana router tersebut didefinisikan. Ini berarti bahwa ketika sebuah URI diakses, router yang cocok pertama akan dieksekusi.

Peringatan

Jika jalur router didefinisikan lebih dari sekali dengan penangan yang berbeda, hanya router yang didefinisikan pertama yang didaftarkan.

Anda dapat memeriksa router yang terdaftar dalam tabel perutean dengan menjalankan perintah spark routes.

Mengubah Prioritas Router

Saat bekerja dengan modul, bisa menjadi masalah jika router dalam aplikasi berisi wildcard. Maka router modul tidak akan diproses dengan benar. Anda dapat mengatasi masalah ini dengan menurunkan prioritas pemrosesan router menggunakan opsi priority. Parameter menerima bilangan bulat positif dan nol. Semakin tinggi angka yang ditentukan dalam priority, semakin rendah prioritas router dalam antrian pemrosesan:

<?php

// First you need to enable processing of the routes queue by priority.
$routes->setPrioritize();

// Config\Routes
$routes->get('(.*)', 'Posts::index', ['priority' => 1]);

// Modules\Acme\Config\Routes
$routes->get('admin', 'Admin::index');

// The "admin" route will now be processed before the wildcard route.

Untuk menonaktifkan fungsionalitas ini, Anda harus memanggil method dengan parameter false:

<?php

$routes->setPrioritize(false);

Catatan

Secara default, semua router memiliki prioritas 0. Bilangan bulat negatif akan diubah menjadi nilai absolut.

Opsi Konfigurasi Router

Class RoutesCollection menyediakan beberapa opsi yang memengaruhi semua router, dan dapat dimodifikasi untuk memenuhi kebutuhan aplikasi Anda. Opsi ini tersedia di app/Config/Routing.php.

Catatan

Berkas konfigurasi app/Config/Routing.php telah ditambahkan sejak v4.4.0. Dalam versi sebelumnya, method setter digunakan di app/Config/Routes.php untuk mengubah pengaturan.

Namespace Default

Saat mencocokkan controller dengan router, router akan menambahkan nilai namespace default ke bagian depan controller yang ditentukan oleh router. Secara default, nilai ini adalah App\Controllers.

Jika Anda mengatur nilai string kosong (''), itu membiarkan setiap router untuk menentukan controller yang sepenuhnya bernama:

<?php

// In app/Config/Routing.php
use CodeIgniter\Config\Routing as BaseRouting;

// ...
class Routing extends BaseRouting
{
    // ...
    public string $defaultNamespace = '';
    // ...
}

// In app/Config/Routes.php
// Controller is \Users
$routes->get('users', 'Users::index');

// Controller is \Admin\Users
$routes->get('users', 'Admin\Users::index');

Jika controller Anda tidak secara eksplisit memiliki namespace, tidak perlu mengubah ini. Jika Anda menamai controller Anda, maka Anda dapat mengubah nilai ini untuk menghemat pengetikan:

<?php

// This can be overridden in app/Config/Routes.php
$routes->setDefaultNamespace('App');

// Controller is \App\Users
$routes->get('users', 'Users::index');

// Controller is \App\Admin\Users
$routes->get('users', 'Admin\Users::index');

Method Default

Pengaturan ini digunakan ketika penangan router hanya memiliki nama controller dan tidak ada nama method yang tercantum. Nilai default adalah index.

// Di app/Config/Routing.php
public string $defaultMethod = 'index';

Catatan

$defaultMethod juga umum dengan Perutean Otomatis. Lihat Perutean Otomatis (Ditingkatkan) atau Perutean Otomatis (Legacy).

Jika Anda mendefinisikan router berikut:

$routes->get('/', 'Home');

method index() dari controller App\Controllers\Home dieksekusi ketika router cocok.

Catatan

Nama method yang dimulai dengan _ tidak dapat digunakan sebagai method default. Namun, mulai v4.5.0, method __invoke diizinkan.

Terjemahkan Garis Penghubung URI

Opsi ini memungkinkan Anda untuk secara otomatis mengganti garis penghubung (-) dengan garis bawah dalam segmen URI controller dan method ketika digunakan dalam Perutean Otomatis, sehingga menghemat entri router tambahan jika Anda perlu melakukannya. Ini diperlukan karena garis penghubung bukan karakter nama class atau method yang valid dan akan menyebabkan error fatal jika Anda mencoba menggunakannya:

<?php

// In app/Config/Routing.php
use CodeIgniter\Config\Routing as BaseRouting;

// ...
class Routing extends BaseRouting
{
    // ...
    public bool $translateURIDashes = true;
    // ...
}

// This can be overridden in app/Config/Routes.php
$routes->setTranslateURIDashes(true);

Catatan

Saat menggunakan Perutean Otomatis (Ditingkatkan), sebelum v4.4.0, jika $translateURIDashes adalah true, dua URI sesuai dengan satu method controller, satu URI untuk garis penghubung (misalnya, foo-bar) dan satu URI untuk garis bawah (misalnya, foo_bar). Ini adalah perilaku yang salah. Sejak v4.4.0, URI untuk garis bawah (foo_bar) tidak dapat diakses.

Gunakan Hanya Router yang Didefinisikan

Sejak v4.2.0, perutean otomatis dinonaktifkan secara default.

Ketika tidak ada router yang didefinisikan yang cocok dengan URI, sistem akan mencoba mencocokkan URI tersebut dengan controller dan method ketika Perutean Otomatis diaktifkan.

Anda dapat menonaktifkan pencocokan otomatis ini, dan membatasi router hanya untuk yang Anda definisikan, dengan mengatur properti $autoRoute ke false:

<?php

// In app/Config/Routing.php
use CodeIgniter\Config\Routing as BaseRouting;

// ...
class Routing extends BaseRouting
{
    // ...
    public bool $autoRoute = false;
    // ...
}

// This can be overridden in app/Config/Routes.php
$routes->setAutoRoute(false);

Peringatan

Jika Anda menggunakan perlindungan CSRF, itu tidak melindungi permintaan GET. Jika URI dapat diakses oleh method GET, perlindungan CSRF tidak akan berfungsi.

Penggantian 404

Ketika halaman tidak ditemukan yang cocok dengan URI saat ini, sistem akan menampilkan tampilan 404 generik. Menggunakan properti $override404 dalam berkas konfigurasi perutean, Anda dapat mendefinisikan class/method controller untuk router 404.

<?php

// In app/Config/Routing.php
use CodeIgniter\Config\Routing as BaseRouting;

// ...
class Routing extends BaseRouting
{
    // ...
    public ?string $override404 = 'App\Errors::show404';
    // ...
}

Anda juga dapat mengubah apa yang terjadi dengan menentukan tindakan yang akan terjadi dengan method set404Override() dalam berkas konfigurasi Router. Nilainya bisa berupa pasangan class/method yang valid, atau Closure:

<?php

// In app/Config/Routes.php
// Would execute the show404 method of the App\Errors class
$routes->set404Override('App\Errors::show404');

// Will display a custom view.
$routes->set404Override(static function () {
    // If you want to get the URI segments.
    $segments = request()->getUri()->getSegments();

    return view('my_errors/not_found.html');
});

Catatan

Mulai v4.5.0, fitur Penggantian 404 mengatur kode status Respons ke 404 secara default. Dalam versi sebelumnya, kodenya adalah 200. Jika Anda ingin mengubah kode status di controller, lihat CodeIgniter\HTTP\Response::setStatusCode() untuk informasi tentang cara mengatur kode status.

Pemrosesan Router berdasarkan Prioritas

Mengaktifkan atau menonaktifkan pemrosesan antrian router berdasarkan prioritas. Penurunan prioritas didefinisikan dalam opsi router. Dinonaktifkan secara default. Fungsionalitas ini memengaruhi semua router. Untuk contoh penggunaan penurunan prioritas lihat Prioritas Router:

<?php

// In app/Config/Routing.php
use CodeIgniter\Config\Routing as BaseRouting;

// ...
class Routing extends BaseRouting
{
    // ...
    public bool $prioritize = true;
    // ...
}

// In app/Config/Routes.php
// to enable
$routes->setPrioritize();

// to disable
$routes->setPrioritize(false);

Catatan

Secara default, semua router memiliki prioritas 0. Bilangan bulat negatif akan diubah menjadi nilai absolut.

Beberapa Segmen URI sebagai Satu Parameter

Baru pada versi 4.5.0.

Ketika opsi ini diaktifkan, placeholder yang cocok dengan beberapa segmen, seperti (:any), akan diteruskan langsung sebagaimana adanya ke satu parameter, meskipun mengandung beberapa segmen.

<?php

// In app/Config/Routing.php
use CodeIgniter\Config\Routing as BaseRouting;

class Routing extends BaseRouting
{
    // ...
    public bool $multipleSegmentsOneParam = true;
    // ...
}

Misalnya router:

<?php

$routes->get('product/(:any)', 'Catalog::productLookup/$1');

akan cocok dengan product/123, product/123/456, product/123/456/789 dan seterusnya. Dan jika URI adalah product/123/456, 123/456 akan diteruskan ke parameter pertama method Catalog::productLookup().

Perutean Otomatis (Ditingkatkan)

Baru pada versi 4.2.0.

Perutean Otomatis (Ditingkatkan) adalah sistem perutean otomatis baru yang lebih aman.

Lihat Auto Routing (Improved) untuk detailnya.

Perutean Otomatis (Legacy)

Penting

Fitur ini hanya ada untuk kompatibilitas mundur. Jangan menggunakannya dalam proyek baru. Bahkan jika Anda sudah menggunakannya, kami merekomendasikan Anda untuk menggunakan Auto Routing (Improved) sebagai gantinya.

Perutean Otomatis (Legacy) adalah sistem perutean dari CodeIgniter 3. Ini dapat secara otomatis merutekan permintaan HTTP berdasarkan konvensi dan mengeksekusi method controller yang sesuai.

Disarankan agar semua router didefinisikan dalam berkas app/Config/Routes.php, atau untuk menggunakan Auto Routing (Improved),

Peringatan

Untuk mencegah kesalahan konfigurasi dan kesalahan pengkodean, kami merekomendasikan agar Anda tidak menggunakan fitur Perutean Otomatis (Legacy). Mudah untuk membuat aplikasi yang rentan di mana filter controller atau perlindungan CSRF dilewati.

Penting

Perutean Otomatis (Legacy) merutekan permintaan HTTP dengan method HTTP apa pun ke method controller.

Mengaktifkan Perutean Otomatis (Legacy)

Sejak v4.2.0, perutean otomatis dinonaktifkan secara default.

Untuk menggunakannya, Anda perlu mengubah pengaturan opsi $autoRoute menjadi true di app/Config/Routing.php:

public bool $autoRoute = true;

Dan atur properti $autoRoutesImproved menjadi false di app/Config/Feature.php:

public bool $autoRoutesImproved = false;

Segmen URI (Legacy)

Segmen dalam URL, mengikuti pendekatan Model-View-Controller, biasanya mewakili:

example.com/class/method/ID

  1. Segmen pertama mewakili class controller yang harus dipanggil.

  2. Segmen kedua mewakili method class yang harus dipanggil.

  3. Segmen ketiga, dan segmen tambahan apa pun, mewakili ID dan variabel apa pun yang akan diteruskan ke controller.

Pertimbangkan URI ini:

example.com/index.php/helloworld/index/1

Dalam contoh di atas, CodeIgniter akan mencoba menemukan controller bernama Helloworld.php dan mengeksekusi method index() dengan meneruskan '1' sebagai argumen pertama.

Lihat Perutean Otomatis (Legacy) dalam Controller untuk info lebih lanjut.

Opsi Konfigurasi (Legacy)

Opsi ini tersedia di berkas app/Config/Routing.php.

Controller Default (Legacy)

Untuk URI Root Situs (Legacy)

Ketika pengguna mengunjungi root situs Anda (yaitu, example.com) controller yang akan digunakan ditentukan oleh nilai yang ditetapkan ke properti $defaultController, kecuali jika ada router yang secara eksplisit ada untuk itu.

Nilai default untuk ini adalah Home yang cocok dengan controller di app/Controllers/Home.php:

public string $defaultController = 'Home';
Untuk URI Direktori (Legacy)

Controller default juga digunakan ketika tidak ada router yang cocok ditemukan, dan URI akan menunjuk ke direktori dalam direktori controller. Misalnya, jika pengguna mengunjungi example.com/admin, jika controller ditemukan di app/Controllers/Admin/Home.php, itu akan digunakan.

Lihat Perutean Otomatis (Legacy) dalam Controller untuk info lebih lanjut.

Method Default (Legacy)

Ini berfungsi mirip dengan pengaturan controller default, tetapi digunakan untuk menentukan method default yang digunakan ketika controller ditemukan yang cocok dengan URI, tetapi tidak ada segmen yang ada untuk method. Nilai default adalah index.

Dalam contoh ini, jika pengguna mengunjungi example.com/products, dan controller Products ada, method Products::listAll() akan dieksekusi:

public string $defaultMethod = 'listAll';

Mengkonfirmasi Router

CodeIgniter memiliki perintah berikut untuk menampilkan semua router.

spark routes

Menampilkan semua router dan filter:

php spark routes

Outputnya seperti berikut:

+---------+---------+---------------+-------------------------------+----------------+---------------+
| Method  | Router  | Nama          | Penangan                      | Filter Sebelum | Filter Sesudah|
+---------+---------+---------------+-------------------------------+----------------+---------------+
| GET     | /       | »             | \App\Controllers\Home::index  |                | toolbar       |
| GET     | feed    | »             | (Closure)                     |                | toolbar       |
+---------+---------+---------------+-------------------------------+----------------+---------------+

Kolom Method menunjukkan method HTTP yang didengarkan oleh router.

Kolom Router menunjukkan jalur router yang akan dicocokkan. Jalur router yang didefinisikan diekspresikan sebagai ekspresi reguler.

Sejak v4.3.0, kolom Nama menunjukkan nama router. » menunjukkan nama sama dengan jalur router.

Penting

Sistem ini tidak sempurna. Untuk router yang berisi pola ekspresi reguler seperti ([^/]+) atau {locale}, Filter yang ditampilkan mungkin tidak benar (jika Anda mengatur pola URI yang rumit untuk filter di app/Config/Filters.php), atau ditampilkan sebagai <unknown>.

Perintah spark filter:check dapat digunakan untuk memeriksa filter yang 100% akurat.

Perutean Otomatis (Ditingkatkan)

Ketika Anda menggunakan Perutean Otomatis (Ditingkatkan), outputnya seperti berikut:

+-----------+-------------------------+---------------+-----------------------------------+----------------+---------------+
| Method    | Router                  | Nama          | Penangan                          | Filter Sebelum | Filter Sesudah|
+-----------+-------------------------+---------------+-----------------------------------+----------------+---------------+
| GET(auto) | product/list/../..[/..] |               | \App\Controllers\Product::getList |                | toolbar       |
+-----------+-------------------------+---------------+-----------------------------------+----------------+---------------+

Method akan seperti GET(auto).

/.. di kolom Router menunjukkan satu segmen. [/..] menunjukkan itu opsional.

Catatan

Ketika perutean otomatis diaktifkan dan Anda memiliki router home, itu juga dapat diakses oleh Home, atau mungkin oleh hOme, hoMe, HOME, dll. tetapi perintah hanya akan menampilkan home.

Jika Anda melihat router dimulai dengan x seperti berikut, itu menunjukkan router yang tidak valid yang tidak akan dirutekan, tetapi controller memiliki method publik untuk perutean.

+-----------+----------------+------+-------------------------------------+----------------+---------------+
| Method    | Router         | Nama | Penangan                            | Filter Sebelum | Filter Sesudah|
+-----------+----------------+------+-------------------------------------+----------------+---------------+
| GET(auto) | x home/foo     |      | \App\Controllers\Home::getFoo       | <unknown>      | <unknown>     |
+-----------+----------------+------+-------------------------------------+----------------+---------------+

Contoh di atas menunjukkan Anda memiliki method \App\Controllers\Home::getFoo(), tetapi tidak dirutekan karena itu adalah controller default (Home secara default) dan nama controller default harus dihilangkan di URI. Anda harus menghapus method getFoo().

Catatan

Sebelum v4.3.4, router yang tidak valid ditampilkan sebagai router normal karena bug.

Perutean Otomatis (Legacy)

Ketika Anda menggunakan Perutean Otomatis (Legacy), outputnya seperti berikut:

+--------+--------------------+---------------+-----------------------------------+----------------+---------------+
| Method | Router             | Nama          | Penangan                          | Filter Sebelum | Filter Sesudah|
+--------+--------------------+---------------+-----------------------------------+----------------+---------------+
| auto   | product/list[/...] |               | \App\Controllers\Product::getList |                | toolbar       |
+--------+--------------------+---------------+-----------------------------------+----------------+---------------+

Method akan menjadi auto.

[/...] di kolom Router menunjukkan sejumlah segmen.

Catatan

Ketika perutean otomatis diaktifkan dan Anda memiliki router home, itu juga dapat diakses oleh Home, atau mungkin oleh hOme, hoMe, HOME, dll. tetapi perintah hanya akan menampilkan home.

Urutkan berdasarkan Penangan

Baru pada versi 4.3.0.

Anda dapat mengurutkan router berdasarkan Penangan:

php spark routes -h

Tentukan Host

Baru pada versi 4.4.0.

Anda dapat menentukan host di URL permintaan dengan opsi --host:

php spark routes --host accounts.example.com

Mendapatkan Informasi Perutean

Di CodeIgniter 4, memahami dan mengelola informasi perutean sangat penting untuk menangani permintaan HTTP secara efektif. Ini melibatkan pengambilan detail tentang controller dan method yang aktif, serta filter yang diterapkan pada router tertentu. Di bawah ini, kita akan menjelajahi cara mengakses informasi perutean ini untuk membantu dalam tugas-tugas seperti pencatatan, debugging, atau mengimplementasikan logika kondisional.

Mengambil Nama Controller/Method Saat Ini

Dalam beberapa kasus, Anda mungkin perlu menentukan controller dan method mana yang telah dipicu oleh permintaan HTTP saat ini. Ini dapat berguna untuk pencatatan, debugging, atau logika kondisional berdasarkan method controller yang aktif.

CodeIgniter 4 menyediakan cara sederhana untuk mengakses nama controller dan method router saat ini menggunakan class Router. Berikut adalah contohnya:

<?php

// Get the router instance.
/** @var \CodeIgniter\Router\Router $router */
$router = service('router');

// Retrieve the fully qualified class name of the controller handling the current request.
$controller = $router->controllerName();

// Retrieve the method name being executed in the controller for the current request.
$method = $router->methodName();

echo 'Current Controller: ' . $controller . '<br>';
echo 'Current Method: ' . $method;

Fungsionalitas ini sangat berguna ketika Anda perlu berinteraksi secara dinamis dengan controller Anda atau mencatat method mana yang menangani permintaan tertentu.

Mendapatkan Filter Aktif untuk Router Saat Ini

Filter adalah fitur canggih yang memungkinkan Anda melakukan operasi seperti otentikasi, pencatatan, dan pemeriksaan keamanan sebelum atau sesudah memproses permintaan HTTP. Untuk mengakses filter aktif untuk router tertentu, Anda dapat menggunakan method CodeIgniter\Router\Router::getFilters() dari class Router.

Method ini mengembalikan daftar filter yang saat ini aktif untuk router yang sedang diproses:

<?php

// Get the router instance.
/** @var \CodeIgniter\Router\Router $router */
$router  = service('router');
$filters = $router->getFilters();

echo 'Active Filters for the Route: ' . implode(', ', $filters);

Catatan

Method getFilters() hanya mengembalikan filter yang didefinisikan untuk router tertentu. Ini tidak termasuk filter global atau yang ditentukan dalam berkas app/Config/Filters.php.

Mendapatkan Opsi Router yang Cocok untuk Router Saat Ini

Ketika kita mendefinisikan router, router tersebut mungkin memiliki parameter opsional: filter, namespace, hostname, subdomain, offset, priority, as. Semuanya telah dijelaskan sebelumnya di atas. Selain itu, jika kita menggunakan addRedirect() kita juga dapat mengharapkan kunci redirect. Untuk mengakses nilai-nilai parameter ini, kita dapat memanggil Router::getMatchedRouteOptions(). Berikut adalah contoh array yang dikembalikan:

<?php

// Get the router instance.
/** @var \CodeIgniter\Router\Router $router */
$router  = service('router');
$options = $router->getMatchedRouteOptions();

echo 'Route name: ' . $options['as'];

print_r($options);

// Route name: api:auth
//
// Array
// (
//     [filter] => api-auth
//     [namespace] => App\API\v1
//     [hostname] => example.com
//     [subdomain] => api
//     [offset] => 1
//     [priority] => 1
//     [as] => api:auth
// )
