Optimisasi

Sebelum men-deploy aplikasi CodeIgniter Anda ke produksi, ada beberapa hal yang dapat Anda lakukan untuk membuat aplikasi Anda berjalan lebih efisien.

Bagian ini menjelaskan fitur-fitur optimisasi yang disediakan oleh CodeIgniter.

spark optimize

Baru pada versi 4.5.0.

Perintah spark optimize melakukan optimisasi berikut:

  • Menghapus Paket Dev

  • Mengaktifkan Config Caching

  • Mengaktifkan FileLocator Caching

Optimisasi Composer

Menghapus Paket Dev

Saat Anda melakukan deploy, jangan lupa untuk menjalankan perintah berikut:

composer install --no-dev

Perintah di atas akan menghapus paket perpustakaan Composer hanya untuk pengembangan yang tidak dibutuhkan di lingkungan produksi. Ini akan sangat mengurangi ukuran folder vendor.

Menentukan Paket yang Akan Ditemukan

Jika Composer Package Auto-Discovery diaktifkan, semua paket Composer akan dipindai saat dibutuhkan. Tetapi tidak perlu memindai paket yang bukan paket CodeIgniter, jadi menentukan paket yang akan dipindai mencegah pemindaian yang tidak perlu.

Lihat Specify Composer Packages.

Config Caching

Penting

Setelah di-cache, nilai konfigurasi tidak akan pernah berubah sampai cache dihapus, bahkan jika file konfigurasi atau .env diubah.

Melakukan caching objek Config dapat meningkatkan kinerja. Namun, cache harus dihapus secara manual saat mengubah nilai Config.

Lihat Caching Konfigurasi.

FileLocator Caching

Melakukan caching jalur file yang ditemukan oleh FileLocator dapat meningkatkan kinerja. Namun, cache harus dihapus secara manual saat menambah/menghapus/mengubah jalur file.

Lihat Caching FileLocator.

PHP Preloading

Dengan PHP preloading, Anda dapat menginstruksikan server untuk memuat file-file penting seperti fungsi dan class ke dalam memori selama startup. Ini berarti elemen-elemen ini siap tersedia untuk semua permintaan, melewati proses pemuatan biasa dan meningkatkan kinerja aplikasi Anda. Namun, ini datang dengan biaya peningkatan penggunaan memori dan memerlukan restart mesin PHP agar perubahan berlaku.

Catatan

Jika Anda ingin menggunakan Preloading, kami menyediakan skrip preload.

Persyaratan

Menggunakan preloading memerlukan satu handler PHP khusus. Biasanya, server web dikonfigurasi untuk menggunakan satu handler PHP, jadi satu aplikasi memerlukan server web khusus. Jika Anda ingin menggunakan preloading untuk beberapa aplikasi di satu server web, konfigurasikan server Anda untuk menggunakan virtual host dengan beberapa handler PHP seperti beberapa PHP-FPM, dengan setiap virtual host menggunakan satu handler PHP. Preloading menyimpan definisi yang relevan dalam memori dengan membaca file yang ditentukan dalam opcache.preload.

Catatan

Lihat Running Multiple Applications with one CodeIgniter Installation untuk membuat satu inti CodeIgniter4 menangani beberapa aplikasi Anda.

Konfigurasi

Buka php.ini atau xx-opcache.ini jika Anda telah memisahkan konfigurasi INI di PHP, dan disarankan untuk mengatur opcache.preload=/path/to/preload.php dan opcache.preload_user=myuser.

Catatan

myuser adalah user yang berjalan di server web Anda. Jika Anda ingin menemukan lokasi konfigurasi INI yang terpisah, cukup jalankan php --ini atau buka file phpinfo() dan cari Additional .ini files parsed.

Pastikan Anda menggunakan instalasi appstarter. Jika menggunakan instalasi manual, Anda harus mengubah direktori di jalur include.

<?php

// ...

class preload
{
    /**
     * @var array Paths to preload.
     */
    private array $paths = [
        [
            'include' => __DIR__ . '/system', // <== change this line to where CI is installed
            // ...
        ],
    ];

    // ...
}

// ...

Deployment ke Layanan Shared Hosting

Penting

index.php tidak lagi berada di root proyek! Ini telah dipindahkan ke dalam folder public, untuk keamanan dan pemisahan komponen yang lebih baik.

Ini berarti Anda harus mengkonfigurasi server web Anda untuk "menunjuk" ke folder public proyek Anda, dan bukan ke root proyek.

Menentukan Document Root

Cara terbaik adalah mengatur document root ke folder public di konfigurasi server:

└── example.com/ (folder proyek)
    └── public/  (document root)

Tanyakan kepada penyedia layanan hosting Anda apakah Anda dapat mengubah document root. Sayangnya, jika Anda tidak dapat mengubah document root, lanjutkan ke cara berikutnya.

Menggunakan Dua Direktori

Cara kedua adalah menggunakan dua direktori, dan menyesuaikan jalur. Satu untuk aplikasi dan yang lainnya adalah document root default.

Unggah konten folder public ke public_html (document root default) dan file lainnya ke direktori untuk aplikasi:

├── example.com/ (untuk aplikasi)
│       ├── app/
│       ├── vendor/ (atau system/)
│       └── writable/
└── public_html/ (document root default)
        ├── .htaccess
        ├── favicon.ico
        ├── index.php
        └── robots.txt

Lihat Install CodeIgniter 4 on Shared Hosting (cPanel) untuk detailnya.

Menambahkan .htaccess

Pilihan terakhir adalah menambahkan .htaccess ke root proyek.

Tidak disarankan untuk menempatkan folder proyek di document root. Namun, jika Anda tidak punya pilihan lain, Anda bisa menggunakan ini.

Tempatkan folder proyek Anda sebagai berikut, di mana public_html adalah document root, dan buat file .htaccess:

└── public_html/     (document root default)
    └── example.com/ (folder proyek)
        ├── .htaccess
        └── public/

Dan edit .htaccess sebagai berikut:

<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteRule ^(.*)$ public/$1 [L]
</IfModule>

<FilesMatch "^\.">
    Require all denied
    Satisfy All
</FilesMatch>
