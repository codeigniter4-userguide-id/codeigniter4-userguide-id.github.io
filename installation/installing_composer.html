Instalasi Menggunakan Composer

  • App Starter

    • Instalasi

    • Konfigurasi Awal

    • Peningkatan (Upgrading)

Composer dapat digunakan dalam dua cara untuk menginstal CodeIgniter4 di sistem Anda.

Penting

CodeIgniter4 membutuhkan Composer 2.0.14 atau yang lebih baru.

Catatan

Jika Anda belum familiar dengan Composer, kami sarankan Anda membaca Penggunaan dasar terlebih dahulu.

Teknik pertama menjelaskan pembuatan proyek kerangka (app starter) menggunakan CodeIgniter4, yang kemudian akan Anda gunakan sebagai dasar untuk aplikasi web baru. Teknik kedua yang dijelaskan di bawah ini memungkinkan Anda menambahkan CodeIgniter4 ke aplikasi web yang sudah ada.

Catatan

Jika Anda menggunakan repositori Git untuk menyimpan kode Anda, atau untuk kolaborasi dengan orang lain, maka folder vendor biasanya akan "diabaikan oleh git". Dalam kasus seperti itu, Anda perlu melakukan composer install (atau composer update jika Anda ingin memperbarui semua dependensi Composer) saat Anda mengkloning repositori ke sistem baru.

App Starter

Repositori CodeIgniter 4 app starter berisi aplikasi kerangka, dengan dependensi composer pada versi framework yang dirilis terbaru.

Teknik instalasi ini cocok untuk pengembang yang ingin memulai proyek baru berbasis CodeIgniter4.

Instalasi

Di folder di atas root proyek Anda:

composer create-project codeigniter4/appstarter project-root

Perintah di atas akan membuat folder project-root.

Jika Anda menghilangkan argumen "project-root", perintah akan membuat folder "appstarter", yang dapat diganti namanya sesuai kebutuhan.

Catatan

Sebelum v4.4.0, autoloader CodeIgniter tidak mengizinkan karakter khusus yang ilegal dalam nama file pada sistem operasi tertentu. Simbol yang dapat digunakan adalah /, _, ., :, \ dan spasi. Jadi jika Anda menginstal CodeIgniter di bawah folder yang berisi karakter khusus seperti (, ), dll., CodeIgniter tidak berfungsi. Sejak v4.4.0, pembatasan ini telah dihapus.

Penting

Saat Anda melakukan deploy ke server produksi, jangan lupa untuk menjalankan perintah berikut:

composer install --no-dev

Perintah di atas akan menghapus paket Composer hanya untuk pengembangan yang tidak dibutuhkan di lingkungan produksi. Ini akan sangat mengurangi ukuran folder vendor.

Menginstal Versi Sebelumnya

Misalnya, Anda mungkin ingin menginstal v4.4.8 setelah v4.5.0 dirilis.

Dalam kasus tersebut, tentukan versi dalam perintah:

composer create-project codeigniter4/appstarter:4.4.8 project-root

Kemudian, buka composer.json di folder root proyek Anda, dan tentukan versi framework:

"require": {
    ...
    "codeigniter4/framework": "4.4.8"
},

Kemudian, jalankan perintah composer update.

Catatan

Saat Anda menggunakan nomor versi tetap seperti "codeigniter4/framework": "4.4.8" di composer.json Anda, perintah composer update tidak akan memperbarui framework ke versi terbaru. Lihat Writing Version Constraints untuk cara menentukan versi.

Konfigurasi Awal

Setelah instalasi, beberapa konfigurasi awal diperlukan. Lihat Konfigurasi Awal untuk detailnya.

Peningkatan (Upgrading)

Setiap kali ada rilis baru, maka dari baris perintah di root proyek Anda:

composer update

Baca instruksi peningkatan dan log perubahan, dan periksa Perubahan yang Melanggar (Breaking Changes) dan Peningkatan (Enhancements).

Meningkatkan ke Versi Tertentu

Misalnya, Anda mungkin ingin meningkatkan dari v4.4.7 ke v4.4.8 setelah v4.5.0 dirilis.

Dalam kasus tersebut, buka composer.json di folder root proyek Anda, dan tentukan versi framework:

"require": {
    ...
    "codeigniter4/framework": "4.4.8"
},

Kemudian, jalankan perintah composer update.
