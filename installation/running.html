Aplikasi CodeIgniter 4 dapat dijalankan dengan beberapa cara berbeda: di-host di server web, menggunakan virtualisasi, atau menggunakan alat baris perintah CodeIgniter untuk pengujian. Bagian ini membahas cara menggunakan setiap teknik, dan menjelaskan beberapa kelebihan dan kekurangannya.

Penting

Anda harus selalu berhati-hati tentang kasus nama file. Banyak pengembang mengembangkan di sistem file yang tidak peka huruf besar/kecil di Windows atau macOS. Namun, sebagian besar lingkungan server menggunakan sistem file yang peka huruf besar/kecil. Jika kasus nama file salah, kode yang berfungsi secara lokal tidak akan berfungsi di server.

Jika Anda baru mengenal CodeIgniter, silakan baca bagian Memulai dari Panduan Pengguna untuk mulai mempelajari cara membangun aplikasi PHP dinamis. Selamat menikmati!

Konfigurasi Awal

Konfigurasi untuk URI Situs Anda

Buka file app/Config/App.php dengan editor teks.

  1. $baseURL

    Atur URL dasar Anda ke $baseURL. Jika Anda membutuhkan lebih banyak fleksibilitas, baseURL dapat diatur dalam file .env sebagai app.baseURL = 'http://example.com/'. Selalu gunakan garis miring di akhir URL dasar Anda!

    Catatan

    Jika Anda tidak mengatur baseURL dengan benar, dalam mode pengembangan, toolbar debug mungkin tidak memuat dengan benar dan halaman web mungkin membutuhkan waktu lebih lama untuk ditampilkan.

  2. $indexPage

    Jika Anda tidak ingin menyertakan index.php dalam URI situs Anda, atur $indexPage ke ''. Pengaturan akan digunakan ketika framework menghasilkan URI situs Anda.

    Catatan

    Anda mungkin perlu mengkonfigurasi server web Anda untuk mengakses situs Anda dengan URL yang tidak berisi index.php. Lihat URL CodeIgniter.

Konfigurasi Pengaturan Koneksi Database

Jika Anda berniat menggunakan database, buka file app/Config/Database.php dengan editor teks dan atur pengaturan database Anda. Atau, ini dapat diatur dalam file .env Anda. Lihat Konfigurasi Database untuk detailnya.

Atur ke Mode Pengembangan

Jika tidak berada di server produksi, atur CI_ENVIRONMENT ke development dalam file .env untuk memanfaatkan alat debugging yang disediakan. Lihat Mengatur Mode Pengembangan untuk detailnya.

Penting

Di lingkungan produksi, Anda harus menonaktifkan tampilan kesalahan dan fungsionalitas khusus pengembangan lainnya. Di CodeIgniter, ini dapat dilakukan dengan mengatur lingkungan ke "production". Secara default, aplikasi akan berjalan menggunakan lingkungan "production". Lihat juga The ENVIRONMENT Constant.

Atur Izin Folder Writable

Jika Anda akan menjalankan situs Anda menggunakan server web (misalnya, Apache atau nginx), Anda perlu memodifikasi izin untuk folder writable di dalam proyek Anda, sehingga dapat ditulis oleh user atau akun yang digunakan oleh server web Anda.

Memeriksa Pengaturan PHP ini

Baru pada versi 4.5.0.

Pengaturan PHP ini mengubah perilaku PHP. CodeIgniter menyediakan perintah untuk memeriksa pengaturan PHP penting.

php spark phpini:check

Kolom Recommended menunjukkan nilai yang direkomendasikan untuk lingkungan produksi. Mereka mungkin berbeda di lingkungan pengembangan.

Catatan

Jika Anda tidak dapat menggunakan perintah spark, Anda dapat menggunakan CheckPhpIni::run(false) di controller Anda.

Misalnya,

<?php

namespace App\Controllers;

use CodeIgniter\Security\CheckPhpIni;

class Home extends BaseController
{
    public function index(): string
    {
        return CheckPhpIni::run(false);
    }
}

Server Pengembangan Lokal

CodeIgniter 4 dilengkapi dengan server pengembangan lokal, memanfaatkan server web bawaan PHP dengan routing CodeIgniter. Anda dapat meluncurkannya, dengan baris perintah berikut di direktori utama:

php spark serve

Ini akan meluncurkan server dan Anda sekarang dapat melihat aplikasi Anda di browser Anda di http://localhost:8080.

Catatan

Server pengembangan bawaan hanya boleh digunakan pada mesin pengembangan lokal. Ini TIDAK PERNAH boleh digunakan di server produksi.

Jika Anda perlu menjalankan situs di host selain localhost, Anda harus terlebih dahulu menambahkan host ke file hosts Anda. Lokasi pasti file bervariasi di setiap sistem operasi utama, meskipun semua sistem tipe unix (termasuk macOS) biasanya akan menyimpan file di /etc/hosts.

Server pengembangan lokal dapat disesuaikan dengan tiga opsi baris perintah:

  • Anda dapat menggunakan opsi CLI --host untuk menentukan host yang berbeda untuk menjalankan aplikasi:

    php spark serve --host example.dev

  • Secara default, server berjalan di port 8080 tetapi Anda mungkin memiliki lebih dari satu situs yang berjalan, atau sudah memiliki aplikasi lain yang menggunakan port tersebut. Anda dapat menggunakan opsi CLI --port untuk menentukan yang berbeda:

    php spark serve --port 8081

  • Anda juga dapat menentukan versi PHP tertentu untuk digunakan, dengan opsi CLI --php, dengan nilainya diatur ke jalur executable PHP yang ingin Anda gunakan:

    php spark serve --php /usr/bin/php7.6.5.4

Hosting dengan Apache

Aplikasi web CodeIgniter4 biasanya di-host di server web. Apache HTTP Server adalah platform "standar", dan diasumsikan dalam sebagian besar dokumentasi kami.

Apache dibundel dengan banyak platform, tetapi juga dapat diunduh dalam bundel dengan mesin database dan PHP dari Bitnami.

Konfigurasi File Konfigurasi Utama

Mengaktifkan mod_rewrite

Modul "mod_rewrite" memungkinkan URL tanpa "index.php" di dalamnya, dan diasumsikan dalam panduan pengguna kami.

Pastikan modul rewrite diaktifkan (tidak dikomentari) di file konfigurasi utama, misalnya, apache2/conf/httpd.conf:

LoadModule rewrite_module modules/mod_rewrite.so

Mengatur Document Root

Juga pastikan bahwa elemen <Directory> document root default mengaktifkan ini juga, dalam pengaturan AllowOverride:

<Directory "/opt/lamp/apache2/htdocs">
    Options Indexes FollowSymLinks
    AllowOverride All
    Require all granted
</Directory>

Hosting dengan VirtualHost

Kami merekomendasikan penggunaan "virtual hosting" untuk menjalankan aplikasi Anda. Anda dapat mengatur alias yang berbeda untuk setiap aplikasi yang Anda kerjakan,

Mengaktifkan vhost_alias_module

Pastikan modul virtual hosting diaktifkan (tidak dikomentari) di file konfigurasi utama, misalnya, apache2/conf/httpd.conf:

LoadModule vhost_alias_module modules/mod_vhost_alias.so

Menambahkan Alias Host

Tambahkan alias host di file "hosts" Anda, biasanya /etc/hosts pada platform tipe unix, atau c:\Windows\System32\drivers\etc\hosts di Windows.

Tambahkan baris ke file tersebut. Ini bisa berupa myproject.local atau myproject.test, misalnya:

127.0.0.1 myproject.local

Mengatur VirtualHost

Tambahkan elemen <VirtualHost> untuk aplikasi web Anda di dalam konfigurasi virtual hosting, misalnya, apache2/conf/extra/httpd-vhost.conf:

<VirtualHost *:80>
    DocumentRoot "/opt/lamp/apache2/myproject/public"
    ServerName   myproject.local
    ErrorLog     "logs/myproject-error_log"
    CustomLog    "logs/myproject-access_log" common

    <Directory "/opt/lamp/apache2/myproject/public">
        AllowOverride All
        Require all granted
    </Directory>
</VirtualHost>

Konfigurasi di atas mengasumsikan folder proyek terletak sebagai berikut:

apache2/
   ├── myproject/      (Folder Proyek)
   │      └── public/  (DocumentRoot untuk myproject.local)
   └── htdocs/

Mulai ulang Apache.

Pengujian

Dengan konfigurasi di atas, aplikasi web Anda akan diakses dengan URL http://myproject.local/ di browser Anda.

Apache perlu di-restart setiap kali Anda mengubah konfigurasinya.

Hosting dengan Subfolder

Jika Anda ingin baseURL seperti http://localhost/myproject/ dengan subfolder, ada tiga cara.

Membuat Symlink

Tempatkan folder proyek Anda sebagai berikut, di mana htdocs adalah document root Apache:

├── myproject/ (folder proyek)
│      └── public/
└── htdocs/

Navigasi ke folder htdocs dan buat symbolic link sebagai berikut:

cd htdocs/
ln -s ../myproject/public/ myproject

Menggunakan Alias

Tempatkan folder proyek Anda sebagai berikut, di mana htdocs adalah document root Apache:

├── myproject/ (folder proyek)
│      └── public/
└── htdocs/

Tambahkan yang berikut ini di file konfigurasi utama, misalnya, apache2/conf/httpd.conf:

Alias /myproject /opt/lamp/apache2/myproject/public
<Directory "/opt/lamp/apache2/myproject/public">
    AllowOverride All
    Require all granted
</Directory>

Mulai ulang Apache.

Menambahkan .htaccess

Pilihan terakhir adalah menambahkan .htaccess ke root proyek.

Tidak disarankan untuk menempatkan folder proyek di document root. Namun, jika Anda tidak punya pilihan lain, seperti di shared server, Anda bisa menggunakan ini.

Tempatkan folder proyek Anda sebagai berikut, di mana htdocs adalah document root Apache, dan buat file .htaccess:

└── htdocs/
    └── myproject/ (folder proyek)
        ├── .htaccess
        └── public/

Dan edit .htaccess sebagai berikut:

<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteRule ^(.*)$ public/$1 [L]
</IfModule>

<FilesMatch "^\.">
    Require all denied
    Satisfy All
</FilesMatch>

Dan hapus pengaturan pengalihan di public/.htaccess:

--- a/public/.htaccess
+++ b/public/.htaccess
@@ -16,16 +16,6 @@ Options -Indexes

-   RewriteCond %{HTTPS} !=on

Hosting dengan mod_userdir (Shared Hosts)

Praktik umum di lingkungan shared hosting adalah menggunakan modul Apache "mod_userdir" untuk mengaktifkan Virtual Host per-user secara otomatis. Konfigurasi tambahan diperlukan untuk memungkinkan CodeIgniter4 dijalankan dari direktori per-user ini.

Berikut ini mengasumsikan bahwa server sudah dikonfigurasi untuk mod_userdir. Panduan untuk mengaktifkan modul ini tersedia di dokumentasi Apache.

Karena CodeIgniter4 mengharapkan server untuk menemukan front controller framework di public/index.php secara default, Anda harus menentukan lokasi ini sebagai alternatif untuk mencari permintaan (bahkan jika CodeIgniter4 diinstal dalam direktori web per-user).

Direktori web user default ~/public_html ditentukan oleh direktif UserDir, biasanya di apache2/mods-available/userdir.conf atau apache2/conf/extra/httpd-userdir.conf:

UserDir public_html

Jadi Anda perlu mengkonfigurasi Apache untuk mencari direktori public CodeIgniter terlebih dahulu sebelum mencoba melayani default:

UserDir "public_html/public" "public_html"

Pastikan untuk menentukan opsi dan izin untuk direktori public CodeIgniter juga. Sebuah userdir.conf mungkin terlihat seperti:

<IfModule mod_userdir.c>
    UserDir "public_html/public" "public_html"
    UserDir disabled root

    <Directory /home/*/public_html>
        AllowOverride All
        Options MultiViews Indexes FollowSymLinks
        <Limit GET POST OPTIONS>
            # Apache <= 2.2:
            # Order allow,deny
            # Allow from all

            # Apache >= 2.4:
            Require all granted
        </Limit>
        <LimitExcept GET POST OPTIONS>
            # Apache <= 2.2:
            # Order deny,allow
            # Deny from all

            # Apache >= 2.4:
            Require all denied
        </LimitExcept>
    </Directory>

    <Directory /home/*/public_html/public>
        AllowOverride All
        Options MultiViews Indexes FollowSymLinks
        <Limit GET POST OPTIONS>
            # Apache <= 2.2:
            # Order allow,deny
            # Allow from all

            # Apache >= 2.4:
            Require all granted
        </Limit>
        <LimitExcept GET POST OPTIONS>
            # Apache <= 2.2:
            # Order deny,allow
            # Deny from all

            # Apache >= 2.4:
            Require all denied
        </LimitExcept>
    </Directory>
</IfModule>

Hosting dengan nginx

nginx adalah server HTTP kedua yang paling banyak digunakan untuk hosting web. Di sini Anda dapat menemukan contoh konfigurasi menggunakan PHP 8.1 FPM (unix sockets) di bawah Ubuntu Server.

default.conf

Konfigurasi ini memungkinkan URL tanpa "index.php" di dalamnya dan menggunakan "404 - File Not Found" CodeIgniter untuk URL yang berakhiran ".php".

server {
    listen 80;
    listen [::]:80;

    server_name example.com;

    root  /var/www/example.com/public;
    index index.php index.html index.htm;

    location / {
        try_files $uri $uri/ /index.php$is_args$args;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;

        # Dengan php-fpm:
        fastcgi_pass unix:/run/php/php8.1-fpm.sock;
        # Dengan php-cgi:
        # fastcgi_pass 127.0.0.1:9000;
    }

    error_page 404 /index.php;

    # menolak akses ke file tersembunyi seperti .htaccess
    location ~ /\. {
        deny all;
    }
}

Bootstrapping Aplikasi

Dalam beberapa skenario, Anda ingin memuat framework tanpa benar-benar menjalankan seluruh aplikasi. Ini sangat berguna untuk pengujian unit proyek Anda, tetapi juga dapat berguna untuk menggunakan alat pihak ketiga untuk menganalisis dan memodifikasi kode Anda. Framework dilengkapi dengan dua skrip bootstrap terpisah khusus untuk skenario ini:

  • system/Test/bootstrap.php: Skrip ini digunakan terutama untuk pengujian unit.

  • system/util_bootstrap.php: Skrip ini digunakan untuk skrip lain yang membutuhkan akses ke framework. Disarankan untuk digunakan dalam skrip yang bukan bagian untuk pengujian karena ini TIDAK akan gagal secara anggun jika terjadi exception.

Sebagian besar jalur ke proyek Anda didefinisikan selama proses bootstrap. Anda dapat menggunakan konstanta yang telah ditentukan sebelumnya untuk mengganti ini, tetapi saat menggunakan default pastikan jalur Anda selaras dengan struktur direktori yang diharapkan untuk metode instalasi Anda.
