Pemecahan Masalah

Berikut adalah beberapa masalah instalasi umum, dan solusi yang disarankan.

  • Bagaimana saya tahu jika instalasi saya berfungsi?

  • Saya harus menyertakan index.php di URL saya

  • Hanya halaman default yang dimuat

  • Tidak ada file input yang ditentukan

  • Aplikasi saya berfungsi dengan baik secara lokal tetapi tidak di server produksi

  • Tutorial memberikan kesalahan 404 di mana-mana :(

  • Ada apa dengan halaman "Whoops!" yang tidak membantu?

  • CodeIgniter Error Logs

Bagaimana saya tahu jika instalasi saya berfungsi?

Dari baris perintah, di root proyek Anda:

php spark serve

http://localhost:8080 di browser Anda seharusnya kemudian menampilkan halaman selamat datang default:

Saya harus menyertakan index.php di URL saya

Jika URL seperti /mypage/find/apple tidak berfungsi, tetapi URL serupa /index.php/mypage/find/apple berfungsi, itu berarti aturan .htaccess Anda (untuk Apache) tidak diatur dengan benar, atau ekstensi mod_rewrite di httpd.conf Apache dikomentari. Lihat Removing the index.php file.

Hanya halaman default yang dimuat

Jika Anda menemukan bahwa tidak peduli apa yang Anda masukkan di URL Anda hanya halaman default yang dimuat, mungkin server Anda tidak mendukung variabel REQUEST_URI yang diperlukan untuk menyajikan URL yang ramah mesin pencari. Sebagai langkah pertama, buka file app/Config/App.php Anda dan cari informasi URI Protocol. Ini akan merekomendasikan Anda untuk mencoba beberapa pengaturan alternatif. Jika masih tidak berfungsi setelah Anda mencoba ini, Anda perlu memaksa CodeIgniter untuk menambahkan tanda tanya (?) ke URL Anda. Untuk melakukan ini, buka file app/Config/App.php Anda dan ubah ini:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class App extends BaseConfig
{
    // ...

    public string $indexPage = 'index.php';

    // ...
}

Menjadi ini:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class App extends BaseConfig
{
    // ...

    public string $indexPage = 'index.php?';

    // ...
}

Tidak ada file input yang ditentukan

Jika Anda melihat "No input file specified", coba ubah aturan rewrite seperti berikut (untuk menambahkan ? setelah index.php):

RewriteRule ^([\s\S]*)$ index.php?/$1 [L,NC,QSA]

Aplikasi saya berfungsi dengan baik secara lokal tetapi tidak di server produksi

Pastikan bahwa kasus nama folder dan file cocok dengan kode.

Banyak pengembang mengembangkan di sistem file yang tidak peka huruf besar/kecil di Windows atau macOS. Namun, sebagian besar lingkungan server menggunakan sistem file yang peka huruf besar/kecil.

Misalnya, ketika Anda memiliki app/Controllers/Product.php, Anda harus menggunakan Product sebagai nama class singkat, bukan product.

Jika kasus nama file salah, file tidak ditemukan di server.

Tutorial memberikan kesalahan 404 di mana-mana :(

Anda tidak dapat mengikuti tutorial menggunakan server web bawaan PHP. Ini tidak memproses file .htaccess yang diperlukan untuk merutekan permintaan dengan benar.

Solusinya: gunakan Apache untuk melayani situs Anda, atau gunakan ekuivalen CodeIgniter bawaan, php spark serve dari root proyek Anda.

Ada apa dengan halaman "Whoops!" yang tidak membantu?

Anda menemukan bahwa aplikasi Anda menampilkan halaman dengan "Whoops!" dan kemudian baris teks "We seem to have hit a snag. Please try again later...".

Itu adalah tanda bahwa Anda berada dalam mode produksi dan telah mengalami kesalahan yang tidak dapat dipulihkan, yang tidak ingin kami tampilkan kepada penonton aplikasi web, demi keamanan yang lebih baik.

Anda dapat melihat kesalahan di file log. Lihat CodeIgniter Error Logs di bawah.

Jika Anda mencapai halaman ini saat mengembangkan, Anda harus mengubah environment Anda ke "development" (di .env). Lihat Mengatur Mode Pengembangan untuk detail lebih lanjut. Setelah itu, muat ulang halaman. Anda akan melihat kesalahan dan jejak balik.
