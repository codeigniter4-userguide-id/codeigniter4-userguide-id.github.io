Bekerja dengan Locale

CodeIgniter menyediakan beberapa alat untuk membantu Anda melokalisasi aplikasi Anda untuk berbagai bahasa. Meskipun lokalisasi penuh suatu aplikasi adalah subjek yang kompleks, mudah untuk menukar string dalam aplikasi Anda dengan bahasa yang didukung yang berbeda.

Mengkonfigurasi Locale

Mengatur Locale Default

Setiap situs akan memiliki bahasa/locale default tempat mereka beroperasi. Ini dapat diatur di app/Config/App.php:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class App extends BaseConfig
{
    // ...

    public string $defaultLocale = 'en';

    // ...
}

Nilai dapat berupa string apa pun yang digunakan aplikasi Anda untuk mengelola string teks dan format lainnya. Disarankan untuk menggunakan kode bahasa BCP 47 <http://www.rfc-editor.org/rfc/bcp/bcp47.txt>. Ini menghasilkan kode bahasa seperti en-US untuk Bahasa Inggris Amerika, atau fr-FR, untuk Bahasa Prancis/Prancis. Pengantar yang lebih mudah dibaca tentang ini dapat ditemukan di situs W3C.

Sistem cukup pintar untuk kembali ke kode bahasa yang lebih umum jika kecocokan persis tidak dapat ditemukan. Jika kode locale diatur ke en-US dan kita hanya memiliki file bahasa yang diatur untuk en maka itu akan digunakan karena tidak ada yang ada untuk en-US yang lebih spesifik. Namun, jika direktori bahasa ada di direktori app/Language/en-US maka itu akan digunakan terlebih dahulu.

Deteksi Locale

Penting

Deteksi locale hanya berfungsi untuk permintaan berbasis web yang menggunakan class IncomingRequest. Permintaan baris perintah tidak akan memiliki fitur ini.

Ada dua metode yang didukung untuk mendeteksi locale yang benar selama permintaan.

  1. Negosiasi Konten: Yang pertama adalah metode "atur dan lupakan" yang secara otomatis akan melakukan negosiasi konten untuk Anda untuk menentukan locale yang benar untuk digunakan.

  2. Dalam Route: Metode kedua memungkinkan Anda untuk menentukan segmen dalam route Anda yang akan digunakan untuk mengatur locale.

Jika Anda perlu mengatur locale secara langsung, lihat Mengatur Locale Saat Ini.

Sejak v4.4.0, IncomingRequest::setValidLocales() telah ditambahkan untuk mengatur (dan mengatur ulang) locale yang valid yang diatur dari pengaturan Config\App::$supportedLocales.

Negosiasi Konten

Anda dapat mengatur negosiasi konten agar terjadi secara otomatis dengan mengatur dua pengaturan tambahan di app/Config/App.php. Nilai pertama memberi tahu class Request bahwa kita ingin menegosiasikan locale, jadi cukup atur ke true:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class App extends BaseConfig
{
    // ...

    public bool $negotiateLocale = true;

    // ...
}

Setelah ini diaktifkan, sistem akan secara otomatis menegosiasikan bahasa yang benar berdasarkan array locale yang telah Anda definisikan di $supportLocales. Jika tidak ada kecocokan yang ditemukan antara bahasa yang Anda dukung, dan bahasa yang diminta, item pertama dalam $supportedLocales akan digunakan. Dalam contoh berikut, locale en akan digunakan jika tidak ada kecocokan yang ditemukan:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class App extends BaseConfig
{
    // ...

    public array $supportedLocales = ['en', 'es', 'fr-FR'];

    // ...
}

Dalam Route

Metode kedua menggunakan placeholder kustom untuk mendeteksi locale yang diinginkan dan mengaturnya pada Request. Placeholder {locale} dapat ditempatkan sebagai segmen dalam route Anda. Jika ada, isi segmen yang cocok akan menjadi locale Anda:

$routes->get('{locale}/books', 'App\Books::index');

Dalam contoh ini, jika pengguna mencoba mengunjungi http://example.com/fr/books, maka locale akan diatur ke fr, dengan asumsi itu dikonfigurasi sebagai locale yang valid.

Jika nilai tidak cocok dengan locale yang valid seperti yang didefinisikan dalam $supportedLocales di app/Config/App.php, locale default akan digunakan sebagai gantinya, kecuali Anda mengatur untuk hanya menggunakan locale yang didukung yang didefinisikan dalam file konfigurasi App:

$routes->useSupportedLocalesOnly(true);

Catatan

Metode useSupportedLocalesOnly() dapat digunakan sejak v4.3.0.

Mengatur Locale Saat Ini

Locale IncomingRequest

Jika Anda ingin mengatur locale secara langsung, Anda dapat menggunakan metode setLocale() di IncomingRequest Class:

/** @var \CodeIgniter\HTTP\IncomingRequest $request */
$request->setLocale('ja');

Sebelum mengatur locale, Anda harus mengatur locale yang valid. Karena setiap upaya untuk mengatur locale yang tidak valid akan mengakibatkan locale default diatur.

Secara default, locale yang valid didefinisikan dalam Config\App::$supportedLocales di app/Config/App.php:

<?php

namespace Config;

use CodeIgniter\Config\BaseConfig;

class App extends BaseConfig
{
    // ...

    public array $supportedLocales = ['en', 'es', 'fr-FR'];

    // ...
}

Catatan

Sejak v4.4.0, IncomingRequest::setValidLocales() telah ditambahkan untuk mengatur (dan mengatur ulang) locale yang valid. Gunakan ini jika Anda ingin mengubah locale yang valid secara dinamis.

Language Locale

Class Language yang digunakan dalam fungsi lang() juga memiliki locale saat ini. Ini diatur ke locale IncomingRequest selama instansiasi.

Jika Anda ingin mengubah locale setelah menginstansiasi class bahasa, gunakan metode Language::setLocale().

/** @var \CodeIgniter\Language\Language $lang */
$lang = service('language');
$lang->setLocale('ja');

Mengambil Locale Saat Ini

Locale saat ini selalu dapat diambil dari objek IncomingRequest, melalui metode getLocale(). Jika controller Anda memperluas CodeIgniter\Controller, ini akan tersedia melalui $this->request:

<?php

namespace App\Controllers;

class UserController extends BaseController
{
    public function index()
    {
        $locale = $this->request->getLocale();
    }
}

Sebagai alternatif, Anda dapat menggunakan class Services untuk mengambil permintaan saat ini:

$locale = service('request')->getLocale();

Lokalisasi Bahasa

Membuat File Bahasa

String bahasa disimpan di direktori app/Language, dengan sub-direktori untuk setiap bahasa yang didukung (locale):

app/
    Language/
        en/
            App.php
        fr/
            App.php

Catatan

File Bahasa tidak memiliki namespace.

Bahasa tidak memiliki konvensi penamaan khusus yang diperlukan. File harus diberi nama secara logis untuk menjelaskan jenis konten yang dipegangnya. Misalnya, katakanlah Anda ingin membuat file yang berisi pesan kesalahan. Anda mungkin menamainya sederhana: Errors.php.

Di dalam file, Anda akan mengembalikan array, di mana setiap elemen dalam array memiliki kunci bahasa dan dapat memiliki string untuk dikembalikan:

<?php

return [
    'languageKey' => 'The actual message to be shown.',
];

Catatan

Anda tidak dapat menggunakan titik (.) di awal dan akhir kunci bahasa.

Ini juga mendukung definisi bersarang:

<?php

return [
    'languageKey' => [
        'nested' => [
            'key' => 'The actual message to be shown.',
        ],
    ],
];

<?php

return [
    'errorEmailMissing'    => 'You must submit an email address',
    'errorURLMissing'      => 'You must submit a URL',
    'errorUsernameMissing' => 'You must submit a username',
    'nested'               => [
        'error' => [
            'message' => 'A specific error message',
        ],
    ],
];

Penggunaan Dasar

Anda dapat menggunakan fungsi helper lang() untuk mengambil teks dari file bahasa mana pun, dengan meneruskan nama file dan kunci bahasa sebagai parameter pertama, dipisahkan oleh titik (.).

Misalnya, untuk memuat string errorEmailMissing dari file bahasa Errors.php, Anda akan melakukan hal berikut:

echo lang('Errors.errorEmailMissing');

Untuk definisi bersarang, Anda akan melakukan hal berikut:

echo lang('Errors.nested.error.message');

Jika kunci bahasa yang diminta tidak ada dalam file untuk locale saat ini (setelah Language Fallback), string akan diteruskan kembali, tidak berubah. Dalam contoh ini, itu akan mengembalikan Errors.errorEmailMissing atau Errors.nested.error.message jika tidak ada.

Mengganti Parameter

Catatan

Fungsi-fungsi berikut semuanya memerlukan ekstensi intl untuk dimuat di sistem Anda agar berfungsi. Jika ekstensi tidak dimuat, tidak ada penggantian yang akan dicoba. Ikhtisar yang bagus dapat ditemukan di Sitepoint.

Anda dapat meneruskan array nilai untuk mengganti placeholder dalam string bahasa sebagai parameter kedua ke fungsi lang(). Ini memungkinkan terjemahan dan pemformatan angka yang sangat sederhana:

<?php

// The language file, Tests.php:
return [
    'apples'      => 'I have {0, number} apples.',
    'men'         => 'The top {1, number} men out-performed the remaining {0, number}',
    'namedApples' => 'I have {number_apples, number, integer} apples.',
];

// Displays "I have 3 apples."
echo lang('Tests.apples', [3]);

Item pertama dalam placeholder sesuai dengan indeks item dalam array, jika numerik:

// Displays "The top 23 men out-performed the remaining 20"
echo lang('Tests.men', [20, 23]);

Anda juga dapat menggunakan kunci bernama untuk membuatnya lebih mudah untuk menjaga hal-hal tetap jelas, jika Anda mau:

// Displays "I have 3 apples."
echo lang('Tests.namedApples', ['number_apples' => 3]);

Jelas, Anda bisa melakukan lebih dari sekadar penggantian angka. Menurut dokumen ICU resmi untuk library yang mendasarinya, jenis data berikut dapat diganti:

  • angka - integer, mata uang, persen

  • tanggal - short, medium, long, full

  • waktu - short, medium, long, full

  • spellout - mengeja angka (yaitu, 34 menjadi tiga puluh empat)

  • ordinal

  • durasi

Berikut adalah beberapa contoh:

<?php

// The language file, Tests.php
return [
    'shortTime'  => 'The time is now {0, time, short}.',
    'mediumTime' => 'The time is now {0, time, medium}.',
    'longTime'   => 'The time is now {0, time, long}.',
    'fullTime'   => 'The time is now {0, time, full}.',
    'shortDate'  => 'The date is now {0, date, short}.',
    'mediumDate' => 'The date is now {0, date, medium}.',
    'longDate'   => 'The date is now {0, date, long}.',
    'fullDate'   => 'The date is now {0, date, full}.',
    'spelledOut' => '34 is {0, spellout}',
    'ordinal'    => 'The ordinal is {0, ordinal}',
    'duration'   => 'It has been {0, duration}',
];

// Displays "The time is now 11:18 PM"
echo lang('Tests.shortTime', [time()]);
// Displays "The time is now 11:18:50 PM"
echo lang('Tests.mediumTime', [time()]);
// Displays "The time is now 11:19:09 PM CDT"
echo lang('Tests.longTime', [time()]);
// Displays "The time is now 11:19:26 PM Central Daylight Time"
echo lang('Tests.fullTime', [time()]);

// Displays "The date is now 8/14/16"
echo lang('Tests.shortDate', [time()]);
// Displays "The date is now Aug 14, 2016"
echo lang('Tests.mediumDate', [time()]);
// Displays "The date is now August 14, 2016"
echo lang('Tests.longDate', [time()]);
// Displays "The date is now Sunday, August 14, 2016"
echo lang('Tests.fullDate', [time()]);

// Displays "34 is thirty-four"
echo lang('Tests.spelledOut', [34]);

// Displays "It has been 408,676:24:35"
echo lang('Tests.ordinal', [time()]);

Anda harus memastikan untuk membaca tentang class MessageFormatter dan pemformatan ICU yang mendasarinya untuk mendapatkan ide yang lebih baik tentang kemampuan apa yang dimilikinya, seperti melakukan penggantian kondisional, pluralisasi, dan banyak lagi. Kedua tautan yang disediakan sebelumnya akan memberi Anda ide yang sangat baik mengenai opsi yang tersedia.

Menentukan Locale

Untuk menentukan locale yang berbeda untuk digunakan saat mengganti parameter, Anda dapat meneruskan locale sebagai parameter ketiga ke fungsi lang().

<?php

// Displays "The time is now 23:21:28 GMT-5"
echo lang('Test.longTime', [time()], 'ru-RU');

// Displays "£7.41"
echo lang('{price, number, currency}', ['price' => 7.41], 'en-GB');
// Displays "$7.41"
echo lang('{price, number, currency}', ['price' => 7.41], 'en-US');

Jika Anda ingin mengubah locale saat ini, lihat Language Locale.

Array Bersarang

File bahasa juga memungkinkan array bersarang untuk memudahkan bekerja dengan daftar, dll...

<?php

// Language/en/Fruit.php

return [
    'list' => [
        'Apples',
        'Bananas',
        'Grapes',
        'Lemons',
        'Oranges',
        'Strawberries',
    ],
];

// Displays "Apples, Bananas, Grapes, Lemons, Oranges, Strawberries"
echo implode(', ', lang('Fruit.list'));

Language Fallback

Jika Anda memiliki sekumpulan pesan untuk locale tertentu, misalnya Language/en/App.php, Anda dapat menambahkan varian bahasa untuk locale tersebut, masing-masing di foldernya sendiri, misalnya Language/en-US/App.php.

Anda hanya perlu memberikan nilai untuk pesan-pesan yang akan dilokalkan secara berbeda untuk varian locale tersebut. Setiap definisi pesan yang hilang akan secara otomatis ditarik dari pengaturan locale utama.

Ini menjadi lebih baik - lokalisasi dapat kembali sepenuhnya ke Bahasa Inggris (en), jika ada pesan baru yang ditambahkan ke framework dan Anda belum memiliki kesempatan untuk menerjemahkannya untuk locale Anda.

Jadi, jika Anda menggunakan locale fr-CA, maka pesan yang dilokalkan pertama-tama akan dicari di direktori Language/fr-CA, kemudian di direktori Language/fr, dan akhirnya di direktori Language/en.

Terjemahan Pesan Sistem

Kami memiliki set "resmi" terjemahan pesan sistem di repositori mereka sendiri.

Anda dapat mengunduh repositori itu, dan menyalin folder Language-nya ke dalam folder app Anda. Terjemahan yang digabungkan akan secara otomatis diambil karena namespace App dipetakan ke folder app Anda.

Sebagai alternatif, praktik yang lebih baik adalah menjalankan perintah berikut di dalam proyek Anda:

composer require codeigniter4/translations

Pesan yang diterjemahkan akan secara otomatis diambil karena folder terjemahan dipetakan dengan tepat.

Menimpa Terjemahan Pesan Sistem

Framework menyediakan Terjemahan Pesan Sistem, dan paket yang Anda instal juga dapat menyediakan terjemahan pesan.

Jika Anda ingin menimpa beberapa pesan bahasa, buat file bahasa di direktori app/Language. Kemudian, kembalikan hanya array yang ingin Anda timpa di dalam file.

Menghasilkan File Terjemahan melalui Command

Baru pada versi 4.5.0.

Anda dapat secara otomatis menghasilkan dan memperbarui file terjemahan di folder app Anda. Perintah akan mencari penggunaan fungsi lang(), menggabungkan kunci terjemahan saat ini di app/Language dengan mendefinisikan locale defaultLocale dari Config\App. Setelah operasi, Anda perlu menerjemahkan kunci bahasa sendiri. Perintah ini mampu mengenali kunci bersarang secara normal File.array.nested.text. Kunci yang sebelumnya disimpan tidak berubah.

php spark lang:find

<?php

// Controllers/Translation/Lang.php
$message  = lang('Text.info.success');
$message2 = lang('Text.paragraph');

// The following will be saved in Language/en/Text.php
return [
    'info' => [
        'success' => 'Text.info.success',
    ],
    'paragraph' => 'Text.paragraph',
];

Catatan

Ketika perintah memindai folder, app/Language akan dilewati.

File bahasa yang dihasilkan kemungkinan besar tidak akan sesuai dengan standar pengkodean Anda. Disarankan untuk memformatnya. Misalnya, jalankan vendor/bin/php-cs-fixer fix ./app/Language jika php-cs-fixer diinstal.

Sebelum memperbarui, dimungkinkan untuk melihat pratinjau terjemahan yang ditemukan oleh perintah:

php spark lang:find --verbose --show-new

Output detail dari --verbose juga menunjukkan daftar kunci yang tidak valid. Contoh:

...

Files found: 10
New translates found: 30
Bad translates found: 5
+------------------------+---------------------------------+
| Kunci Buruk            | Jalur File                      |
+------------------------+---------------------------------+
| ..invalid_nested_key.. | app/Controllers/Translation.php |
| .invalid_key           | app/Controllers/Translation.php |
| TranslationBad         | app/Controllers/Translation.php |
| TranslationBad.        | app/Controllers/Translation.php |
| TranslationBad...      | app/Controllers/Translation.php |
+------------------------+---------------------------------+

Semua operasi selesai!

Untuk pencarian yang lebih akurat, tentukan locale atau direktori yang diinginkan untuk dipindai.

php spark lang:find --dir Controllers/Translation --locale en --show-new

Informasi detail dapat ditemukan dengan menjalankan perintah:

php spark lang:find --help

Sinkronisasi File Terjemahan melalui Command

Baru pada versi 4.6.0.

Anda mungkin perlu membuat file untuk bahasa lain setelah Anda selesai menerjemahkan untuk bahasa saat ini. Anda dapat menggunakan perintah spark lang:find untuk membantu ini. Namun, mungkin tidak mendeteksi semua terjemahan, terutama yang memiliki parameter yang diatur secara dinamis seperti lang('App.status.' . $key, ['payload' => 'John'], 'en').

Untuk memastikan tidak ada terjemahan yang terlewat, sebaiknya salin file bahasa yang sudah selesai dan terjemahkan secara manual. Pendekatan ini mempertahankan kunci unik apa pun yang mungkin terlewatkan oleh perintah.

Yang perlu Anda lakukan hanyalah mengeksekusi:

// Tentukan locale untuk terjemahan baru/yang diperbarui
php spark lang:sync --target ru

// atau atur locale asli
php spark lang:sync --locale en --target ru

Sebagai hasilnya, Anda akan menerima file dengan kunci terjemahan. Jika ada kunci duplikat di locale target, kunci tersebut akan disimpan.

Peringatan

Kunci yang tidak cocok dalam terjemahan baru akan dihapus!
