HTTP Responses

Class Response memperluas HTTP Message Class dengan metode yang hanya sesuai untuk server yang merespons klien yang memanggilnya.

Bekerja dengan Response

Sebuah class Response diinstansiasi untuk Anda dan diteruskan ke controller Anda. Ini dapat diakses melalui $this->response. Ini adalah instance yang sama yang dikembalikan oleh Services::response(). Kami menyebutnya instance response global.

Seringkali Anda tidak perlu menyentuh class secara langsung, karena CodeIgniter mengurus pengiriman header dan body untuk Anda. Ini bagus jika halaman berhasil membuat konten yang diminta. Ketika ada yang salah, atau Anda perlu mengirim kode status yang sangat spesifik kembali, atau bahkan memanfaatkan caching HTTP yang kuat, itu ada untuk Anda.

Mengatur Output

Ketika Anda perlu mengatur output skrip secara langsung, dan tidak bergantung pada CodeIgniter untuk mendapatkannya secara otomatis, Anda melakukannya secara manual dengan metode setBody. Ini biasanya digunakan bersamaan dengan pengaturan kode status response:

<?php

$this->response->setStatusCode(404)->setBody($body);

Frasa alasan ('OK', 'Created', 'Moved Permanently') akan secara otomatis ditambahkan, tetapi Anda dapat menambahkan alasan kustom sebagai parameter kedua dari metode setStatusCode():

<?php

$this->response->setStatusCode(404, 'Nope. Not here.');

Anda dapat mengatur format array menjadi JSON atau XML dan mengatur header tipe konten ke mime yang sesuai dengan metode setJSON() dan setXML(). Biasanya, Anda akan mengirimkan array data untuk dikonversi:

<?php

$data = [
    'success' => true,
    'id'      => 123,
];

return $this->response->setJSON($data);

// or
return $this->response->setXML($data);

Mengatur Header

setHeader()

Seringkali, Anda perlu mengatur header untuk response. Class Response membuatnya sangat mudah untuk dilakukan, dengan metode setHeader().

Parameter pertama adalah nama header. Parameter kedua adalah nilai, yang bisa berupa string atau array nilai yang akan digabungkan dengan benar saat dikirim ke klien.

<?php

$this->response->setHeader('Location', 'http://example.com')
    ->setHeader('WWW-Authenticate', 'Negotiate');

Menggunakan fungsi-fungsi ini alih-alih menggunakan fungsi PHP native memungkinkan Anda untuk memastikan bahwa tidak ada header yang dikirim terlalu dini, menyebabkan kesalahan, dan memungkinkan pengujian.

Penting

Sejak v4.6.0, jika Anda mengatur header menggunakan fungsi PHP native header() dan kemudian menggunakan class Response untuk mengatur header yang sama, yang sebelumnya akan ditimpa.

Catatan

Metode ini hanya mengatur header ke instance response. Jadi, jika Anda membuat dan mengembalikan instance response lain (misalnya, jika Anda memanggil redirect()), header yang diatur di sini tidak akan dikirim secara otomatis.

appendHeader()

Jika header ada dan dapat memiliki lebih dari satu nilai, Anda dapat menggunakan metode appendHeader() dan prependHeader() untuk menambahkan nilai ke akhir atau awal daftar nilai, masing-masing. Parameter pertama adalah nama header, sedangkan yang kedua adalah nilai yang akan ditambahkan atau diawali.

<?php

$this->response->setHeader('Cache-Control', 'no-cache')
    ->appendHeader('Cache-Control', 'must-revalidate');

removeHeader()

Header dapat dihapus dari response dengan metode removeHeader(), yang mengambil nama header sebagai satu-satunya parameter. Ini tidak peka huruf besar/kecil.

<?php

$this->response->removeHeader('Location');

Redirect

Jika Anda ingin membuat redirect, gunakan fungsi redirect().

Ini mengembalikan instance RedirectResponse. Ini adalah instance yang berbeda dari instance response global yang dikembalikan oleh Services::response().

Peringatan

Jika Anda mengatur Cookies atau Response Headers sebelum Anda memanggil redirect(), mereka diatur ke instance response global, dan mereka tidak secara otomatis disalin ke instance RedirectResponse. Untuk mengirimnya, Anda perlu memanggil metode withCookies() atau withHeaders() secara manual.

Penting

Jika Anda ingin mengarahkan ulang, instance RedirectResponse harus dikembalikan dalam metode Controller atau Controller Filter. Perhatikan bahwa metode __construct() atau initController() tidak dapat mengembalikan nilai apa pun. Jika Anda lupa mengembalikan RedirectResponse, tidak ada pengalihan yang akan terjadi.

Redirect ke Jalur URI

Ketika Anda ingin meneruskan jalur URI (relatif terhadap baseURL), gunakan redirect()->to():

// Go to specific URI path. "admin/home" is the URI path relative to baseURL.
return redirect()->to('admin/home');

Catatan

Jika ada fragmen di URL Anda yang ingin Anda hapus, Anda dapat menggunakan parameter refresh dalam metode. Seperti return redirect()->to('admin/home', null, 'refresh');.

Redirect ke Route yang Didefinisikan

Ketika Anda ingin meneruskan nama route atau Controller::method untuk reverse routing, gunakan redirect()->route():

// Go to a named route. "user_gallery" is the route name, not a URI path.
return redirect()->route('user_gallery');

Ketika meneruskan argumen ke dalam fungsi, itu diperlakukan sebagai nama route atau Controller::method untuk reverse routing, bukan URI relatif/lengkap, memperlakukan itu sama dengan menggunakan redirect()->route():

// Go to a named/reverse-routed URI.
return redirect('named_route');

Redirect Kembali

Ketika Anda ingin mengarahkan ulang kembali, gunakan redirect()->back():

// Go back to the previous page.
return redirect()->back();

// Keep the old input values upon redirect so they can be used by the `old()` function.
return redirect()->back()->withInput();

// Set a flash message.
return redirect()->back()->with('foo', 'message');

Catatan

redirect()->back() tidak sama dengan tombol "kembali" browser. Ini membawa pengunjung ke "halaman terakhir yang dilihat selama Session" ketika Session tersedia. Jika Session belum dimuat, atau tidak tersedia, maka versi HTTP_REFERER yang telah dibersihkan akan digunakan.

Redirect dengan Cookies

Jika Anda mengatur Cookies sebelum Anda memanggil redirect(), mereka diatur ke instance response global, dan mereka tidak secara otomatis disalin ke instance RedirectResponse.

Untuk mengirim Cookies, Anda perlu memanggil metode withCookies() secara manual.

// Copies all cookies from global response instance.
return redirect()->back()->withCookies();

Redirect dengan Header

Jika Anda mengatur Response Headers sebelum Anda memanggil redirect(), mereka diatur ke instance response global, dan mereka tidak secara otomatis disalin ke instance RedirectResponse.

Untuk mengirim Header, Anda perlu memanggil metode withHeaders() secara manual.

// Copies all headers from the global response instance.
return redirect()->back()->withHeaders();

Kode Status Redirect

Kode status HTTP default untuk permintaan GET adalah 302. Namun, ketika menggunakan HTTP/1.1 atau yang lebih baru, 303 digunakan untuk permintaan POST/PUT/DELETE dan 307 untuk semua permintaan lainnya.

Anda dapat menentukan kode status:

// Redirect to a URI path relative to baseURL with status code 301.
return redirect()->to('admin/home', 301);

// Redirect to a route with status code 308.
return redirect()->route('user_gallery', [], 308);

// Redirect back with status code 302.
return redirect()->back(302);

Catatan

Karena bug, di v4.3.3 atau versi sebelumnya, kode status response redirect yang sebenarnya mungkin berubah bahkan jika kode status telah ditentukan. Lihat ChangeLog v4.3.4.

Jika Anda tidak tahu kode status HTTP untuk pengalihan, disarankan untuk membaca Redirections in HTTP.

Paksa Unduh File

Class Response menyediakan cara sederhana untuk mengirim file ke klien, meminta browser untuk mengunduh data ke komputer Anda. Ini mengatur header yang sesuai untuk mewujudkannya.

Parameter pertama adalah nama yang Anda inginkan untuk file yang diunduh, parameter kedua adalah data file.

Jika Anda mengatur parameter kedua ke null dan $filename adalah jalur file yang ada dan dapat dibaca, maka kontennya akan dibaca sebagai gantinya.

Jika Anda mengatur parameter ketiga ke boolean true, maka tipe MIME file yang sebenarnya (berdasarkan ekstensi nama file) akan dikirim, sehingga jika browser Anda memiliki handler untuk tipe tersebut - ia dapat menggunakannya.

Contoh:

<?php

$data = 'Here is some text!';
$name = 'mytext.txt';

return $this->response->download($name, $data);

Jika Anda ingin mengunduh file yang ada dari server Anda, Anda perlu meneruskan null secara eksplisit untuk parameter kedua:

<?php

// Contents of photo.jpg will be automatically read
return $this->response->download('/path/to/photo.jpg', null);

Gunakan metode opsional setFileName() untuk mengubah nama file saat dikirim ke browser klien:

<?php

return $this->response->download('awkwardEncryptedFileName.fakeExt', null)->setFileName('expenses.csv');

Catatan

Objek response HARUS dikembalikan agar unduhan dapat dikirim ke klien. Ini memungkinkan response untuk melewati semua filter setelah sebelum dikirim ke klien.

Buka File di Browser

Beberapa browser dapat menampilkan file seperti PDF. Untuk memberi tahu browser agar menampilkan file alih-alih menyimpannya, panggil metode DownloadResponse::inline().

<?php

$data = 'Here is some text!';
$name = 'mytext.txt';

return $this->response->download($name, $data)->inline();

HTTP Caching

Dibangun dalam spesifikasi HTTP adalah alat yang membantu klien (seringkali browser web) meng-cache hasilnya. Digunakan dengan benar, ini dapat menyebabkan peningkatan kinerja yang besar pada aplikasi Anda karena akan memberi tahu klien bahwa mereka tidak perlu menghubungi server sama sekali karena tidak ada yang berubah. Dan Anda tidak bisa lebih cepat dari itu.

Ini ditangani melalui header Cache-Control dan ETag. Panduan ini bukan tempat yang tepat untuk pengantar menyeluruh tentang semua kekuatan header cache, tetapi Anda bisa mendapatkan pemahaman yang baik di Google Developers.

Secara default, semua objek response yang dikirim melalui CodeIgniter memiliki caching HTTP dimatikan. Opsi dan keadaan yang tepat terlalu bervariasi bagi kami untuk dapat membuat default yang baik selain mematikannya. Sangat mudah untuk mengatur nilai Cache sesuai kebutuhan Anda, melalui metode setCache():

<?php

$options = [
    'max-age'  => 300,
    's-maxage' => 900,
    'etag'     => 'abcde',
];
$this->response->setCache($options);

Array $options hanya mengambil array pasangan kunci/nilai yang, dengan beberapa pengecualian, ditetapkan ke header Cache-Control. Anda bebas mengatur semua opsi persis seperti yang Anda butuhkan untuk situasi spesifik Anda. Meskipun sebagian besar opsi diterapkan ke header Cache-Control, ia secara cerdas menangani opsi etag dan last-modified ke header yang sesuai.

Parameter:

  • $name (array|Cookie|string) -- Nama cookie atau array asosiatif dari semua parameter yang tersedia untuk metode ini atau instance CodeIgniter\Cookie\Cookie

  • $value (string) -- Nilai cookie

  • $expire (int) -- Waktu kedaluwarsa cookie dalam detik. Jika diatur ke 0, cookie hanya akan berlaku selama browser terbuka

  • $domain (string) -- Domain cookie

  • $path (string) -- Jalur cookie

  • $prefix (string) -- Prefiks nama cookie. Jika diatur ke '', nilai default dari app/Config/Cookie.php akan digunakan

  • $secure (bool) -- Apakah hanya mentransfer cookie melalui HTTPS. Jika diatur ke null, nilai default dari app/Config/Cookie.php akan digunakan

  • $httponly (bool) -- Apakah hanya membuat cookie dapat diakses untuk permintaan HTTP (tanpa JavaScript). Jika diatur ke null, nilai default dari app/Config/Cookie.php akan digunakan

  • $samesite (string) -- Nilai untuk parameter cookie SameSite. Jika diatur ke '', tidak ada atribut SameSite yang akan diatur pada cookie. Jika diatur ke null, nilai default dari app/Config/Cookie.php akan digunakan

Return type:

void

Catatan

Sebelum v4.2.7, nilai default $secure dan $httponly adalah false karena bug, dan nilai-nilai ini dari app/Config/Cookie.php tidak pernah digunakan.

Mengatur cookie yang berisi nilai yang Anda tentukan ke instance Response.

Ada dua cara untuk meneruskan informasi ke metode ini agar cookie dapat diatur: Metode Array, dan Parameter Diskrit:

Metode Array

Menggunakan metode ini, array asosiatif diteruskan sebagai parameter pertama:

<?php

$cookie = [
    'name'     => 'The Cookie Name',
    'value'    => 'The Value',
    'expire'   => 86500,
    'domain'   => '.some-domain.com',
    'path'     => '/',
    'prefix'   => 'myprefix_',
    'secure'   => true,
    'httponly' => false,
    'samesite' => 'Lax',
];

$response->setCookie($cookie);

Hanya name dan value yang wajib. Untuk menghapus cookie, atur dengan value kosong.

expire diatur dalam detik, yang akan ditambahkan ke waktu saat ini. Jangan sertakan waktu, melainkan hanya jumlah detik dari sekarang yang Anda inginkan agar cookie valid. Jika expire diatur ke nol, cookie hanya akan berlaku selama browser terbuka.

Catatan

Tetapi jika value diatur ke string kosong dan expire diatur ke 0, cookie akan dihapus.

Untuk cookie seluruh situs terlepas dari bagaimana situs Anda diminta, tambahkan URL Anda ke domain dimulai dengan titik, seperti ini: .domain-anda.com

path biasanya tidak diperlukan karena metode ini mengatur jalur root.

prefix hanya diperlukan jika Anda perlu menghindari tabrakan nama dengan cookie lain yang namanya sama untuk server Anda.

Flag secure hanya diperlukan jika Anda ingin menjadikannya cookie aman dengan mengaturnya ke true.

Nilai samesite mengontrol bagaimana cookie dibagikan antara domain dan sub-domain. Nilai yang diizinkan adalah 'None', 'Lax', 'Strict' atau string kosong ''. Jika diatur ke string kosong, atribut SameSite default akan diatur.

Parameter Diskrit

Jika Anda lebih suka, Anda dapat mengatur cookie dengan meneruskan data menggunakan parameter individual:

<?php

$response->setCookie($name, $value, $expire, $domain, $path, $prefix, $secure, $httponly, $samesite);
deleteCookie($name = ''[, $domain = ''[, $path = '/'[, $prefix = '']]])
Parameter:

  • $name (mixed) -- Nama cookie atau array parameter

  • $domain (string) -- Domain cookie

  • $path (string) -- Jalur cookie

  • $prefix (string) -- Prefiks nama cookie

Return type:

void

Menghapus cookie yang ada.

Catatan

Ini juga hanya mengatur cookie browser untuk menghapus cookie.

Hanya name yang wajib.

prefix hanya diperlukan jika Anda perlu menghindari tabrakan nama dengan cookie lain yang namanya sama untuk server Anda.

Berikan prefix jika cookie hanya boleh dihapus untuk subset tersebut. Berikan nama domain jika cookie hanya boleh dihapus untuk domain tersebut. Berikan nama path jika cookie hanya boleh dihapus untuk jalur tersebut.

Jika salah satu parameter opsional kosong, maka cookie dengan nama yang sama akan dihapus di seluruh yang berlaku.

Contoh:

<?php

$response->deleteCookie($name);

Catatan

Hanya name yang wajib. Jika prefix ditentukan, itu akan ditambahkan ke nama cookie.

Jika tidak ada value yang diberikan, metode hanya memeriksa keberadaan cookie yang disebutkan. Jika value diberikan, maka metode memeriksa bahwa cookie ada, dan bahwa ia memiliki nilai yang ditentukan.

Contoh:

<?php

if ($response->hasCookie($name)) {
    // ...
}
getCookie($name = ''[, $prefix = ''])
Parameter:

  • $name (string) -- Nama cookie

  • $prefix (string) -- Prefiks nama cookie

Return type:

Cookie|Cookie[]|null

Mengembalikan cookie yang disebutkan, jika ditemukan, atau null. Jika tidak ada name yang diberikan, mengembalikan array objek Cookie.

Contoh:

<?php

$cookie = $response->getCookie($name);
getCookies()
Return type:

Cookie[]

Mengembalikan semua cookie yang saat ini diatur dalam instance Response. Ini adalah cookie apa pun yang telah Anda tentukan secara khusus untuk diatur selama permintaan saat ini saja.
