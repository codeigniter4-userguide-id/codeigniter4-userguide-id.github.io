View Cells

Banyak aplikasi memiliki fragmen view kecil yang dapat diulang dari halaman ke halaman, atau di berbagai tempat di halaman. Ini seringkali berupa kotak bantuan, kontrol navigasi, iklan, formulir login, dll. CodeIgniter memungkinkan Anda untuk mengkapsulasi logika untuk blok presentasi ini dalam View Cells. Mereka pada dasarnya adalah mini-view yang dapat disertakan dalam view lain. Mereka dapat memiliki logika yang dibangun untuk menangani logika tampilan khusus cell. Mereka dapat digunakan untuk membuat view Anda lebih mudah dibaca dan dipelihara dengan memisahkan logika untuk setiap cell ke dalam class-nya sendiri.

Simple dan Controlled Cells

CodeIgniter mendukung dua jenis View Cells: simple dan controlled.

Simple View Cells dapat dihasilkan dari class dan metode pilihan Anda dan tidak harus mengikuti aturan apa pun, kecuali harus mengembalikan string.

Controlled View Cells harus dihasilkan dari class yang memperluas class Codeigniter\View\Cells\Cell yang menyediakan kemampuan tambahan yang membuat View Cells Anda lebih fleksibel dan lebih cepat digunakan.

Memanggil View Cell

Tidak peduli jenis View Cell apa yang Anda gunakan, Anda dapat memanggilnya dari view mana pun dengan menggunakan fungsi helper view_cell().

Parameter pertama adalah (1) nama class dan metode (Simple Cell) atau (2) nama class dan metode opsional (Controlled Cell) yang akan dipanggil, dan parameter kedua adalah array atau string parameter yang akan diteruskan ke metode:

// In a View.

// Simple Cell
<?= view_cell('App\Cells\MyClass::myMethod', ['param1' => 'value1', 'param2' => 'value2']) ?>

// Controlled Cell
<?= view_cell('App\Cells\MyCell', ['param1' => 'value1', 'param2' => 'value2']) ?>

String yang dikembalikan Cell akan disisipkan ke dalam view di mana fungsi view_cell() dipanggil.

Penghapusan Namespace

Baru pada versi 4.3.0.

Jika Anda tidak menyertakan namespace lengkap untuk class, ia akan mengasumsikan dapat ditemukan di namespace App\Cells. Jadi, contoh berikut akan mencoba menemukan class MyClass di app/Cells/MyClass.php. Jika tidak ditemukan di sana, semua namespace akan dipindai hingga ditemukan, mencari di dalam subdirektori Cells dari setiap namespace:

// In a View.
<?= view_cell('MyClass::myMethod', ['param1' => 'value1', 'param2' => 'value2']) ?>

Meneruskan Parameter sebagai String Kunci/Nilai

Anda juga dapat meneruskan parameter sebagai string kunci/nilai:

// In a View.
<?= view_cell('MyClass::myMethod', 'param1=value1, param2=value2') ?>

Simple Cells

Simple Cells adalah class yang mengembalikan string dari metode yang dipilih. Contoh cell Pesan Peringatan sederhana mungkin terlihat seperti ini:

<?php

namespace App\Cells;

class AlertMessage
{
    public function show(array $params): string
    {
        return "<div class=\"alert alert-{$params['type']}\">{$params['message']}</div>";
    }
}

Anda akan memanggilnya dari dalam view seperti:

// In a View.
<?= view_cell('AlertMessage::show', ['type' => 'success', 'message' => 'The user has been updated.']) ?>

Selain itu, Anda dapat menggunakan nama parameter yang cocok dengan variabel parameter dalam metode untuk keterbacaan yang lebih baik. Ketika Anda menggunakannya dengan cara ini, semua parameter harus selalu ditentukan dalam panggilan view cell:

// In a View.
<?= view_cell('Blog::recentPosts', 'category=codeigniter, limit=5') ?>

<?php

// In a Cell.

namespace App\Cells;

class Blog
{
    // ...

    public function recentPosts(string $category, int $limit): string
    {
        $posts = $this->blogModel->where('category', $category)
            ->orderBy('published_on', 'desc')
            ->limit($limit)
            ->get();

        return view('recentPosts', ['posts' => $posts]);
    }
}

Controlled Cells

Baru pada versi 4.3.0.

Controlled cells memiliki dua tujuan utama: (1) untuk membuatnya secepat mungkin untuk membangun cell, dan (2) menyediakan logika tambahan dan fleksibilitas ke view Anda, jika diperlukan.

Class harus memperluas CodeIgniter\View\Cells\Cell. Mereka harus memiliki file view di folder yang sama. Sesuai konvensi, nama class harus dalam PascalCase diakhiri dengan Cell dan view harus versi snake_case dari nama class, tanpa sufiks. Misalnya, jika Anda memiliki class MyCell, file view harus my.php.

Catatan

Sebelum v4.3.5, file view yang dihasilkan berakhir dengan _cell.php. Meskipun v4.3.5 dan yang lebih baru akan menghasilkan file view tanpa sufiks _cell, file view yang ada masih akan ditemukan dan dimuat.

Membuat Controlled Cell

Pada tingkat paling dasar, yang perlu Anda implementasikan dalam class hanyalah properti publik. Properti ini akan tersedia secara otomatis untuk file view.

Mengimplementasikan AlertMessage dari atas sebagai Controlled Cell akan terlihat seperti ini:

<?php

// app/Cells/AlertMessageCell.php

namespace App\Cells;

use CodeIgniter\View\Cells\Cell;

class AlertMessageCell extends Cell
{
    public $type;
    public $message;
}

// app/Cells/alert_message.php
<div class="alert alert-<?= esc($type, 'attr') ?>">
    <?= esc($message) ?>
</div>

// Called in main View:
<?= view_cell('AlertMessageCell', 'type=warning, message=Failed.') ?>

Catatan

Jika Anda menggunakan properti bertipe, Anda harus mengatur nilai awal:

<?php

// app/Cells/AlertMessageCell.php

namespace App\Cells;

use CodeIgniter\View\Cells\Cell;

class AlertMessageCell extends Cell
{
    public string $type    = '';
    public string $message = '';
}

Menghasilkan Cell melalui Command

Anda juga dapat membuat controlled cell melalui command bawaan dari CLI. Command-nya adalah php spark make:cell. Ia mengambil satu argumen, nama cell yang akan dibuat. Nama harus dalam PascalCase, dan class akan dibuat di direktori app/Cells. File view juga akan dibuat di direktori app/Cells.

php spark make:cell AlertMessageCell

Menggunakan View yang Berbeda

Anda dapat menentukan nama view kustom dengan mengatur properti view dalam class. View akan ditemukan seperti view biasanya:

<?php

namespace App\Cells;

use CodeIgniter\View\Cells\Cell;

class AlertMessageCell extends Cell
{
    public $type;
    public $message;

    protected string $view = 'my/custom/view';
}

Menyesuaikan Rendering

Jika Anda memerlukan kontrol lebih atas rendering HTML, Anda dapat mengimplementasikan metode render(). Metode ini memungkinkan Anda untuk melakukan logika tambahan dan meneruskan data ekstra ke view, jika diperlukan. Metode render() harus mengembalikan string.

Untuk memanfaatkan fitur lengkap Controlled Cells, Anda harus menggunakan $this->view() alih-alih fungsi helper view() biasa:

<?php

namespace App\Cells;

use CodeIgniter\View\Cells\Cell;

class AlertMessageCell extends Cell
{
    public $type;
    public $message;

    public function render(): string
    {
        return $this->view('my/custom/view', ['extra' => 'data']);
    }
}

Properti Terhitung (Computed Properties)

Jika Anda perlu melakukan logika tambahan untuk satu atau lebih properti, Anda dapat menggunakan properti terhitung. Ini memerlukan pengaturan properti menjadi protected atau private dan mengimplementasikan metode publik yang namanya terdiri dari nama properti yang diapit oleh get dan Property:

// In a View. Initialize the protected properties.
<?= view_cell('AlertMessageCell', ['type' => 'note', 'message' => 'test']) ?>

<?php

// app/Cells/AlertMessageCell.php

namespace App\Cells;

use CodeIgniter\View\Cells\Cell;

class AlertMessageCell extends Cell
{
    protected $type;
    protected $message;
    private $computed;

    public function mount(): void
    {
        $this->computed = sprintf('%s - %s', $this->type, $this->message);
    }

    public function getComputedProperty(): string
    {
        return $this->computed;
    }

    public function getTypeProperty(): string
    {
        return $this->type;
    }

    public function getMessageProperty(): string
    {
        return $this->message;
    }
}

// app/Cells/alert_message.php
<div>
    <p>type - <?= esc($type) ?></p>
    <p>message - <?= esc($message) ?></p>
    <p>computed: <?= esc($computed) ?></p>
</div>

Penting

Anda tidak dapat mengatur properti yang dideklarasikan sebagai private selama inisialisasi cell.

Metode Presentasi

Terkadang Anda perlu melakukan logika tambahan untuk view, tetapi Anda tidak ingin meneruskannya sebagai parameter. Anda dapat mengimplementasikan metode yang akan dipanggil dari dalam view cell itu sendiri. Ini dapat membantu keterbacaan view Anda:

<?php

// app/Cells/RecentPostsCell.php

namespace App\Cells;

use CodeIgniter\View\Cells\Cell;

class RecentPostsCell extends Cell
{
    protected $posts;

    public function linkPost($post): string
    {
        return anchor('posts/' . $post->id, $post->title);
    }
}

// app/Cells/recent_posts.php
<ul>
    <?php foreach ($posts as $post): ?>
        <li><?= $this->linkPost($post) ?></li>
    <?php endforeach ?>
</ul>

Melakukan Logika Penyiapan

Jika Anda perlu melakukan logika tambahan sebelum view dirender, Anda dapat mengimplementasikan metode mount(). Metode ini akan dipanggil tepat setelah class diinstansiasi, dan dapat digunakan untuk mengatur properti tambahan atau melakukan logika lain:

<?php

namespace App\Cells;

use CodeIgniter\View\Cells\Cell;

class RecentPostsCell extends Cell
{
    protected $posts;

    public function mount(): void
    {
        $this->posts = model('PostModel')->orderBy('created_at', 'DESC')->findAll(10);
    }
}

Anda dapat meneruskan parameter tambahan ke metode mount() dengan meneruskannya sebagai array ke fungsi helper view_cell(). Parameter apa pun yang dikirim yang cocok dengan nama parameter metode mount() akan diteruskan:

<?php

// app/Cells/RecentPostsCell.php

namespace App\Cells;

use CodeIgniter\View\Cells\Cell;

class RecentPostsCell extends Cell
{
    protected $posts;

    public function mount(?int $categoryId): void
    {
        $this->posts = model('PostModel')
            ->when(
                $categoryId,
                static fn ($query, $categoryId) => $query->where('category_id', $categoryId),
            )
            ->orderBy('created_at', 'DESC')
            ->findAll(10);
    }
}

// Called in main View:
<?= view_cell('RecentPostsCell', ['categoryId' => 5]) ?>

Cell Caching

Anda dapat meng-cache hasil panggilan view cell dengan meneruskan jumlah detik untuk meng-cache data sebagai parameter ketiga. Ini akan menggunakan engine cache yang sedang dikonfigurasi:

// Cache the view for 5 minutes
<?= view_cell('App\Cells\Blog::recentPosts', 'limit=5', 300) ?>

Anda dapat memberikan nama kustom untuk digunakan alih-alih yang dihasilkan secara otomatis jika Anda mau, dengan meneruskan nama baru sebagai parameter keempat:

// Cache the view for 5 minutes
<?= view_cell('App\Cells\Blog::recentPosts', 'limit=5', 300, 'newcacheid') ?>
