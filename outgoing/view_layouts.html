View Layouts

  • Membuat Layout

  • Menggunakan Layout di Views

  • Merender View

  • Menyertakan View Partials

CodeIgniter mendukung sistem layout yang sederhana, namun sangat fleksibel, yang memudahkan penggunaan satu atau lebih layout halaman dasar di seluruh aplikasi Anda. Layout mendukung bagian konten yang dapat disisipkan dari view apa pun yang sedang dirender. Anda dapat membuat layout yang berbeda untuk mendukung satu kolom, dua kolom, halaman arsip blog, dan lainnya. Layout tidak pernah dirender secara langsung. Sebaliknya, Anda merender view, yang menentukan layout yang ingin diperluas.

Membuat Layout

Layout adalah view seperti yang lainnya. Satu-satunya perbedaan adalah tujuan penggunaannya. Layout adalah satu-satunya file view yang akan menggunakan metode renderSection(). Metode ini berfungsi sebagai placeholder untuk konten.

Contoh: app/Views/default.php:

<!doctype html>
<html>
<head>
    <title>Layout Saya</title>
</head>
<body>
    <?= $this->renderSection('content') ?>
</body>
</html>

Metode renderSection() memiliki dua argumen: $sectionName dan $saveData. $sectionName adalah nama bagian yang digunakan oleh view anak mana pun untuk menamai bagian konten. Jika argumen boolean $saveData diatur ke true, metode ini menyimpan data untuk panggilan berikutnya. Jika tidak, metode ini membersihkan data setelah menampilkan konten.

Contoh: app/Views/welcome_message.php:

<!doctype html>
<html>
<head>
    <title><?= $this->renderSection('page_title', true) ?></title>
</head>
<body>
    <h1><?= $this->renderSection('page_title') ?><h1>
    <p><?= $this->renderSection('content') ?></p>
</body>
</html>

Catatan

$saveData dapat digunakan sejak v4.4.0.

Menggunakan Layout di Views

Kapan pun sebuah view ingin disisipkan ke dalam layout, ia harus menggunakan metode extend() di bagian atas file:

<?= $this->extend('default') ?>

Metode extend() mengambil nama file view apa pun yang ingin Anda gunakan. Karena mereka adalah view standar, mereka akan ditemukan seperti sebuah view. Secara default, ia akan mencari di direktori View aplikasi, tetapi juga akan memindai namespace lain yang didefinisikan PSR-4. Anda dapat menyertakan namespace untuk menemukan view di direktori View namespace tertentu:

<?= $this->extend('Blog\Views\default') ?>

Semua konten dalam view yang memperluas layout harus disertakan dalam panggilan metode section($name) dan endSection(). Setiap konten di antara panggilan ini akan disisipkan ke dalam layout di mana pun panggilan renderSection($name) yang cocok dengan nama bagian tersebut ada.

Contoh: app/Views/some_view.php:

<?= $this->extend('default') ?>

<?= $this->section('content') ?>
    <h1>Halo Dunia!</h1>
<?= $this->endSection() ?>

endSection() tidak memerlukan nama bagian. Ia secara otomatis tahu bagian mana yang harus ditutup.

Bagian dapat berisi bagian bersarang:

<?= $this->extend('default') ?>

<?= $this->section('content') ?>
    <h1>Halo Dunia!</h1>
    <?= $this->section('javascript') ?>
       let a = 'a';
    <?= $this->endSection() ?>
<?= $this->endSection() ?>

Merender View

Merender view dan layout-nya dilakukan persis seperti view lainnya yang akan ditampilkan dalam controller:

<?php

namespace App\Controllers;

class MyController extends BaseController
{
    public function index()
    {
        return view('some_view');
    }
}

Ini merender View app/Views/some_view.php dan jika itu memperluas default, Layout app/Views/default.php juga digunakan secara otomatis. Renderer cukup pintar untuk mendeteksi apakah view harus dirender sendiri, atau jika memerlukan layout.

Menyertakan View Partials

View partial adalah file view yang tidak memperluas layout apa pun. Mereka biasanya menyertakan konten yang dapat digunakan kembali dari view ke view. Saat menggunakan layout view, Anda harus menggunakan $this->include() untuk menyertakan view partial apa pun.

<?= $this->extend('default') ?>

<?= $this->section('content') ?>
    <h1>Halo Dunia!</h1>

    <?= $this->include('sidebar') ?>
<?= $this->endSection() ?>

Saat memanggil metode include(), Anda dapat meneruskan semua opsi yang sama seperti saat merender view normal, termasuk direktif cache, dll.
