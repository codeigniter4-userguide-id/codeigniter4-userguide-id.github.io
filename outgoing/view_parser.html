View Parser dapat melakukan substitusi teks sederhana untuk pseudo-variabel yang terdapat dalam file view Anda. Ini dapat mengurai variabel sederhana atau pasangan tag variabel.

Nama pseudo-variabel atau konstruksi kontrol diapit dalam kurung kurawal, seperti ini:

<html>
<head>
    <title>{blog_title}</title>
</head>
<body>
    <h3>{blog_heading}</h3>

    {blog_entries}
        <h5>{title}</h5>
        <p>{body}</p>
    {/blog_entries}

</body>
</html>

Variabel-variabel ini bukanlah variabel PHP yang sebenarnya, melainkan representasi teks biasa yang memungkinkan Anda untuk menghilangkan PHP dari template Anda (file view).

Catatan

CodeIgniter tidak mengharuskan Anda untuk menggunakan class ini karena menggunakan PHP murni di halaman view Anda (misalnya menggunakan View renderer) membuatnya berjalan sedikit lebih cepat. Namun, beberapa developer lebih suka menggunakan beberapa bentuk template engine jika mereka bekerja dengan desainer yang mereka rasa akan menemukan kebingungan bekerja dengan PHP.

Menggunakan View Parser Class

Metode paling sederhana untuk memuat class parser adalah melalui service-nya:

<?php

$parser = service('parser');

Alternatifnya, jika Anda tidak menggunakan class Parser sebagai renderer default Anda, Anda dapat menginstansinya secara langsung:

<?php

$parser = new \CodeIgniter\View\Parser();

Kemudian Anda dapat menggunakan salah satu dari tiga metode rendering standar yang disediakannya: render(), setVar() dan setData(). Anda juga dapat menentukan delimiter secara langsung, melalui metode setDelimiters().

Penting

Menggunakan Parser, template view Anda hanya diproses oleh Parser itu sendiri, dan tidak seperti skrip PHP view konvensional. Kode PHP dalam skrip tersebut diabaikan oleh parser, dan hanya substitusi yang dilakukan.

Ini disengaja: file view tanpa PHP.

Apa yang Dilakukannya

Class Parser memproses "skrip PHP/HTML" yang disimpan di jalur view aplikasi. Skrip ini tidak boleh mengandung PHP.

Setiap parameter view (yang kami sebut sebagai pseudo-variabel) memicu substitusi, berdasarkan jenis nilai yang Anda berikan untuknya. Pseudo-variabel tidak diekstraksi ke dalam variabel PHP; sebaliknya nilainya diakses melalui sintaks pseudo-variabel, di mana namanya direferensikan di dalam kurung kurawal.

Class Parser menggunakan array asosiatif secara internal, untuk mengumpulkan pengaturan pseudo-variabel hingga Anda memanggil render(). Ini berarti nama pseudo-variabel Anda harus unik, atau pengaturan parameter yang lebih baru akan menimpa yang sebelumnya.

Ini juga berdampak pada nilai parameter escaping untuk konteks yang berbeda di dalam skrip Anda. Anda harus memberikan setiap nilai yang di-escape nama parameter yang unik.

Parser Templates

Anda dapat menggunakan metode render() untuk mengurai (atau merender) template sederhana, seperti ini:

<?php

$data = [
    'blog_title'   => 'My Blog Title',
    'blog_heading' => 'My Blog Heading',
];

return $parser->setData($data)->render('blog_template');

Parameter view diteruskan ke setData() sebagai array asosiatif data untuk diganti dalam template.

Dalam contoh di atas, template akan berisi dua variabel: {blog_title} dan {blog_heading}

Parameter pertama untuk render() berisi nama template, di mana blog_template adalah nama file template view Anda.

Penting

Jika ekstensi file dihilangkan, maka view diharapkan berakhir dengan ekstensi .php.

Opsi Konfigurasi Parser

Beberapa opsi dapat diteruskan ke metode render() atau renderString().

  • cache - waktu dalam detik, untuk menyimpan hasil view; diabaikan untuk renderString()

  • cache_name - ID yang digunakan untuk menyimpan/mengambil hasil view yang di-cache; default ke viewpath; diabaikan untuk renderString()

  • saveData - true jika parameter data view harus dipertahankan untuk panggilan berikutnya; default adalah true

  • cascadeData - true jika pengaturan pseudo-variabel harus diteruskan ke substitusi bersarang; default adalah true

<?php

return $parser->render('blog_template', [
    'cache'      => HOUR,
    'cache_name' => 'something_unique',
]);

Variasi Substitusi

Ada tiga jenis substitusi yang didukung: sederhana, perulangan, dan bersarang. Substitusi dilakukan dalam urutan yang sama dengan penambahan pseudo-variabel.

Substitusi sederhana yang dilakukan oleh parser adalah penggantian satu-ke-satu pseudo-variabel di mana parameter data yang sesuai memiliki nilai skalar atau string, seperti dalam contoh ini:

<?php

$template = '<head><title>{blog_title}</title></head>';
$data     = ['blog_title' => 'My ramblings'];

return $parser->setData($data)->renderString($template);
// Result: <head><title>My ramblings</title></head>

Parser membawa substitusi jauh lebih jauh dengan "pasangan variabel", digunakan untuk substitusi bersarang atau perulangan, dan dengan beberapa konstruksi lanjutan untuk substitusi kondisional.

Ketika parser dieksekusi, umumnya akan

  • menangani setiap substitusi kondisional

  • menangani setiap substitusi bersarang/perulangan

  • menangani substitusi tunggal yang tersisa

Substitusi Perulangan

Substitusi perulangan terjadi ketika nilai untuk pseudo-variabel adalah array sekuensial dari array, seperti array pengaturan baris.

Kode contoh di atas memungkinkan variabel sederhana untuk diganti. Bagaimana jika Anda ingin seluruh blok variabel diulang, dengan setiap iterasi berisi nilai baru? Pertimbangkan contoh template yang kami tunjukkan di bagian atas halaman:

<html>
<head>
    <title>{blog_title}</title>
</head>
<body>
    <h3>{blog_heading}</h3>

    {blog_entries}
        <h5>{title}</h5>
        <p>{body}</p>
    {/blog_entries}

</body>
</html>

Dalam kode di atas Anda akan melihat sepasang variabel: {blog_entries} data... {/blog_entries}. Dalam kasus seperti ini, seluruh bagian data antara pasangan ini akan diulang beberapa kali, sesuai dengan jumlah baris dalam elemen "blog_entries" dari array parameter.

Mengurai pasangan variabel dilakukan menggunakan kode identik yang ditunjukkan di atas untuk mengurai variabel tunggal, kecuali, Anda akan menambahkan array multi-dimensi yang sesuai dengan data pasangan variabel Anda. Pertimbangkan contoh ini:

<?php

$data = [
    'blog_title'   => 'My Blog Title',
    'blog_heading' => 'My Blog Heading',
    'blog_entries' => [
        ['title' => 'Title 1', 'body' => 'Body 1'],
        ['title' => 'Title 2', 'body' => 'Body 2'],
        ['title' => 'Title 3', 'body' => 'Body 3'],
        ['title' => 'Title 4', 'body' => 'Body 4'],
        ['title' => 'Title 5', 'body' => 'Body 5'],
    ],
];

return $parser->setData($data)->render('blog_template');

Nilai untuk pseudo-variabel blog_entries adalah array asosiatif sekuensial dari array. Tingkat luar tidak memiliki kunci yang terkait dengan setiap "baris" bersarang.

Jika data "pasangan" Anda berasal dari hasil database, yang sudah merupakan array multi-dimensi, Anda cukup menggunakan metode database getResultArray():

<?php

$query = $db->query('SELECT * FROM blog');

$data = [
    'blog_title'   => 'My Blog Title',
    'blog_heading' => 'My Blog Heading',
    'blog_entries' => $query->getResultArray(),
];

return $parser->setData($data)->render('blog_template');

Jika array yang Anda coba ulangi berisi object alih-alih array, parser akan terlebih dahulu mencari metode asArray() pada object. Jika ada, metode tersebut akan dipanggil dan array yang dihasilkan kemudian diulang seperti yang dijelaskan di atas. Jika tidak ada metode asArray(), object akan di-cast sebagai array dan properti publiknya akan tersedia untuk Parser.

Ini sangat berguna dengan class Entity, yang memiliki metode asArray() yang mengembalikan semua properti publik dan terproteksi (dikurangi properti _options) dan membuatnya tersedia untuk Parser.

Substitusi Bersarang

Substitusi bersarang terjadi ketika nilai untuk pseudo-variabel adalah array asosiatif dari nilai, seperti record dari database:

<?php

$data = [
    'blog_title'   => 'My Blog Title',
    'blog_heading' => 'My Blog Heading',
    'blog_entries' => [
        [
            'title' => 'Title 1',
            'body'  => 'Body 1',
        ],
    ],
];

return $parser->setData($data)->render('blog_template');

Nilai untuk pseudo-variabel blog_entries adalah array asosiatif. Pasangan kunci/nilai yang didefinisikan di dalamnya akan diekspos di dalam perulangan pasangan variabel untuk variabel tersebut.

Sebuah blog_template.php yang mungkin berfungsi untuk di atas:

<h1>{blog_title} - {blog_heading}</h1>
{blog_entries}
    <div>
        <h2>{title}</h2>
        <p>{body}</p>
    </div>
{/blog_entries}

Jika Anda ingin pseudo-variabel lain dapat diakses di dalam scope blog_entries, maka pastikan opsi cascadeData diatur ke true.

Komentar

Anda dapat menempatkan komentar di template Anda yang akan diabaikan dan dihapus selama parsing dengan membungkus komentar dalam simbol {# #}.

{# Komentar ini dihapus selama parsing. #}
{blog_entry}
    <div>
        <h2>{title}</h2>
        <p>{body}</p>
    </div>
{/blog_entry}

Data Bertingkat (Cascading Data)

Dengan substitusi bersarang dan perulangan, Anda memiliki opsi untuk mengalirkan pasangan data ke dalam substitusi bagian dalam.

Contoh berikut tidak terpengaruh oleh cascading:

<?php

$template = '{name} lives in {locations}{city} on {planet}{/locations}.';

$data = [
    'name'      => 'George',
    'locations' => [
        ['city' => 'Red City', 'planet' => 'Mars'],
    ],
];

return $parser->setData($data)->renderString($template);
// Result: George lives in Red City on Mars.

Contoh ini memberikan hasil yang berbeda, tergantung pada cascading:

<?php

$template = '{locations}{name} lives in {city} on {planet}{/locations}.';

$data = [
    'name'      => 'George',
    'locations' => [
        ['city' => 'Red City', 'planet' => 'Mars'],
    ],
];

return $parser->setData($data)->renderString($template, ['cascadeData' => false]);
// Result: {name} lives in Red City on Mars.

// or

return $parser->setData($data)->renderString($template, ['cascadeData' => true]);
// Result: George lives in Red City on Mars.

Mencegah Parsing

Anda dapat menentukan bagian halaman agar tidak diurai dengan pasangan tag {noparse} {/noparse}. Apa pun di bagian ini akan tetap persis seperti adanya, tanpa substitusi variabel, perulangan, dll., yang terjadi pada markup di antara kurung kurawal.

{noparse}
    <h1>Kode Tidak Tersentuh</h1>
{/noparse}

Logika Kondisional

Class Parser mendukung beberapa kondisional dasar untuk menangani sintaks if, else, dan elseif. Semua blok if harus ditutup dengan tag endif:

{if $role=='admin'}
    <h1>Selamat Datang, Admin!</h1>
{endif}

Blok sederhana ini diubah menjadi berikut selama parsing:

<?php if ($role === 'admin'): ?>
    <h1>Welcome, Admin!</h1>
<?php endif ?>

Semua variabel yang digunakan dalam pernyataan if harus telah diatur sebelumnya dengan nama yang sama. Selain itu, ini diperlakukan persis seperti kondisional PHP standar, dan semua aturan PHP standar akan berlaku di sini. Anda dapat menggunakan operator perbandingan apa pun yang biasa Anda gunakan, seperti ==, ===, !==, <, >, dll.

{if $role=='admin'}
    <h1>Selamat Datang, Admin</h1>
{elseif $role=='moderator'}
    <h1>Selamat Datang, Moderator</h1>
{else}
    <h1>Selamat Datang, Pengguna</h1>
{endif}

Peringatan

Di latar belakang, kondisional diurai menggunakan eval(), jadi Anda harus memastikan bahwa Anda berhati-hati dengan data pengguna yang digunakan dalam kondisional, atau Anda dapat membuka aplikasi Anda terhadap risiko keamanan.

Mengubah Delimiter Kondisional

Jika Anda memiliki kode JavaScript seperti berikut di template Anda, Parser akan menimbulkan kesalahan sintaks karena ada string yang dapat diinterpretasikan sebagai kondisional:

<script type="text/javascript">
    var f = function() {
        if (hasAlert) {
            alert('{message}');
        }
    }
</script>

Dalam kasus tersebut, Anda dapat mengubah delimiter untuk kondisional dengan metode setConditionalDelimiters() untuk menghindari salah tafsir:

<?php

$parser->setConditionalDelimiters('{%', '%}');

Dalam kasus ini, Anda akan menulis kode di template Anda:

{% if $role=='admin' %}
    <h1>Selamat Datang, Admin</h1>
{% else %}
    <h1>Selamat Datang, Pengguna</h1>
{% endif %}

Meng-escape Data

Secara default, semua substitusi variabel di-escape untuk membantu mencegah serangan XSS pada halaman Anda. Metode esc() CodeIgniter mendukung beberapa konteks yang berbeda, seperti html umum, ketika berada di attr HTML, di css, dll. Jika tidak ada yang lain ditentukan, data akan diasumsikan berada dalam konteks HTML. Anda dapat menentukan konteks yang digunakan dengan menggunakan filter esc():

{ user_styles | esc(css) }
<a href="{ user_link | esc(attr) }">{ title }</a>

Akan ada saat-saat ketika Anda benar-benar membutuhkan sesuatu untuk digunakan dan TIDAK di-escape. Anda dapat melakukan ini dengan menambahkan tanda seru ke kurung kurawal pembuka dan penutup:

{! unescaped_var !}

Filter

Setiap substitusi variabel tunggal dapat memiliki satu atau lebih filter yang diterapkan padanya untuk memodifikasi cara disajikannya. Ini tidak dimaksudkan untuk secara drastis mengubah output, tetapi menyediakan cara untuk menggunakan kembali data variabel yang sama tetapi dengan presentasi yang berbeda. Filter esc yang dibahas di atas adalah salah satu contohnya. Tanggal adalah kasus penggunaan umum lainnya, di mana Anda mungkin perlu memformat data yang sama secara berbeda di beberapa bagian pada halaman yang sama.

Filter adalah perintah yang datang setelah nama pseudo-variabel, dan dipisahkan oleh simbol pipa, |:

// -55 ditampilkan sebagai 55
{ value|abs }

Jika parameter mengambil argumen, argumen tersebut harus dipisahkan oleh koma dan diapit dalam tanda kurung:

{ created_at|date(Y-m-d) }

Beberapa filter dapat diterapkan ke nilai dengan menggabungkan beberapa filter bersama. Filter tersebut diproses secara berurutan, dari kiri ke kanan:

{ created_at|date_modify(+5 days)|date(Y-m-d) }

Filter yang Tersedia

Filter berikut tersedia saat menggunakan parser:

Filter

Argumen

Deskripsi

Contoh

abs

Menampilkan nilai absolut dari sebuah angka.

{ v|abs }

capitalize

Menampilkan string dalam format kalimat: semua huruf kecil dengan huruf pertama dikapitalisasi.

{ v|capitalize}

date

format (Y-m-d)

string, pemformatan yang kompatibel dengan PHP date.

{ v|date(Y-m-d) }

date_modify

nilai yang akan ditambah/ dikurangi

string, yang kompatibel dengan strtotime, untuk memodifikasi tanggal seperti +5 day atau -1 week.

{ v|date_modify(+1 day) }

default

nilai default

Menampilkan nilai default jika variabel empty().

{ v|default(just in case) }

esc

html, attr, css, js

Menentukan konteks untuk meng-escape data.

{ v|esc(attr) }

excerpt

frasa, radius

Mengembalikan teks dalam radius kata dari frasa yang diberikan. Sama dengan fungsi helper excerpt.

{ v|excerpt(green giant, 20) }

highlight

frasa

Menyorot frasa tertentu dalam teks menggunakan tag '<mark></mark>'.

{ v|highlight(view parser) }

highlight_code

Menyorot contoh kode dengan HTML/CSS.

{ v|highlight_code }

limit_chars

limit

Membatasi jumlah karakter hingga $limit.

{ v|limit_chars(100) }

limit_words

limit

Membatasi jumlah kata hingga $limit.

{ v|limit_words(20) }

local_currency

mata uang, locale, pecahan

Menampilkan versi mata uang yang dilokalkan. Nilai "currency" adalah kode mata uang ISO 4217 3 huruf.

{ v|local_currency(EUR,en_US) }

local_number

tipe, presisi, locale

Menampilkan versi angka yang dilokalkan. "tipe" bisa salah satu dari: decimal, currency, percent, scientific, spellout, ordinal, duration.

{ v|local_number(decimal,2,en_US) }

lower

Mengonversi string menjadi huruf kecil.

{ v|lower }

nl2br

Mengganti semua karakter baris baru (n) menjadi tag HTML <br/>.

{ v|nl2br }

number_format

place

Membungkus fungsi PHP number_format untuk digunakan dalam parser.

{ v|number_format(3) }

prose

Mengambil bagian teks dan menggunakan metode auto_typography() untuk mengubahnya menjadi prosa yang lebih indah dan mudah dibaca.

{ v|prose }

round

place, tipe

Membulatkan angka ke tempat yang ditentukan. Tipe ceil dan floor dapat diteruskan untuk menggunakan fungsi tersebut sebagai gantinya.

{ v|round(3) } { v|round(ceil) }

strip_tags

karakter yang diizinkan

Membungkus PHP strip_tags. Dapat menerima string dari tag yang diizinkan.

{ v|strip_tags(<br>) }

title

Menampilkan versi "title case" dari string, dengan semua huruf kecil, dan setiap kata dikapitalisasi.

{ v|title }

upper

Menampilkan string dalam semua huruf kapital.

{ v|upper }

Lihat PHP's NumberFormatter untuk detail yang relevan dengan filter "local_number".

Filter Kustom

Anda dapat dengan mudah membuat filter Anda sendiri dengan mengedit app/Config/View.php dan menambahkan entri baru ke array $filters. Setiap kunci adalah nama yang digunakan filter dalam view, dan nilainya adalah PHP callable yang valid:

<?php

namespace Config;

use CodeIgniter\Config\View as BaseView;

class View extends BaseView
{
    public $filters = [
        'foo'        => '\Some\Class::methodName',
        'str_repeat' => 'str_repeat', // native php function
    ];

    // ...
}

Parser Plugins

Plugin memungkinkan Anda untuk memperluas parser, menambahkan fitur kustom untuk setiap proyek. Mereka bisa berupa PHP callable apa pun, membuatnya sangat sederhana untuk diimplementasikan. Dalam template, plugin ditentukan oleh tag {+ +}:

{+ foo +} inner content {+ /foo +}

Contoh ini menunjukkan plugin bernama foo. Ini dapat memanipulasi konten apa pun antara tag pembuka dan penutupnya. Dalam contoh ini, ia dapat bekerja dengan teks " inner content ". Plugin diproses sebelum substitusi pseudo-variabel terjadi.

Meskipun plugin seringkali terdiri dari pasangan tag, seperti yang ditunjukkan di atas, mereka juga bisa berupa tag tunggal, tanpa tag penutup:

{+ foo +}

Tag pembuka juga dapat berisi parameter yang dapat menyesuaikan cara kerja plugin. Parameter direpresentasikan sebagai pasangan kunci/nilai:

{+ foo bar=2 baz="x y" +}

Parameter juga bisa berupa nilai tunggal:

{+ include somefile.php +}

Plugin yang Disediakan

Plugin berikut tersedia saat menggunakan parser:

Plugin

Argumen

Deskripsi

Contoh

current_url

Alias untuk fungsi helper current_url.

{+ current_url +}

previous_url

Alias untuk fungsi helper previous_url.

{+ previous_url +}

siteURL

Alias untuk fungsi helper site_url.

{+ siteURL "login" +}

mailto

Mendaftarkan Plugin

Paling sederhana, yang perlu Anda lakukan untuk mendaftarkan plugin baru dan membuatnya siap digunakan adalah menambahkannya ke app/Config/View.php, di bawah array $plugins. Kunci adalah nama plugin yang digunakan dalam file template. Nilainya adalah PHP callable yang valid, termasuk metode class statis:

<?php

namespace Config;

use CodeIgniter\Config\View as BaseView;

class View extends BaseView
{
    public $plugins = [
        'foo' => '\Some\Class::methodName',
    ];

    // ...
}

Anda juga dapat menggunakan closure, tetapi ini hanya dapat didefinisikan dalam konstruktor file config:

<?php

namespace Config;

use CodeIgniter\Config\View as BaseView;

class View extends BaseView
{
    public $plugins = [];

    public function __construct()
    {
        $this->plugins['bar'] = static fn (array $params = []) => $params[0] ?? '';

        parent::__construct();
    }

    // ...
}

Jika callable berdiri sendiri, itu diperlakukan sebagai tag tunggal, bukan tag buka/tutup. Ini akan diganti dengan nilai pengembalian dari plugin:

<?php

namespace Config;

use CodeIgniter\Config\View as BaseView;

class View extends BaseView
{
    public $plugins = [
        'foo' => '\Some\Class::methodName',
    ];

    // ...
}

/*
 * Tag is replaced by the return value of Some\Class::methodName() static function.
 * {+ foo +}
 */

Jika callable dibungkus dalam array, itu diperlakukan sebagai pasangan tag buka/tutup yang dapat beroperasi pada konten apa pun di antara tagnya:

<?php

namespace Config;

use CodeIgniter\Config\View as BaseView;

class View extends BaseView
{
    public $plugins = [
        'foo' => ['\Some\Class::methodName'],
    ];

    // ...
}

// {+ foo +} inner content {+ /foo +}

Catatan Penggunaan

Jika Anda menyertakan parameter substitusi yang tidak direferensikan dalam template Anda, parameter tersebut diabaikan:

<?php

$template = 'Hello, {firstname} {lastname}';
$data     = [
    'title'     => 'Mr',
    'firstname' => 'John',
    'lastname'  => 'Doe',
];

return $parser->setData($data)->renderString($template);
// Result: Hello, John Doe

Jika Anda tidak menyertakan parameter substitusi yang direferensikan dalam template Anda, pseudo-variabel asli ditampilkan dalam hasilnya:

<?php

$template = 'Hello, {firstname} {initials} {lastname}';
$data     = [
    'title'     => 'Mr',
    'firstname' => 'John',
    'lastname'  => 'Doe',
];

return $parser->setData($data)->renderString($template);
// Result: Hello, John {initials} Doe

Jika Anda memberikan parameter substitusi string ketika array diharapkan, yaitu, untuk pasangan variabel, substitusi dilakukan untuk tag pasangan variabel pembuka, tetapi tag pasangan variabel penutup tidak dirender dengan benar:

<?php

$template = 'Hello, {firstname} {lastname} ({degrees}{degree} {/degrees})';
$data     = [
    'degrees'   => 'Mr',
    'firstname' => 'John',
    'lastname'  => 'Doe',
    'titles'    => [
        ['degree' => 'BSc'],
        ['degree' => 'PhD'],
    ],
];

return $parser->setData($data)->renderString($template);
// Result: Hello, John Doe (Mr{degree} {/degrees})

Fragmen View

Anda tidak harus menggunakan pasangan variabel untuk mendapatkan efek iterasi di view Anda. Dimungkinkan untuk menggunakan fragmen view untuk apa yang akan ada di dalam pasangan variabel, dan untuk mengontrol iterasi di controller Anda alih-alih di view.

Contoh dengan iterasi yang dikontrol dalam view:

$template = '<ul>{menuitems}
    <li><a href="{link}">{title}</a></li>
{/menuitems}</ul>';

$data = [
    'menuitems' => [
        ['title' => 'Link Pertama', 'link' => '/first'],
        ['title' => 'Link Kedua', 'link' => '/second'],
    ]
];

return $parser->setData($data)->renderString($template);

Hasil:

<ul>
    <li><a href="/first">Link Pertama</a></li>
    <li><a href="/second">Link Kedua</a></li>
</ul>

Contoh dengan iterasi yang dikontrol dalam controller, menggunakan fragmen view:

<?php

$temp      = '';
$template1 = '<li><a href="{link}">{title}</a></li>';
$data1     = [
    ['title' => 'First Link', 'link' => '/first'],
    ['title' => 'Second Link', 'link' => '/second'],
];

foreach ($data1 as $menuItem) {
    $temp .= $parser->setData($menuItem)->renderString($template1);
}

$template2 = '<ul>{menuitems}</ul>';
$data      = [
    'menuitems' => $temp,
];

return $parser->setData($data)->renderString($template2);

Hasil:

<ul>
    <li><a href="/first">Link Pertama</a></li>
    <li><a href="/second">Link Kedua</a></li>
</ul>
