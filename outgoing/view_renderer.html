Menggunakan View Renderer

Fungsi view() adalah fungsi kenyamanan yang mengambil instance dari service renderer, mengatur data, dan merender view. Meskipun ini seringkali persis seperti yang Anda inginkan, Anda mungkin menemukan saat-saat di mana Anda ingin bekerja dengannya secara lebih langsung. Dalam kasus tersebut, Anda dapat mengakses service View secara langsung:

$view = service('renderer');

Atau, jika Anda tidak menggunakan class View sebagai renderer default Anda, Anda dapat menginstansiasikannya secara langsung:

$view = new \CodeIgniter\View\View();

Penting

Anda harus membuat service hanya di dalam controller. Jika Anda membutuhkan akses ke class View dari sebuah library, Anda harus mengaturnya sebagai dependency di constructor library Anda.

Kemudian Anda dapat menggunakan salah satu dari tiga metode standar yang disediakannya: render(), setVar() dan setData().

Apa yang Dilakukan

Class View memproses skrip HTML/PHP konvensional yang disimpan di jalur view aplikasi, setelah mengekstrak parameter view ke dalam variabel PHP, yang dapat diakses di dalam skrip. Ini berarti nama parameter view Anda harus berupa nama variabel PHP yang sah.

Class View menggunakan array asosiatif secara internal, untuk mengakumulasi parameter view sampai Anda memanggil render()-nya. Ini berarti nama parameter (atau variabel) Anda harus unik, atau pengaturan variabel yang lebih baru akan menimpa yang sebelumnya.

Ini juga berdampak pada escaping nilai parameter untuk konteks yang berbeda di dalam skrip Anda. Anda harus memberikan setiap nilai yang di-escape nama parameter yang unik.

Tidak ada makna khusus yang melekat pada parameter yang nilainya adalah array. Terserah Anda untuk memproses array dengan tepat dalam kode PHP Anda.

Mengatur Parameter View

Metode setVar() mengatur parameter view.

$view->setVar('name', 'Joe', 'html');

Metode setData() mengatur beberapa parameter view sekaligus.

$view->setData(['name' => 'George', 'position' => 'Boss']);

Rantai Metode (Method Chaining)

Metode setVar() dan setData() dapat dirantai (chainable), memungkinkan Anda untuk menggabungkan sejumlah panggilan yang berbeda bersama-sama dalam sebuah rantai:

$view->setVar('one', $one)
    ->setVar('two', $two)
    ->render('myView');

Meng-escape Data

Ketika Anda meneruskan data ke fungsi setVar() dan setData(), Anda memiliki opsi untuk meng-escape data untuk melindungi dari serangan cross-site scripting. Sebagai parameter terakhir dalam salah satu metode, Anda dapat meneruskan konteks yang diinginkan untuk meng-escape data. Lihat di bawah untuk deskripsi konteks.

Jika Anda tidak ingin data di-escape, Anda dapat meneruskan null atau 'raw' sebagai parameter terakhir ke setiap fungsi:

$view->setVar('one', $one, 'raw');

Jika Anda memilih untuk tidak meng-escape data, atau Anda meneruskan instance objek, Anda dapat secara manual meng-escape data di dalam view dengan fungsi esc(). Parameter pertama adalah string yang akan di-escape. Parameter kedua adalah konteks untuk meng-escape data (lihat di bawah):

<?= esc($object->getStat()) ?>

Konteks Escaping

Secara default, fungsi esc() dan, pada gilirannya, fungsi setVar() dan setData() mengasumsikan bahwa data yang ingin Anda escape dimaksudkan untuk digunakan dalam HTML standar. Namun, jika data dimaksudkan untuk digunakan dalam Javascript, CSS, atau dalam atribut href, Anda akan membutuhkan aturan escaping yang berbeda agar efektif. Anda dapat meneruskan nama konteks sebagai parameter kedua. Konteks yang valid adalah 'html', 'js', 'css', 'url', dan 'attr':

<a href="<?= esc($url, 'url') ?>" data-foo="<?= esc($bar, 'attr') ?>">Beberapa Tautan</a>

<script>
    var siteName = '<?= esc($siteName, 'js') ?>';
</script>

<style>
    body {
        background-color: <?= esc('bgColor', 'css') ?>
    }
</style>

Opsi View Renderer

Beberapa opsi dapat diteruskan ke metode render() atau renderString():

  • $options

    • cache - waktu dalam detik, untuk menyimpan hasil view; diabaikan untuk renderString().

    • cache_name - ID yang digunakan untuk menyimpan/mengambil hasil view yang di-cache; defaultnya adalah $viewPath; diabaikan untuk renderString().

    • debug - dapat diatur ke false untuk menonaktifkan penambahan kode debug untuk Debug Toolbar.

  • $saveData - true jika parameter data view harus dipertahankan untuk panggilan selanjutnya.

Catatan

$saveData seperti yang didefinisikan oleh interface harus berupa boolean, tetapi class yang mengimplementasikan (seperti View di bawah) dapat memperluas ini untuk menyertakan nilai null.

Referensi Class

class CodeIgniter\View\View
render($view[, $options[, $saveData = false]])
Parameter:

  • $view (string) -- Nama berkas sumber view

  • $options (array) -- Array opsi, sebagai pasangan kunci/nilai

  • $saveData (boolean|null) -- Jika true, akan menyimpan data untuk digunakan dengan panggilan lain. Jika false, akan membersihkan data setelah merender view. Jika null, menggunakan pengaturan konfigurasi.

Kembali:

Teks yang dirender untuk view yang dipilih

Return type:

string

Membangun output berdasarkan nama berkas dan data apa pun yang telah diatur:

echo $view->render('myview');
renderString($view[, $options[, $saveData = false]])
Parameter:

  • $view (string) -- Isi view yang akan dirender, misalnya konten yang diambil dari database

  • $options (array) -- Array opsi, sebagai pasangan kunci/nilai

  • $saveData (boolean|null) -- Jika true, akan menyimpan data untuk digunakan dengan panggilan lain. Jika false, akan membersihkan data setelah merender view. Jika null, menggunakan pengaturan konfigurasi.

Kembali:

Teks yang dirender untuk view yang dipilih

Return type:

string

Membangun output berdasarkan fragmen view dan data apa pun yang telah diatur:

echo $view->renderString('<div>My Sharona</div>');

Peringatan

Ini dapat digunakan untuk menampilkan konten yang mungkin telah disimpan dalam database, tetapi Anda perlu menyadari bahwa ini adalah potensi kerentanan keamanan, dan bahwa Anda harus memvalidasi data tersebut, dan mungkin meng-escape-nya dengan tepat!

setData([$data[, $context = null]])
Parameter:

  • $data (array) -- Array string data view, sebagai pasangan kunci/nilai

  • $context (string) -- Konteks yang akan digunakan untuk escaping data.

Kembali:

Renderer, untuk method chaining

Return type:

CodeIgniter\View\RendererInterface.

Mengatur beberapa bagian data view sekaligus:

$view->setData(['name' => 'George', 'position' => 'Boss']);

Konteks escape yang didukung: html, css, js, url, atau attr atau raw. Jika 'raw', tidak ada escaping yang akan terjadi.

Setiap panggilan menambahkan ke array data yang diakumulasikan objek, sampai view dirender.

setVar($name[, $value = null[, $context = null]])
Parameter:

  • $name (string) -- Nama variabel data view

  • $value (mixed) -- Nilai data view ini

  • $context (string) -- Konteks yang akan digunakan untuk escaping data.

Kembali:

Renderer, untuk method chaining

Return type:

CodeIgniter\View\RendererInterface.

Mengatur satu bagian data view:

$view->setVar('name', 'Joe', 'html');

Konteks escape yang didukung: html, css, js, url, attr atau raw. Jika 'raw', tidak ada escaping yang akan terjadi.

Jika Anda menggunakan variabel data view yang sebelumnya telah Anda gunakan untuk objek ini, nilai baru akan menggantikan nilai yang sudah ada.
