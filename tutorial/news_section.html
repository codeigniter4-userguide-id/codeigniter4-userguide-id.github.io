Pada bagian terakhir, kita membahas beberapa konsep dasar framework dengan menulis sebuah class yang mereferensikan halaman statis. Kita membersihkan URI dengan menambahkan aturan routing kustom. Sekarang saatnya untuk memperkenalkan konten dinamis dan mulai menggunakan database.

Membuat Database untuk Digunakan

Instalasi CodeIgniter mengasumsikan bahwa Anda telah menyiapkan database yang sesuai, seperti yang diuraikan dalam persyaratan. Dalam tutorial ini, kami menyediakan kode SQL untuk database MySQL, dan kami juga mengasumsikan bahwa Anda memiliki klien yang sesuai untuk mengeluarkan perintah database (mysql, MySQL Workbench, atau phpMyAdmin).

Anda perlu membuat database ci4tutorial yang dapat digunakan untuk tutorial ini, dan kemudian mengkonfigurasi CodeIgniter untuk menggunakannya.

Menggunakan klien database Anda, sambungkan ke database Anda dan jalankan perintah SQL di bawah ini (MySQL):

CREATE TABLE news (
    id INT UNSIGNED NOT NULL AUTO_INCREMENT,
    title VARCHAR(128) NOT NULL,
    slug VARCHAR(128) NOT NULL,
    body TEXT NOT NULL,
    PRIMARY KEY (id),
    UNIQUE slug (slug)
);

Juga, tambahkan beberapa catatan seed. Untuk saat ini, kami hanya akan menunjukkan kepada Anda pernyataan SQL yang diperlukan untuk membuat tabel, tetapi Anda harus menyadari bahwa ini dapat dilakukan secara terprogram setelah Anda lebih familiar dengan CodeIgniter; Anda dapat membaca tentang Migrasi dan Seeds untuk membuat pengaturan database yang lebih berguna nanti.

Catatan menarik: "slug", dalam konteks penerbitan web, adalah teks pendek yang ramah pengguna dan SEO yang digunakan dalam URL untuk mengidentifikasi dan mendeskripsikan sumber daya.

Catatan seed mungkin seperti ini:

INSERT INTO news VALUES
(1,'Elvis terlihat','elvis-sighted','Elvis terlihat di kafe internet Podunk. Sepertinya dia sedang menulis aplikasi CodeIgniter.'),
(2,'Jangan bilang begitu!','say-it-isnt-so','Para ilmuwan menyimpulkan bahwa beberapa programmer memiliki selera humor.'),
(3,'Kafeinasi, Ya!','caffeination-yes','Toko kopi terbesar di dunia buka toko kopi di tempat hanya untuk staf.');

Menghubungkan ke Database Anda

File konfigurasi lokal, .env, yang Anda buat saat menginstal CodeIgniter, seharusnya memiliki pengaturan properti database yang tidak dikomentari dan diatur dengan benar untuk database yang ingin Anda gunakan. Pastikan Anda telah mengkonfigurasi database Anda dengan benar seperti yang dijelaskan dalam Konfigurasi Database:

database.default.hostname = localhost
database.default.database = ci4tutorial
database.default.username = root
database.default.password = root
database.default.DBDriver = MySQLi

Menyiapkan Model Anda

Alih-alih menulis operasi database langsung di controller, kueri harus ditempatkan di model, sehingga dapat dengan mudah digunakan kembali nanti. Model adalah tempat di mana Anda mengambil, menyisipkan, dan memperbarui informasi di database Anda atau penyimpanan data lainnya. Mereka menyediakan akses ke data Anda. Anda dapat membaca lebih lanjut tentangnya di Menggunakan Model CodeIgniter.

Membuat NewsModel

Buka direktori app/Models dan buat file baru bernama NewsModel.php dan tambahkan kode berikut.

<?php

namespace App\Models;

use CodeIgniter\Model;

class NewsModel extends Model
{
    protected $table = 'news';
}

Kode ini terlihat mirip dengan kode controller yang digunakan sebelumnya. Ini membuat model baru dengan memperluas CodeIgniter\Model dan memuat library database. Ini akan membuat class database tersedia melalui objek $this->db.

Menambahkan Metode NewsModel::getNews()

Sekarang database dan model telah diatur, Anda akan membutuhkan metode untuk mendapatkan semua postingan kita dari database kita. Untuk melakukan ini, lapisan abstraksi database yang disertakan dengan CodeIgniter - Query Builder - digunakan dalam CodeIgniter\Model. Ini memungkinkan untuk menulis 'kueri' Anda sekali dan membuatnya berfungsi pada semua sistem database yang didukung. Class Model juga memungkinkan Anda untuk dengan mudah bekerja dengan Query Builder dan menyediakan beberapa alat tambahan untuk membuat pekerjaan dengan data menjadi lebih sederhana. Tambahkan kode berikut ke model Anda.

    /**
     * @param false|string $slug
     *
     * @return array|null
     */
    public function getNews($slug = false)
    {
        if ($slug === false) {
            return $this->findAll();
        }

        return $this->where(['slug' => $slug])->first();
    }

Dengan kode ini, Anda dapat melakukan dua kueri yang berbeda. Anda bisa mendapatkan semua catatan berita, atau mendapatkan item berita berdasarkan slug-nya. Anda mungkin telah memperhatikan bahwa variabel $slug tidak di-escape sebelum menjalankan kueri; Query Builder melakukannya untuk Anda.

Dua metode yang digunakan di sini, findAll() dan first(), disediakan oleh class CodeIgniter\Model. Mereka sudah tahu tabel yang akan digunakan berdasarkan properti $table yang kita atur di class NewsModel, sebelumnya. Mereka adalah metode pembantu yang menggunakan Query Builder untuk menjalankan perintah mereka pada tabel saat ini, dan mengembalikan array hasil dalam format pilihan Anda. Dalam contoh ini, findAll() mengembalikan array dari array.

Menampilkan Berita

Sekarang kueri telah ditulis, model harus diikat ke view yang akan menampilkan item berita kepada pengguna. Ini bisa dilakukan di controller Pages kita yang dibuat sebelumnya, tetapi demi kejelasan, controller News baru didefinisikan.

Menambahkan Aturan Routing

Modifikasi file app/Config/Routes.php Anda, sehingga terlihat sebagai berikut:

<?php

// ...

use App\Controllers\News; // Add this line
use App\Controllers\Pages;

$routes->get('news', [News::class, 'index']);           // Add this line
$routes->get('news/(:segment)', [News::class, 'show']); // Add this line

$routes->get('pages', [Pages::class, 'index']);
$routes->get('(:segment)', [Pages::class, 'view']);

Ini memastikan permintaan mencapai controller News alih-alih langsung menuju controller Pages. Baris kedua $routes->get() merutekan URI dengan slug ke metode show() di controller News.

Membuat News Controller

Buat controller baru di app/Controllers/News.php.

<?php

namespace App\Controllers;

use App\Models\NewsModel;

class News extends BaseController
{
    public function index()
    {
        $model = model(NewsModel::class);

        $data['news_list'] = $model->getNews();
    }

    public function show(?string $slug = null)
    {
        $model = model(NewsModel::class);

        $data['news'] = $model->getNews($slug);
    }
}

Melihat kode, Anda mungkin melihat beberapa kesamaan dengan file yang kita buat sebelumnya. Pertama, ia memperluas BaseController yang memperluas class inti CodeIgniter, Controller, yang menyediakan beberapa metode pembantu, dan memastikan bahwa Anda memiliki akses ke objek Request dan Response saat ini, serta class Logger, untuk menyimpan informasi ke disk.

Selanjutnya, ada dua metode, satu untuk melihat semua item berita, dan satu untuk berita item tertentu.

Selanjutnya, fungsi model() digunakan untuk membuat instance NewsModel. Ini adalah fungsi pembantu. Anda dapat membaca lebih lanjut tentangnya di Global Functions and Constants. Anda juga bisa menulis $model = new NewsModel();, jika Anda tidak menggunakannya.

Anda dapat melihat bahwa variabel $slug diteruskan ke metode model di metode kedua. Model menggunakan slug ini untuk mengidentifikasi item berita yang akan dikembalikan.

Menyelesaikan Metode News::index()

Sekarang data diambil oleh controller melalui model kita, tetapi belum ada yang ditampilkan. Hal berikutnya yang harus dilakukan adalah, meneruskan data ini ke view. Modifikasi metode index() agar terlihat seperti ini:

<?php

namespace App\Controllers;

use App\Models\NewsModel;

class News extends BaseController
{
    public function index()
    {
        $model = model(NewsModel::class);

        $data = [
            'news_list' => $model->getNews(),
            'title'     => 'News archive',
        ];

        return view('templates/header', $data)
            . view('news/index')
            . view('templates/footer');
    }

    // ...
}

Kode di atas mendapatkan semua catatan berita dari model dan menetapkannya ke sebuah variabel. Nilai untuk judul juga ditetapkan ke elemen $data['title'] dan semua data diteruskan ke view. Anda sekarang perlu membuat view untuk merender item berita.

Membuat File View news/index

Buat app/Views/news/index.php dan tambahkan potongan kode berikutnya.

<h2><?= esc($title) ?></h2>

<?php if ($news_list !== []): ?>

    <?php foreach ($news_list as $news_item): ?>

        <h3><?= esc($news_item['title']) ?></h3>

        <div class="main">
            <?= esc($news_item['body']) ?>
        </div>
        <p><a href="/news/<?= esc($news_item['slug'], 'url') ?>">View article</a></p>

    <?php endforeach ?>

<?php else: ?>

    <h3>No News</h3>

    <p>Unable to find any news for you.</p>

<?php endif ?>

Catatan

Kita kembali menggunakan esc() untuk membantu mencegah serangan XSS. Tapi kali ini kita juga meneruskan "url" sebagai parameter kedua. Itu karena pola serangan berbeda tergantung pada konteks di mana output digunakan.

Di sini, setiap item berita diulang dan ditampilkan kepada pengguna. Anda bisa melihat kita menulis template kita dalam PHP yang dicampur dengan HTML. Jika Anda lebih suka menggunakan bahasa template, Anda dapat menggunakan View Parser CodeIgniter atau parser pihak ketiga.

Menyelesaikan Metode News::show()

Halaman ikhtisar berita sekarang selesai, tetapi halaman untuk menampilkan item berita individual masih belum ada. Model yang dibuat sebelumnya dibuat sedemikian rupa sehingga dapat dengan mudah digunakan untuk fungsionalitas ini. Anda hanya perlu menambahkan beberapa kode ke controller dan membuat view baru. Kembali ke controller News dan perbarui metode show() dengan yang berikut:

<?php

namespace App\Controllers;

use App\Models\NewsModel;
use CodeIgniter\Exceptions\PageNotFoundException;

class News extends BaseController
{
    // ...

    public function show(?string $slug = null)
    {
        $model = model(NewsModel::class);

        $data['news'] = $model->getNews($slug);

        if ($data['news'] === null) {
            throw new PageNotFoundException('Cannot find the news item: ' . $slug);
        }

        $data['title'] = $data['news']['title'];

        return view('templates/header', $data)
            . view('news/view')
            . view('templates/footer');
    }
}

Jangan lupa untuk menambahkan use CodeIgniter\Exceptions\PageNotFoundException; untuk mengimpor class PageNotFoundException.

Alih-alih memanggil metode getNews() tanpa parameter, variabel $slug diteruskan, sehingga akan mengembalikan item berita tertentu.

Membuat File View news/view

Satu-satunya hal yang tersisa untuk dilakukan adalah membuat view yang sesuai di app/Views/news/view.php. Masukkan kode berikut ke dalam file ini.

<h2><?= esc($news['title']) ?></h2>
<p><?= esc($news['body']) ?></p>

Arahkan browser Anda ke halaman "news" Anda, yaitu, localhost:8080/news, Anda akan melihat daftar item berita, yang masing-masing memiliki tautan untuk menampilkan hanya satu artikel.
